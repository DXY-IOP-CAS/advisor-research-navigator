# advisor-research-navigator

导师研究方向调研与学习路线生成工具。

这个项目的目标很具体：给定一位导师的姓名、机构路径和官网 URL，尽量从公开资料出发，整理出一组可读、可复核、能指导学生入门的 Markdown 文档。它不是报考推荐系统，也不评价导师或学生是否匹配；它更像一个把公开材料、论文线索和学习路径放到同一张桌面上的工作流。

## 项目做什么

主流程从最小输入开始：导师姓名、机构路径和官网 URL。项目会围绕公开资料、论文记录和学习需求，整理出五份相互衔接的 Markdown 文档。它们不是五份并列报告，而是一条从“看见材料”走向“能开始学习”的路线。

1. `00_材料导读.md`：先告诉读者这套材料怎么读。它说明五份文档的阅读顺序、遇到卡点时该回到哪里，以及引用、来源和“需人工复核”等标记该怎样理解。

2. `01_基础画像.md`：建立事实底座。它回答“我们讨论的是哪位老师、资料从哪里来、身份和论文集合有多稳、哪些地方还不能说死”，避免后面的领域判断建立在混乱来源上。

3. `02_领域地图.md`：建立专业坐标。它不写大而全的领域综述，而是帮助学生知道导师方向大致处在哪片领域，旁边有哪些容易混淆的方向，进入论文前要先抓住哪些概念。

4. `03_论文路线.md`：把论文从清单变成问题链。它不按年份或引用数罗列论文，而是看每篇论文在研究推进中承担什么角色：哪些是信号底座，哪些是机制推进，哪些是平台支撑，哪些还需要复核。

5. `04_学习向导.md`：把论文路线倒推成学习路线。它从学生已有的数学、物理和课程基础出发，说明为了读懂目标论文、核心图和平台链路，需要先补哪些概念，并把这些补课转成进组前可以请教的问题。

这五份文档的共同目标不是替学生做结论，而是把“我看不懂导师方向”拆成一组可以继续学习、核查和提问的步骤。

## 当前示范材料和学习子项目

仓库里已经包含两类张老师相关内容，分别回答两个问题：主项目能生成什么，生成之后怎样继续学。

项目生成的示范输出在：

```text
output/中国科学院大学/中科院物理研究所/超快物质科学中心/张鹏举/
```

这个目录下的 `00_材料导读.md` 到 `04_学习向导.md` 是当前示范成品；`_internal/` 保存蓝图、证据记录和工作流内部状态。README 只说明材料位置，不展开张老师的具体信息，因为这组内容仍在继续校准。

如果只想看“最终给学生读的材料”，先看该目录根部的五份 Markdown；如果想理解项目如何把事实材料转成文档，再看 `_internal/blueprint.md` 等内部文件。

围绕这五份成品继续学习的子项目在：

```text
张鹏举学习/
```

这个目录不是主程序输出目录，而是一个后续学习子项目。它承接 `00-04` 成品文档，展示拿到导师调研材料之后，怎样继续用 AI 辅助学习：先确定入口和学习路线，再把原文材料拆成讲义，按小节追问、复述、校准概念，最后留下学习记录和规范。它已经随仓库提交，便于张老师或项目读者看到“生成文档之后如何一步步弄懂”的过程；其中具体内容还在推进中，README 暂不展开细节。

## 主要目录

```text
advisor-research-navigator/
├── src/phase1/              # 阶段 1 脚本：初始化、采集、合并、风险门和基础画像验证
├── quality-workbench/       # 文档标准和支撑方法的工作台，目前是方法设计的主要入口
├── output/                  # 导师调研产出；张老师示范成品也在这里
├── 张鹏举学习/              # 学习子项目：承接 00-04，记录 AI 辅助学习过程
├── docs/                    # 项目计划、上下文交接和端到端回归记录
├── tests/                   # 阶段 1、入口和文档 smoke 测试
└── QUICKSTART.md            # 新窗口或新执行者的快速入口
```

`archive/` 是旧版存档，只供人工溯源；自动执行或二次开发时不要把它当作当前资料来源。

## 使用入口

新接手这个项目时，建议先读：

1. `QUICKSTART.md`
2. `docs/上下文交接.md`
3. `src/phase1/pipeline.md`

阶段 1 的技术入口是：

```bash
python src/phase1/phase1_init.py \
  --university "<大学>" \
  --institute "<学院或研究所>" \
  --department "<部门>" \
  --name "<中文名>" \
  --official-url "<导师官网 URL>"
```

后续脚本会围绕生成的 `prof_dir` 继续运行。完整步骤和验证命令以 `src/phase1/pipeline.md` 与 `QUICKSTART.md` 为准。

## 项目边界

- 所有来源必须可追溯；缺失来源应标为 `[未找到]`，弱推断应标明需要人工复核。
- 项目不输出导师评价、报考建议、匹配度判断或推荐意见。
- 机械验证只能检查结构、链接、格式和部分 smoke 规则，不能替代人工学术判断。
- 示例材料可以帮助理解输出形态，但不能直接当作所有导师的固定模板。
