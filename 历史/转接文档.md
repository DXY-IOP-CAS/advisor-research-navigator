## 6. pilot-test 目录结构（已建好）

```
练习册/project-pilot/pilot-test/
├── .claude/
│   ├── settings.json       ← 阶段3：写 Hook
│   └── skills/project-pilot/  ← 阶段2：写 SKILL.md
└── 产出/                    ← 阶段1：建空目录（待建占位文件）
```

**Hook 加载机制关键事实**：
- Claude Code 启动时根据**当前工作目录**决定加载哪份 `.claude/settings.json`
- 子目录的 settings.json **只在该子目录及其后代生效**
- 父目录的 settings.json 继续在父目录及其他子目录生效
- **两者不冲突**——子目录的 Hook 优先级更高（项目局部覆盖全局）

**因此 pilot-test 是隔离测试区**，Hook 不会污染根项目。但需要在 pilot-test 子目录里新开会话才能让 Hook 生效。
