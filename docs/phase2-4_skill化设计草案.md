# Phase 2-4 Skill 化设计草案

## 目标

把阶段 2-4 从一次性写作任务沉淀为可复用的调研技能和质量门。目标不是让模型套模板，而是让后续 agent 能稳定执行同一套认知流程：

1. 从 `01_基础画像.md` 识别导师当前单位、当前方向、履历阶段和论文线索。
2. 通过 fresh search 补足领域背景，而不是拼贴阶段一。
3. 把导师履历、当前领域、论文路线、学生学习路径层层递进地串起来。
4. 用 deterministic smoke 先挡住格式、来源、禁用评价语等硬错误，再由人工或 agent 做内容质量复核。

## 四份文档的递进关系

`01_基础画像.md` 是事实底座：锁定身份、履历阶段、论文集合、来源风险。论文按学术领域和阶段划分，是为了观察导师在身份、单位、合作网络或研究条件变化时，研究方向是否出现微调、延续或转向。

`02_领域脉络.md` 是申请视角的当前方向地图：先快速建立导师人生历程框架，再把重点放在当前单位和当前方向。它要回答当前方向位于哪个一级学科、二级学科、更小问题域，领域现状和前沿是什么，导师从什么时候开始进入或靠近这条线，哪些旧履历与当前方向直接相关。

`03_论文定位.md` 是专业视角的研究路线解释：基于阶段二的领域地图，把论文分成当前主线、技术前史、早期背景、旁支、弱证据。它要说明导师具体在做什么、用什么方法、研究路线如何展开，并把需要掌握的知识点输出给阶段四。

`04_学习讲义.md` 是教育视角的入门桥梁：基于阶段二和阶段三，倒推学生需要从哪些数学、物理、方法、领域概念开始，按什么顺序读资源和论文，怎样一步步接近导师当前前沿。

## Skill 架构

当前复用入口在 `.claude/skills/research-advisor/`：

- `SKILL.md`：四阶段路由和硬约束。
- `references/phase2-field-map.md`：阶段二领域脉络流程。
- `references/phase3-paper-position.md`：阶段三论文定位流程。
- `references/phase4-learning-guide.md`：阶段四学习讲义流程。
- `references/evidence-rules.md`：来源层级、fresh search、证据强弱、禁止动作。
- `references/quality-gates.md`：共享质量门和分阶段质量门。
- `assets/templates/02_领域脉络.md`、`03_论文定位.md`、`04_学习讲义.md`：只提供稳定章节结构，不替代内容理解。
- `scripts/verify_phase_docs.py`：确定性 smoke，检查文件、章节、来源标记和禁用评价语。
- `tests/test_verify_phase_docs.py`：脚本级回归测试。

## 方法借鉴

阶段二借鉴 scoping review、science mapping、concept mapping：先定范围，再找概念和论文簇，最后形成领域关系图。这里不复制学术综述模板，只借用它们避免漏掉上位学科、子领域、关键方法和证据空洞。

阶段四借鉴 backward design 和 constructive alignment：先定义学生最终要能读懂什么、解释什么、判断什么，再倒推需要学的概念、工具、资源和检查点。这里不复制课程设计模板，只借用“目标先行、活动对齐目标”的思想。

## 执行原则

每次写阶段 2-4 都必须重新检索当前领域资料。阶段一只能提供起点和线索，不能单独支撑阶段二。

遇到领域理解薄弱、方向转折证据不足、论文归类含糊时，优先继续检索或标 `需人工复核`，不要用顺滑但空泛的叙述补洞。

需要独立视角时，可以用 subagent 做外部领域扫描、论文分类复核或质量审查；主线程保留最终取舍和写作责任，避免把 subagent 结论无校验地搬进正文。

确定性脚本只做烟测。真正质量要看：是否消化了资料、是否解释出领域发展脉络、是否把论文放进研究路线、是否能让学生按讲义接近前沿。
