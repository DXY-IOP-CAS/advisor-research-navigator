# Phase 2-4 Skill 化设计草案

## 目标

把阶段 2-4 从一次性写作任务沉淀为可复用的调研技能和质量门。目标不是让模型套模板，而是让后续 agent 能稳定执行同一套认知流程：

1. 从 `01_基础画像.md` 识别导师当前单位、当前方向、履历阶段和论文线索。
2. 通过 fresh search 补足领域背景，而不是拼贴阶段一。
3. 把导师履历、当前领域、论文路线、学生学习路径层层递进地串起来。
4. 用 deterministic smoke 先挡住格式、来源、禁用评价语等硬错误，再由人工或 agent 做内容质量复核。

## 第一性原理

学生真正缺的不是“导师资料汇总”，而是一条从陌生到能理解导师当前研究的认知路径。四份文档都必须服务这条路径：先知道老师是谁，再知道当前领域是什么，再知道老师在这个领域具体做什么，最后知道自己该怎么学到能读懂相关论文。

因此，项目的核心不是为了实现功能而实现功能，也不是把 AI 检索到的材料堆成章节。每个阶段都要回答一个不可替代的问题：

- 阶段一：这位老师的身份、履历、论文集合和来源风险是否可靠？
- 阶段二：这位老师当前方向所在领域是什么，领域如何发展到这里，老师位于哪里？
- 阶段三：这位老师当前研究内容是什么，相关论文群如何构成研究路线？
- 阶段四：学生如何从现有基础一步步学到能理解该方向和这些论文？

如果某份文档无法回答自己的核心问题，即使格式完整、字数充足，也不能算合格。

## 规模化目标

当前项目先服务单个学生申请单个导师，但设计目标必须面向批量化。近期目标是覆盖中国科学院物理研究所一个中心或一个研究所的导师；中期目标是覆盖中国科学院大学相关专业；长期目标是扩展到全国高校和科研院所的物理、化学、材料、光学等专业导师。

因此，四阶段文档不能依赖某个 agent 的临场发挥，也不能只为某一位导师定制。每个阶段都必须有稳定输入、稳定输出、证据规则、失败处理和质量门。允许不同学科在内容上差异很大，但不允许同一阶段的认知职责漂移：阶段一锁事实，阶段二建领域地图，阶段三定位论文，阶段四生成学习路径。

批量化时最危险的问题不是单篇文档写得不够漂亮，而是不同导师之间质量良莠不齐、证据标准不一、章节职责混乱。为避免这种问题，后续所有优化都要优先沉淀到 skill、模板、验证脚本或计划书中，再反映到单个导师文档。

## 四份文档的递进关系

`01_基础画像.md` 是事实底座：锁定身份、履历阶段、论文集合、来源风险。论文按学术领域和阶段划分，是为了观察导师在身份、单位、合作网络或研究条件变化时，研究方向是否出现微调、延续或转向。

`02_领域脉络.md` 是申请视角的当前方向地图：先快速建立导师人生历程框架，再把重点放在当前单位和当前方向。它要回答当前方向位于哪个一级学科、二级学科、更小问题域，领域现状和前沿是什么，导师从什么时候开始进入或靠近这条线，哪些旧履历与当前方向直接相关。

`03_论文定位.md` 是当前研究内容与论文群内部逻辑：基于阶段二的领域地图，深入导师自己的相关论文，建立“研究对象—核心过程—方法平台—论文角色—相邻论文关系”的阅读坐标系。它不是给每篇论文写摘要，也不是把所有论文详细拆解，而是让学生读完后能定位任意一篇当前相关论文在导师研究路线中的位置。

`04_学习讲义.md` 是教育视角的入门桥梁：基于阶段二和阶段三，倒推学生需要从哪些数学、物理、方法、领域概念开始，按什么顺序读资源和论文，怎样一步步接近导师当前前沿。

## Skill 架构

当前复用入口在 `.claude/skills/research-advisor/`：

- `SKILL.md`：四阶段路由和硬约束。
- `references/phase2-field-map.md`：阶段二领域脉络流程。
- `references/phase3-paper-position.md`：阶段三论文定位流程。
- `references/phase4-learning-guide.md`：阶段四学习讲义流程。
- `references/evidence-rules.md`：来源层级、fresh search、证据强弱、禁止动作。
- `references/quality-gates.md`：共享质量门和分阶段质量门。
- `assets/templates/02_领域脉络.md`、`03_论文定位.md`、`04_学习讲义.md`：只提供稳定章节结构，不替代内容理解。
- `scripts/verify_phase_docs.py`：确定性 smoke，检查文件、章节、来源标记和禁用评价语。
- `tests/test_verify_phase_docs.py`：脚本级回归测试。

## Skill 写作原则

参考官方/通用 skill 设计经验后，本项目按以下原则维护 harness：

1. `SKILL.md` 只放触发、路由、硬约束和第一性原理，保持短而稳定。
2. 细节放入 `references/`，按阶段渐进加载，避免每次任务都把所有规则塞进上下文。
3. 重复且脆弱的检查交给 `scripts/`，例如章节、来源标记、禁用评价语；脚本只做确定性 smoke，不冒充学术质量判断。
4. 模板放入 `assets/templates/`，只固定章节骨架，不替代内容理解。
5. 每次实际失败都要沉淀回 skill、模板、脚本或计划书，而不是只在当前对话里解释。
6. 中英文使用分层：触发描述、文件名、脚本参数保留必要英文；认知规则、质量标准、阶段职责尽量中文化，方便用户审查。
7. 复杂写作先查证再合成。遇到领域判断、论文关系、学习路线拿不准时，优先检索专业来源或请用户补充判断，不闭门造车。

## 方法借鉴

阶段二借鉴 scoping review、science mapping、concept mapping：先定范围，再找概念和论文簇，最后形成领域关系图。这里不复制学术综述模板，只借用它们避免漏掉上位学科、子领域、关键方法和证据空洞。

阶段四借鉴 backward design 和 constructive alignment：先定义学生最终要能读懂什么、解释什么、判断什么，再倒推需要学的概念、工具、资源和检查点。这里不复制课程设计模板，只借用“目标先行、活动对齐目标”的思想。

## 执行原则

每次写阶段 2-4 都必须重新检索当前领域资料。阶段一只能提供起点和线索，不能单独支撑阶段二。

阶段二必须先让读者理解领域本身，再把导师放回领域中。合格的 `02_领域脉络.md` 不能只是导师履历和近期论文的重排，而应解释大领域为什么发展到当前小方向、关键技术为什么出现、当前难点在哪里、导师当前课题组处在哪个节点。

遇到领域理解薄弱、方向转折证据不足、论文归类含糊时，优先继续检索或标 `需人工复核`，不要用顺滑但空泛的叙述补洞。

需要独立视角时，可以用 subagent 做外部领域扫描、论文分类复核或质量审查；主线程保留最终取舍和写作责任，避免把 subagent 结论无校验地搬进正文。

确定性脚本只做烟测。真正质量要看：是否消化了资料、是否解释出领域发展脉络、是否把论文放进研究路线、是否能让学生按讲义接近前沿。

阶段三和阶段四不能只反复消化阶段一、阶段二。阶段三应继续检索当前论文、论文引言、官方讲座/新闻/报告、综述资料，以核对老师当前研究路线；阶段四应检索教材、讲义、综述和课程材料，以保证学习路线符合专业共识。检索结果要经过筛选，优先官方、DOI/期刊、arXiv、ORCID、Google Scholar、OpenAlex、教材/讲义和综述，不把二手网页当强证据。

最终文档应采用类似学术笔记的引用风格。正文避免堆裸 URL，可使用 `[P1]`、`[R2]`、`[O1]` 等引用键或隐藏 Markdown 链接，完整链接集中放在“参考文献与来源”。这样既保留可核查性，又保证文稿可读性。

## 当前边界

这份设计还没有替代阶段一的 Python 流水线。阶段一先保留现有 `src/phase1/` 和 `verify_profile.py` / `risk_gate.py`，后续再把阶段一也纳入同一套 skill 规范和质量报告中。

在正式生成张鹏举的 `02_领域脉络.md`、`03_论文定位.md`、`04_学习讲义.md` 前，应先按该 skill 执行 fresh search 和证据分级，再开始写正文。
