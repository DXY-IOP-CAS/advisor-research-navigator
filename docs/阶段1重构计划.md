### 新增：`src/gs_browser.py` — 核心改动

替代 `gs_scraper.py`。用 Playwright 控制真实 Chrome 爬 GS。

**功能**：
- 接收 GS user_id → 打开 profile 页 → 翻页爬全部论文 + 引用指标
- 输出格式完全兼容 `gs_scraper.py` 现有的 JSON schema（`works`、`metrics`、`blocked`）
- 爬不到返回 `blocked: true`，pipeline 自动降级 OA

**实现骨架**：

```python

    f"--user-data-dir=./.cache/chrome-profile",
]

async def scrape_profile(user_id: str, max_pages: int = 5):
    async with async_playwright() as pw:
        browser = await pw.chromium.launch(
            headless=True,  # Chrome headless=new 模式
            args=STEALTH_ARGS,
        )
        context = await browser.new_context(
            viewport={"width": 1280, "height": 900},
            locale="en-US",
            timezone_id="America/New_York",
        )
        # 屏蔽 webdriver 检测
        await context.add_init_script("""
            Object.defineProperty(navigator, 'webdriver', {get: () => undefined});
        """)
        page = await context.new_page()
```

## 物理所批量处理策略

### 第一步：准备导师名单

物理所官网（`http://english.iphy.ac.cn/` 或 `http://www.iop.cas.cn/`）找到全部导师列表。估计 80-120 位。

名单格式：

```csv
name,affiliation,gs_id,oa_id,orcid,url
张鹏举,中科院物理所,ls7XuGoAAAAJ,A5000914228,0000-0001-7746-2113,http://...
...
```

其中 `gs_id`、`oa_id`、`orcid` 可以为空，Step 2 广域搜索自动找。

### 第二步：分批次运行

```
第 1 批：导师 1-5  → 约 30 分钟（6 页/人 × 12s/page + 官网 + OA + arXiv）
           └── 暂停 5 分钟
第 2 批：导师 6-10 → 约 30 分钟
           └── 暂停 5 分钟
...
~20 批 = 约 12 小时完成全部
```

**运行时机**：建议留一晚跑，或者分多天跑。

**断点续跑**：每完成一位导师写入缓存。中途中断后重新运行自动跳过已完成的。

### 第三步：缓存管理

第一次全量跑之后，30 天内新增/修改导师名单时只跑新增的。30 天后运行自动刷新缓存。
