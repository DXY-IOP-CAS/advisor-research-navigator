### 已知候选工具（需调研）

以下工具在先前调研中已发现，但未深入评估。需部署测试后判断是否可用。

**1. `academic-research-mcp`** ← 看起来最可能直接替代大部分脚本
- GitHub: https://github.com/alisoroushmd/academic-research-mcp
- 能力：Google Scholar（搜索+作者 profile）+ OpenAlex + S2 + arXiv + ORCID + CrossRef + Unpaywall
- 25 个工具、无 API key 要求、本地缓存
- **需核实**：GS 作者 profile 爬取的稳定性和论文列表完整性、中文作者覆盖

**2. `scholarly` Python 库**
- PyPI: `scholarly`，2K+ stars
- 能力：GS 搜索、作者 profile、引用追踪、內建代理轮换（ScraperAPI/FreeProxy）
- **需核实**：免费代理模式的可靠性（社区反馈 free proxy 经常超时）

**3. `google-scholar-search-mcp`（LWaetzig）**
- GitHub: https://github.com/LWaetzig/google-scholar-search-mcp
- 基于 scholarly，內建限速 5-15s、重试 3 次、代理配置
- **需核实**：与 academic-research-mcp 的互补性

**4. `professor-fit-analyser`（voidful/academic-skills）**
- GitHub: https://github.com/voidful/academic-skills
- 7 步方法论框架：Snapshot → Timeline → Taxonomy → Read Papers → Synthesize → Future → Fit
- **需核实**：方法论能否借鉴到阶段 2/3

### 调研方法

1. **部署 + 实测**：对每个候选工具，用张鹏举的 GS ID 跑一次完整流程，看论文数、质量、稳定性
2. **不瞎拼瞎凑**：评估后选出 1-2 个核心工具，替换或补充现有脚本。不是全部装上来
3. **评估维度**：论文覆盖度、GS 封锁频率、中文作者兼容性、维护活跃度、依赖复杂度

## 关键约束

- **禁止自己造轮子**：能用现成工具解决的问题绝不自写
- **禁止缓存学者数据**：每次调研必须全新，除非用户明确要求缓存
- **微调适配，不是直接套用**：工具要调成符合我们需求的样子
- **结果导向**：最终产物是一个能在新窗口稳定运行的阶段 1 流水线
