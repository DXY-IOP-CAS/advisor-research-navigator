## 模型使用约定

平时开发、研究判断、规则设计、论文路线分析、学习路径设计、阶段 2-4 写作或质量修订，默认使用当前高质量模型。这个项目最容易被低质量推理带偏：如果论文方向、证据强弱、领域边界、学习路径判断、文件编码或验证链路错了，后续返工成本会高于省下的额度。

不再主动建议为了省额度切换到低成本模型。即使任务看起来只是批量性、重复性、低智力、可验证的机械工作，也优先由当前高质量模型执行并完成验证。只有当用户明确要求使用低成本模型时，才把任务限制在边界清楚、有清单、有验证命令的机械工作；不得让低成本模型承担研究方向判断、证据强弱判断、论文主线重写、学习路线设计、长期规则修改或上下文交接。低成本模型完成机械任务后，Agent 必须提醒用户切回高质量模型做内容审查、研究判断和下一步设计；如果发现验证失败、编码异常或上下文压缩风险，应立即停止低成本模型流程。

## 四份文档的第一性原理

学生真正缺的不是“导师资料汇总”，而是一条从陌生到能理解导师当前研究的认知路径。

四份文档回答四个不可替代的问题：

1. `01_基础画像.md`：这位老师是谁，身份、履历、论文集合和来源风险是否可靠？
2. `02_领域地图.md`：这位老师当前方向所在领域是什么，领域如何发展到这里，老师位于哪里？
3. `03_论文路线.md`：这位老师当前研究内容是什么，相关论文群如何构成研究路线？
4. `04_学习向导.md`：学生如何从现有基础一步步学到能理解该方向和这些论文？

如果某份文档无法回答自己的核心问题，即使格式完整、字数充足，也不能算合格。

## 阶段边界

阶段一是事实底座。论文按履历阶段划分，不是为了按年份归档，而是为了观察导师在身份、单位、合作网络或研究条件变化时，研究方向是否微调、延续或转向。

阶段二是当前方向地图。它先快速回顾导师学术路径，帮助学生建立人物和时间框架；随后重点讲当前单位和当前方向。早期方向只作为当前方向的前史和能力来源处理，不平均展开。

阶段三是当前研究内容与论文群内部逻辑。它基于阶段二的领域地图，深入导师自己的相关论文，建立“研究对象—核心过程—方法平台—论文角色—相邻论文关系”的阅读坐标系。它不是论文摘要合集，也不是逐篇深挖所有论文。学生读完后，应能定位任意一篇当前相关论文在导师研究路线中的位置。

阶段四是学习路径。它从阶段二和阶段三倒推，说明学生需要补哪些数学、物理、方法、领域概念，按什么顺序读资源和论文，怎样逐步接近导师当前前沿。它不是通用课程表，也不替代教材正文。

## 长期路线：当前 Harness 与未来教学 Agent 的边界

当前项目的目标是把导师研究方向 Harness 做稳：用户输入姓名、机构和官网 URL，产出四份可验证、可复用、可批量扩展的 Markdown 文档。阶段 4 的定位是学习路径、小字典和知识地图，帮助学生从接近小白逐步接近“能理解导师当前方向、能开始读关键论文、能满足进组前认知准备”的状态。

未来计划是单独开发教学 Agent，用来接续阶段 4：根据学生当前水平进行互动教学、诊断弱点、安排练习、解释教材和论文。这个教学 Agent 是后续独立项目，不应提前塞进当前 Harness。当前 Harness 完全弄好，并且全自动批量化跑完整个中科院物理所之后，才进入教学 Agent 项目的开发。当前项目必须可落地、可验证、可批量复用，不能走马观花或用漂亮文字敷衍。

## 信息检索与引用规则

阶段二到四不能只反复消化前文。阶段一和前序文档可以作为可靠输入源，但仍必须做 fresh search。

优先

1. 导师官网、课题组、机构主页、CV、项目/基金页面。
2. DOI、期刊/出版社、arXiv、Google Scholar、OpenAlex、ORCID。
3. 近年综述、教程、教材、课程讲义。
4. Wikipedia/百科只用于术语初筛，不能单独支撑技术判断。

正文采用论文式上标引用，避免裸 URL 噪声。优先用 `<sup><a href="#o1">[O1]</a></sup>`、`<sup><a href="#p2">[P2]</a></sup>`、`<sup><a href="#r3">[R3]</a></sup>` 等可点击引用键，完整链接集中放在“参考文献与资料”；文末参考文献表编号列使用 `<a id="p3"></a>[P3]` 形式提供跳转锚点。来源缺失写 `[未找到]`；合理但证据不充分写 `需人工复核`。
