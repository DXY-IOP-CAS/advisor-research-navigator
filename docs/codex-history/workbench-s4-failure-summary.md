## 反复出现的问题

1. 分析停在原则层：能说“可视化要服务理解”，但没有把规则落到某一份 `00-04` 文档的某一节、某个读者卡点和某种图形选择。
2. 图像变成装饰：新增 Mermaid 或图示只是把标题连起来，没有减少读者需要记住的关系，也没有改变写作动作。
3. 场景缺失：没有先说清楚读者正在读哪段材料、卡在哪里、图要帮他看见什么关系，就直接开始画图。
4. 渲染边界不清：没有说明图放在哪里、用 Mermaid 还是图片、相对路径如何写、在 Markdown/PDF/Codex 里能否实际显示。
5. 反馈后只修辞性承认问题：道歉和“你说得对”没有进入新的检查点，下一轮仍按同一种模板继续写。

## 以后处理可视化时的硬门槛

每个候选图都先写出这五项，写不出就不要画：

```text
所在文档和小节：
读者当前卡点：
图要呈现的关系：
为什么纯文字不够：
验证方式：
```

图形选择必须来自关系类型：

| 关系类型 | 优先形式 | 不适合的形式 |
|:---|:---|:---|
| 3-6 个概念之间的依赖 | 小型关系图或定位表 | 长 Mermaid 流程图 |
| 学习路径或论文路线 | 纵向路线、阶段表 | 横向大流程图 |
| 两种说法、两条路线、两个证据层级 | 双列表或对照表 | 把对照画成节点图 |
| 多个对象按同一维度比较 | 矩阵表 | 分散的卡片或段落 |
| 单个复杂图的读图顺序 | 带编号的注释清单 | 重新画一张泛化图 |

## 落地检查

改 `quality-workbench/支撑方法/可视化与理解.md` 或任何 `00-04` 成品前，先确认本次任务的主层级：

```text
S4 支撑方法 / D00-D04 文档标准 / 真实导师成品 / project skill / verifier
```

一次只改一个主层级。发现另一个层级也需要改时，写入“后续回收”，不要顺手扩大范围。

完成后至少检查：

- 目标段落是否真的因为图而更容易读。
- 图是否有明确标题、说明和证据边界。
- Mermaid 或图片是否能在目标载体中显示；不能验证时，必须明说。
- 图没有替代事实核查、论文理解或人工学术判断。
- 新规则能被下一位执行 AI 复用，不依赖这次聊天上下文。

## 停止条件

如果修改后只能得到“原则更完整”“分类更整齐”“看起来更像方法论”，但不能指出下一次写 `00-04` 时会改变哪一步动作，就停止继续润色。应回到具体导师样本、具体章节和具体读者卡点上重新判断。
