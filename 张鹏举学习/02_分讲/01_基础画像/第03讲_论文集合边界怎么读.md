# 第 03 讲：论文集合边界怎么读

这一讲继续学 `01_基础画像.md`，只处理一个问题：论文表到底能用来做什么，不能用来做什么。

## 这一讲解决什么

`01_基础画像.md` 里的论文表不是“代表作推荐表”，也不是“当前主线论文清单”。它是多源合并后的事实范围，用来告诉我们：

- 当前能查到哪些论文记录。
- 哪些记录来自 Google Scholar、OpenAlex、arXiv 或 DOI 页面。
- 哪些论文明显不属于这位老师，已经被排除。
- 哪些记录仍有元数据风险或显示截断。
- 后续 `02/03/04` 应该在什么范围内继续判断。

所以这一讲不要问“哪篇最厉害”，而要问：

> 这张论文表能支持什么判断？哪些判断必须等 `02` 和 `03` 继续完成？

## 论文表能支持什么

| 能支持的事 | 具体含义 |
|:---|:---|
| 论文集合范围 | 当前多源检索后有哪些候选记录，明显误归属已经排除 |
| 时间变化 | 论文主题怎样随履历阶段变化 |
| 方向线索 | 哪些阶段开始出现 XUV、阿秒、液相光电子谱、液体水 ICD 等线索 |
| 风险提示 | 哪些 DOI、ORCID、题名截断或来源记录需要后续复核 |
| 后续入口 | 帮助 `02` 定领域，帮助 `03` 选论文角色，帮助 `04` 倒推学习缺口 |

## 论文表不能直接支持什么

| 不能直接支持的事 | 为什么 |
|:---|:---|
| 不能直接评价导师强弱 | 本项目不做导师评价，也不把论文数量、引用数写成申请建议 |
| 不能直接代表当前主线 | 论文表包含早期训练、合作论文、平台论文、背景论文和当前论文 |
| 不能直接替代原论文阅读 | 题名、期刊和引用数不能说明论文内部问题、方法和图表证据 |
| 不能直接证明每条元数据都无误 | 多源记录可能有截断、误配、ORCID 差异或平台显示问题 |
| 不能直接生成学习路线 | 学习路线要等 `03` 确认目标论文和核心图后，再由 `04` 倒推 |

## 关键读法

第一，看论文表时先看“阶段”和“主题变化”，不要一上来就看引用数。引用数可以提示影响力，但在这里不是学习主线。

第二，把论文表当成待解释的材料。`01` 只告诉你“这些记录在当前证据下可以进入视野”；真正解释它们怎样组成当前问题链，要等 `03_论文路线.md`。

第三，遇到元数据异常时，不要马上删掉或全盘接受。先标成风险：它可能是平台记录问题，也可能提示同名、ORCID 或作者归属需要复核。

## 苏格拉底式自检

请你回答这 4 个问题：

1. 为什么 `01` 的论文表不能直接当作“代表作清单”？
2. 如果一篇论文引用数很高，但属于早期阶段，它对理解当前方向可能有什么用？又有什么限制？
3. 如果某条论文记录来自 Google Scholar，但 DOI 或 ORCID 信息有异常，你应该怎么处理？
4. 后续进入 `03_论文路线.md` 时，你会怎样使用 `01` 里的论文表？

## 费曼输出

合上论文表后，用 4 句话复述：

```text
`01` 里的论文表主要帮我……
它不能直接告诉我……
遇到元数据异常时，我应该……
后面读 `03` 时，我会用这张表来……
```

## 这一讲的出口

你能说清下面这句话，就可以结束 `01_基础画像` 的第一阶段学习，准备进入 `02_领域地图.md`：

> `01` 的论文表是事实范围和风险边界，不是代表作推荐表。它帮助我看履历阶段、主题变化和来源风险；真正的当前主线、论文角色和学习入口，要等 `02` 和 `03` 在这个事实底座上继续判断。
