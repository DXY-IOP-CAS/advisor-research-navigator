# 学习区规范

这份规范只服务导师方向学习区。当前文件夹名是 `张鹏举学习/`，当前学习对象是张鹏举老师；但这套方法不只服务这一位导师。以后换成其他导师，也应保留同样的学习内核：先从原文档抽主线，再按主线拆子任务，讨论后必须回到主线。

## 第一原则

这个文件夹只有一个功能：

> 帮助学习者看懂当前导师的研究方向、核心概念、论文路线和需要补的专业知识。

不要在这里记录项目设计、模板修改、verifier、harness、skill、workbench、规则回收或正式成品开发任务。那些事情属于其他窗口或其他 AI。

## AI 辅导学习的固定流程

以后任何 AI 接手这个文件夹，都不能直接凭聊天印象开始讲概念。必须先从原文档建立主线，再把主线拆成学习子任务。

固定顺序如下：

```text
1. 先锁定本轮原文档。
2. 重新读取原文档。
3. 检查或生成 `00_主线表.md`。
4. 锁定或生成本轮 `02_分讲/` 主阅读材料。
5. 检查或生成本轮多轮对话安排。
6. 按主阅读材料逐段学习，不用 03 记录替代 02 分讲。
7. 每个子任务只解决一个主问题。
8. 子任务中可以讨论、追问、稍微发散。
9. 每次发散后必须回到原文主线和本轮分讲。
10. 本轮结束时留下可复述的出口句、理解校准和下一步入口。
```

这里的“原文档”指 `01_原文/` 里的 `00-04` 快照。分讲、记录和对话都不能替代原文档。分讲只是帮助理解原文主线的脚手架。

“继续学习”也要走这个顺序。不能因为目录里已有分讲，或者聊天里刚讨论过某个概念，就跳过原文和主线表。

### 学习区 Harness：整文地图闸门

学习区的最小 Harness 不是测试脚本，而是一组开讲前必须通过的闸门。它要防止 AI 抓住一个例子反复展开，却没有带学习者学完整份成品文档。

每进入一份 `00-04` 原文学习，AI 必须先生成或核对“整文学习地图”：

```text
当前原文：
这份文档的主问题：
全文重要内容块：
内容块之间的理解依赖：
哪些内容只是入口例子或支撑材料：
由浅入深的学习顺序：
整份文档读完后的学生出口：
当前这一轮处在整文地图哪一块：
```

没有这张地图，不能直接讲概念、论文、图或术语。已有 `00_主线表.md` 但只覆盖局部例子的，也视为没有通过闸门。

整文地图还要区分“入口例子”和“整份主线”。例如液体水 ICD 可以作为张鹏举材料里的重要入口，但不能替代 `02_领域地图.md` 的全部领域定位，也不能替代 `03_论文路线.md` 的全部论文问题链。

学习中如果出现这些信号，AI 必须停讲并回到整文地图：

```text
用户说混乱、认知过载、看不懂路线；
连续两轮都在同一个例子里扩展；
把 `02` 的领域入口当成 `03` 或 `04` 的学习主线；
用户指出 AI 一直在绕局部例子；
AI 说不出当前讲解对应整份原文哪一块。
```

## 02 分讲主阅读闸门

每轮正式学习必须有一个当前主阅读版本，位置只能在 `02_分讲/<原文编号与主题>/`。AI 开讲前必须明示：

```text
本轮原文：
整份文档主问题：
整份文档学习路线：
本轮主线位置：
本轮主阅读：
本次只读哪几段：
暂时不读哪几段：
读完要能说：
```

如果本轮子任务已有分讲，AI 必须先沿着这篇分讲带读，不能只把它当背景检查。带读的最小动作是：引用或转述分讲当前小节的核心句，解释这一小节解决什么，再让学习者复述一句。不能在未带读分讲正文前，连续追问理解验收。

如果本轮子任务没有分讲，AI 必须先从 `01_原文/` 和 `00_主线表.md` 生成一篇可顺读的 `第 N 讲_*.md` 草稿，再开始辅导。这个草稿要满足“分讲怎么写”的要求：开头说明解决什么、暂时不解决什么、读完能说什么；正文按理解依赖推进；结尾保留少量自检句。没有 02 分讲时，不允许只生成 `03_学习记录/` 然后直接问答。

`03_学习记录/` 的作用是记录本轮安排、用户原始理解和校准结果。它不能承担主阅读功能，也不能成为“没有写 02 分讲”的替代品。学习记录里出现了更清楚的解释，下一步应提取到对应分讲；在提取前，它只能作为过程记录，不作为复习入口。

## 主线先于子任务

每学一份原文档，先问三件事：

```text
这份原文档到底想让我看懂什么？
它的主线从哪里开始，到哪里结束？
哪些内容是主干，哪些只是例子或支撑材料？
```

没有完成这一步，不要直接新增分讲，也不要直接讲某个术语。术语讲解必须挂回原文主线。

主线必须落成文件，而不是只在对话里口头说明。每份正在学习的原文，都应在对应分讲目录里有一份：

```text
02_分讲/<原文编号与主题>/00_主线表.md
```

这份表至少包括：

```text
原文位置：
这一段在原文里承担的任务：
学习时要解决的问题：
学完要能说：
暂时不要做：
```

主线表必须从原文抽出。抽取时先看原文标题层级、段落推进、表格和图的功能、证据与待复核边界。不要从已有分讲标题、学习记录、聊天摘要或 AI 自己想讲的顺序倒推。

主线表必须覆盖整份原文，至少能让学习者看到：

```text
全文主问题；
全部重要内容块；
理解依赖；
入口例子和支撑材料；
整文学习顺序；
整文出口；
当前轮次位置；
```

如果主线表只列出局部概念链，比如只围绕某一篇论文、某一个图组或某一个机制例子展开，就不能作为整份原文的主线表。

例如学习 `02_领域地图.md` 时，主线应先压成类似这样的链条：

```text
张鹏举老师当前方向
  -> 位于阿秒/XUV 超快谱学和复杂体系电子动力学的交叉处
  -> 工具包括 HHG、XUV、阿秒脉冲、光电子谱和符合测量
  -> 对象从分子扩展到液体水、液相分子和固体
  -> 问题集中在电子运动、弛豫、电荷转移、ICD 和电子-核运动耦合
  -> 实验平台重要，因为没有光源、样品和谱仪，很多机制看不见
  -> 液体水 ICD 是一个具体入口，因为它把工具、对象、机制和平台难点压在同一个例子里
```

只有先有这条主线，后面讲 HHG、PES、极端条件、ICD、平台论文，才不会乱跳。

## 子任务必须服从主线

主线确定后，再拆子任务。子任务顺序不能按文件创建时间，也不能按 AI 临时想到什么讲什么，而要按理解依赖来排。

一个合格的子任务要写清：

```text
本轮原文：
本轮主线位置：
本轮只解决：
暂时不展开：
学完要能说：
如果跑偏，回到：
```

示例：

```text
本轮原文：
01_原文/02_领域地图.md

本轮主线位置：
从“为什么需要极端条件和光电子谱”进入“液体水 ICD 这个具体例子”。

本轮只解决：
ICD 是什么，为什么液体水 ICD 适合作为入口。

暂时不展开：
不读 2025 论文细图，不讲完整非绝热动力学，不讲机制论文和平台论文分类。

学完要能说：
液体水 ICD 为什么能把光电子谱、低能电子、液相环境、电子弛豫和核运动竞争连在一起。

如果跑偏，回到：
02_领域地图.md 里“为什么先从液体水 ICD 切入”的主线。
```

如果一个子任务里出现了旁支问题，可以短暂解释，但必须说明它服务主线哪一环。服务不了主线的旁支，先记入学习记录，不继续展开。

## 每轮对话先有安排

进入一轮新的学习前，必须在 `03_学习记录/` 里生成或更新本轮对话安排。它不是主阅读材料，也不是项目设计文档，而是本轮教学控制台。

这份记录至少写清：

```text
本轮原文：
主线表位置：
本轮只解决：
暂时不展开：
对话推进表：
用户每轮需要输出：
AI 每轮只校准什么：
跑偏时怎么收回：
本轮保留句：
下一步入口：
```

如果没有这份安排，不要直接开讲。尤其不要在用户说“继续学习”后，立刻凭上一轮聊天印象讲概念。

这份安排必须引用本轮 `02_分讲/` 主阅读文件，并写清本次从哪一小节开始。如果对应分讲还不存在，先生成分讲，再写学习记录；不能先用学习记录承接一整轮概念讲解。

## 每轮结束要回到原文

每轮学习结束时，不只问“懂了吗”，而要检查三件事：

```text
这轮对应原文哪一段或哪一个判断？
这轮解决了主线里的哪一环？
我现在能不能用自己的话复述这一环？
这一环在整份文档地图里的上一块和下一块是什么？
```

如果复述不出来，就不能进入下一讲。先回到本轮分讲或原文相关段落。

学习记录也要按这个逻辑写。每份记录至少保留：

```text
本轮原文：
本轮主线位置：
讨论中发散了什么：
发散怎样回到主线：
本轮保留句：
下一步入口：
```

这样新来的 AI 才能知道学习不是随便聊天，而是在沿着原文主线推进。

## 文件角色

学习区只有五类材料。

| 类型 | 放在哪里 | 判断标准 |
|:---|:---|:---|
| 入口 | `00_入口/` | 告诉读者从哪开始、现在学到哪里、下一步读什么 |
| 原文 | `01_原文/` | 从正式输出复制来的对照材料，不在这里改写 |
| 分讲 | `02_分讲/` | 真正给人顺读、复习、补概念的学习材料 |
| 学习记录 | `03_学习记录/` | 用户回答、理解验收、问答校准和卡点追溯 |
| 规范与历史 | `90_规范与历史/` | 学习区使用规范、迁移对照、历史版本 |

新增文件前先判断它属于哪一类。判断不清时，先在学习记录里标记为待判断材料，不要冒充分讲；但正式辅导前仍必须先锁定或生成对应的 `02_分讲` 主阅读材料，不能用这条学习记录承接整轮讲解。项目设计内容不放进本文件夹。

## 命名规则

`第 N 讲` 只给分讲使用。它必须能顺着读，能回答一个清楚的专业学习问题。

`第 N 轮` 只给学习记录使用。它可以保留当时的回答、错词、追问和校准，不要求像文章一样完整。

正式原文只放在 `01_原文/`，文件名保留 `00-04` 编号。不要把原文副本、学习讲义和理解验收放在同一层。

旧入口不使用 `99_` 这种方式降级。旧材料要么合并进当前入口，要么移入 `90_规范与历史/历史版本/`。

## 分讲怎么写

一篇分讲只解决一个学习问题。标题要像学生真的会问的话，例如“液体水 ICD 为什么适合作为入口”，不要用“第 02 轮理解验收”这种过程标题。

每篇分讲开头五行内说清三件事：

```text
这一讲解决什么？
暂时不解决什么？
读完以后要能说出哪句话？
这一讲属于整份原文地图中的哪一块？
```

正文要先给直觉，再给术语。比如先说“用足够短的光去看电子运动”，再引入“阿秒”；先说“把电子打出来再读它带出的信息”，再引入“光电子谱”。

分讲结尾只保留 3-4 个自检句。自检句要能让读者用自己的话补完，不要把一大串问题清单当作学习出口。

## 学习记录怎么写

学习记录不是主阅读材料。它的作用是让后来的人知道：当时用户怎么理解、哪里被校准、哪些解释已经提取成分讲。

每份学习记录开头都要写清：

```text
这是一份学习记录，不是主阅读材料。
当前主阅读版本在哪里。
已提取到哪里；如果没有提取，为什么还保留。
```

记录正文可以保留问答过程，但不要变成杂乱聊天日志。最少要保留三件事：

- 用户原始理解；
- 校准或补充；
- 下一步学习入口。

如果记录承担本轮多轮对话安排，还要保留对话推进表。后续真正发生问答后，再把用户原始理解和校准补进去。不要把未校准的解释直接提取成分讲。

## 同一内容只能有一个主阅读版本

同一个解释可以在学习记录里出现原始版本，但主阅读版本只能放在一篇分讲里。入口和总览只放摘要和链接，不继续扩写。

如果某个过程记录里出现了更好的解释，处理方式是：

1. 把可复用解释提取到对应分讲。
2. 原记录保留在 `03_学习记录/`，用于追溯。
3. README 或学习路线只链接主阅读版本。

这样以后回来复习时，不会在三四个文件里反复读到同一段话。

## 禁止放入本文件夹的内容

下面这些内容不放进 `张鹏举学习/`：

- 项目开发计划；
- 文档标准设计；
- 模板、skill、verifier、harness 的修改建议；
- 对正式成品质量体系的回收规则；
- 和学习张老师专业方向无关的项目管理记录。

如果学习过程中出现这类想法，先在对话里说明“这不属于张鹏举学习区”，不要写进本文件夹。

## 整理检查清单

每次新增或整理文件前，先过一遍这张清单。

新增分讲时检查：

- 标题是不是学生会问的专业问题，而不是过程标题。
- 开头是否说清这一讲解决什么、暂时不解决什么、读完能说哪句话。
- 是否只有一个主阅读版本；重复解释是否已经从入口或记录里降级。
- 结尾是否有少量自检，而不是一大串题目。

新增学习记录时检查：

- 开头是否说明这是记录，不是主阅读材料。
- 是否写清当前主阅读版本在哪里。
- 是否保留用户原始理解、校准和下一步。
- 是否写明“已提取到哪里”；如果没有提取，要说明为什么保留。

新增历史版本时检查：

- 是否有冻结头，说明来源、保存原因和当前替代文件。
- 是否明确“不是当前主阅读材料”。
- 是否放在 `90_规范与历史/历史版本/`，而不是入口区。

最后再检查一遍：这个文件是否真的帮助学习张鹏举老师的研究方向。如果不是，就不要放进这个文件夹。
