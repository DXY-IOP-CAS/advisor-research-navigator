# 导师方向学习区 AGENTS.md

## 一句话定位

这个目录是导师研究方向学习区。当前实例是张鹏举老师，但这里沉淀的方法不只服务这一位导师；以后换成其他导师，也应沿用同一套“原文主线 -> 子任务 -> 讨论 -> 回主线”的学习方法。

这个目录只服务一个目标：帮助学习者看懂当前导师的研究方向、核心概念、论文路线和需要补的专业知识。

这里不是项目设计工作台，不承担正式成品开发、质量规则回收、模板修改、skill 修改、verifier、harness、workbench 或其他工程任务。

## 开始前必须读

新窗口或新 AI 进入本目录后，先读：

1. `README.md`
2. `90_规范与历史/00_学习区规范.md`
3. 当前要学习的 `01_原文/*.md`

不要直接根据聊天印象或分讲标题开始讲概念。

## 学习辅导总流程

每次辅导学习，都按这个顺序。这里的“继续学习”不是直接开讲，而是先完成学习准备动作：

```text
1. 锁定本轮原文档。
2. 重新读取原文档，不用聊天印象、分讲标题或旧记录替代原文。
3. 检查对应分讲目录下是否已有 `00_主线表.md`。
4. 如果没有主线表，先生成；如果主线表不是从原文结构抽出的，先修正。
5. 在 `03_学习记录/` 里生成或更新本轮多轮对话安排。
6. 再开始正式讲解，每个子任务只解决一个主问题。
7. 子任务中可以讨论、追问、稍微发散。
8. 每次发散后必须回到原文主线。
9. 本轮结束时留下可复述的出口句、用户理解校准和下一步入口。
```

这里的原文档指 `01_原文/` 里的 `00-04` 快照。分讲、学习记录和对话都不能替代原文档。

### 主线表和多轮对话安排是开讲前置条件

进入任何一份原文学习前，必须先检查两类文件：

```text
主线表：02_分讲/<原文编号与主题>/00_主线表.md
多轮对话安排：03_学习记录/<原文编号与主题>/第NN轮_*.md
```

主线表必须来自原文的真实结构。生成时先读原文标题、段落功能、表格和图的作用，再抽出学习顺序。不能从已有分讲标题、上一轮聊天、AI 的概括习惯或“看起来合理”的教学顺序倒推。

多轮对话安排必须写清：

```text
本轮原文：
主线表位置：
本轮只解决：
暂时不展开：
对话推进表：
用户每轮需要输出：
AI 每轮只校准什么：
跑偏时怎么收回：
本轮保留句：
下一步入口：
```

如果这两类文件缺失、过时或与原文不一致，先补齐或修正，再开始辅导。不要等用户提醒才生成。

## 每轮开讲前必须写清

每一轮正式讲解前，先给出这六项：

```text
本轮原文：
本轮主线位置：
本轮只解决：
暂时不展开：
学完要能说：
如果跑偏，回到：
```

如果写不出这六项，说明还没有准备好开讲。先回到原文档抽主线。

如果没有完成“读原文 -> 主线表 -> 多轮对话安排”，也说明还没有准备好开讲。此时不要用临场解释替代文件化准备。

## 学习顺序服从理解依赖

学习顺序按原文主线和理解依赖安排，不按文件创建时间、不按 AI 临时想到的顺序安排。

例如学习 `02_领域地图.md` 时，顺序应是：

```text
先理解张老师到底研究什么
再理解为什么需要阿秒、XUV、光电子谱和极端条件
再进入液体水 ICD 这个具体例子
再读液体水 ICD 的机制链和论文图
再区分机制论文和平台论文
```

不要在学习者还没学 ICD 前，就要求他判断机制论文和平台论文。

## 允许发散，但必须收回

学习过程中可以讨论旁支问题，例如：

- HHG 是否类似共振；
- 光电子谱是不是光电效应；
- 极端条件有没有现实意义；
- 液体水为什么比气相更难；
- ICD 为什么会释放低能电子。

但每个旁支都必须回答：

```text
这个问题帮助理解原文主线的哪一环？
如果它不服务本轮主线，是否应该先记入学习记录，而不是继续展开？
```

## 文件角色

| 目录 | 作用 |
|:---|:---|
| `00_入口/` | 当前学习路线和方向总览 |
| `01_原文/` | 五份正式材料快照，是学习主线来源 |
| `02_分讲/` | 面向学习者的主阅读讲义 |
| `03_学习记录/` | 用户回答、理解验收、校准和追溯 |
| `90_规范与历史/` | 学习区规范、迁移对照、历史入口 |

不要新增项目设计目录，不要把开发备忘写进本学习区。

## 学习记录要求

学习记录不是聊天流水账。每份记录至少保留：

```text
本轮原文：
本轮主线位置：
讨论中发散了什么：
发散怎样回到主线：
用户原始理解：
校准或补充：
本轮保留句：
下一步入口：
```

如果某段解释已经提取成分讲，要写清“已提取到哪里”。

如果本轮是一个新的学习段落，学习记录还要先承担“多轮对话安排”的作用。它不是项目计划，而是本轮教学控制台：规定本轮从主线表哪一行开始、分几轮问答、每轮要求用户说出什么、AI 只校准什么、哪些问题先暂存不展开。没有这个安排，不要直接进入长篇讲解。

## 禁止事项

不要在本目录内写：

- 项目开发计划；
- 文档标准设计；
- 模板、skill、verifier、harness 的修改建议；
- 对正式成品质量体系的回收规则；
- 和学习张老师专业方向无关的项目管理记录。

如果学习过程中出现这些想法，直接说明“这不属于张鹏举学习区”，不要写进本目录。
