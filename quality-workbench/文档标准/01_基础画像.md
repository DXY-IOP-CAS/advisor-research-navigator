# D01 基础画像标准：建立可回看的事实底座

状态：按 D00、M0 和张鹏举样本试写反补后的工作台标准。本文只规定成品 `01_基础画像.md` 怎样写；脚本、template、skill、verifier 和 harness 等正式回收另开任务。

`01_基础画像.md` 是五份导师材料中的事实底座。它回答三件事：这位老师是谁；资料和论文集合稳不稳；哪些事实、线索和风险会被 `02-04` 继续使用或反向复核。

它不负责讲完整研究方向，不负责解释论文路线，不负责做学习计划，也不做导师评价。学生第一遍读 `01`，只需要拿到可信身份、履历阶段、论文集合边界和风险意识；读完 `02-04` 后，再回到 `01` 检查后续理解有没有踩在不稳事实上。

成品 `01` 固定为八节：

```text
画像定位
  -> 资料与身份核查
  -> 学术履历
  -> 方向入口
  -> 论文集合
  -> 公开线索与风险
  -> 主体自检
  -> 来源与核查
```

这八节不是旧脚本字段的重新命名，而是一条事实使用路径：先告诉学生怎样读事实画像，再判断资料能不能合并阅读，再进入履历和论文集合，最后把事实底座收束成自检和来源核查。

<a id="d01-exit-card"></a>
## D01 文档出口卡

| 项 | 标准要求 |
|:---|:---|
| 学生入口 | 已经读过 `00`，知道五份材料的阅读顺序和来源符号，但还不知道这位老师的身份、履历、论文集合和风险是否站得住。 |
| 本文任务 | 建立后续 `02-04` 可以回看的事实底座：身份、机构、履历阶段、论文集合、公开线索、来源边界和人工复核点。 |
| 本文不做 | 不评价导师，不写个人选择判断，不把履历写成传记，不把方向入口写成领域综述，不把论文集合筛成代表作，不解释论文路线或学习路径。 |
| 本文出口 | 学生能说出这位老师是谁；资料为什么可以或不可以暂时合并阅读；履历和论文集合怎样按阶段读；哪些判断可暂用，哪些必须 `需人工复核`；后续读 `02-04` 时回哪里核查事实。 |
| 交给下一份 | `02_领域地图.md` 拿走可信身份、当前机构、履历阶段、论文集合边界、最低方向入口词和风险清单。 |
| 反向修正 | `02/03/04` 如果发现领域定位、论文路线或目标论文依赖了不稳事实，必须回修 `01` 的风险说明、论文归属或阶段解释。 |

成品 `01` 开头不要先放旧脚本运行信息、数据库流水、长免责声明或导师方向大总结。第一屏要让学生知道：这份文档是事实底座，第一遍只抓身份、阶段、论文集合和风险。

<a id="d01-input-stop"></a>
## 写作前输入与停写条件

成品 `01` 可以消费阶段一脚本、Fact Pack 或现有检索结果，但不能被旧脚本章节结构绑架。写作前，执行 AI 必须确认：

```text
官方身份入口是否存在；
中文名、英文名、当前机构和官网 URL 是否能互相支撑；
作者档案和论文集合来自哪些来源；
同名噪声、机构滞后、ORCID 差异、单源论文、题名截断等风险是否已经记录；
完整论文集合是否保留为事实审计资产；
正文短引用和来源表是否能支撑关键事实。
```

停写与降级条件：

```text
如果身份锁定不能同时说明“支持什么”和“仍不支持什么”，不能写稳定身份结论；只能写暂用身份、复核点和后续不可依赖点。
如果论文集合只是数据库合并结果，没有同名排除、弱线索和待复核记录，不能写成“全部论文已确认”；只能写候选集合和风险边界。
如果方向入口只能靠官网方向词或少量论文题名，不能写成当前研究主线；只能写成进入 `02` 的最低入口词，并说明证据弱。
如果引用数、h-index、人才项目、头衔或合作网络被写成价值判断，必须删掉或降级为资料口径说明。
如果完整论文表质量受脚本限制，成品可以加读法提示和风险边界，但不能手工删改事实表来制造整洁感。
```

## D01 最小落地产物

D01 标准必须让执行 AI 在写作前完成下面七类内部产物。缺任何一项，`01` 就会退回旧问题：事实很多，但学生不知道哪些能用、哪些要查。

| 槽位 | 执行者内部必须写清什么 | 成品里怎样出现 | 缺失时回哪里 |
|:---|:---|:---|:---|
| 文档出口卡 | 学生入口、本文任务、本文不做、读完输出、交给 `02` 的产物 | `画像定位` 的短说明，不出现“出口卡”字样 | 出口关 / S2 |
| 证据入口 | 官方身份、作者档案、论文元数据、公开线索、数据库指标各自能支撑什么、不能支撑什么 | `资料与身份核查` 和 `来源与核查` 的证据表 | 证据关 / S1 |
| 章节推进链 | 八节怎样从画像定位一路接到来源与核查 | 章节顺序和段落承接，不出现“推进链”字样 | 台阶关 / S2/S3 |
| 关系负担与支撑规格入口 | 身份链、资料状态、履历阶段、论文集合、公开线索和风险用什么表格、矩阵或证据卡承载 | 必要短表、证据边界卡或阶段导航 | 支撑关 / S4 |
| 段落任务要求 | 每节怎样承接、核查、限定、交接和让学生输出 | 每节开头和表后收束句 | 成文关 / S3 |
| 学生输出任务 | 学生合上 `01` 后能说出身份、资料状态、履历阶段、论文集合边界和风险回看位置 | `主体自检` 的检查表 | 输出关 / S2 |
| 失败回退表 | 成品 `01` 写偏时回哪一关、哪份支撑方法、补什么产物 | 只留在标准和审查中，不进入学生正文 | 输出与倒查关 / S1-S4 |

这些槽位是标准层要求，不是成品章节标题。成品保持八节结构；标准负责保证八节背后有可审查产物。学生正文不得出现“文档出口卡”“章节推进链”“失败回退表”“S1-S4”等工作台语言。

## 章节推进链

| 章节 | 上一节交给我什么 | 本节推进什么动作 | 默认支撑 | 学生输出 | 下一节拿走什么 | 失败回退 |
|:---|:---|:---|:---|:---|:---|:---|
| 画像定位 | `00` 给出的阅读顺序和证据符号 | 建立阅读契约：`01` 是事实底座，第一遍只抓事实轮廓和风险 | 短段，必要时一行读法链 | 说出 `01` 用来确认“我在看谁、资料稳不稳” | 资料与身份核查拿走读法和边界 | 写成传记或方向综述，回 S2/S3 |
| 资料与身份核查 | 学生知道要先确认事实底座 | 判断资料能否暂时合并到同一位老师名下阅读 | 资料状态表、身份证据链表 | 说出身份判断靠哪些来源，哪些仍需复核 | 学术履历拿走可信主体和资料边界 | 链接堆砌或只说“已核验”，回 S1 |
| 学术履历 | 已有可信主体和资料边界 | 把履历写成后续可用的阶段索引 | 履历阶段表 + 阶段读法短段 | 说出主要阶段、当前阶段和阶段线索 | 方向入口拿走当前阶段和方向线索 | 写成传记或成就叙事，回 S3 |
| 方向入口 | 学生知道履历阶段 | 给进入 `02` 前的最低方向词，不解释完整领域 | 入口词表或短卡 | 说出 1-5 个入口词、来源和证据边界 | 论文集合拿走阶段和方向词作为读法提示 | 抢 `02` 领域地图职责，回 S2/S3 |
| 论文集合 | 已有阶段索引和入口词 | 保留完整论文事实，教学生按阶段粗读论文集合 | 阶段读法 + 完整论文表 | 说出论文集合规模、阶段分布、当前阶段信号和风险 | 公开线索与风险拿走论文事实和待交叉线索 | 筛成代表作或讲论文路线，回 S1/S3 |
| 公开线索与风险 | 学生已经看过论文事实 | 用公开线索补强或暴露冲突 | 公开线索表、风险提示；线索不足时用缺口说明 | 区分补强事实、弱线索和冲突 | 主体自检拿走完整主体事实线 | 写成人脉/平台/实力判断，回 S1/S3 |
| 主体自检 | 主体事实线已走完 | 把身份、履历、论文集合和风险转成输出型检查 | 自检表 | 判断自己能否带事实底座进入 `02` | 来源与核查拿走需要细查的来源问题 | 自检像考试或无回退，回 S2 |
| 来源与核查 | 学生知道哪些事实要复核 | 给出完整来源入口和关键事实核查方法 | 按用途分组来源表；必要时风险核查表 | 能从关键判断跳到来源，并说出来源支撑什么、不支撑什么 | `02-04` 共享事实核查习惯 | 只列 URL 或 bibliography，回 S1/S3 |

审查 D01 时，先看这条推进链。若任一节删掉后前后仍能顺读，说明该节没有承担独立事实动作。

<a id="d01-support"></a>
## 关系负担与支撑形式

| 关系负担 | 默认支撑形式 | 升级或降级条件 | 禁止做法 |
|:---|:---|:---|:---|
| 资料状态和读法 | 资料状态表 | 风险类型多时拆成风险表 | 把脚本运行日志放进正文 |
| 身份是否能合并阅读 | 身份证据链表 | 冲突影响后续文档时，用证据边界卡 | 只写“已核验”，不说明依据和边界 |
| 履历阶段 | 履历阶段表 + 每阶段读法 | 阶段多且方向转折明显时，可用短时间线 | 写成传记、成就叙事或评价 |
| 方向入口词 | 入口词表或短卡 | 词之间易混时，交给 `02` 做相邻方向比较 | 在 `01` 里画领域地图 |
| 完整论文集合 | 阶段读法 + 完整论文表 | 表太长时加阶段导航、字段瘦身或附录区，不删论文事实 | 筛成代表作或在 `01` 判论文角色 |
| 公开线索和风险 | 公开线索表、风险提示 | 线索不足时写缺口说明，不强行凑表 | 写成合作网络价值判断 |
| 主体自检 | 自检表 | 无需升级图 | 只问“是否理解”，不给回看位置 |
| 来源核查 | 按用途分组来源表 | 风险复杂时加“判断 / 来源 / 不支撑什么 / 处理”表 | 只列 URL，或保留未被正文消费的来源 |

每个表格必须被正文消费。表前要说明学生正在查什么关系，表后要拿走表里的结果继续推进。表格删掉后如果正文理解不受影响，说明它只是格式装饰。

### 关键表格骨架

资料状态表：

| 项目 | 当前状态 | 读者怎么看 |
|:---|:---|:---|
| 当前身份 | <中文名>（<英文名或 [未找到]>），<当前机构 / 部门> | 先确认后文讨论的是同一位老师 |
| 官方入口 | <官网 URL> | 身份、职务、履历和公开方向优先回官网核查 |
| 核查时间 | <YYYY-MM-DD 或本轮生成日期> | 公开信息会变化，后续重要使用前要重新检索 |
| 论文集合 | <论文数> 篇，来自 <GS/OA/arXiv/DOI 等> | 第一遍只看规模、阶段分布和风险 |
| 数据源覆盖 | <哪些源可用，哪些源缺失> | 知道哪些判断较稳，哪些要复核 |
| 主要风险 | <同名噪声 / ORCID 差异 / 单源论文 / 题名截断等> | 后文遇到相关判断时保留疑问 |

身份证据链表：

| 核查点 | 主要来源 | 支撑什么 | 不支撑什么 | 风险处理 |
|:---|:---|:---|:---|:---|
| 官方身份 | 官网、机构页、实验室页、UCAS / 学院页 | 当前机构、职位、姓名写法 | 不单独证明全部论文归属 | 来源缺失写 `[未找到]` |
| 联系方式与 ID | 邮箱域名、ORCID、个人主页链接 | 官网身份和作者档案是否能连上 | 不证明每篇论文都属于本人 | 不匹配时标 `需人工复核` |
| 作者档案 | Google Scholar、OpenAlex、ORCID | 论文集合候选和同名消歧线索 | 数据库不能单独锁定身份 | 消歧可疑时保守使用 |
| 论文指纹 | 题名主题、合作者、机构署名、年份连续性 | 单篇论文是否应归入本文论文集合 | 单个线索不能证明归属 | 单源或弱匹配论文不承担主线 |
| 同名排除 | 明显不同机构、方向、合作者或作者页 | 哪些资料不能混进来 | 不证明剩余资料全都稳 | 记录排除理由 |

来源与核查表：

| 编号 | 类型 | 资料名称 | 支撑内容 | 不支撑什么 | 链接 |
|:---|:---|:---|:---|:---|:---|
| <a id="o1"></a>[O1] | Official / 官网 | <资料名称> | <支撑身份、机构、履历或官方方向> | <不能证明完整论文路线或学习路径> | <URL> |
| <a id="p1"></a>[P1] | Paper / 论文 | <论文题名> | <支撑论文题名、作者、发表信息、研究对象> | <不能单独证明导师整体方向> | <DOI / 期刊页> |
| <a id="o2"></a>[O2] | Database / 作者档案 | <GS / OpenAlex / ORCID> | <支撑作者档案和论文候选> | <不能单独消除同名风险> | <URL> |

当前 D01 成品不新增 `[D#]`。数据库和作者档案暂时沿用 D00 的 `[O#] [P#] [R#] [B#]` 符号系统：正文短引用可用 `[O#]` 承接，来源表的 `类型` 列写明 `Database / 作者档案`。是否引入独立 `[D#]`，等样本试写、D00 反修和 verifier 回收时统一决定。

<a id="section-design"></a>
## 小节设计

每节按同一组问题审查：

```text
本节出口：
上一节交给：
本节动作：
证据入口：
关系负担与支撑：
正文推进：
学生输出：
下一节拿走：
失败回退：
容易写偏：
```

### 画像定位

本节出口：学生知道 `01` 是事实底座，第一遍只抓身份、履历阶段、论文集合和风险；读完 `02-04` 后要回到这里核查事实。正文用短段承接 `00`，写本文任务、第一遍读法、回看读法和使用边界。容易写偏：传记开头、方向综述、长免责声明、内部 pipeline 术语。

### 资料与身份核查

本节出口：学生知道这批资料能否暂时合并到同一位老师名下阅读，也知道哪些风险不能当成已解决。先给资料状态，再给身份证据链，最后写后文使用边界。官方身份锁定和论文归属消歧是两步；不能用 Google Scholar、OpenAlex、ORCID 等作者档案反推当前身份。

### 学术履历

本节出口：学生能按阶段理解老师经历，而不是记一串年份；后续能用阶段索引阅读论文集合和方向入口。履历只做阶段索引和阶段读法，不写成传记、成就叙事或当前主线解释。

### 方向入口

本节出口：学生带着 1-5 个最低方向词进入 `02`，并知道这些词只是入口，不是完整领域地图。证据集中或不足时，宁可只给 1-2 个高置信入口词，也不要为了数量凑弱词。每个词必须有一句新手解释、来源依据和证据边界。

### 论文集合

本节出口：学生知道论文集合覆盖范围、阶段分布、当前阶段信号和主要风险；后续可以用它复核 `02/03/04`，但不把它误读成代表作列表。

完整论文表不是数据库原样搬运。进入学生正文前必须先定字段和边界：

- 必要字段：年份、题名、作者位置或作者名单摘要、来源类型、发表位置或 DOI、阶段、风险标记。
- 排序：优先按履历阶段分组，再按年份倒序或正序；同一阶段内不要按引用数排序。
- 记录状态：来源弱、题名截断、同名风险或归属不稳的论文标为 `候选 / 需人工复核`，不承担方向主线判断。
- 长表处理：正文先给阶段导航和读法；表格过长时可放在本节末尾的“完整论文表”区或折叠区，但不能删成代表作清单。
- 字段瘦身：可以删数据库技术字段、重复 ID 和内部脚本字段；不能删题名、年份、来源、风险标记这些事实核查字段。

### 公开线索与风险

本节出口：学生知道论文之外的公开线索怎样补强事实，哪些只是弱线索或冲突，不能单独证明当前主线。公开线索稀疏时，本节仍保留标题，但降级为短说明，不强行凑表。最小写法是：说明已检查哪些公开入口、没有找到可支撑正文判断的线索、后续哪些判断不能依赖公开线索。

### 主体自检

本节出口：学生能判断自己是否抓住 `01` 的事实底座，答不上时知道回哪一节。本节不新增事实来源，只消费前文判断和风险。自检只检查身份来源、阶段、论文集合读法、公开线索和风险，不检查专业理解深度。

### 来源与核查

本节出口：学生能从关键事实回到来源条目，说明来源类型、支撑内容、不支撑内容和风险处理。只列正文真正使用的来源和会影响身份、履历、论文集合、公开线索的关键资料。缺失来源写 `[未找到]`，弱判断写 `需人工复核`。

<a id="confirmed-rules"></a>
## 已确认规则

- `01` 是事实底座，不写成传记、领域综述、论文路线、学习路线或导师评价。
- 文首必须有目录和显式 anchor。
- 成品章节固定为 `画像定位 / 资料与身份核查 / 学术履历 / 方向入口 / 论文集合 / 公开线索与风险 / 主体自检 / 来源与核查`。
- `01` 支持两次阅读：第一遍粗读事实轮廓，读完 `02-04` 后回看核查事实和风险。
- `资料与身份核查` 必须写出资料状态和身份证据链，不能只写“已核验”。
- 身份结论必须说明来源支持什么、不支持什么。
- “论文指纹”必须解释为题名主题、合作者、机构署名、年份连续性等可核对线索；不能用单个线索锁定论文归属。
- `学术履历` 只做阶段索引和阶段读法，不写传记和成就评价。
- `方向入口` 只给进入 `02` 的最低方向词，不解释完整领域，不判断当前主线。
- 方向入口词不强行凑数；证据集中时可以只有 1-2 个高置信入口词，并说明为什么不扩展。
- 完整论文集合是事实审计资产，不能筛成代表作；长表前必须有阶段读法提示，长表可用阶段导航、字段瘦身、风险标记和附录 / 折叠区降低阅读负担。
- `公开线索与风险` 只补强事实或暴露冲突，不能写成人脉、平台、实力或个人选择判断。
- 公开线索缺失时写缺口和复核入口，不强行凑公开关系分析。
- 引用数、h-index、人才项目、头衔和项目经费不得写成导师评价；如需保留，只写来源口径和事实边界。
- `主体自检` 只检查 `01` 的事实底座，不提前要求学生理解领域、论文路线或学习路径。
- `来源与核查` 的来源表必须说明“支撑内容”和“不支撑什么”，这是 D01 相对 D00 的增强列；是否回收进 D00，等样本验证后统一处理。
- 当前 D01 不新增 `[D#]`；数据库和作者档案使用 D00 现有来源符号，在来源表 `类型` 列标明 `Database / 作者档案`。
- 第一次出现的机构缩写、领域缩写和专业术语必须展开并给短解释。
- 复杂 Mermaid 默认不用；`01` 优先使用短表、证据边界卡和必要的短时间线。

## 成品试写时验证

先验证 D01 标准自身：

- 七个最小落地产物是否都能在标准中找到。
- 八节是否都有出口、承接、动作、支撑、输出、交接和失败回退。
- 身份、论文集合、公开线索和来源边界是否被分开，不再混成“快速阅读结论”。
- 学生输出是否可检查，不使用抽象的“了解 / 掌握”。
- 失败回退是否指向 S1-S4 和必补产物。
- 没有把 `02/03/04` 的研究内容、论文路线和学习准备提前搬进 `01`。
