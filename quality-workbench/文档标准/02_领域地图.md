## 文档职责

`02_领域地图.md` 是专业范围图。它回答四个问题：导师当前方向属于哪个大领域；这个大领域内部有哪些主要板块；导师方向在其中大概处于什么范围和位置；学生进入论文路线前，需要先抓住哪些概念和相邻方向。

它承接 `01_基础画像.md`，但不复述履历和论文长表。`01` 告诉学生“老师是谁、资料是否可靠、论文集合是什么”；`02` 把这些事实放进学科地图，让学生从“我看到一堆方向词”走到“我知道这些词在专业里大概放在哪里”。

`02` 也不承担论文路线分析。它可以提到近期论文或目标论文群，但只用来定位问题域、研究对象、方法平台或相邻方向。论文之间怎样推进研究问题，留给 `03`。

读完 `02` 后，学生应能用自己的话说清：

- 这位老师当前方向属于哪个大领域、子领域和具体问题域。
- 这个方向在大领域里大概是主流大方向、稳定子领域、小问题域，还是前沿交叉点。
- 旁边有哪些相邻方向，哪些和导师强相关，哪些只是背景。
- 后续进入 `03` 时，应该带着哪几个领域问题去看论文。
- 哪些定位判断有强来源支撑，哪些只是初步范围感，需要继续核查。

<a id="general-principles"></a>
## 通用设计原则

### 先给地形，再讲路线

`02` 的读者刚读完 `01`，手里有导师身份、履历、论文集合和少量方向词。此时最缺的是专业地形感：这个词属于哪片大专业，旁边有哪些方向，它是大方向、子方向还是具体问题。

正文应先让学生看到大图，再逐步收紧到导师位置。合适的顺序是：

```text
01 的方向线索 -> 领域全景 -> 范围定位 -> 概念入口 -> 03 的论文问题
```

如果一开始就解释术语，学生容易把每个词当成孤立词条。领域全景图的作用是把这些词放回同一张地图。

### 比例感可以粗，证据边界必须清楚

学生需要知道某个方向在学科里大概有多大。这里的“大”和“小”优先表达认知范围，不追求精确统计。

默认使用这组范围标签：

| 标签 | 含义 | 适合来源 |
|:---|:---|:---|
| 主流大方向 | 学科中长期稳定存在的大板块 | 学科分类、院系方向、课程体系、综述 |
| 稳定子领域 | 大方向下经常出现的研究分支 | 综述、课程、期刊专题、会议分会 |
| 具体问题域 | 论文层面反复出现的研究问题 | 近期论文、综述小节、课题组方向 |
| 前沿交叉点 | 多个方向或方法交汇形成的活跃问题 | 综述、近期论文、会议专题、项目介绍 |
| 工具 / 平台 | 支撑研究的实验、计算、数据或理论方法 | 方法论文、平台介绍、课程或教程 |

不要写无来源百分比。只有在检索式、学科统计、会议分类或数据库结果可复核时，才使用数字。没有数字证据时，可以用“大 / 中 / 小”“主流 / 稳定 / 小众 / 新兴”表达范围感，并在来源区说明判断依据。

### 主图解决主理解任务

每份 `02` 应有一张核心领域范围图。它的任务是让学生一眼看到：

- 上位学科和主要分支。
- 导师相关区域。
- 相邻方向和背景区域。
- 粗略范围大小。
- 强相关、弱相关和背景的区别。

主图可以是 SVG，也可以是 Mermaid `treemap`、`venn` 或其他能被当前导出环境稳定渲染的图。普通表格、文字链条和长流程图不能替代主图。

### 图下必须有读法

图形不能让学生自己猜。每个图旁边都要用短段说明：

- 这张图回答哪个读者问题。
- 面积、颜色、边框、箭头或距离分别代表什么。
- 哪些判断有来源支撑，哪些只是帮助理解的视觉权重。
- 学生看完后应能复述哪一句定位判断。

如果删掉图后正文几乎没有损失，这张图就没有完成任务。

### 概念解释只服务下一步阅读

`02` 的概念入口只解释到“能进入 `03`”的程度。每个概念优先回答三件事：

- 先这样理解：给新手一句话。
- 为什么重要：它连接导师位置或论文问题。
- 后续去哪看：在 `03` 的哪类论文、图或问题中会再次出现。

公式推导、课程先修和学习顺序留给 `04_学习向导.md`。

### 时间线按需使用

有些领域必须看时间变化才能理解当前位置，例如新实验平台出现、计算方法成熟、数据集开放、材料体系扩展或政策议题改变。遇到这种情况，可以加入短时间线。

时间线讲领域发展，不讲导师履历。导师个人论文怎样组成路线，留给 `03`。

<a id="visual-toolkit"></a>
## 图形工具箱

`02` 的图形选择应先问“学生需要看见什么关系”，再选形式。Markdown 可以承载 Mermaid，也可以引用 SVG。SVG 更适合画自由布局的范围地图，Mermaid 更适合批量生成和轻量维护。两者可以并存：主图用 SVG，辅助图用 Mermaid 或表格。

### 领域范围图

用途：建立大专业、主要分支、导师区域和相邻方向的空间感。

推荐形式：SVG 气泡图、嵌套圆图、手绘感地图、treemap。优先用于 `领域全景`。

适合表达：

- 哪些方向是大板块，哪些是小问题域。
- 导师相关方向在图中位置。
- 哪些相邻领域容易在论文里出现。
- 哪些区域只是背景。

批量化输入建议：

| 字段 | 含义 | 示例 |
|:---|:---|:---|
| `name` | 节点名 | 凝聚态物理 |
| `level` | 层级 | 大学科 / 一级方向 / 子领域 / 问题域 |
| `scale` | 范围感 | 大 / 中 / 小 / 交叉点 |
| `relation` | 与导师关系 | 当前主线 / 强相关 / 相邻 / 背景 |
| `evidence` | 来源依据 | 学科分类、综述、课程、论文、官网 |
| `note` | 图下解释 | 为什么这样放 |

面积和大小可以表达视觉权重，但必须在图例写清：视觉权重用于帮助理解，不代表精确统计；有数字证据时另行标注来源。

### 范围套层图

用途：把导师方向从大学科逐层收紧到具体问题域。

推荐形式：SVG 嵌套块、Mermaid `block`、简洁层级树。优先用于 `范围定位`。

套层图不能只写“甲 -> 乙 -> 丙”。每一层旁边要有范围标签和证据提示：

```text
物理学                  大学科
└─ 凝聚态物理            主流大方向，强相关
   └─ 量子材料            稳定子领域，强相关
      └─ 超快动力学        具体问题域，当前主线
         └─ 导师切入点      由近期论文和官网方向支撑
```

### 方向重心图

用途：区分当前主线、强相关方向、相邻背景和弱线索。

推荐形式：二维气泡图、Mermaid `quadrantChart`、定位矩阵。适合用于 `范围定位` 或 `导师相关方向说明`。

坐标轴必须有学科含义，例如：

- 横轴：与导师当前工作的相关度。
- 纵轴：领域成熟度或主流程度。

如果坐标轴只是装饰，应改用表格。

### 交叉关系图

用途：表达两个或多个领域真实交汇时形成的问题域。

推荐形式：Mermaid `venn` 或 SVG 重叠圆。只在确实存在交叉时使用。

常见场景：

- 一个方向同时依赖材料体系和测量平台。
- 一个问题域由物理、化学、计算或工程方法共同支撑。
- 文科社科中，一个议题处在方法流派和研究对象的交叉处。

交叉图不能拿来表达普通层级关系。包含关系用范围图或套层图更清楚。

### 发展脉络图

用途：解释领域为什么走到当前问题。

推荐形式：Mermaid `timeline`、竖向时间轴、小型阶段图。只在时间变化能降低理解难度时使用。

时间轴应回答：

- 哪个对象、平台、方法或问题推动了方向变化。
- 当前前沿为什么会出现。
- 学生后面看论文时为什么会反复遇到这些词。

### 概念关系图

用途：解释 3-7 个核心概念之间的依赖、并列、支撑或对象-方法关系。

推荐形式：小型 Mermaid `mindmap`、概念图、二列表。适合用于 `概念入口`。

概念图不能替代领域范围图。它只负责让学生知道少数核心词怎样互相连接。

## 小节设计

本节先放每个小节要解决的问题。后续逐节讨论时，再补正文走法、图表样式、读完检查和容易写偏的地方。

| 小节 | 要解决的问题 | 推荐图文形态 | 容易写偏的地方 |
|:---|:---|:---|:---|
| 地图定位 | 学生为什么需要专业范围感，第一遍读到什么程度即可继续？ | 短开头 + 读者出口能力 | 写成空泛导语，或提前讲太多论文细节 |
| 方向线索 | 这张地图从 `01` 的哪些事实出发？ | 线索短表，列官网方向、当前单位、近期论文关键词 | 变成身份核查或完整论文清单 |
| 领域全景 | 这个大专业有哪些主要板块，导师方向在哪一片？ | SVG 领域范围图；可用 treemap / 气泡图备选 | 只画层级链条，或编造精确比例 |
| 范围定位 | 导师方向从大领域怎样收紧到具体问题域？ | 范围套层图 + 方向重心表 | 把弱线索写成强定位；只写名词不解释范围 |
| 概念入口 | 进入 `03` 前最少要认识哪些概念？ | 4-7 个概念卡片或小型概念图 | 写成百科、课程大纲或术语堆 |
| 论文入口 | `03` 应从哪些领域问题进入论文路线？ | 2-4 个问题入口，连接目标论文群或核心图 | 提前写论文路线，和 `03` 重复 |
| 主体自检 | 学生怎样判断自己可以进入 `03`？ | 自检短表：能否复述全景、定位、概念、证据边界 | 检查题太空，学生答完仍不知道缺什么 |
| 来源与参考资料 | 领域判断和概念解释从哪里来？ | 按来源类型分组；每条说明支撑内容和限制 | 链接堆积；单一来源支撑过大结论 |

### 批量化生成时的最低输入

为了让不同导师能复刻同一质量底线，`02` 写作前应先准备一份轻量字段表。它不需要成为正式成品，但要约束 AI 不乱画图。

| 字段 | 作用 | 缺失时的处理 |
|:---|:---|:---|
| 导师当前方向一句话 | 决定地图中心 | 回到 `01` 和 fresh search 补证据 |
| 上位学科和主要分支 | 生成领域全景 | 查学科分类、院系方向、课程和综述 |
| 导师相关子领域 | 标出高亮区域 | 用近期论文、官网方向、综述交叉确认 |
| 相邻方向 | 防止学生误以为只有一条线 | 从综述、课程目录、会议专题提取 |
| 范围标签 | 给出大 / 中 / 小 / 交叉点的直觉 | 没证据时写“初步范围感，需人工复核” |
| 关键概念 | 连接到 `03` 的论文问题 | 控制在 4-7 个 |
| 时间脉络 | 判断是否需要发展图 | 不影响理解时省略 |
| 图源依据 | 支撑图形布局和高亮 | 写入来源区或证据核对表 |

<a id="open-questions"></a>
## 待确认问题

- `领域范围图` 的默认实现选 SVG 还是 Mermaid `treemap`。SVG 表现力强，Mermaid 更容易批量生成；真实导出环境稳定性需要试跑。
- 范围大小用“大 / 中 / 小 / 交叉点”是否足够，还是需要增加“主流 / 稳定 / 新兴 / 小众”等成熟度标签。
- `方向线索` 是否会和 `01_基础画像.md` 里的方向入口重复。当前判断是可以保留，但必须只摘线索，不展开身份和履历。
- `概念入口` 控制在 4-7 个词是否适合所有学科。理工科通常可行，文科社科可能要按“对象 / 问题 / 方法 / 理论传统”组织。
- 时间线是否作为条件小节出现。当前建议不放入固定大纲，只在 `领域全景` 或 `范围定位` 中按需加入。
