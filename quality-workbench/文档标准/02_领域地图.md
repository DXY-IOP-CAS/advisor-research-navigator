<a id="workbench-role"></a>
## 文档职责

`02_领域地图.md` 是领域坐标。它回答三个问题：导师当前方向属于哪个大领域；这个领域内部怎样从大问题走到小问题；导师当前工作大致落在哪个位置。

它承接 `01_基础画像.md`，但不复述履历和论文长表。`01` 已经告诉学生“老师是谁、资料是否可靠、论文集合是什么”；`02` 只拿这些事实做定位，把学生从“我看到一些方向词”带到“我知道这些词在领域里大概放在哪里”。

`02` 不承担论文路线分析。它可以提到近期论文或目标论文群，但只用来定位问题域、对象、方法或平台；论文之间怎样推进研究问题，留给 `03`。

读完 `02` 后，学生应能用自己的话说清：

- 这位老师当前方向属于哪个大领域和小问题域。
- 这个问题域中最少需要先认识哪些对象、方法、平台或物理量。
- 老师当前位置和 `01` 中的当前阶段、近期论文、公开方向之间怎样互相支撑。
- 进入 `03` 时，应该带着哪几个领域问题去看论文。

| 地图定位 | 我为什么要先看领域地图，而不是直接读论文？ | 承接 `01` 的方向入口词；说明本节只建立领域坐标；告诉学生先看层级、概念和导师位置，不急着追每篇论文细节 | 知道 `02` 是进入论文前的坐标图 | 不写领域综述，不复述 `01`，不提前分析论文路线 |

| 导师位置 | 这位老师在地图中的具体位置在哪里？ | 把 `01` 的当前阶段、官方方向、近期论文和公开线索放回领域层级；用定位矩阵说明证据强弱 | 知道导师当前位置是由哪些事实支撑的 | 不做导师评价，不把弱线索写成强定位 |
| 论文入口 | 后面读论文时，应带着哪些领域问题进去？ | 从领域位置提取 2-4 个进入 `03` 的问题；说明这些问题怎样对应论文群、目标论文或核心图 | 知道 `03` 要看什么问题链 | 只交接问题，不解释论文之间的完整路线 |
| 读后自检 | 我怎么判断自己真的有了领域坐标？ | 用短表检查领域层级、核心概念、导师位置、论文入口和证据意识；答不上则回到对应小节 | 判断自己能不能进入 `03` | 不要求学生已经懂论文细节或完整理论 |
| 来源与参考资料 | 领域层级、概念和导师位置分别由哪些来源支撑？ | 按领域综述 / 教材课程 / 论文 / 官方资料分组；每条说明支撑什么、不能证明什么 | 能沿来源复核领域定位和概念解释 | 不只堆链接，不用单个来源支撑过大结论 |

<a id="general-principles"></a>
## 通用设计原则

### 先定位，再展开

`02` 的读者刚读完 `01`，手里只有身份、履历、论文集合和少量方向入口词。正文应先回答“这些词放在哪里”，再解释“它们大概是什么意思”。如果一开始就讲概念定义，学生很容易把术语当成孤立词条。

合适的顺序是：

```text
01 的方向入口词 -> 领域层级 -> 核心概念 -> 导师位置 -> 03 的论文问题
```

### 分层要有依据

领域分层不能凭感觉切。每一层至少要来自一种可说明的依据：

- 官方方向或机构页面。
- 近期论文题名、摘要、关键词或图示。
- 领域综述、教材、课程或权威背景资料。
- 学术履历中的当前阶段和研究环境。

分层证据弱时，正文要写成“可作为初步定位”或标 `需人工复核`。不要把一个论文题名里的词直接写成完整领域结构。

### 图表只承担一个理解动作

`02` 可以有图，但图必须让学生留下领域位置判断。优先级建议是：

1. 层级表：适合说明从大领域到小问题域。
2. 定位矩阵：适合说明导师位置由哪些事实支撑。
3. 小型概念图：适合说明 3-6 个概念之间的关系。
4. Mermaid：只在确实存在流程、平台链路或因果路径时使用。

不建议使用抽象象限图，除非两个坐标轴都有明确学科含义、来源依据和读法说明。没有边标、没有证据、没有读后判断的图，会让学生看起来更迷糊。

### 概念解释要短

核心概念只解释到“能继续读 `03`”的程度。每个概念优先回答三件事：

- 先这样理解：给新手一句话。
- 为什么重要：它连接导师位置或论文问题。
- 后续去哪看：在 `03` 的哪类论文、图或问题中会再次出现。

概念解释不要扩成百科条目。更深的公式、推导、课程学习顺序留给 `04_学习向导.md`。

### 证据边界要贴在定位旁边

领域定位最容易出现顺滑幻觉。正文每次从事实推出领域位置时，都要让读者看见证据强弱：

- 官网或官方方向能支撑“官方表述”。
- 近期论文能支撑“当前问题域线索”。
- 综述、教材和课程能支撑“领域共识背景”。
- 单篇论文、讲座标题、新闻稿只能作弱线索。

<a id="section-design"></a>
## 小节设计

本节先放每个小节要解决的问题。后续逐节讨论时，再补正文走法、图表用法、读完检查和容易写偏的地方。

| 小节 | 要解决的问题 | 目前先定的写法方向 |
|:---|:---|:---|
| 地图定位 | 学生为什么需要领域坐标，第一遍读到什么程度即可继续？ | 用短开头承接 `01`，说明 `02` 不求读懂全部背景，只先建立坐标 |
| 领域层级 | 大领域、二级方向、小问题域和导师切入点怎样分清？ | 用层级表或小层级图；每层都写“为什么这样分” |
| 核心概念 | 哪些概念是进入 `03` 前必须先挂住的？ | 选少量概念，按对象、方法、平台、物理量或问题类型分组 |
| 导师位置 | 导师当前位置怎样由 `01` 的事实和外部领域资料共同支撑？ | 用定位矩阵，区分直接证据、交叉证据和弱线索 |
| 论文入口 | `03` 应从哪些领域问题进入论文路线？ | 提取少量问题，不提前解释论文链 |
| 读后自检 | 学生怎样判断自己可以进入 `03`？ | 用自检表，答不上时回到对应小节 |
| 来源与参考资料 | 领域判断和概念解释从哪里来？ | 按来源类型分组，写清支撑内容和限制 |

<a id="open-questions"></a>
## 待确认问题

- `地图定位` 是否需要单独成节，还是并入 `领域层级` 的开头。
- `领域层级` 的最终标题是否过抽象；可能替代为“方向层级”或“问题层级”。
- `核心概念` 应控制在几个词以内，才不会变成小百科。
- `导师位置` 是否需要一个固定定位矩阵；如果导师资料很少，矩阵会不会显得机械。
- `论文入口` 是否足够必要；它的价值在于把 `02` 自然接到 `03`，但不能重复 `03` 的论文路线。
- `02` 的主图应优先用层级表、定位矩阵还是小型概念图，需要用张鹏举样本试写判断。
