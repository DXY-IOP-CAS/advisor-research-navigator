# D02 领域地图标准：建立专业范围感和方向坐标

状态：按 D00、D01、M0 和 S1-S4 反补后的重构稿。本文只规定成品 `02_领域地图.md` 怎样写；真实样本、template、skill、verifier 和 harness 等正式回收另开任务。

`02_领域地图.md` 是五份导师材料中的专业坐标层。它回答三件事：这位老师的当前方向应先放在哪片大领域里读；这片领域内部有哪些必须分清的层级、相邻方向和问题域；学生进入 `03_论文路线.md` 前，需要带走哪些概念、范围判断和论文阅读问题。

它不负责重复 `01` 的身份、履历和论文长表，不负责写完整领域综述，不负责判定论文路线，不负责做学习计划，也不做导师评价。学生读完 `02`，应从“我看到一些方向词和论文题名”走到“我知道这些词大概处在什么专业范围里，哪些只是背景，哪些会影响后面读论文”。

成品 `02` 固定为八节：

```text
地图定位
  -> 方向线索
  -> 领域全景
  -> 范围定位
  -> 概念入口
  -> 论文入口
  -> 主体自检
  -> 来源与核查
```

这八节不是八块百科条目，而是一条专业坐标建立路径：先说明为什么要看地图，再交代地图从哪些事实和资料出发，随后给出大领域和相邻方向，逐步收紧到导师当前位置，最后把少量概念和问题交给 `03`。

## D02 文档出口卡

| 项 | 标准要求 |
|:---|:---|
| 学生入口 | 已经读过 `00` 和 `01`，知道导师身份、当前机构、履历阶段、论文集合边界和最低方向入口词，但还缺专业范围感。 |
| 本文任务 | 把 `01` 给出的事实线索放进可核查的领域坐标：大领域、子领域、问题域、相邻方向、入口概念和交给 `03` 的论文阅读问题。 |
| 本文不做 | 不评价导师，不复述 `01` 的身份和履历，不写百科式领域综述，不提前分析论文路线，不设计学习计划，不用论文关键词或官网方向词直接替代领域判断。 |
| 本文出口 | 学生能用 2-4 句话说出导师方向大致处在哪片领域；能区分大领域、子领域、问题域、方法入口和相邻方向；能说明哪些判断证据较稳、哪些只是线索；能带着 2-4 个领域问题进入 `03`。 |
| 交给下一份 | `03_论文路线.md` 拿走领域位置、相邻方向边界、4-7 个入口概念、2-4 个论文阅读问题、待复核的领域假设。 |
| 反向修正 | `03` 如果发现论文内容不支持 `02` 的定位，必须回修 `02` 的范围定位、相邻方向边界、概念入口和来源核查。 |

成品 `02` 开头不要写成“大领域介绍”或“导师方向总结”。第一屏要让学生知道：这份文档是帮助他建立专业坐标，不是替他读完领域综述。

## 写作前输入与停写条件

成品 `02` 必须从 `01` 的事实底座出发，但不能只靠 `01` 写出来。写作前，执行 AI 必须确认：

```text
`01` 是否已经给出可信身份、当前机构、履历阶段、论文集合边界、最低方向入口词和风险清单；
导师官网、机构页、课题组页或官方方向描述能支撑哪些入口词；
近期论文、代表论文或论文集合能支撑哪些研究对象、方法入口和问题域；
是否查到综述、课程资料、教材章节、学会 / 期刊 / 会议分类、领域背景页等领域资料；
相邻方向和容易混用的术语是否至少有一类来源支撑；
每个领域定位判断能说清“来源支持什么、不支持什么”；
`_internal/blueprint.md` 是否已经更新或准备更新 D02 字段：导师当前方向一句话、领域定位假设、相邻方向边界、入口概念候选、交给 D03 的论文问题、可视化计划和关键证据风险；
正文短引用和来源表是否能支撑关键领域判断。
```

停写与降级条件：

```text
如果 `01` 身份或论文集合边界不稳，`02` 只能写候选领域线索，不能写稳定领域定位。
如果只有官网方向词、论文题名关键词或数据库标签，不能写领域地图；先补综述、课程、学科分类或领域背景资料。
如果找不到相邻方向，不能把导师方向写成孤立单线；先补领域资料，至少说明已查入口和缺口。
如果大领域、子领域、问题域、方法入口混在一起分不清，不能直接画范围图；先用层级表和证据边界拆开。
如果视觉图会暗示证据没有支撑的比例、中心性、距离、上下位或因果关系，必须降级为文字、表格或证据边界卡。
如果已经开始逐篇解释论文怎样组成路线，必须退回 D03 边界；`02` 只交接论文阅读问题。
```

| 槽位 | 执行者内部必须写清什么 | 成品里怎样出现 | 缺失时回哪里 |
|:---|:---|:---|:---|
| 文档出口卡 | 学生入口、本文任务、本文不做、读完输出、交给 `03` 的产物 | `地图定位` 的短说明，不出现“出口卡”字样 | 出口关 / S2 |
| 领域资料资格卡 | 官网方向、论文资料、综述 / 课程 / 背景资料、相邻方向资料分别能支撑什么、不能支撑什么 | `方向线索` 和 `来源与核查` 的证据表 | 证据关 / S1 |
| 章节推进链 | 八节怎样从地图定位一路接到来源与核查 | 章节顺序和段落承接，不出现“推进链”字样 | 台阶关 / S2/S3 |
| 关系负担与支撑规格入口 | 大领域、子领域、问题域、相邻方向、概念关系和论文问题用什么表格、范围图、层级图或比较图承载 | 必要短表、领域范围图、边界表、概念卡或问题交接表 | 支撑关 / S4 |
| 领域层级判断 | 大领域、稳定子领域、具体问题域、方法 / 平台、相邻方向和背景区域怎样区分 | `领域全景` 和 `范围定位` 的分层说明 | 证据关 / S1；支撑关 / S4 |
| 概念入口 | 进入 `03` 前最少需要的 4-7 个概念、每个概念的短解释、用途和后文出现位置 | `概念入口` 的概念卡或小表 | 输出关 / S2；成文关 / S3 |
| 论文问题交接 | 2-4 个带到 `03` 的领域问题，以及这些问题来自哪层范围定位 | `论文入口` 的问题交接表 | 台阶关 / S2/S3 |
| 蓝图与关键判断交接 | 哪些定位假设、相邻边界、概念候选、可视化计划、证据风险进入 `_internal/blueprint.md` 或关键判断核查表；哪些只保留为 `需人工复核` | 不进入学生正文；通过 `方向线索`、`范围定位`、`论文入口` 和 `来源与核查` 被正文消费 | 证据关 / S1；台阶关 / S2/S3 |
| 失败回退表 | 成品 `02` 写偏时回哪一关、哪份支撑方法、补什么产物 | 只留在标准和审查中，不进入学生正文 | 输出与倒查关 / S1-S4 |

这些槽位是标准层要求，不是成品章节标题。成品保持八节结构；标准负责保证八节背后有可审查产物。学生正文不得出现“文档出口卡”“章节推进链”“S1-S4”“资料资格卡”等工作台语言。

## 章节推进链

| 章节 | 上一节交给我什么 | 本节推进什么动作 | 默认支撑 | 学生输出 | 下一节拿走什么 | 失败回退 |
|:---|:---|:---|:---|:---|:---|:---|
| 地图定位 | `00/01` 给出的阅读顺序和事实底座 | 建立阅读契约：`02` 用来建立专业坐标，第一遍只抓范围、边界和问题 | 短段，必要时一行读法链 | 说出 `02` 不是领域百科，而是进入论文路线前的地图 | 方向线索拿走读法和边界 | 写成领域导论或导师总结，回 S2/S3 |
| 方向线索 | 学生知道要先从事实线索出发 | 摘出 `01`、官方方向、近期论文和领域资料中的定位线索，并说明证据边界 | 线索表、来源用途表 | 说出这张地图从哪些线索出发，哪些还只是候选 | 领域全景拿走可用线索和风险 | 只堆关键词或重写 `01`，回 S1/S3 |
| 领域全景 | 已有线索和资料资格边界 | 建立大领域、主要分支、相邻方向和背景区域的粗地图 | 分层正文 + 领域范围图或层级图 | 说出这片专业地形有哪些主要块，导师相关区域在哪里 | 范围定位拿走大地形和相邻方向 | 百科化或图越证据，回 S1/S4 |
| 范围定位 | 学生已有大领域地形 | 把导师方向从大领域收紧到子领域、问题域、方法入口或交叉点 | 范围套层表 / 定位矩阵 / 证据边界卡 | 说出导师当前方向大致落在哪一层，为什么这样判断 | 概念入口拿走必须解释的少数概念 | 弱线索写成定论，回 S1 |
| 概念入口 | 已有导师当前位置 | 给进入 `03` 前最少需要的 4-7 个概念，不展开成课程 | 概念卡、对象-问题-方法表、小型概念图 | 用自己的话解释入口概念，并知道它们后面会在哪里出现 | 论文入口拿走概念和问题域 | 百科词条化或课程化，回 S2/S3 |

| 来源与核查 | 学生知道哪些领域判断要复核 | 给出完整来源入口和关键判断边界 | 按用途分组来源表、关键判断核查表 | 能从关键定位判断跳到来源，并说出来源支撑什么、不支撑什么 | `03` 共享领域证据边界 | 只列链接或 bibliography，回 S1/S3 |

审查 D02 时，先看这条推进链。若任一节删掉后前后仍能顺读，说明该节没有承担独立动作。

<a id="d02-support"></a>
## 关系负担与支撑形式

D02 的支撑重点是“范围和边界”。表格、图、卡片都不能只是换一种形式展示名词；它们必须降低学生同时处理大领域、子领域、问题域、相邻方向、概念和证据边界的负担。

| 关系负担 | 默认支撑形式 | 升级或降级条件 | 禁止做法 |
|:---|:---|:---|:---|
| 线索来自哪里、能支撑什么 | 方向线索表、来源用途表 | 来源冲突或弱线索多时，加证据边界卡 | 把官网词、论文关键词、数据库标签直接写成领域结论 |
| 大领域、分支、相邻方向的范围感 | 分层正文 + 非比例领域范围图 | 证据只支持上下位关系时，用层级树；证据不足时降级为文字和缺口说明 | 用面积、中心、距离、重叠暗示无证据的规模或重要性 |
| 大领域到问题域的逐层收紧 | 范围套层表、层级树 | 方法 / 平台与问题域混杂时，拆成“研究对象 / 问题 / 方法”三列 | 只写名词链条，不说明每层依据 |
| 当前主线、强相关、相邻、背景的区别 | 定位矩阵或边界表 | 两个连续维度都有来源时才用坐标图 | 用坐标轴装饰，或把弱线索放在中心位置 |
| 相邻方向容易混用 | 相邻方向比较表 | 差异来自两个明确维度时，可用比较图或二维图 | 没有真实交集却画 Venn；没有连续维度却画坐标 |
| 入口概念之间的关系 | 概念卡、对象-问题-方法表 | 概念之间有命题关系时，用带动词边的小概念图 | 只连概念名，不写关系词 |
| 论文阅读问题从哪里来 | 问题交接表 | 只在问题之间存在前后依赖时，用小问题链 | 逐篇分析论文角色，抢 `03` 职责 |
| 证据强弱和待复核点 | 证据边界卡、关键判断核查表 | 风险贯穿多节时，在自检中加回看位置 | 用颜色深浅或粗线假装可信度 |

每个支撑形式都必须被正文消费。表前或图前要提出学生正在卡的关系，表后或图后要拿走其中一个结果继续推进。图表删掉后如果正文理解不受影响，说明它只是装饰。

### 领域范围图规格

`02` 可以有核心领域范围图，但不能把“必须有图”误解为“必须画复杂图”。进入正文前，先写清这张图的规格：

```text
学生输出：看完图后能说出导师方向先放在哪片领域读。
关系中的对象：大领域、主要分支、问题域、相邻方向、导师入口。
证据允许的视觉语义：阅读位置、背景 / 强相关 / 当前主线、需复核方向。
禁止使用的视觉语义：精确面积、真实距离、中心性、重要性排序、因果关系。
图下读法：先看大领域，再看导师相关区域，再看相邻方向和需复核点。
图后消费：把粗位置交给 `范围定位`，把相邻方向交给边界比较。
```

如果写不出这张规格，先不要画图。回到 S1 补资料，或回到 S2 明确学生输出。

### 关键表格骨架

方向线索表：

| 线索来源 | 看到什么 | 可以支撑什么 | 不支撑什么 | 处理方式 |
|:---|:---|:---|:---|:---|
| `01` 方向入口词 | <词或短句> | 进入 `02` 的最低检索入口 | 不单独证明当前领域定位 | 作为候选线索，需领域资料交叉 |
| 官方方向 | <官网 / 机构页表述> | 导师公开自述或机构口径 | 不证明论文群真实路线 | 与近期论文和综述互查 |
| 近期论文 | <研究对象 / 方法 / 问题词> | 当前问题域和方法入口线索 | 不单独定义大领域 | 用于范围定位和 `03` 候选问题 |
| 领域资料 | <综述 / 课程 / 分类> | 大领域、分支、相邻方向和概念解释 | 不证明导师本人采用该表述 | 用于领域全景和概念入口 |

范围定位表：

| 层级 | 本文暂定说法 | 来源依据 | 证据边界 | 学生怎么读 |
|:---|:---|:---|:---|:---|
| 大领域 | <大领域> | <综述 / 课程 / 分类 / 学科页> | <不支撑精确比例或导师贡献> | 先把方向放进这片地形 |

## 小节设计

每节按同一组问题审查：

```text
本节出口：
上一节交给：
本节动作：
证据入口：
关系负担与支撑：
正文推进：
学生输出：
下一节拿走：
失败回退：
容易写偏：
```

### 地图定位

本节出口：学生知道 `02` 是专业坐标层，第一遍只抓领域范围、相邻方向、入口概念和交给 `03` 的问题，不急着读懂论文细节。

上一节交给：`00` 的阅读顺序和 `01` 的事实底座。学生知道导师是谁，也知道论文集合和方向入口词还需要进一步解释。

本节动作：建立阅读契约，说明为什么读完事实画像后不能直接进入论文路线，而要先建立领域坐标。

证据入口：本节通常不展开具体来源，只说明后文会区分官方线索、论文线索和领域资料。若提到具体方向判断，必须有短引用或留到后文。

关系负担与支撑：学生容易把 `01` 的方向入口词当成完整领域定位。可用一两句话写清“入口词不是地图”；不要在开头塞大表。

正文推进：

1. 承接 `01`：学生已经知道导师事实和论文集合，但方向词仍是散的。
2. 写读者困难：不知道这些词属于大领域、子领域、问题域、方法还是背景。
3. 写本文任务：把这些词放进专业范围，分清相邻方向和证据边界。
4. 写第一遍读法：先抓粗坐标和后续问题，不追求读完领域。
5. 写使用边界：不评价导师，不替代领域综述，不提前写论文路线。

学生输出：用一句话说出“我读 `02` 是为了知道这位老师的方向先放在哪片专业范围里，而不是为了读完整个领域”。

下一节拿走：方向线索拿走读法和边界，开始列出地图从哪里来。

失败回退：写成领域导论，回 S2/S3；写成导师方向总结，退回 `01` 和 D02 边界；出现评价语，回 S3 删除。

容易写偏：

- 一开头讲大领域历史。
- 把导师当前方向写成定论，但没有说明来源。
- 使用“领域生态”“研究范式”等抽象词，却不给学生读法。
- 提前解释论文贡献。

### 方向线索

本节出口：学生知道后面的地图从哪些线索出发，也知道每类线索只能支撑到什么程度。

上一节交给：学生已经知道 `02` 要建专业坐标，需要先看地图依据。

本节动作：把 `01` 的方向入口词、官方方向、近期论文、领域资料拆成不同来源角色。只列线索，不展开成身份核查或论文分析。

证据入口：严格执行 S1。官网方向可以支撑官方表述，不能单独支撑论文路线；论文题名可以提示对象和方法，不能单独定义大领域；综述和课程可以解释领域结构，不能证明导师本人采用某个表述。

关系负担与支撑：多类来源容易被混成一段顺滑文字。默认用方向线索表，列出“可以支撑什么 / 不支撑什么 / 处理方式”。

正文推进：

1. 承接地图定位：地图不是凭空画出来的，也不是只靠一个方向词。
2. 列线索来源：`01` 入口词、官方方向、近期论文、领域资料。
3. 对每类线索写支持范围和边界。
4. 标出弱线索、冲突和 `[未找到]`，不在本节硬解释。
5. 把可用线索交给 `领域全景`。

学生输出：说出至少三类线索分别能支撑什么，并指出哪一类线索还需要后文复核。

下一节拿走：领域全景拿走可用于画大地形的线索和证据边界。

失败回退：只堆关键词，回 S1/S3；重写身份核查，退回 D01；逐篇分析论文，退回 D03；没有领域资料，停写补资料。

容易写偏：

- 把官网方向词当成最终领域定位。
- 把数据库关键词当成学科分类。
- 把论文题名相似当成问题链。
- 为了让地图顺滑，省略弱线索和冲突。

### 领域全景

本节出口：学生能看见大领域、主要分支、相邻方向和导师相关区域的粗地图，知道这张图只能提供阅读位置，不能提供精确比例或评价。

上一节交给：可用线索、领域资料和证据边界。

本节动作：建立专业地形。先说明大领域是什么，再列主要分支和相邻方向，最后标出导师相关区域。

证据入口：本节必须优先使用综述、课程、教材章节、学会 / 期刊 / 会议分类、领域背景资料等能支撑领域结构的来源。导师官网和论文只用来标出相关区域，不单独决定大地形。

关系负担与支撑：学生需要同时处理大领域、分支、相邻方向和导师入口。默认用分层正文 + 非比例领域范围图；如果证据只支持上下位关系，用层级树；如果证据不足，降级为分层表和缺口说明。

正文推进：

1. 用一段话定义大领域，不写成百科开篇。
2. 列出与导师方向有关的主要分支和相邻方向。
3. 说明导师方向先放在哪个区域读，哪些区域只是背景。
4. 对关键分支至少写出它关心的对象、问题、方法或可观测量，说明这会怎样影响后面读论文。
5. 给领域范围图或层级图，并写清读法和证据边界。
6. 把图里的导师相关区域交给 `范围定位`。

学生输出：用 2-3 句话说出这片领域的大地形、导师相关区域和容易混用的相邻方向。

下一节拿走：范围定位拿走大地形和相邻方向，进一步收紧到导师当前位置。

失败回退：写成百科综述，回 S3；图没有正文消费，回 S4；图暗示无证据比例或中心性，回 S1/S4；没有相邻方向，回 S1 补资料。

容易写偏：

- 大段解释领域历史和全部分支。
- 主图只有漂亮形状，没有图下读法。
- 把背景方向画得和当前主线一样强。
- 用“热门”“前沿”“重要”等词替代来源判断。

### 范围定位

本节出口：学生能把导师方向从大领域逐层收紧到子领域、具体问题域、方法 / 平台或交叉点，并知道每层依据和边界。

上一节交给：大地形、相邻方向和导师相关区域。

本节动作：把“导师方向在这片领域里”进一步拆成可核查的层级判断。每一层都要回答：这是什么层级、依据是什么、不能推出什么。

证据入口：结合官方方向、近期论文、论文摘要 / 方法 / 图表、综述或课程中的问题域。不能只用题名关键词拼接名词链。

关系负担与支撑：大领域、子领域、问题域、方法 / 平台容易混在一起。默认用范围定位表；必要时加范围套层图或定位矩阵。弱证据用证据边界卡，不画成确定位置。

正文推进：

1. 承接领域全景：学生已经知道大地形，现在需要找到导师入口。
2. 按层级写：大领域 -> 子领域 -> 问题域 -> 方法 / 平台 -> 导师切入点。
3. 每层说明来源依据和证据边界。
4. 每个关键层级至少说明它关心什么对象、问题、方法、可观测量或瓶颈，以及这个说明会让 `03` 重点看论文的哪一类内容。
5. 明确相邻方向：哪些强相关，哪些只是背景，哪些待复核。
6. 收束成 1-2 句定位判断，并标出可能被 `03` 修正的地方。

学生输出：用一段短话说出导师方向的层级定位，并指出至少一个不能过度解读的边界。

下一节拿走：概念入口拿走学生理解这段定位所需的少数概念。

失败回退：只写名词链条，回 S3/S4；弱线索写成强定位，回 S1；相邻方向不区分，回 S1/S4；开始判断论文代表性，退回 D03。

容易写偏：

- “A 属于 B，B 属于 C”但不解释每层对读论文有什么用。
- 把方法、平台、对象和问题域混成同一层。
- 为了显得准确，编造精确比例、中心位置或成熟度。
- 把 `01` 的方向入口词原样搬过来。

### 概念入口

本节出口：学生能复述进入 `03` 前最低限度的 4-7 个概念，知道每个概念为什么和导师当前位置有关，以及后面会在哪里再次遇到。

上一节交给：导师范围定位、问题域、方法入口和相邻方向。

本节动作：筛出少数入口概念，给新手可用的短解释。概念解释只服务后续论文阅读，不扩成教材章节。

证据入口：概念解释优先来自综述、课程、教材或领域背景资料；论文可以说明概念在导师问题域中的用法，但不能替代基础解释。

关系负担与支撑：概念之间可能是对象、问题、方法、测量、材料 / 数据或理论关系。默认用概念卡或“对象 / 问题 / 方法”表；只有概念之间存在明确命题关系时，才用带动词边的小概念图。

正文推进：

1. 承接范围定位：学生已经知道方向在哪，但还缺少进入论文的词。
2. 说明筛选原则：只保留 `03` 会反复遇到、且影响理解问题域的概念。
3. 每个概念写“一句新手解释 / 为什么重要 / 后面在哪里出现 / 本节不展开什么”。
4. 概念之间容易混用时，加一行边界说明。
5. 把概念交给 `论文入口` 的问题。

学生输出：不用查资料，能用自己的话解释每个入口概念，并说出它会帮助后面看哪类论文问题。

下一节拿走：论文入口拿走这些概念，转成 `03` 的问题交接。

失败回退：写成百科词条，回 S2/S3；概念过多，回 S2 限制输出；没有后文用途，删掉或移出；公式和课程先修过多，退回 D04。

容易写偏：

- 每个概念都写成一段维基百科式定义。
- 为了显得专业，加入后文不会用到的术语。
- 用“掌握”“深入理解”等空泛动词。
- 把 `04` 的学习路径提前搬进来。

### 论文入口

本节出口：学生带着 2-4 个领域问题进入 `03`，知道后面读论文要观察什么，但还不预设论文路线答案。

上一节交给：范围定位和入口概念。

本节动作：把领域坐标转成论文阅读问题。问题要来自前文的领域层级、相邻方向、方法入口或概念关系，而不是凭空提出。

证据入口：问题可以由近期论文题名、摘要、图表、综述问题、官方方向共同提示；但本节不判断哪篇论文承担什么角色。

关系负担与支撑：学生容易把领域问题、论文角色和学习任务混在一起。默认用问题交接表，明确“`03` 应该看什么 / `02` 不在这里做什么”。

正文推进：

1. 承接概念入口：现在不是继续讲概念，而是把概念变成读论文的问题。
2. 提出 2-4 个问题，每个问题都回到前文某层定位。
3. 写清 `03` 将用论文群回答什么。
4. 写清 `02` 不预判论文路线、不选目标论文、不讲论文贡献。
5. 把待复核的领域假设也交给 `03`。

学生输出：说出后面读 `03` 时要带着哪几个问题，以及这些问题分别来自哪层领域定位。

下一节拿走：主体自检拿走完整领域坐标和问题入口。

失败回退：逐篇解释论文路线，退回 D03；问题太多，回 S2 限制输出；问题和前文定位无关，回 S3 重写承接；把问题写成学习计划，退回 D04。

容易写偏：

- “后面要重点读这些论文”但没有说明领域问题。
- 已经开始给论文分入口、方法、目标角色。
- 把概念清单换个标题叫论文问题。
- 问题只来自论文题名相似。

### 主体自检

本节出口：学生能判断自己是否已经有领域坐标，可以进入 `03`；答不上时知道回哪一节补。

上一节交给：领域全景、范围定位、概念入口和论文问题。

本节动作：把 `02` 的专业坐标转成输出型检查。自检不考知识面，而是检查学生能否复述范围、边界、概念和证据。

证据入口：本节不新增事实，只消费前文判断。涉及证据时，要求学生指出来源类型或回看位置。

关系负担与支撑：自检项、合格标准和回退位置适合表格，不用图。

正文推进：

1. 承接论文入口：进入 `03` 前先检查自己是否带着正确问题。
2. 说明最低合格线：能复述粗坐标、相邻方向、概念和证据边界即可，不要求读懂论文。
3. 给自检表：大领域、范围定位、相邻方向、概念入口、论文问题、证据边界。
4. 每项都写答不上回哪里。
5. 引出来源与核查：证据边界不清时要回来源。

学生输出：完成自检，并指出至少一个自己需要回看的位置。

下一节拿走：来源与核查拿走学生需要复核的关键判断。

失败回退：自检只问“是否理解”，回 S2；没有回退位置，回 S3；检查论文细节，退回 D03；检查学习计划，退回 D04。

容易写偏：

- 写成考试题。
- 问“是否掌握领域背景”。
- 没有告诉学生答不上该回哪节。
- 把来源核查省略成“见参考资料”。

### 来源与核查

本节出口：学生能从关键领域判断回到来源条目，说明来源类型、支撑内容、不支撑内容和风险处理。

上一节交给：学生已经知道哪些领域判断可能需要复核。

本节动作：把正文短引用落到完整来源清单，并让来源服务关键判断，而不是堆链接。

证据入口：严格执行 S1。只列正文真正使用的来源和会影响领域全景、范围定位、概念解释、相邻方向边界的关键资料。缺失来源写 `[未找到]`，弱判断写 `需人工复核`。

关系负担与支撑：来源编号、类型、资料名称、支撑内容、不支撑什么、链接适合表格。来源较多时按官方资料、论文资料、综述与背景资料、教材与基础资料分组。

正文推进：

1. 承接主体自检：自检中答不上证据边界时，从这里回查。
2. 说明来源分组：官方资料、论文资料、综述 / 背景资料、教材 / 基础资料各自用途不同。
3. 给关键判断核查表或来源表，必须包含支撑内容和不支撑什么。
4. 标出 `[未找到]`、`需人工复核` 和弱线索。
5. 说明 `03` 可能会反向修正哪些定位。

学生输出：看到一个定位判断，能沿短引用找到来源，并说出这条来源支撑什么、不支撑什么。

下一节拿走：本节是 `02` 的收束；`03` 沿用这些领域证据边界。

失败回退：只列 URL，回 S1/S3；正文没有消费的 bibliography，删除；没有“不支撑什么”，回 S1；来源支撑不了定位，回到前文降级判断。

容易写偏：

- 把参考资料写成大而全 bibliography。
- 只列论文，不列领域资料。
- 只写“来源：官网 / 论文”，不写支撑什么。
- 用来源数量营造可信感。

<a id="confirmed-rules"></a>
## 已确认规则

- `02` 是专业坐标层，不是领域百科、导师评价、论文路线或学习计划。
- 文首必须有目录和显式 anchor。
- 成品章节固定为 `地图定位 / 方向线索 / 领域全景 / 范围定位 / 概念入口 / 论文入口 / 主体自检 / 来源与核查`。
- `02` 承接 `01`，但不能只靠 `01` 写成；必须补领域资料、相邻方向和概念来源。
- `方向线索` 只列地图依据和证据边界，不重做身份核查，不重列完整论文表。
- 官网方向词、论文题名关键词、数据库标签都只能作为线索，不能单独定义领域地图。
- `领域全景` 优先解决大领域、主要分支、相邻方向和导师相关区域，不写大而全综述。
- `范围定位` 必须区分大领域、子领域、问题域、方法 / 平台、相邻方向和背景区域。
- 每层定位都要说明来源依据、证据边界和这一层关心的对象、问题、方法或可观测量，不能只写名词链条。
- 领域范围图可以使用，但必须是非比例阅读图；不得暗示无证据的精确比例、中心性、距离、重要性或因果关系。
- 图前要提出学生卡点，图下要教读并标证据边界，图后要消费图里的结果。
- 相邻方向不是装饰项；缺相邻方向时，必须写已查入口和缺口，不能把导师方向写成孤立单线。
- `概念入口` 控制在 4-7 个概念；只解释到能进入 `03`，公式推导、课程先修和学习顺序留给 `04`。
- 每个入口概念都要写“先这样理解 / 为什么重要 / 后面在哪里再遇到 / 本节不展开什么”。
- `论文入口` 只交接 2-4 个领域问题和待复核假设，不逐篇分析论文角色，不选择目标论文。
- `主体自检` 检查领域坐标、相邻方向、概念、论文问题和证据边界，不检查论文细节或学习计划。
- `来源与核查` 必须列出来源类型、支撑内容、不支撑什么和链接；来源编号使用显式 anchor，正文短引用要能回跳；缺失来源写 `[未找到]`，弱判断写 `需人工复核`。
- D02 的领域定位假设、相邻方向边界、入口概念候选、论文问题、可视化计划和关键证据风险必须同步到 `_internal/blueprint.md` 或关键判断核查表，不能只留在正文里。
- 第一次出现的机构缩写、领域缩写和专业术语必须展开并给短解释。
- 不使用“值得关注”“方向热门”“影响力很大”“平台资源好”“适合申请”“匹配度”等评价或申请建议。

先验证 D02 标准自身：

- 八个最小落地产物是否都能在标准中找到。
- 八节是否都有出口、承接、动作、支撑、输出、交接和失败回退。
