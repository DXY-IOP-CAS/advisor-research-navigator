# 03 论文路线

状态：候选标准稿。先作为 `03_论文路线.md` 的样板，用来校准“文档标准怎样从真实失败出发”。通过真实导师成品试压前，不回收进模板、skill reference 或 verifier。

## 中心问题

`03_论文路线.md` 要解决的失败很具体：执行 AI 拿到一组论文后，容易按年份或题名逐篇写摘要。这样的文档看起来信息多，学生读完却只记得几篇论文名，说不出这些论文共同推进了什么问题，也不知道哪篇论文该先读、哪条关系只是弱线索。

`03` 的任务，是把论文集合加工成学生能复述的问题链。这里的“问题链”指学生能听懂的一组判断，不按时间线或引用网络组织：

```text
这位老师当前方向主要在问什么；
哪些论文进入这条主线；
每篇论文承担什么角色；
这些角色怎样让问题往前推进；
哪些关系有证据，哪些需要人工复核；
读完后要把哪些概念、图和平台链路交给 04。
```

如果一份 `03` 只让学生得到论文清单、论文摘要或研究关键词，它还没有完成任务。

## 文档职责

`03_论文路线.md` 是论文问题链标准。它承接 `02_领域地图.md` 给出的领域问题，把导师相关论文组织成一条或几条有边界的研究路线。

它要做五件事：

- 确定当前问题：这组论文主要回答哪个研究问题。
- 筛出路线论文：哪些论文进入主线，哪些只是背景、旁支或弱线索。
- 说明论文角色：每篇论文在问题链中承担什么任务。
- 解释推进关系：这些论文怎样共同推进研究问题。
- 交给 `04`：目标论文、核心图、平台链路、概念缺口和适合继续学习的问题。

它不做三件事：

- 不重新做身份核查。身份、履历、论文归属和论文集合风险属于 `01_基础画像.md`。
- 不重新讲领域全景。大领域、相邻方向和概念入口属于 `02_领域地图.md`。
- 不写学习计划。课程基础、补课顺序、读图训练和进组前学习任务属于 `04_学习向导.md`。

读完 `03` 后，学生应能用 3 到 5 句话复述论文路线，并能指出哪篇是目标论文、哪些是主线支撑、哪些只是旁支或需人工复核的关系。

## 接力关系

`03` 不能孤立写。它需要接住前两份材料，也要给 `04` 留下可使用的学习对象。

| 来源 | 交给 `03` 什么 | `03` 怎样使用 |
|:---|:---|:---|
| `01_基础画像.md` | 已验证论文集合、履历阶段、身份和论文归属风险 | 决定哪些论文可进入路线，哪些必须降级为风险 |
| `02_领域地图.md` | 当前领域问题、相邻方向、少数入口概念 | 把论文放进具体问题域，避免重写领域综述 |
| `_internal/blueprint.md` | 导师当前方向一句话、目标论文、回看论文、核心图或平台链路 | 固定 `03` 的路线中心，避免临场凑论文 |
| fresh search | 论文全文、摘要、DOI 页面、项目页、数据集、综述和导师主页更新 | 核对论文关系、核心图、平台链路和证据边界 |

如果这些输入不足，`03` 要暂停写作或降级表达。目标论文不清、论文归属不稳、论文之间缺少关系证据时，不能靠顺滑文字补出一条路线。

## 候选大纲

成品 `03` 可以先按下面结构写。标题可以根据导师方向微调，但章节职责不要随意改。

```markdown
# <姓名>论文路线

## 目录
- [路线定位](#route-scope)
- [从领域问题进入论文](#field-to-papers)
- [论文篮](#paper-basket)
- [主线问题](#main-question)
- [论文角色](#paper-roles)
- [推进关系](#route-logic)
- [目标论文入口](#target-paper)
- [旁支与需复核关系](#side-branches)
- [交给 04 的学习对象](#handoff-to-learning)
- [读后自检](#paper-route-self-check)
- [来源与参考资料](#sources)
```

这份大纲先从读者要解决的问题进入论文，再把论文筛成路线。不要先抛完整论文表，也不要先按年份排列。

## 小节设计

每一节只推进一个动作。执行 AI 写作前，应先确认这一节接住什么、产出什么、不能写成什么。

| 小节 | 读者问题 | 本节动作 | 读完留下什么 | 容易写偏 |
|:---|:---|:---|:---|:---|
| 路线定位 | 我为什么要看论文路线？ | 说明 `03` 只回答论文群怎样构成研究路线 | 知道本文的用途是路线分析 | 写成空泛导语，或提前讲完整路线 |
| 从领域问题进入论文 | `02` 的领域问题怎样接到论文？ | 用 1-3 个问题承接 `02`，作为筛论文的入口 | 用问题筛论文，不用年份带读 | 重新写领域地图 |
| 论文篮 | 哪些论文会进入路线？ | 列出主线、目标、回看、旁支、弱线索论文，并说明筛选理由 | 知道路线论文有筛选理由 | 变成完整论文清单或摘要表 |
| 主线问题 | 这些论文共同在问什么？ | 把论文篮收束成一句当前问题 | 能用一句话说出路线中心 | 把几个关键词拼成方向判断 |
| 论文角色 | 每篇论文承担什么任务？ | 判断论文是问题提出、方法支撑、平台建设、关键验证、目标入口还是旁支 | 能看出论文分工 | 只贴标签，不解释依据 |
| 推进关系 | 这些角色怎样接上？ | 用正文或小图说明论文之间的推进、支撑或并行关系 | 能用 3 到 5 句话复述路线 | 把时间顺序误当研究路线 |
| 目标论文入口 | 为什么从这篇论文进入当前方向？ | 说明目标论文的地位、核心图和阅读价值 | 知道后续学习先盯哪篇、哪张图 | 把目标论文写成完整精读笔记 |
| 旁支与需复核关系 | 哪些内容不能放进主线？ | 标出证据不足、方向较远、论文归属不稳或只作背景的材料 | 不把弱线索当结论 | 为了路线完整强行合并旁支 |
| 交给 04 的学习对象 | 后面该学什么？ | 提取概念缺口、核心图、平台链路和可请教问题 | `04` 能据此设计学习桥 | 抢写课程计划 |
| 读后自检 | 我是否真的看懂路线？ | 用具体问题检查主线、角色、证据和交接 | 学生知道能否进入 `04` | 自检题只问“是否理解” |
| 来源与参考资料 | 路线判断靠什么来源？ | 按判断类型列来源和限制 | 能复核论文关系和风险 | 链接堆积，没说明支撑什么 |

## 把论文篮变成问题链

`03` 写作的关键动作是把论文篮加工成问题链。逐篇扩写只在解释角色和证据时使用，不能成为正文主线。

### 先定论文篮

论文篮只收当前路线需要的论文。来源可以包括 `01` 的已验证论文集合、蓝图目标论文、导师主页近期方向、论文全文、DOI 页面、arXiv、数据集或项目页。

论文篮至少分成五类：

| 类型 | 进入正文的方式 |
|:---|:---|
| 目标论文 | 作为当前方向入口，重点说明为什么选它 |
| 主线论文 | 支撑当前问题链，解释角色和关系 |
| 回看论文 | 帮学生理解目标论文前的关键一步 |
| 旁支论文 | 只在需要区分方向时出现，不强行并入主线 |
| 弱线索 / 需复核论文 | 标明证据不足，通常不承担主线判断 |

论文篮不等于完整论文表。完整论文集合在 `01`，`03` 只使用能解释当前路线的部分。

### 正文前先写论文角色卡

`03` 最容易失败的地方，是执行 AI 还没读懂论文角色，就开始写顺滑正文。正式写正文前，应先在 `_internal/analysis/` 这类内部分析位置完成论文角色卡；如果只是口头判断，不算完成。不要把角色卡放进 `_internal/evidence/`，那里只放关键判断证据表，旧 verifier 会按固定表头检查。

角色卡不用写给学生看，但必须能让审稿人判断这篇论文能承担什么、不该承担什么。每张卡至少回答：

```text
论文题名：
来源 / DOI：
研究对象是什么：
核心问题是什么：
用了什么方法或平台：
主要结果支撑什么判断：
和目标论文的关系是什么：
它在路线中的角色：
证据强度：
不能过度写成什么：
```

角色卡完成后，再决定哪些内容进入正文。学生只需要看到被加工后的问题链；执行 AI 必须先留下可审查的中间判断。没有角色卡时，`03` 很容易把“相邻语言”“平台边界”“前史能力”和“目标机制”写成同等层级，读者会以为所有论文都在推进同一条主线。

### 再定主线问题

主线问题要来自论文证据，不能只由题名关键词拼出来。执行 AI 应优先看这些线索：

- 目标论文和回看论文共同研究的对象、机制、方法或平台。
- 多篇论文反复出现的问题表述、实验对象、数据类型或核心图。
- 导师主页、课题组介绍或项目页对当前方向的描述。
- 综述或引言中对该问题位置的解释。

主线问题要写成学生能复述的话。例如：

```text
这组论文主要在问：某个对象在某种实验或理论条件下，为什么会出现某种可观测现象，以及怎样用特定方法把它测出来或解释清楚。
```

避免写成“围绕某领域开展研究”。这种句子太宽，无法筛论文，也无法指导 `04`。

### 再判论文角色

论文角色要帮助学生看出分工，不能停在漂亮标签。常用角色可以先从这组里选：

| 角色 | 判断依据 |
|:---|:---|
| 问题提出 | 论文明确提出后续主线要解决的问题或现象 |
| 对象底座 | 论文给出研究对象、样品、体系或数据基础 |
| 方法 / 平台支撑 | 论文提供测量、计算、建模、数据或实验平台 |
| 关键验证 | 论文验证目标问题中的核心判断 |
| 目标入口 | 当前最适合学生进入导师方向的论文 |
| 延伸旁支 | 与导师方向相关，但不直接推进当前主线 |
| 弱线索 | 只有题名、关键词或单一来源支持，需要人工复核 |

角色判断必须在正文附近给出理由。理由可以很短，但要说清“为什么它承担这个角色”。

### 写推进关系

推进关系有几种，不要把它们都画成流程图。

| 关系类型 | 适合写法 | 使用边界 |
|:---|:---|:---|
| 问题链 | 短正文 + 小路线图 | 只有论文之间确实有先后、支撑或验证关系时使用 |
| 平台链 | 平台或方法链路说明 | 只在同一实验、计算或数据平台贯穿多篇论文时使用 |
| 方法比较 | 对比表或矩阵 | 几篇论文处理相近问题，但方法或对象不同 |
| 并行分支 | 分支短表 | 论文相关但不能合成单一路线时使用 |

如果证据只支持“这些论文属于相近方向”，就写成并行分支，不要强行写成问题链。

## 图文与表格使用

`03` 不默认画图。先判断学生需要少背哪段关系，再决定形式。

优先使用一张路线矩阵：

```text
论文 -> 问题 -> 方法 / 平台 -> 路线角色 -> 证据强度 -> 交给 04 的学习缺口
```

矩阵的作用是让学生看出论文分工。矩阵之后必须用正文收束成 3 到 5 句话。只放矩阵不解释，仍然是资料堆。

小型路线图只在关系有方向时使用。图中箭头必须写清语义：问题推进、方法支撑、平台输入、实验验证或阅读入口。横向长 Mermaid 流程图通常不适合 `03`，因为节点一多就变成论文名串联。

证据边界优先放在判断旁边。论文角色、目标论文选择、旁支降级这些位置容易误判，应该用短句标出证据强弱，不需要把证据画成网络。

## 证据边界

`03` 的每个路线判断都要能追到来源。不同判断的证据要求不同。

| 判断 | 最低证据要求 | 写作边界 |
|:---|:---|:---|
| 论文属于导师 | 来自 `01` 已验证集合，或官网 / Scholar / DOI / ORCID 交叉支持 | 归属不稳时不进入主线 |
| 论文进入当前路线 | 题名、摘要、全文、导师主页或项目页能支撑与当前问题有关 | 只有关键词相似时写弱线索 |
| 论文角色 | 论文内容能说明它提供对象、方法、验证、目标入口或旁支关系 | 不能只按年份推断角色 |
| 论文之间有推进关系 | 引用、方法延续、同一平台、同一问题、目标论文引言或导师表述能支持 | 证据不足时写并行关系 |
| 目标论文适合作入口 | 论文能集中体现当前问题、核心图或平台链路，且来源可核查 | 不能只因发表时间最新就设为目标 |

弱推断要写 `需人工复核`。来源缺失写 `[未找到]`。不要用“可能说明”“似乎体现”这类模糊词掩盖证据不足。

## 读后自检

`03` 的自检要检查学生是否真的看懂了论文路线。

成品中至少保留这些问题：

1. 用一句话说出这组论文共同推进的问题。
2. 选出目标论文，并说明它为什么适合作为当前入口。
3. 说出两篇主线论文之间的关系：问题推进、方法支撑、平台共用、关键验证，还是证据不足。
4. 指出一篇旁支或弱线索论文，说明为什么它不能直接进入主线。
5. 列出进入 `04` 前必须补的 2 到 4 个概念、图或方法。

如果学生答不出来，先回看 `03` 的主线问题、论文篮、角色判断和推进关系，不要把失败归因于学生基础差。

## 写偏风险

这些坏写法要在审稿时直接删或重写。

| 坏写法 | 为什么坏 | 应改成 |
|:---|:---|:---|
| 按年份逐篇摘要 | 学生只记论文名，看不到问题链 | 先定主线问题，再按角色组织论文 |
| 用发表时间代替路线 | 时间先后不等于研究推进 | 写清问题、方法或平台怎样接上 |
| 给论文贴标签但不解释 | 标签不能帮助学生复述 | 每个角色旁写一句判断依据 |
| 把旁支并入主线 | 会放大弱证据，制造假路线 | 单独列旁支和需复核关系 |
| 画长流程图 | 箭头容易无语义，显示也不稳定 | 用矩阵 + 短正文，必要时用小图 |
| 抢写学习计划 | 和 `04` 重复，削弱 `03` 职责 | 只交出学习对象，不安排课程顺序 |

## 成品试写时验证

这份标准要用真实导师成品试压。试压时先看张鹏举样本，因为现有成品已经暴露出论文路线、核心图和学习向导之间的质量问题。

试压时只问四件事：

- 成品 `03` 能不能让学生说出论文群共同推进的问题。
- 目标论文、回看论文、旁支论文是否有清楚边界。
- 图表是否真正降低论文关系理解负担。
- `03` 是否把 `04` 需要的概念、核心图和平台链路交出去，而没有提前写学习计划。

试压通过前，不把本标准回收到正式模板、reference 或 verifier。

### 张鹏举样本的第一轮观察

张鹏举旧版 `03_论文路线.md` 已经有不少可用判断，例如把目标论文定为 2025 年液体水 ICD 竞争机制论文，把 2022 年液体水 ICD 首次观测作为回看论文，也能区分液相信号底座、分子阿秒语言和平台边界。但旧稿把这些判断摊成许多重复板块：前面讲问题链，后面又讲当前主线、当前主线关系、前史积累和学习准备。读者容易看到很多材料，却分不清哪一节才是论文路线的主推进。

第一轮重写后，`03` 的主线应收紧为四步：

```text
液相信号底座
  -> 液体水 ICD 从首次识别到竞争机制
  -> 分子阿秒论文补时间、相位和波包语言
  -> 平台论文说明后续实验边界
```

学习准备不再留在 `03` 内展开。`03` 只把目标论文、回看论文、核心图组、概念缺口和平台链路交给 `04`。课程补法、读图训练、进组前讨论稿由 `04_学习向导.md` 承担。

旧 `verify_phase_docs.py` 仍按早期章节名检查 `03`，例如要求“先抓住论文在回答什么问题”“当前主线论文”“从论文路线倒推出的学习准备”等旧标题。这个 smoke 失败说明正式 verifier 尚未回收新标准，不能反过来要求成品恢复旧结构。后续回收时，应先用张鹏举新版 `03` 确认结构稳定，再更新 verifier 的章节检查。

## 待确认问题

- 不同学科的论文路线是否都适合用“问题链”表述。若不适合，应允许写成平台链、方法比较或并行分支。
- 目标论文是否每位导师都必须有。当前判断是必须有当前入口，但它可以是一篇论文、一组论文或一个核心图组。
- 路线矩阵的字段是否需要按理工、文社科分别调整。
- `03` 是否需要固定“目标论文入口”小节，还是可以在“推进关系”中自然收束到目标论文。

## 后续回收
