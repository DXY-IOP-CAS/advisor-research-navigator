# 00 材料导读

状态：正在讨论。

## 文档职责

`00_材料导读.md` 是读者入口。它不负责讲具体研究内容，而是教学生如何使用这套材料。

它要回答：

- 这套材料是什么，不是什么。
- 五份文档怎样配合，为什么这样读。
- 读完后应该形成什么输出。
- 来源、引用和风险标记怎样核查。

它不应提前替 `02/03/04` 讲复杂研究内容，也不应变成五份文档摘要合集。

## 已确认大纲

章节标题要短，服务扫描。正文负责解释完整含义。大纲不能只列标题，还要让审查者一眼看出每节承担什么功能、内部大概怎样展开、读完应留下什么、和前后章节怎样衔接。

这里的大纲不是最终成品正文，而是给人和执行 AI 看的“整篇设计图”。它只做粗略设计：一眼看出这份导读为什么分成这些节、每节内部大致怎样写、哪里不能越界。更细的叙事顺序、图文选择和常见错误放在后面的“小节设计”里展开。

## 通用设计原则

这一层只讲所有小节共用的方法，不绑定某一个章节。它之后可以回收到写作规则或审查清单；具体到每节怎么写，放在“小节设计”里。

### 正文节奏

每节正文都要让学生知其然，也知其所以然。写作时内部要暗含四步，但不要把它们写成僵硬小标题：

1. 读者处境：学生为什么会在这里卡住。
2. 本节动作：这一节让读者完成哪个阅读或判断动作。
3. 设计原因：为什么这个动作在整条阅读路径中必要。
4. 出口能力：读完后读者应留下什么判断、问题或下一步动作。

如果一节正文出现长串并列、递进或因果关系，优先用短段、列表或短表拆开。目标不是让版面显得丰富，而是让学生不用在一大坨文字里自己猜逻辑关系。

### 图文配合

`00` 的图文不负责解释具体研究机制，只负责降低“怎么读这套材料”的理解负担。因此优先使用短表格，不优先使用 Mermaid。

适合 `00` 的视觉构件：

- 可能误解 -> 实际定位。
- 学生困难 -> 对应文档。
- 阅读轮次 -> 阅读动作 -> 读完应抓住什么。
- 阅读成果 -> 自查标准。
- 引用标记 -> 来源类型 -> 需要怎样核查。

每个视觉构件必须先回答一个读者问题。若删掉图表后正文理解几乎不受影响，说明这个图表只是装饰，应删掉或改成更有信息密度的短表。

## 小节设计

本节用于逐节细化。大纲总览解决“整篇结构是否顺”和“每节大概做什么”；通用设计原则解决“所有小节共用什么方法”；这里记录每个小节自己的中心思想、叙事顺序、图文选择、出口检查和常见错误，方便后续把标准转成模板、skill 规则或人工审查清单。

每个小节尽量使用同一套内部检查项：

- 中心思想：这一节存在的根本目的。
- 内部叙事顺序：正文应按什么逻辑展开。
- 可用图文：是否需要表格、层级树、文本链路或 Mermaid。
- 出口检查：读者读完这一节后应能说出什么或做什么。
- 常见错误：执行 AI 最容易写偏的地方。

### 材料定位

中心思想：这一节不只是免责声明，而是建立读者和材料之间的“阅读契约”。读者读完后应知道：自己不需要一开始就懂所有术语和论文；这套材料会把零散资料整理成可核查的阅读路径；它帮助自己进入导师研究方向，但不替自己评价导师、做申请决策或替代论文教材。

内部叙事顺序：

1. 读者起点：说明本文默认读者是第一次接触该导师方向的学生，可能只有基础课背景，对导师主页、论文题名和领域术语都不熟。
2. 读者困境：真正困难不只是“资料不够”，而是资料太散、术语太密、顺序不明、证据强弱不明。
3. 材料承诺：这套材料把官网、履历、论文和学习资源整理成一条从事实到领域、从论文到学习准备的阅读路径，并尽量让关键判断可追溯。
4. 使用姿势：第一遍不要求读者读懂所有细节，而是先知道应该按什么顺序读、每一步抓什么、遇到不确定判断时怎样保留疑问。
5. 使用边界：不评价导师，不给申请建议，不替代原论文、教材和课程训练。
6. 出口判断：读者读完本节，应愿意带着“先按路线建立理解，再回到来源核查”的心态进入 `阅读顺序`。

可用图文：最多使用一个“可能误解 / 实际定位”短表，用来纠偏预期。表格不只写“不做什么”，还要写“应该怎样用”，例如“把它当申请结论”对应“把它当阅读路线图”。不使用流程图，不列五份文档详细职责。

出口检查：读者应能用一句话说出：“我现在拿到的是帮助我进入某位导师研究方向的阅读路线图；第一遍不需要马上判断导师或读懂所有论文，而是先跟着材料建立可核查的理解。”

常见错误：

- 一开头就堆导师方向的专业术语。
- 把本节写成免责声明。
- 把五份文档职责全塞进本节，抢走 `阅读顺序` 的功能。
- 直接使用“认知路径”这类内部抽象词，却不解释学生要怎么用。
- 只写“本材料不评价导师、不替代教材”，却没有告诉学生它正面提供什么帮助。

### 阅读顺序

中心思想：这一节把“材料定位”中的阅读契约落成可执行路线。它不是文件清单，而是解释五份文档如何共同解决新手理解导师方向时的不同困难，以及为什么要先粗读 `01`、再读 `02/03/04`、最后回看 `01`。

内部叙事顺序：

1. 承接上一节：既然这套材料是阅读路线图，本节就回答“路线怎么走”。
2. 先讲五份文档各自解决的困难：`00` 解决怎么读，`01` 解决事实和来源，`02` 解决领域位置，`03` 解决论文问题链，`04` 解决学习准备。
3. 再解释为什么第一遍先粗读 `01`：学生需要一个人物、机构、履历和论文集合的事实底座，但此时不要急着判断当前方向。
4. 接着解释为什么读 `02/03/04`：领域地图补空间坐标，论文路线补问题推进，学习向导把前沿倒推成学习任务。
5. 最后解释为什么要回看 `01`：当学生已经理解领域、论文和学习路径后，再回看履历、论文分组和来源风险，才能避免把早期方向、旁支论文和当前主线混在一起。
6. 用短表把阅读轮次、对应文档和阅读动作压缩成可扫描版本。

可用图文：优先使用“阅读轮次 / 文档 / 阅读动作 / 抓住什么”短表。可以用一行文本链路表达 `01 粗读 -> 02 领域 -> 03 论文 -> 04 学习 -> 01 回看`，但不建议使用 Mermaid 流程图；这个顺序太线性，画成图容易占空间却不增加理解。

出口检查：读者应能说清两点：第一，五份文档不是五篇并列报告，而是一条阅读路径；第二，`01_基础画像.md` 要读两次，第一次是建立事实底座，第二次是用更深的领域理解回头核查事实和风险。

常见错误：

- 只列文件名和作用，没解释为什么按这个顺序读。
- 把 `阅读顺序` 写成 `文件定位`，导致学生知道每份文档是什么，却不知道怎么使用。
- 省略最后回看 `01`，让事实核查只停留在开头。
- 在这里提前展开导师具体研究内容，抢走 `02/03/04` 的职责。
- 用横向很长的流程图表达简单顺序，导致 PDF 中显示很小、信息增量很低。

## 已确认规范

- 文档标题和章节标题要短。
- 文首必须有目录。
- 目录使用显式 anchor，避免中文标题在 Markdown/PDF 工具中跳转不稳定。
- `阅读顺序` 合并原来的“文件定位”。
- `阅读成果` 替代原来突兀的“起步讨论入口”。
- 使用边界分两层：`材料定位` 简短说明不做什么；`来源核查` 具体说明如何避免误信。
