# S3 文稿设计：把判断写成章节推进

状态：工作台重构稿。用于规定 `00-04` 和支撑方法怎样把带证据边界的判断、学习块和局部支撑，写成读者能一路跟下去的章节和段落。

S3 只管一个问题：

```text
S1 已经给出哪些判断能写，S2 已经给出学生要完成什么动作；现在怎样把它们排成章节、段落、证据语气和局部支撑。
```

S3 不做润色课，也不做“去 AI 味”清单。它要拦住一种更根本的失败：资料和原则都对，正文却像分类抽屉。学生每节都能看懂几句，合起来仍然说不出本文怎样把他从入口带到出口。

## 这份方法拦住什么失败

本项目的文稿问题远不止句子难看。更常见的是：每一节都有材料，每一段都有判断，读者却不知道上一节为什么出现、下一节为什么接上、哪个判断最重要、哪里只是弱线索。

反复出现的失败有这些：

- 章节像抽屉。`身份`、`领域`、`论文`、`学习资源` 被并排摆开，缺少从上一节到下一节的转换。
- 段落在空转。句子很顺，但只是在解释“这很重要”“这很复杂”“这提供基础”，没有新增判断、关系、证据边界或读者动作。
- 证据语气被抹平。官网事实、论文摘要判断、新闻线索、数据库关键词和人工复核点被写成同一种确定语气。
- 机械过渡代替真实接力。段落靠连接词排在一起，实际没有前后依赖。
- 图表、例子和引用漂浮。表格只是把字段切开，图只是把名词连起来，来源只堆在文末，后文没有消费它们。
- AI 味来自结构偷懒。很多句子看似规整，换一位导师也能成立，因为它没有使用本导师证据、本阶段判断和学生出口。

S3 的工作，是把这些失败变成写作门槛。每一节进入正文前都要能说清：

```text
本节接住什么？
本节使用哪些带边界判断？
本节让学生多理解什么？
本节需要什么局部支撑？
本节输出什么给下一节？
```

说不清这些，就不能继续扩写正文。

<a id="s3-source-to-judgment"></a>

## 调研资料怎样改变本方法

S3 只吸收能改变文稿动作的写作资料。

| 来源方向 | 可迁移判断 | 本项目吸收成什么动作 | 边界 |
|:---|:---|:---|:---|
| Gopen & Swan 的读者期待 | 读者会根据句子位置、段落重心和前后衔接判断重点。 | 每节开头交代承接和本节动作；段落结尾留下可被下一段使用的判断。 | 这条规则不要求固定句式；硬套“承上启下”模板，仍然会空。 |
| Mensh & Kording 的论文结构规则 | 好文章需要一个中心贡献，每个部分都要服务这个中心。 | 每份文档先写学生出口，再让章节沿着出口推进；与出口无关的材料降级或移走。 | 本项目是导师材料，不能照搬科研论文结构；这里只迁移“中心动作约束结构”。 |
| Weinberger 等关于科学写作的经验规则 | 短、紧、简单的句子能减少读者负担；多余修饰会遮住判断。 | 删除“重要、复杂、基础、显著”等空词，改写成具体证据、关系或读者动作。 | 简短不等于粗糙；该解释的术语和证据边界仍要展开。 |
| Purdue OWL 的段落组织 | 段落需要统一性、发展和连贯；一段不应混多个任务。 | 每段只做一个主要动作：承接、引入判断、解释关系、标证据边界、教读图、交接或要求输出。 | 这是通用段落规则，项目里还要受 S1 证据边界和 S2 学习出口约束。 |
| Schimel 的科学写作故事结构 | 研究写作要让读者看到问题、行动和结果的推进。 | 每节先给读者卡点，再写本节如何处理卡点，结尾交出一个可用结果。 | 这里的“故事”指问题推进，不能写成戏剧化或煽动性叙事。 |
| CARS 模型 | 学术引入常通过领域、缺口和本文动作来建立阅读位置。 | 章节入口可用“已有、缺口、本节动作”三步建立位置，避免直接抛资料。 | 不要求每节都写成论文 introduction；只在读者需要定位时使用。 |
| 图形写作规则 | 图要服务读者、信息和媒介，不能只为好看。 | S3 只安排局部支撑位置；真正图形类型和箭头语义交给 S4。 | 图文形式不能越过 S1 证据边界。 |

这些来源共同给出一个方法判断：文稿质量来自结构上的诚实。资料支撑到哪里，章节就推进到哪里；学生需要跨哪一步，段落就只服务那一步。

<a id="s3-input"></a>

## S3 从 S1 和 S2 接什么

S3 不从空白页开始写正文。它接住前两份方法给出的产物。

S1 交给 S3 的是正文材料的资格：

```text
稳定事实；
交叉判断；
弱线索；
冲突和缺口；
需要人工复核的位置；
这些判断能进入哪份文档。
```

S2 交给 S3 的是读者要走的学习动作：

```text
学生入口；
本块只解决什么；
暂时不展开什么；
读完要输出什么；
输出失败时回哪里；
下一节或下一份文档拿走什么。
```

S3 把两者合在一起，形成正文设计：

```text
哪条判断先进；
哪条判断需要解释；
哪条判断只能限定表达；
哪处需要表格、例子、证据卡或图；
哪段之后让学生输出或回看；
哪节交给下一节使用。
```

如果没有 S1 的证据边界，S3 会把弱线索写成漂亮结论。如果没有 S2 的学习出口，S3 会把正文写成顺滑资料库。

<a id="s3-writing-chain"></a>

## 成文推进链

S3 按这条链写正文：

```text
带边界判断 -> 本文主句 -> 章节接力 -> 段落任务 -> 局部支撑 -> 证据语气 -> 输出倒查
```

| 关口 | 要回答的问题 | 合格产物 | 不合格时停在哪里 |
|:---|:---|:---|:---|
| 带边界判断 | 哪些内容有资格进入正文？ | S1 判断和证据等级 | 回到 S1，别润色 |
| 本文主句 | 这份文档要把学生带到哪个出口？ | 一句可检查的文档动作 | 回到 S2，别排目录 |
| 章节接力 | 章节按什么依赖关系推进？ | 章节接力卡 | 章节可换序时停下 |
| 段落任务 | 每段具体做什么？ | 段落动作标签或等价自审 | 一段混多个任务时拆分 |
| 局部支撑 | 哪个卡点需要表、图、例子或证据卡？ | 支撑规格 | 说不出卡点就不用支撑 |
| 证据语气 | 判断写多稳？ | 直接事实、交叉判断、弱线索、复核点的不同写法 | 语气抹平时回 S1 |
| 输出倒查 | 学生能否完成本节或本文输出？ | 输出任务和回看位置 | 输出空泛时回 S2/S3 |

这条链让文稿设计可审查。审稿时要追问两层：句子是否顺，顺着读下去以后学生拿到了什么判断或动作。

<a id="s3-section-handoff"></a>

## 章节怎样接力

章节不能当材料容器。每节要接住前面交来的产物，处理一个新问题，再把结果交给下一节。

写每节前，先填这张卡：

```text
本节要干什么：
上一节交给我什么：
本节使用哪些 S1 判断：
本节承接哪个 S2 学习块或学生卡点：
本节暂时不展开什么：
本节需要什么局部支撑：
本节用什么证据语气：
本节交给下一节什么：
```

以 `03_论文路线.md` 的中段为例：

```text
本节要干什么：把论文按角色放进问题链。
上一节交给我什么：已经确认的相关论文集合和每篇论文大致问题。
本节使用哪些 S1 判断：论文归属稳定、摘要和正文能支撑粗角色判断；部分新闻线索只作旁支。
本节承接哪个 S2 学习块：学生要用 3 到 5 句话复述论文群怎样分工。
本节暂时不展开什么：每篇论文全部公式和完整背景。
本节需要什么局部支撑：论文角色表；如果关系复杂，再交给 S4 判断是否需要结构关系图。
本节用什么证据语气：角色判断写依据组合；弱线索写成“提示可能相关”。
本节交给下一节什么：目标论文、入口论文、平台论文和还需复核的旁支。
```

章节接力失败时，常见表现是每节都像完整说明，但删掉或调换一节后文档仍然能读。那说明章节没有形成依赖，只是把资料按标签分箱。

<a id="s3-paragraph-job"></a>

## 段落怎样承担任务

段落只做一个主要动作。常用动作有七个：

| 段落动作 | 它应该做什么 | 坏信号 |
|:---|:---|:---|
| 承接 | 接住上一段留下的判断或问题。 | 只靠连接词开头，没有真实依赖。 |
| 定位 | 告诉读者本段把内容放在哪个范围里。 | 一上来抛术语或论文名。 |
| 引入判断 | 给出本段要让读者接受的判断。 | 只说“很重要”“值得关注”。 |
| 解释关系 | 说明两个概念、论文、证据或学习任务怎样相连。 | 把多个名词排成一串。 |
| 标证据边界 | 说明这句话靠什么证据、稳到什么程度。 | 来源全在文末，正文语气全一样。 |
| 嵌入支撑 | 教读者怎样看表、图、例子或证据卡。 | 表图出现后正文不再使用它。 |
| 交接输出 | 把本段产物交给下一段、下一节或学生输出。 | 段落结尾只是总结空话。 |

写完每段后问：

```text
这一段接住前面什么？
这一段新增了哪一个判断、关系、边界或动作？
这一段有没有混进别的任务？
下一段会怎样使用它？
学生读完这一段能多说出什么？
```

如果一段只能回答“它让文章显得完整”，就删掉、拆开或移到内部材料。文稿密度来自每段都完成一个可见推进，不能靠塞更多信息制造。

<a id="s3-evidence-voice"></a>

## 证据语气怎样进入正文

S3 不重新判断证据强弱，但要把 S1 的证据边界写到正确位置。判断附近没有边界，学生就会把所有句子当成同等确定。

| S1 证据位置 | S3 正文写法 | 不该写成 |
|:---|:---|:---|
| 稳定事实 | 短句直接陈述，附近标来源。 | 长篇解释成“可能”“似乎”，反而削弱事实。 |
| 交叉判断 | 写清来源组合如何共同支撑。 | 装成单一事实，隐藏推理过程。 |
| 弱线索 | 加限定语，靠近判断写明只提示补查。 | 写成主线结论或章节标题。 |
| 冲突信息 | 写出冲突影响哪项判断，必要时降级。 | 用顺滑解释把冲突抹平。 |
| 需人工复核 | 明确标出复核原因和影响范围。 | 放到文末，让读者读正文时完全不知道风险。 |
| 暂不进入 | 留在内部记录，不进正文主线。 | 为了“完整”硬塞进段落。 |

正文语气要和章节功能一致。`01` 的稳定身份事实可以简短；`02` 的领域定位常常需要写依据组合；`03` 的论文角色如果只读到摘要和题名，就只能写粗角色；`04` 的学习路径如果资源接不到目标论文，就不能写成主线。

<a id="s3-local-support"></a>

## 局部支撑怎样嵌入

表格、例子、证据卡和图都要嵌在正文推进里。S3 只决定“这里是否需要局部支撑、支撑回答什么问题”，具体图形类型由 S4 进一步判断。

局部支撑按这个顺序出现：

```text
正文提出学生卡点
  -> 局部支撑回答卡点
  -> 支撑旁边写读法或证据边界
  -> 后文使用这个结果
```

| 卡点 | S3 可安排的支撑 | 交给 S4 的部分 |
|:---|:---|:---|
| 重复字段太多 | 表格或短清单 | 不需要图。 |
| 两个维度共同影响判断 | 矩阵候选 | 维度是否成立、矩阵是否必要。 |
| 领域位置难想象 | 范围定位图候选 | 证据是否足够支持位置表达。 |
| 论文角色分工难看清 | 角色表，必要时结构关系图候选 | 节点和连线语义。 |
| 先后、机制、学习依赖混在一起 | 路线图候选 | 箭头表示时间、机制还是学习依赖。 |
| 判断强弱容易误读 | 证据边界卡 | 是否需要视觉卡，还是正文限定语足够。 |

局部支撑不能浮在章节开头当装饰。它出现前，正文要先提出学生正在卡的关系；它出现后，后文要拿走它的结果继续推进。

<a id="s3-doc-roles"></a>

## 五份文档的文稿主线

S3 落到五份成品时，不给统一模板，只规定文稿推进方向。

| 文档 | 最容易写坏的样子 | S3 要改成什么推进 |
|:---|:---|:---|
| `00_材料导读.md` | 写成项目说明、证据规则和免责声明的集合。 | 从学生拿到材料的困惑出发，依次建立阅读顺序、自检、来源理解和回看路径。 |
| `01_基础画像.md` | 写成身份、履历、论文、指标的字段仓库。 | 先建立“我在看谁”和“资料稳不稳”，再让履历和论文集合成为后续 `02/03` 的输入。 |
| `02_领域地图.md` | 写成领域百科或概念表。 | 先给大领域范围感，再收束到导师方向，随后把关键概念和待核查假设交给 `03`。 |
| `03_论文路线.md` | 写成论文列表、年份线或支持 `02` 的材料筛选。 | 回到论文集合本身，先分析论文在问什么，再写问题链、角色分工和反向复核点。 |
| `04_学习向导.md` | 写成通用课程和资源推荐清单。 | 从学生基础课出发，逐步桥接到目标论文、核心图、方法缺口和进组前问题。 |

五份文档的语气也不同。`00` 要像使用说明，`01` 要像事实核查，`02` 要像领域定位，`03` 要像论文问题链分析，`04` 要像学习桥。混用语气会让学生分不清当前正在完成哪一步。

## 坏稿怎样改

坏稿常见写法：

```text
导师方向涉及凝聚态物理、超快光谱、强关联体系、非平衡动力学等内容。
这些方向在现代物理中十分重要，并与材料科学、量子调控和新型功能材料有关。
```

这段的问题在于缺少关系。学生只看到名词和重要性声明。

先按 S3 写成文卡：

```text
本段接住什么：01 的履历和论文集合里反复出现的方向词。
本段新增什么：区分大领域、观察工具和问题域。
证据边界：这里只做粗定位，后续还要由 03 的论文分析反向核查。
局部支撑：如果文字压不住范围感，再交给 S4 判断是否需要范围定位图。
交给下一段什么：学生先带着“范围、工具、问题域”三个层次进入领域定位。
```

再写正文：

```text
从 01 的履历和论文集合看，学生会先碰到一串相近的词：凝聚态物理、超快光谱、强关联体系、非平衡动力学。这里先不追每个词的完整定义，先分清它们在阅读里的位置。

凝聚态物理是较大的学科范围，关注大量粒子组成的材料和物质状态。超快光谱更像观察工具，用很短的光脉冲追踪材料在极短时间内怎样响应。强关联体系和非平衡动力学更靠近问题域：材料内部粒子相互作用很强，受到激发后会出现难以用简单模型解释的动态行为。

因此，本节先给一个粗略位置感：导师方向可先放在凝聚态物理里的超快物质科学附近阅读。这个定位还要在 03 回到论文集合后复核；如果论文群显示问题域更窄或方向偏移，02 要回写修正。
```

改后的段落没有追求华丽。它交给学生三个可用结果：大领域、观察工具、问题域；也留下了证据边界和后续复核位置。

<a id="s3-quality-check"></a>

## 质量检查

使用或重构 S3 后，用这组问题审查。答不上时停在成文层，不继续润色句子。

| 检查 | 合格表现 |
|:---|:---|
| 输入承接 | 正文能说清使用了哪些 S1 判断和 S2 学习块。 |
| 文档主句 | 这份文档有一个可检查的学生出口。 |
| 章节接力 | 每节能说出接住什么、推进什么、交出什么。 |
| 段落任务 | 每段只做一个主要动作。 |
| 证据语气 | 稳定事实、交叉判断、弱线索、冲突和复核点写法不同。 |
| 局部支撑 | 表、图、例子和证据卡都回答具体卡点，并被后文使用。 |
| 反 AI 味 | 删除空泛重要性、机械转折、模板总结和可替换导师的套话。 |
| 输出倒查 | 学生输出失败时能定位到 S1、S2、S3 或 S4 的具体问题。 |

最低停写条件：

```text
没有 S1 证据边界，不写确定判断。
没有 S2 学生出口，不排目录。
章节可以随意换序，停在章节接力。
段落只让文章显得完整，删掉或拆开。
图表没有被正文提出和消费，删掉或降级。
弱线索被写成主线，回到 S1 降级。
输出失败只能怪学生基础差，回到 S2/S3 倒查。
```
