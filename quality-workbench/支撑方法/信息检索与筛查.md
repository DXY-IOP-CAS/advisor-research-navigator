# S1 信息检索与筛查：让资料获得正文资格

状态：工作台重构稿。它是四份支撑方法里的第一块地基，负责回答公开资料怎样从搜索结果、数据库记录、论文页面、新闻和课程资料，变成可以进入 `00-04` 的正文判断。

本方法只管一件事：

```text
这条资料能支撑哪一个判断，不能支撑什么，应该写进哪份文档，证据不够时停在哪里补查。
```

读这份方法时先抓三句话：

```text
搜到的资料只是候选，不能直接进正文。
每条资料都要接到一个具体判断，并说明它不支撑什么。
证据边界决定正文位置：稳定事实进主线，弱线索进限定语，结构性弱判断进复核点。
```

文章节奏、学生练习和图文形式分别交给 S2、S3 和 S4。S1 交出去的产物应当是带边界的判断，而非一堆看起来有用的链接。

## 本项目反复出现的痛点

导师调研最容易在信息层出错。后面的文稿越顺，前面的错越难被学生发现。

用户反复指出的核心痛点是：资料没有经过资格审查就进了正文。官网、数据库、新闻、讲座、论文题名都能被放进段落里，读者看到很多来源，却看不出哪些资料真的支撑了身份、论文归属、领域定位或学习路径。

`01_基础画像.md` 的目标是尽量获得老师的全面事实信息。旧写法容易把 Google Scholar、OpenAlex、ORCID、arXiv 和 DOI 页面合并成论文集合，却没有说明同名、旧机构、数据库缺漏、机构滞后和主题漂移怎样处理。只把条目合并成表格，不能说明这些论文属于同一个人，也不能说明哪些资料已经锁定、哪些还要复核。

`02_领域地图.md` 的目标是尽量获得领域全局，再判断老师处在哪片大领域、小领域或前沿位置。旧写法容易把论文关键词、官网方向词或新闻里的平台名直接写成领域坐标。学生会得到一个看似专业的位置，但这个位置没有经过综述、教材、课程体系、相邻方向和近期论文的共同校验。

`03_论文路线.md` 的目标是回到相关履历阶段的论文集合，尽量全面理解论文内容，再找出问题链和角色分工。旧写法容易拿 `02` 的领域定位当靶子，只挑能支持这个定位的论文，然后把它们写成自洽路线。这会掩盖 `02` 可能偏宽、偏窄或偏错的问题。

`04_学习向导.md` 的目标是从学生已有基础课接到导师当前重点、目标论文和核心图。旧写法容易凭 AI 常识列课程顺序，或者把学习资源当推荐清单，却没有核查这些资源怎样连接到具体论文、图、方法或平台链路。

支撑方法本身也出过同样的问题。文末列了 Cochrane、PRISMA-S、PRESS、ACRL、OpenAlex、ORCID、Crossref 等资料，正文仍然按常识写原则。参考资料没有改变检索动作、证据语气或停写条件，就只是装饰。

还有一类反复出现的伤害是冲突被顺滑语言抹平。官网和数据库机构不一致、论文题名和履历阶段对不上、新闻方向和论文摘要不一致时，正文却给出整齐结论。这里应当降级或留复核点，不能用好看的句子补洞。

S1 的出口是：

```text
每个关键判断都能说出来源实际支撑了什么、不支撑什么、证据等级是什么、怎样进入正文、哪里还要人工复核。
```

<a id="s1-chain"></a>

## 资料资格链

检索的起点是文档问题，终点是带边界的正文判断。关键词和链接只是中间材料。S1 按这条资格链推进：

```text
文档问题
  -> 检索计划
  -> 候选资料篮
  -> 消歧和排除
  -> 来源能支撑什么
  -> 证据等级
  -> 冲突和缺口
  -> 正文资格
  -> 复核记录
```

| 关口 | 要回答的问题 | 合格产物 | 不合格时停在哪里 |
|:---|:---|:---|:---|
| 文档问题 | 这份文档需要判断什么 | 按 `00-04` 拆出的检索问题 | 问题还停在“找资料” |
| 检索计划 | 哪些来源能回答这个问题 | 来源类型、关键词、检索日期和预期证据 | 只有关键词，没有来源策略 |
| 候选资料篮 | 找到了哪些可能有用的资料 | 按身份、论文、领域、路线、学习资源分篮 | 搜到什么就写什么 |
| 消歧和排除 | 哪些资料属于本人，哪些是噪声 | 保留、排除、待复核记录 | 同名、旧机构、相邻主题混在一起 |
| 来源能支撑什么 | 资料实际证明了什么 | 资料到判断卡 | 来源只在文末出现 |
| 证据等级 | 判断能写多稳 | 直接证据、交叉证据、弱线索、需人工复核 | 所有来源同一种语气 |
| 冲突和缺口 | 哪些地方互相矛盾或缺资料 | 降级说明和补查任务 | 冲突被解释成整齐故事 |
| 正文资格 | 这条判断怎样进入哪份文档 | 稳定事实、限定判断、证据边界卡或暂不进入 | 弱线索承担结构判断 |
| 复核记录 | 后续怎样审查这条判断 | 关键判断证据表或文档内复核点 | 审稿只能凭感觉 |

这条链的要点是：资料先获得资格，再进入正文。任何一步跳过，后面的写作都可能变成资料转述。

<a id="s1-doc-questions"></a>

## 先按五份文档拆问题

同一条资料在不同文档里能承担的责任不同。导师官网可以锁定身份，也可以提供公开方向入口；它不能单独完成领域层级、论文路线或学习路径判断。检索前先按 `00-04` 拆问题。

| 文档 | S1 要帮它回答什么 | 优先来源 | 常见风险 |
|:---|:---|:---|:---|
| `00_材料导读.md` | 读者怎样理解来源符号、缺失、复核和五份材料的边界 | 本项目证据规则、成品来源表、关键判断核对表 | 把证据规则写成项目内黑话，学生看不懂 |
| `01_基础画像.md` | 老师是谁、身份是否锁定、履历和论文集合边界在哪里 | 官网、机构页、课题组页、简历、ORCID、Google Scholar、OpenAlex、DOI/期刊页 | 同名、旧机构、数据库噪声、论文归属过度合并 |
| `02_领域地图.md` | 当前方向处在哪片大领域、小问题域和相邻方向 | 官网方向、近期论文、综述、教材、课程讲义、权威领域页面 | 把论文关键词或新闻标题直接写成领域定位 |
| `03_论文路线.md` | 当前相关履历阶段的论文群怎样形成问题链 | 论文正文、题名、摘要、图表、DOI/期刊页、arXiv、综述、组内新闻或报告 | 按 `02` 的假设只挑支持材料，忽略论文集合本身 |
| `04_学习向导.md` | 学生从基础课到目标论文要补哪些概念、方法和读图能力 | 教材、课程讲义、综述、教程、目标论文、核心图 | 凭 AI 常识自造学习顺序，或写成通用课程表 |

拆完问题后，检索记录至少保留：

```text
文档：
判断问题：
检索日期：
检索词：
检索来源：
保留资料：
排除资料：
待复核资料：
本轮仍未回答的问题：
```

这份记录不一定进入成品正文，但它决定正文能写多稳。

<a id="s1-card"></a>

## 资料到判断卡

每条重要资料进入正文前，先过一张卡。卡片不追求长，追求把“来源”和“判断”接上。

```text
文档位置：
要支撑的判断：
候选来源：
来源实际支撑了什么：
来源不支撑什么：
证据等级：
正文写法：
需要补查或人工复核：
```

示例：

```text
文档位置：02_领域地图.md / 当前方向定位
要支撑的判断：导师当前方向可先放在某个二级领域中的某个小问题域阅读。
候选来源：官网方向、近三到五年论文、领域综述、课程讲义。
来源实际支撑了什么：官网给公开方向入口，论文给研究对象和方法重复模式，综述和课程给领域层级。
来源不支撑什么：单篇新闻不能定义领域位置；数据库关键词不能说明问题链。
证据等级：官网 + 论文 + 综述交叉证据。
正文写法：写成“适合先放在……附近阅读”，并说明依据组合。
需要补查或人工复核：如果论文群显示位置偏窄或偏宽，03 反向修正 02。
```

这张卡也适用于 `03`。如果 `02` 说某方向可能是当前重点，`03` 仍要回到论文集合本身，逐篇判断论文的问题、对象、方法、图表和相互关系。`02` 的定位只能给阅读假设，不能代替论文分析。

<a id="s1-source-roles"></a>

## 不同来源能承担什么

来源等级不能写成固定排名。ACRL 的信息素养框架强调，权威要看问题场景。导师官网对身份很强，对专业领域层级未必足够；综述对领域脉络很强，对某位导师论文归属没有决定权。S1 先看“这个来源能回答哪个问题”。

| 来源类型 | 适合支撑 | 风险边界 |
|:---|:---|:---|
| 导师官网、机构页、课题组页、简历 | 当前身份、机构、公开方向、履历入口 | 官网方向常是概括词，不能单独写成领域地图或论文路线 |
| ORCID、Google Scholar、OpenAlex、作者档案 | 作者 ID、论文候选、同名消歧线索 | 数据库可能合并错误、遗漏、机构滞后；必须和官网、DOI、题名、合作者交叉 |
| DOI、期刊页、出版社页、arXiv | 论文题名、作者、摘要、出版元数据、开放全文入口 | 元数据能核论文存在和归属线索，不能自动说明论文角色 |
| 综述、教材、课程讲义、教程 | 领域层级、概念入口、学习先修、常见方法 | 领域共识不能直接替代导师论文群分析 |
| 新闻、项目页、讲座、报告 | 近期主题、平台线索、研究动向、术语提示 | 多为弱线索，适合提示补查，不能单独承担结构判断 |
| Wikipedia、百科、博客 | 术语初筛和检索词扩展 | 不能单独支撑技术判断、履历判断、论文路线或学习顺序 |

Digital Inquiry Group 的 lateral reading 思路对这里有直接用处：不要在单个网页内部寻找全部答案，要离开该网页，用其他来源核查发布者、上下文和同一事实。用在导师调研里，就是每个关键判断都要离开原始页面横向核查。

<a id="s1-core-checks"></a>

## 核心核查对象

### 身份锁定

身份锁定要回答“这些资料是否指向同一位老师”。至少交叉：

```text
中文名 / 英文名 / 拼音
当前机构和部门
邮箱或个人主页
ORCID / Google Scholar / OpenAlex
合作者和论文题名主题
旧机构和履历年份
```

可以稳定写入 `01` 的判断：

```text
官网和机构页共同指向当前身份；作者档案和论文题名主题与官网方向一致；同名风险已记录。
```

停写条件：

```text
英文名、机构、作者 ID 或论文主题无法互相支撑时，不生成稳定论文集合。
```

### 论文集合

论文集合要回答“哪些论文可以进入当前导师材料”。不要只按数据库导出。每篇重要论文至少看：

```text
题名
作者列表
DOI 或 arXiv
机构署名
年份
合作者
摘要或全文入口
和当前方向的关系
```

论文集合的正文资格分三层：

| 资格 | 能做什么 |
|:---|:---|
| 归属稳定 | 可进入 `01` 论文表和后续分析候选 |
| 归属稳定但方向关系弱 | 可作为前史、旁支或背景，不能直接进入当前主线 |
| 归属或题名存疑 | 标 `需人工复核`，暂不承担结构判断 |

`03` 的论文路线必须从“归属稳定”的集合里重新筛当前相关论文。它不能只挑能支撑 `02` 定位的论文。

### 领域定位

领域定位要回答“学生先把导师方向放在哪片知识地图里”。这类判断不能只靠导师官网和论文关键词。较稳的组合是：

```text
官网公开方向
近期论文重复出现的对象、方法和问题
领域综述或教材给出的学科层级
课程或讲义给出的概念组织方式
相邻方向的边界
```

正文可以写成：

```text
从官网方向、近期论文模式和领域综述看，学生适合先把该方向放在……附近阅读。
```

如果只有新闻稿、讲座标题或数据库关键词，就写成线索，不写成领域结论。

### 论文路线

论文路线要回答“相关论文怎样共同推进问题”。S1 在这里不替 S3 写文稿，但要给出可分析的证据边界。

每篇进入 `03` 的论文，至少记录：

```text
它问什么问题
研究对象或体系是什么
用了什么方法、平台或模型
结果改变了什么
和其他论文是什么关系
证据来自题名、摘要、正文、图表还是外部新闻
```

如果只看题名和摘要，论文角色只能写到粗定位；如果要判断“核心入口”“平台论文”“问题推进”，通常要回到论文正文、图表、introduction 或领域综述。

### 学习资源

学习资源要回答“学生补哪段知识才接得上目标论文”。资源资格看三点：

```text
是否来自教材、课程讲义、综述或教程型资料
是否能解释目标论文里的概念、方法或图
是否适合学生当前基础
```

`04` 不能凭通用课程名直接排序。高数、线代、普物、量子、统计、光学或凝聚态内容，都要说明它连接到哪篇论文、哪张图、哪个概念或哪个实验链路。

<a id="s1-evidence-voice"></a>

## 证据等级和正文语气

正文里的语气要跟证据等级一致。

| 等级 | 判定方式 | 正文语气 |
|:---|:---|:---|
| 直接证据 | 来源直接说明该事实 | 可以稳定陈述，并给来源 |
| 交叉证据 | 多个来源从不同角度支持同一判断 | 写出依据组合，不装成单一事实 |
| 弱线索 | 单源、新闻、讲座、项目题名、数据库标签或相邻论文提示 | 写成线索、可能入口或待核查主题 |
| 需人工复核 | 弱线索会影响身份、领域、论文路线或学习顺序 | 明确标注，不能承担主线 |
| 暂不进入 | 来源与判断关系说不清 | 留在工作记录，不进入正文 |

正文写法示例：

```text
稳定事实：官网列出的当前机构为……
交叉判断：官网方向、近期论文题名和综述资料共同支持把方向先放在……
弱线索：讲座题目提示该主题可能是近期关注点，需回到论文或组内资料复核。
暂不进入：数据库关键词只用于扩展检索词，不支撑领域层级。
```

证据语气要在判断附近出现。读者不能只到文末参考资料里猜这句话有多稳。

<a id="s1-conflict"></a>

## 冲突、缺口和降级

冲突不一定是坏消息。它能告诉学生哪里还不能确信，也能告诉后续写作者哪里要补查。

| 冲突或缺口 | 处理方式 |
|:---|:---|
| 官网和数据库机构不一致 | 以官网或机构页作为当前身份入口，数据库机构写成收录滞后或待复核 |
| Google Scholar 和 OpenAlex 论文数差异大 | 写覆盖差异，不合成一个确定数字 |
| ORCID 论文少于数据库 | ORCID 只作身份线索，不能用缺失条目否定论文归属 |
| DOI 题名和正文来源表不一致 | 回到 Crossref、DataCite、出版社或 DOI 页面核题名 |
| 论文主题和履历阶段对不上 | 回查作者、机构署名、合作者和 DOI；仍不稳则降级 |
| 新闻方向和论文摘要不一致 | 新闻只保留为公开线索，不进入当前主线 |
| 学习资源与目标论文连接不上 | 换资源或标为背景读物，不放进学习主线 |

降级有三种做法：

```text
改成限定语：适合先作为线索阅读。
标成复核点：需人工复核。
移出正文主线：留在内部记录或参考资料。
```

强行解释冲突，会让文档显得完整，但会伤害学生判断力。

<a id="s1-doc-entry"></a>

## 怎样进入 `00-04`

S1 的产物进入五份文档时，要按文档职责落点。

| S1 产物 | 进入哪里 | 写法 |
|:---|:---|:---|
| 来源符号、证据等级、复核说明 | `00_材料导读.md` | 教学生怎样读 `[O#]`、`[P#]`、`[R#]`、`[B#]`、`[未找到]` 和 `需人工复核` |
| 身份、履历、论文集合、同名风险 | `01_基础画像.md` | 稳定事实直接写，冲突和缺口写在资料边界里 |
| 领域层级、相邻方向、概念入口 | `02_领域地图.md` | 写出官网、论文模式和领域资料的依据组合 |
| 当前相关论文、论文角色、弱旁支 | `03_论文路线.md` | 先回到论文集合全面分析，再写论文问题链 |
| 学习资源、先修概念、目标论文连接 | `04_学习向导.md` | 每项学习任务都连接到论文、图、方法或平台链路 |

进入正文前再问：

```text
这条判断是事实、解释、假设、线索还是复核点？
它会不会改变后续章节结构？
它有没有足够证据承担这个结构位置？
学生读完会不会把弱线索误认为确定结论？
```

如果答案不稳，先留在证据表或内部记录，不进入正文主线。

<a id="s1-handoff"></a>

## 和其他支撑方法怎样交接

S1 交给 S2 的核心是学生需要处理的证据难度。比如哪些概念缺背景、哪些判断需要学生知道“这是线索，还没到结论”、哪些复核点会影响阅读信心。

S1 交给 S3 的核心是正文语气和段落任务。稳定事实可以短句陈述；交叉判断要说明依据组合；弱线索要靠近判断写边界；冲突要变成复核段落或脚注式说明。

S1 交给 S4 的核心是证据能支持的关系。证据只能支撑粗定位，就不能画精细坐标；证据只能支撑先后顺序，就不要画成机制链；证据边界本身需要读者看见时，才使用证据卡。

S1 也会反向影响 M0。重构 S2-S4 后，如果发现某类资料资格链反复出现，就回到 M0 提炼成总方法；不要在 S1 里独自扩成总论。

<a id="s1-quality-check"></a>

## 质量检查

重构或使用 S1 后，按这组问题审查。答不上时停在信息层，不继续润色正文。

| 检查 | 合格表现 |
|:---|:---|
| 真实失败 | 本文先说明要拦住哪类信息错误 |
| 文档问题 | 已按 `00-04` 拆出检索和判断问题 |
| 检索计划 | 来源类型、关键词和检索日期可追溯 |
| 候选篮 | 身份、论文、领域、路线、学习资源分开存放 |
| 消歧 | 同名、旧机构、合作者和题名主题做过交叉 |
| 资料到判断 | 每个关键判断都有资料到判断卡或等价记录 |
| 证据语气 | 直接证据、交叉证据、弱线索和复核点写法不同 |
| 冲突处理 | 冲突被降级、解释或留作复核点 |
| 正文资格 | 弱线索没有承担结构性判断 |
| 交接 | S1 明确交给 S2、S3、S4 什么 |
| 参考资料 | 参考资料改变了本方法的动作或停写条件 |

最低停写条件：

```text
身份锁定不稳，不写稳定论文集合。
论文归属不稳，不写论文路线。
领域资料不足，不写领域定位。
只靠新闻或讲座，不写当前主线。
学习资源接不到目标论文，不写学习路径主线。
参考资料说不清怎样改变判断，就移出正文主线。
```

<a id="s1-reference-judgment"></a>

## 参考资料怎样改变本方法

S1 只吸收能改变执行动作的外部资料。

| 来源方向 | 可迁移判断 | 本项目吸收成的规则 | 边界 |
|:---|:---|:---|:---|
| Cochrane Handbook Chapter 4 | 系统检索要先规划问题、来源和策略，检索过程要记录 | 导师调研先拆 `00-04` 问题，再写检索计划和候选篮 | 本项目属于导师调研，目标是形成可审查判断，不追求穷尽全部文献 |
| PRISMA-S | 检索报告要能复查来源、策略和记录 | 关键检索要留下日期、检索词、来源、保留和排除结果 | 成品正文不展示完整检索流水 |
| PRESS | 检索策略本身需要被审查 | 重要判断写不出来时，先审查检索问题和关键词，再决定是否补正文 | 不要求每次都有第二人正式审稿 |
| ACRL Framework | 权威取决于问题场景 | 来源按“能回答什么问题”使用，不把官网、数据库、综述排成一条固定万能等级 | 具体学科仍要看导师和论文证据 |
| Digital Inquiry Group lateral reading | 单个网页不能自证，要横向核查 | 官网、新闻、数据库和百科都要离开原页面交叉核查 | 横向核查不能替代论文正文阅读 |
| OpenAlex Authors documentation | 作者实体元数据可用于消歧，也可能需要纠错 | OpenAlex 作为论文候选和同名线索，不能单独锁定论文集合 | 数据库算法结果不等于项目证据结论 |
| ORCID for researchers | 持久身份标识能辅助区分研究者 | ORCID 可辅助锁身份和作者档案，但缺失条目不能否定论文归属 | 很多研究者 ORCID 不完整 |
| Crossref / DataCite metadata | DOI 元数据可核题名、作者、出版信息 | DOI 来源用于核论文存在、题名和来源表准确性 | 元数据不说明论文在导师路线里的角色 |
