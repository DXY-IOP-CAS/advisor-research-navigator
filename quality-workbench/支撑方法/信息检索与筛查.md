<a id="info-main-question"></a>

## 中心问题

信息层要防止两种错误：查得不全，把缺口写成结论；筛得不准，把噪声写成事实。导师材料一旦在身份、论文归属或领域定位上错了，后续文字越顺，误导性越强。

本文件只追一个问题：

```text
一条搜索结果怎样变成可以写进正文的判断？
```

搜索结果只是候选。候选要经过消歧、交叉核查、证据分级和冲突处理，才允许进入正文。

<a id="info-chain"></a>

## 信息加工链

信息工作按这条链推进：

```text
拆问题
  -> 宽检索
  -> 建候选篮
  -> 消歧和排除
  -> 证据分级
  -> 冲突降级
  -> 生成正文判断
```

每一步都有产物：

| 步骤 | 产物 | 下一步怎样用 |
|:---|:---|:---|
| 拆问题 | 身份、论文、领域、课程等检索问题 | 决定去哪里找资料 |
| 宽检索 | 官网、数据库、论文、综述、课程、新闻等候选来源 | 建候选篮，不急着解释 |
| 建候选篮 | 候选身份、候选论文、候选领域词、候选课程资料 | 进入消歧 |
| 消歧和排除 | 保留、排除、待复核记录 | 避免同名和噪声进正文 |
| 证据分级 | 直接证据、交叉证据、弱线索、需人工复核 | 控制正文语气 |
| 冲突降级 | 冲突说明和复核点 | 防止假确定结论 |
| 生成正文判断 | 带证据边界的事实或分析句 | 交给 `01/02` 正文 |

最危险的位置，是从宽检索直接跳到正文。执行 AI 搜到一堆资料后，越想快写，越容易把候选当确认。

<a id="info-questions"></a>

## 先拆问题再检索

检索前先写问题，不先堆关键词。

| 问题 | 优先来源 | 辅助来源 | 不能单独支撑什么 |
|:---|:---|:---|:---|
| 这位老师是谁 | 官网、机构页、课题组页、简历 | 新闻、会议页、个人主页 | 研究方向判断 |
| 哪些论文属于本人 | Google Scholar、ORCID、OpenAlex、DOI、期刊页 | arXiv、Semantic Scholar、会议页 | 当前方向结论 |
| 履历阶段怎样分 | 简历、机构页、论文署名、年份 | 新闻、项目页 | 评价导师强弱 |
| 当前方向在哪里 | 官网方向、近期论文、综述、课程体系 | 讲座、新闻、项目题名 | 领域层级 |
| 学习路径靠什么定 | 教材、课程讲义、综述、目标论文 | 通用百科、博客 | 专业共识 |

关键词从问题里来：

```text
姓名：中文名、英文名、拼音、常见缩写
机构：当前机构、旧机构、课题组
论文：题名、DOI、合作者、期刊、年份
方向：中文词、英文词、缩写、同义词
领域资料：review、lecture notes、course、textbook、tutorial
```

检索记录至少保留检索日期、检索问题、检索词、检索源、保留结果、排除结果和未解决问题。

<a id="info-candidate-to-claim"></a>

## 候选怎样进入判断

以 `02_领域地图.md` 的领域定位为例，执行 AI 可能先看到这些候选：

```text
官网方向：超快物质科学、凝聚态相关方向
近期论文：出现 ultrafast、strongly correlated、materials dynamics 等词
综述资料：超快光谱常用于观察材料的非平衡响应
新闻 / 讲座：提到某个新材料或平台
```

这些候选还不能直接写成“导师研究某某前沿交叉领域”。先分层：

| 候选信息 | 证据位置 | 能进入正文的方式 |
|:---|:---|:---|
| 官网当前方向 | 直接证据 | 说明当前公开方向入口 |
| 近期论文反复出现的对象和方法 | 交叉证据 | 支撑“小问题域”和 `03` 论文路线 |
| 综述或课程中的领域说明 | 直接 / 交叉证据 | 解释大领域、相邻方向和学习入口 |
| 新闻、讲座、项目标题 | 弱线索 | 提示近期主题，不单独定义领域层级 |

正文判断应带着证据边界：

```text
从官网方向和近期论文看，导师当前材料更适合先放在凝聚态物理中的超快物质科学附近阅读。这个定位来自官网方向、近期论文题名 / 摘要与领域综述的交叉；新闻和讲座只作为近期主题线索，不单独支撑领域层级。
```

这句话同时交代位置、依据和限制。它才是信息层交给正文的合格产物。

<a id="info-evidence"></a>

## 证据分级和冲突处理

正文里的判断要能回到证据等级。

| 等级 | 含义 | 可写法 |
|:---|:---|:---|
| 直接证据 | 来源直接说出了这个事实 | “官网列出当前机构为……” |
| 交叉证据 | 多个来源互相支持，但没有单一来源完整说明 | “官网身份、作者档案和论文指纹互相支撑……” |
| 弱线索 | 单源、元数据、新闻、讲座或相邻线索 | “可作为近期主题线索” |
| 需人工复核 | 会影响身份、论文主线、领域定位或学习路径的弱判断 | 直接标 `需人工复核` |

冲突先降级，再解释：

| 冲突 | 处理 |
|:---|:---|
| 官网和数据库机构不一致 | 以官网当前身份为主，数据库机构标收录滞后或需复核 |
| GS 和 OpenAlex 论文数差异大 | 写清覆盖差异，不合成一个确定数 |
| 论文主题和履历阶段不匹配 | 回查合作者、机构署名和 DOI；仍不稳则标 `需人工复核` |
| 领域词来自新闻稿，论文中不明显 | 降级为公开线索，不进入当前主线 |
| 综述领域层级和论文关键词不一致 | 找更贴近论文群的综述或课程资料 |

冲突本身也是信息。把冲突写出来，比给出假确定结论更可靠。

<a id="info-doc-roles"></a>

## `01` 和 `02` 的分工

`01_基础画像.md` 处理“这个老师是谁，哪些公开信息可以放在同一个事实底座里读”。

`01` 写作前至少锁住：

```text
官方身份
英文名和作者 ID
履历阶段
论文集合
同名风险
公开交叉线索
数据库质量限制
```

`02_领域地图.md` 处理“老师当前方向处在哪个领域层级，学生怎样从大领域进入小问题域”。

`02` 写作前至少锁住：

```text
一级学科
二级方向
小问题域
方法和平台
相邻方向
当前阶段
领域共识来源
```

两份文档的边界要清楚：`01` 不急着解释完整领域，`02` 不能只复述 `01` 的论文关键词。`01` 交出事实底座，`02` 用事实底座和领域资料建立专业坐标。

<a id="info-writing-entry"></a>

## 正文落点

筛查完成后，信息进入正文有四种落点：

| 信息状态 | 正文落点 | 写法 |
|:---|:---|:---|
| 稳定事实 | 身份、履历、论文集合 | 明确写出并标来源 |
| 交叉判断 | 当前方向、论文主线、领域位置 | 写出依据组合 |
| 弱线索 | 新闻、讲座、单源项目、元数据异常 | 保留限定语，不承担结构判断 |
| 需人工复核 | 影响后续结构的弱判断 | 暂停强结论，标复核点 |

如果某条弱信息会决定 `02` 领域地图、`03` 论文路线或 `04` 学习路径，先继续查，别让它进入主线。

<a id="info-quality-check"></a>

## 质量检查

生成或修订 `01/02` 前，执行者至少回答：

| 检查 | 通过标准 |
|:---|:---|
| 身份 | 当前身份有官网或机构页支撑 |
| 同名 | 英文名、作者 ID、机构、合作者和题名主题做过交叉 |
| 论文 | 论文集合按 DOI、题名、合作者、机构署名核查 |
| 阶段 | 履历阶段能解释论文分组 |
| 当前方向 | 主要来自当前或最近相关阶段 |
| 领域 | `02` 的领域层级有综述、教材、课程或权威领域资料支撑 |
| 弱线索 | 新闻、讲座和项目只作为辅助线索 |
| 冲突 | 冲突来源被降级或标复核 |
| 正文 | 每个关键判断能回到证据等级 |

答不上这些问题时，先补信息层，不继续润色正文。
