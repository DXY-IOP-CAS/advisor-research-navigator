<a id="workbench-role"></a>
## 文档职责

这份文档只管信息层：怎样找资料、筛资料、消除同名和噪声、判断证据强弱，再把结果交给 `01_基础画像.md` 和 `02_领域地图.md`。

它不替代五份成品文档，也不直接规定最终正文话术。它解决一个更早的问题：成品开始写之前，执行者手里的信息是否足够完整、足够可靠、足够清楚地标出风险。

`01` 和 `02` 都靠信息层托底：

- `01` 需要确认老师是谁、论文是不是本人、履历阶段和论文阶段能否对上。
- `02` 需要确认老师当前方向处在哪个领域层级、哪些概念是领域共识、哪些只是从少数论文推出来的线索。

如果信息层错了，后续文档写得越顺，误导性越强。

<a id="borrowed-principles"></a>
## 借鉴原则

外部成熟方法给本项目提供几个约束，但不能照搬成系统综述流程。

| 来源 | 可借鉴的点 | 本项目怎么用 |
|:---|:---|:---|
| Cochrane Handbook | 检索前先规划问题、来源和策略，检索后记录过程 | 生成 `01/02` 前先写清要找什么、去哪里找、找不到怎么办 |
| PRISMA-S | 检索过程应可报告、可复查 | 记录数据库、关键词、检索日期、补充检索和限制条件 |
| PRESS | 检索策略本身需要被审查 | 检查关键词是否漏掉英文名、缩写、旧机构、方向同义词 |
| ORCID / OpenAlex / Crossref | 作者、论文和机构元数据可用于消歧 | 用 ID、机构、合作者、DOI、论文题名和年份交叉判断 |
| ACRL 信息素养框架 | 权威取决于问题场景 | 官网可支撑身份，论文可支撑研究内容，新闻只能辅助 |
| 横向阅读 | 不让单个网页自证 | 看到一个判断后，跳到其他来源核查它是否能对上 |

<a id="workflow"></a>
## 总流程

信息工作按这个顺序走：

```text
问题拆解 -> 宽检索 -> 候选集 -> 消歧筛查 -> 证据分级 -> 冲突处理 -> 信息分析 -> 成品写作
```

每一步只回答自己的问题。

| 步骤 | 要回答的问题 | 输出物 |
|:---|:---|:---|
| 问题拆解 | 这次要确认哪些身份、论文和领域问题？ | 检索问题清单 |
| 宽检索 | 哪些来源可能包含相关信息？ | 候选来源和候选论文 |
| 候选集 | 哪些信息先保留下来等待判断？ | 候选身份、候选论文、候选领域词 |
| 消歧筛查 | 哪些属于本人，哪些是同名或噪声？ | 保留、排除、待复核记录 |
| 证据分级 | 每条判断靠什么支撑？ | 直接证据、交叉证据、弱证据、需人工复核 |
| 冲突处理 | 来源互相矛盾时怎样降级？ | 风险说明和复核点 |
| 信息分析 | 事实集合能推出什么，不能推出什么？ | `01/02` 写作输入 |
| 成品写作 | 怎样把信息写成学生能读的文档？ | `01/02` 正文 |

不要跳过“候选集”和“消歧筛查”。直接从搜索结果进入正文，是最容易产生幻觉的位置。

<a id="phase1-info-task"></a>
## `01` 的信息任务

`01_基础画像.md` 的信息目标是确认“这个老师是谁，哪些论文和公开信息可以放在同一个事实底座里阅读”。

`01` 写作前至少要完成这些核查：

| 核查对象 | 需要找什么 | 判断重点 |
|:---|:---|:---|
| 官方身份 | 官网、机构页、课题组页、简历、邮箱、职务 | 当前机构和姓名写法是否稳定 |
| 英文名与 ID | 英文名、ORCID、Google Scholar、OpenAlex、个人主页链接 | 多个身份入口是否指向同一人 |
| 履历阶段 | 时间、机构、职位、研究环境变化 | 阶段边界是否能解释论文分组 |
| 论文集合 | GS、OpenAlex、arXiv、DOI、期刊页 | 论文是否属于本人，是否有同名噪声 |
| 论文指纹 | 题名主题、合作者、机构署名、年份连续性 | 单篇论文是否能归入本人集合 |
| 公开交叉线索 | 合作者、合作机构、讲座、新闻、项目 | 是否补强身份和履历阶段判断 |
| 数据质量 | 引用、h-index、缺 DOI、单源论文、收录延迟 | 哪些数字只能当数据库记录 |

`01` 的核心风险：

- 把同名作者论文混入本人论文集合。
- 把早期方向、旁支论文或合作者方向当成当前方向。
- 把数据库指标写成导师评价。
- 把新闻稿、讲座标题或项目标题当作强证据。
- 只看官网简介，不检查论文集合和作者档案是否对得上。

<a id="phase2-info-task"></a>
## `02` 的信息任务

`02_领域地图.md` 的信息目标是确认“老师当前方向处在哪个领域层级，学生应该怎样从大领域进入小问题域”。

`02` 写作前至少要完成这些核查：

| 核查对象 | 需要找什么 | 判断重点 |
|:---|:---|:---|
| 一级学科 | 机构方向、论文分类、综述、课程体系 | 不能把小技术写成整个学科 |
| 二级学科 | 综述、教材、课程讲义、权威领域页面 | 领域名称是否属于专业共识 |
| 小问题域 | 近期论文、综述小节、会议主题、课题组描述 | 是否贴近导师当前阶段 |
| 方法和平台 | 论文方法、实验装置、理论模型、数据来源 | 方法是否被误写成研究目标 |
| 前史关系 | 早期履历、早期论文、领域发展脉络 | 早期方向怎样影响当前方向 |
| 当前阶段 | 当前机构、近期论文、项目和讲座线索 | 后续 `03/04` 应关注哪一段 |

`02` 的核心风险：

- 只依赖 `01` 的论文表，缺少领域综述、教材或课程材料。
- 把导师论文题名里的关键词直接拼成领域地图。
- 把百科词条改写成领域说明。
- 把合作者领域、早期方向或单篇论文方向写成当前主线。
- 领域层级过粗，学生看完仍不知道老师具体处在什么问题域。

<a id="search-strategy"></a>
## 检索策略

检索前先拆概念，再搜。不要只用“导师姓名 + 研究方向”一组词。

### 检索概念

| 概念 | 例子 | 用途 |
|:---|:---|:---|
| 人名 | 中文名、英文名、拼音、常见缩写 | 找身份入口和作者档案 |
| 当前机构 | 大学、学院、研究所、课题组 | 锁定当前身份 |
| 旧机构 | 博士、博士后、前任单位 | 划分履历阶段 |
| 方向词 | 中文词、英文词、缩写、同义词 | 连接论文和领域资料 |
| 论文题名 | 近期论文、最高相关论文、目标论文 | 反查 DOI、期刊页和相关综述 |
| 合作者 | 高频共作者、课题组成员、合作机构 | 交叉核查身份和论文归属 |
| 领域资源 | review、lecture notes、course、textbook、tutorial | 建立 `02/04` 的专业共识 |

### 来源顺序

按问题选择来源，不按“看起来权威”一刀切。

| 问题 | 优先来源 | 辅助来源 |
|:---|:---|:---|
| 当前身份 | 官网、机构页、课题组页、简历 | 新闻、会议页、个人主页 |
| 论文归属 | Google Scholar、ORCID、OpenAlex、DOI、期刊页 | arXiv、Semantic Scholar、会议页 |
| 作者消歧 | ORCID、邮箱域名、机构署名、合作者、题名主题 | 数据库作者页、新闻页 |
| 领域定位 | 综述、教材、课程讲义、权威领域页面 | 论文 introduction、百科词条 |
| 平台链路 | 论文方法、课题组平台页、机构新闻、项目页 | 讲座摘要、新闻稿 |

百科和通用问答页面只能帮助识别术语，不能单独支撑领域判断。

### 检索记录

每次检索至少留下这些记录。现在先放在 workbench 规则里，后续再决定是否变成 `_internal/` 模板。

| 项目 | 记录什么 |
|:---|:---|
| 检索日期 | 公开信息会变化 |
| 检索问题 | 这次要确认身份、论文、领域还是平台 |
| 检索词 | 中文、英文、缩写、旧机构、论文题名 |
| 检索源 | 官网、GS、OpenAlex、ORCID、arXiv、Crossref、综述、课程 |
| 命中结果 | 保留哪些候选，为什么 |
| 排除结果 | 排除哪些候选，为什么 |
| 未解决问题 | 哪些地方标 `需人工复核` |

<a id="screening-disambiguation"></a>
## 筛查与消歧

筛查时先保守，后解释。不能为了让材料完整，把弱匹配塞进确定事实。

### 身份消歧

同名风险较高时，至少检查五类线索：

- 当前机构和官网身份。
- 邮箱域名、ORCID 或个人主页链接。
- 作者档案中的机构和研究主题。
- 论文题名主题、合作者和机构署名。
- 年份是否和履历阶段连续。

多个线索一致时，才能把候选身份合并阅读。只靠姓名一致，不足以合并。

### 论文筛查

论文进入本人集合前，先判断它的证据来源：

| 情况 | 处理 |
|:---|:---|
| GS / ORCID / 官网论文列表中出现 | 可作为强候选，再用 DOI 或期刊页补元数据 |
| OpenAlex 作者页出现，题名和合作者也对得上 | 可保留，标明数据库来源 |
| arXiv-only，有 DOI 或能与其他来源匹配 | 可保留，记录匹配依据 |
| arXiv-only，无 DOI、无合作者交叉、题名主题游离 | 降级或排除 |
| 单源出现且会影响当前方向判断 | 标 `需人工复核`，不要写成确定结论 |

### 领域筛查

领域词进入 `02` 前，也要筛查。

| 领域词来源 | 可用方式 |
|:---|:---|
| 官网方向和当前课题组描述 | 可作为当前方向入口，但仍要用论文核查 |
| 近期论文题名 / 摘要 | 可提示小问题域，不能单独定义领域层级 |
| 综述和教材 | 可支撑领域层级和学习路径 |
| 课程讲义 | 可支撑入门顺序和先修关系 |
| 新闻和讲座 | 可提示近期主题，只能作弱线索 |

<a id="evidence-grading"></a>
## 证据分级

正文里的判断要能回到证据等级。

| 等级 | 含义 | 可写法 |
|:---|:---|:---|
| 直接证据 | 来源直接说出了这个事实 | “官网列出当前机构为……” |
| 交叉证据 | 多个来源互相支持，但没有单一来源完整说明 | “官网身份、作者档案和论文指纹互相支撑……” |
| 弱证据 | 只有单源、元数据或相邻线索支撑 | “可作为初步线索” |
| 需人工复核 | 会影响身份、当前方向、论文主线或学习路径的弱判断 | 直接标 `需人工复核` |

证据等级不能被文风抹平。弱证据写得再顺，也还是弱证据。

<a id="conflict-handling"></a>
## 冲突处理

来源冲突时，先降级，不要解释圆满。

| 冲突 | 处理 |
|:---|:---|
| 官网和数据库机构不一致 | 以官网当前身份为主，数据库机构标收录滞后或需复核 |
| GS 和 OpenAlex 论文数差异大 | 写清覆盖差异，不强行合并成一个确定数 |
| 论文主题和履历阶段不匹配 | 回查合作者、机构署名和 DOI；仍不稳则标 `需人工复核` |
| 领域词来自新闻稿，论文中不明显 | 降级为公开线索，不进入当前主线 |
| 综述领域层级和论文关键词不一致 | 先找更贴近论文群的综述或课程资料 |

冲突本身也是信息。把冲突写出来，比给出假确定结论更有价值。

<a id="analysis-entry"></a>
## 信息分析入口

完成筛查后，再进入分析。分析的任务是判断哪些信息能进入哪份文档，哪些只能留作风险提示。

| 输入 | 进入哪里 | 使用方式 |
|:---|:---|:---|
| 身份、履历、论文集合 | `01` | 建立事实底座和风险边界 |
| 当前方向入口词 | `01` / `02` | `01` 只给入口，`02` 建立领域坐标 |
| 领域层级和综述 | `02` | 建立一级、二级、小问题域 |
| 近期论文群 | `02` / `03` | `02` 定位，`03` 分析问题链 |
| 教材、课程、讲义 | `04` | 设计从基础到论文的学习路径 |

如果某条信息只能靠弱证据支撑，但会决定后续文档结构，先暂停写作，继续检索或标 `需人工复核`。

<a id="execution-checklist"></a>
## 执行清单

生成或试写 `01/02` 前，执行者至少回答这些问题：

- 当前身份是否有官网或机构页支撑？
- 英文名、ORCID、GS、OpenAlex、arXiv 是否存在同名风险？
- 论文集合是否按 DOI、题名、合作者、机构署名做过交叉核查？
- 哪些论文是单源或弱匹配？
- 履历阶段是否能解释论文分组？
- 当前方向是否主要来自当前或最近相关阶段？
- `02` 的领域层级是否有综述、教材、课程或权威领域资料支撑？
- 新闻、讲座和项目是否只作为辅助线索使用？
- 每个关键判断是否有证据等级？
- 影响结构的弱判断是否标了 `需人工复核`？

答不上这些问题时，不要继续润色正文。
