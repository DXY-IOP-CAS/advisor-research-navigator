<a id="learning-main-question"></a>

## 中心问题

学习设计要解决“看过但不会用”的问题。学生读导师材料时，最危险的状态是觉得自己看懂了，合上文档却说不出导师方向、论文问题链、证据风险和下一步学习缺口。

本项目采用一条学习动作链：

```text
建立主线 -> 少量输入 -> 主动输出 -> 自检卡点 -> 回退补洞 -> 继续下一块
```

这条链来自两部分：学习科学中关于认知负荷、检索练习、自我解释和元认知校准的研究；用户自己的经验，即“主线剧情 + 费曼输出 + 苏格拉底式小问题”。这些原则不直接写成教育学讲义，而要变成学生能做的阅读动作。

<a id="learning-action-chain"></a>

## 学习动作链

每份文档至少承担链条中的一个动作。

| 动作 | 学生要做什么 | 文档怎样支撑 |
|:---|:---|:---|
| 建立主线 | 知道这一轮只解决什么问题 | 开头写清读者起点、本文任务和读完出口 |
| 少量输入 | 只接收当前必要信息 | 概念、论文、图表按主线分批出现 |
| 主动输出 | 不看原文复述一句判断、一个链条或一个问题 | 每份文档设置小输出 |
| 自检卡点 | 发现自己哪里只是看懂表面 | 提供“答不上就回看哪一节” |
| 回退补洞 | 回到事实、领域、论文或课程缺口补一小块 | 五份文档之间提供回看路径 |
| 继续下一块 | 带着上一步输出进入下一份材料 | 文末给下一步入口 |

资料堆只给输入。学习路径要让学生输出，并且在输出失败时知道回哪里修。

<a id="learning-block"></a>

## 学习块怎样设计

学习块是一段可进入、可完成、可回看的阅读任务。它避免学生一边读文档，一边查旁支，一边开 AI 无限发散。

一个学习块写成六句话：

```text
本轮只读哪份材料：
本轮只解决什么问题：
暂时不展开什么：
读完要输出什么：
答不上时回看哪里：
下次从哪里继续：
```

以 `03_论文路线.md` 为例：

```text
本轮只读 03 的论文问题链。
本轮只解决：这些论文围绕什么问题分工推进。
暂时不展开：每篇论文的全部背景和所有公式。
读完要输出：用 3 到 5 句话复述问题链。
答不上时回看：论文角色图和代表论文说明；问题本身听不懂就回 02。
下次继续：带着这段复述进入 04，倒推要补的课程知识和核心图读法。
```

这就是费曼输出和苏格拉底式提问的落点。文档不只告诉学生“去复述”，还告诉他复述什么、卡住时回哪里。

<a id="learning-doc-roles"></a>

## 五份文档怎样承担学习任务

| 文档 | 学习任务 | 读完输出 | 回退位置 |
|:---|:---|:---|:---|
| `00_材料导读.md` | 建立阅读顺序和自检方式 | 说出五份材料怎样读，卡住时回看哪里 | 回到对应文档入口 |
| `01_基础画像.md` | 建立事实底座 | 说出老师是谁、资料是否锁定、论文集合有什么风险 | 回到来源和证据说明 |
| `02_领域地图.md` | 建立专业范围感 | 说出导师方向处在哪片领域，旁边有什么方向 | 回到领域定位和关键概念 |
| `03_论文路线.md` | 建立论文问题链 | 用 3 到 5 句话复述论文群怎样分工 | 回到论文角色和证据边界 |
| `04_学习向导.md` | 从基础课走到目标论文 | 列出课程缺口、论文图读法和进组前问题 | 回到 `02/03/01` 的相应位置 |

这里的输出承担校准作用，用来发现“我以为懂了”的位置。

<a id="learning-integration"></a>

## 怎样融入正文

学习原则要藏进正文动作里，避免把成品写成学习方法课。

推荐写法：

| 内部原则 | 正文里的自然写法 |
|:---|:---|
| 控制认知负荷 | “第一遍只抓当前方向和证据风险，不要求读懂每篇论文。” |
| 主动输出 | “读完本节，试着不看原文说出这位老师当前方向的大致位置。” |
| 自我解释 | “如果你说不出为什么这篇论文属于方法角色，回看论文角色说明。” |
| 元认知校准 | “答不上这个问题，说明你还缺领域坐标，先回到 02。” |
| 单任务沉浸 | “这一轮只解决论文问题链，旁支术语先记到稍后查。” |

AI 辅助学习时，可以让学生用一个轻量提示开局：

```text
本轮主线：
本轮只解决：
我已经知道：
我卡在：
不要展开：
最后请给我：
```

这个提示的作用是管住发散，让 AI 也围绕同一个学习块工作。

<a id="learning-quality-check"></a>

## 质量检查

每份文档写完后，用学生输出倒查：

| 检查 | 通过标准 |
|:---|:---|
| 主线 | 学生知道这一轮只解决什么问题 |
| 输入负荷 | 概念、论文和图表没有一次性倒满 |
| 输出任务 | 文档给出具体输出，不停在“了解”“掌握” |
| 自检 | 答不上时能找到回看位置 |
| 回退 | 五份文档之间有清楚回退路径 |
| 继续 | 读完后知道下一份材料拿走什么 |
| 语言 | 不把教育学术语直接丢给学生 |

如果学生无法完成输出，先回看文档是否铺好了台阶，不先归因于学生基础差。
