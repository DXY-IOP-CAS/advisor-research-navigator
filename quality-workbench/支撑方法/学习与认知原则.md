# S2 学习与认知原则：把看过材料变成会用材料

状态：工作台重构稿。用于规定 `00-04` 怎样把公开资料、论文判断和学习资源写成学生能读、能复述、能自检、能回退的阅读任务。

S2 只管一个问题：

```text
学生读完这一份材料以后，能多做什么；如果做不到，应该回到哪一层补洞。
```

它接住 S1 交出的带边界判断，再把这些判断改成学习动作。资料已经经过检索、筛查和证据分级，仍然不能直接塞进正文。学生需要的是一段能走的路：知道这轮读什么，先放下什么，读完说出什么，答不上时回哪一节，带着什么进入下一份文档。

## 这份方法拦住什么失败

导师材料最容易制造一种假理解：学生一路看下来，觉得每个词都见过，合上文档却说不出这位老师当前在什么方向、论文群解决什么问题、哪些判断证据不稳、自己下一步要补什么。旧稿把“认知负荷、检索练习、自我解释、元认知”写出来了，但这些词没有稳定进入 `00-04` 的章节推进，所以执行时又回到资料堆和板块拼接。

项目里反复出现的学习层失败有这些：

- 文档给了很多概念、论文和来源，学生只能记住名词，不能复述关系。
- 输出任务停在“了解”“掌握”“熟悉”，没有可检查的学生动作。
- `04_学习向导.md` 变成通用课程清单，没有接到目标论文、核心图、方法或平台链路。
- 学生卡住时只知道“再查资料”，不知道回到 `01` 的证据边界、`02` 的领域坐标、`03` 的论文问题链，还是 `04` 的课程缺口。
- 章节之间没有接力。上一节没有留下可用产物，下一节只能重新堆内容。

S2 要把这些失败改成写作前的硬问题。每一份成品、每一个主要章节、每一个学习块都要能回答：

```text
我要干什么？
上一节交给了我什么？
我还需要调研、筛查和分析什么？
这一步能让学生多理解什么？
学生读完要输出什么？
学生答不上时回哪里？
下一节会拿走什么？
```

这组问题里的“调研、筛查和分析”不能省。省掉它，学习块会变成上节摘要；省掉学生输出，学习块会变成作者自说自话。

<a id="s2-source-to-judgment"></a>

## 调研资料怎样改变本方法

S2 不写学习科学综述，只吸收能改变 `00-04` 写法的判断。

| 来源方向 | 可迁移判断 | 本项目吸收成什么动作 | 边界 |
|:---|:---|:---|:---|
| 有效学习技术研究 | 主动提取比单纯重读更容易暴露理解问题，分布式练习也比临时重读可靠。 | 每份文档都要给学生一个合上材料也能完成的小输出，例如一句身份判断、三到五句论文问题链、一个学习缺口清单。 | 这不等同于给学生考试；输出只用于校准理解和回退。 |
| How People Learn II | 先验知识、迁移和元认知会影响学生能否把新材料接到已有知识。 | 每份文档开头要说明学生入口，正文里要给出“答不上说明缺哪块”的回看路径。 | 不能把学生差异泛化成固定标签；只能按任务和缺口调整台阶。 |
| 认知负荷理论 | 新手处理陌生信息时，工作记忆容易被过多元素压垮。 | 每个学习块只解决一个问题；概念、论文、图表分批进入；旁支先暂存。 | 控制负荷不代表降低学术内容，只是改进入顺序。 |
| ICAP 框架 | 学习活动从被动接收走向主动处理、自我建构和互动，理解通常更深。 | 学生不能只读；要复述、解释关系、指出证据边界，必要时用 AI 或同伴追问自己的回答。 | 不要求成品正文讲 ICAP 名词，只转成可见学习动作。 |
| 自我解释研究 | 让学习者解释“为什么这一步成立、为什么这个关系成立”能促进理解，但提示强度要适配水平。 | 自检问题要问原因和关系，例如“为什么这篇论文能当入口论文”“为什么这条证据只能算线索”。 | 学生完全没有背景时，需要先给短解释，再让他自我解释。 |
| 建构性对齐 | 目标、学习活动和评价方式应指向同一类表现。 | 文档入口、正文任务、读完输出、回看路径必须对齐；不能入口说看论文路线，输出却只让学生列术语。 | 这套思想来自课程设计，迁移到文档时只保留“出口和活动一致”。 |
| 偏好标签证据评估 | 按“视觉型、听觉型”等偏好匹配教学缺少强证据。 | 图、表、正文、路线图按任务选择，不按学生标签选择。学生需要看见关系时才用图。 | 这不否定图文配合，只否定把图当作万能适配。 |
| 注意残留研究 | 未完成任务会干扰下一项任务，频繁切换会降低当下任务质量。 | 阅读块要写“暂时不展开什么”；旁支资料进入暂存清单，别让学生边读边无限检索。 | 导师调研与办公任务实验不同，迁移时只取“减少未完成旁支干扰”的设计启发。 |

这些来源共同给出一个方法判断：学习设计的核心要放在入口、单一任务、可见输出和失败回路上，不能只给学生更多内容。

<a id="s2-action-chain"></a>

## 学习动作链

S2 把每份文档写成一条学习动作链：

```text
入口定位 -> 任务收窄 -> 主动提取 -> 自我解释 -> 失败诊断 -> 回退补洞 -> 交给下一份
```

| 动作 | 要回答的问题 | 落到文档里怎样写 |
|:---|:---|:---|
| 入口定位 | 学生进来时已经知道什么，还不知道什么？ | 开头用短段说明本文件接住前文哪项产物，本轮只带学生跨哪一步。 |
| 任务收窄 | 这轮只处理哪一个理解问题？ | 明确暂时不展开的内容，避免论文、概念、课程资源一次性涌入。 |
| 主动提取 | 学生合上文档能说出什么？ | 设置一句话、三到五句话、一个小表或一个问题清单作为输出。 |
| 自我解释 | 学生能否说明为什么这个判断成立？ | 追问“为什么这条证据足够”“为什么这篇论文承担这个角色”“为什么这个概念要先学”。 |
| 失败诊断 | 答不上说明哪里缺口还没有补？ | 把失败分回事实、领域、论文、方法、图读法或课程基础。 |
| 回退补洞 | 学生应该回到哪份文档、哪类证据或哪段学习任务？ | 在自检附近写清回看位置，不用泛泛说“回顾前文”。 |
| 交给下一份 | 下一份文档要拿走什么产物？ | 文末交出一项可用产物，例如领域坐标、问题链、目标论文、核心图或课程缺口。 |

这条链的关键是可检查。学生读完后如果只能说“我大概了解了”，S2 没有通过。合格输出要能被另一位读者追问、修正和带到下一份材料里。

<a id="s2-learning-block"></a>

## 学习块怎样设计

学习块是一段可以完成的阅读任务。它的大小由学生能否完成输出决定，不能由作者手里有多少资料决定。

写每个学习块前，先填这张卡：

```text
本块要干什么：
上一块交给我什么：
我还要调研、筛查或分析什么：
本块暂时不展开什么：
学生读完要输出什么：
输出失败时回哪里：
下一块拿走什么：
```

以 `03_论文路线.md` 为例，合格的学习块应接近这样：

```text
本块要干什么：判断当前相关论文群怎样推进问题链。
上一块交给我什么：02 给出的领域位置和问题候选，只能当阅读假设。
我还要调研、筛查或分析什么：回到相关履历阶段的论文集合，读题名、摘要、方法、对象、图表和相互引用，先归纳论文自身在问什么。
本块暂时不展开什么：每篇论文的完整推导、全部背景和所有旁支术语。
学生读完要输出什么：用 3 到 5 句话说明论文群怎样分工，指出哪些论文是入口、核心、平台、背景或弱线索。
输出失败时回哪里：分不清论文角色就回本文件的论文角色说明；问题本身听不懂就回 02；论文归属有疑问就回 01。
下一块拿走什么：04 拿走目标论文、核心图、概念缺口、方法缺口和进组前问题。
```

这个例子里，`02` 没有资格替 `03` 选好结论。`02` 给的是领域坐标和阅读假设；`03` 还要重新面对论文集合本身。论文读完后发现 `02` 的小领域判断偏宽、偏窄或偏错，就把它写成反向复核点。

<a id="s2-doc-roles"></a>

## 五份文档各自让学生跨哪一步

五份成品的学习任务连续，但每份都要自包含。学生可能只读其中一两份，所以每份开头都要交代本文件解决的问题、证据边界和读完出口。

| 文档 | 学生入口 | 本文件让学生跨的一步 | 读完输出 | 输出失败回哪里 |
|:---|:---|:---|:---|:---|
| `00_材料导读.md` | 学生拿到一套陌生材料，不知道先读哪份。 | 学会使用后面四份文档，理解来源、缺口、复核和回看方式。 | 说出五份材料的阅读顺序、卡住时回哪里、哪些判断需要复核。 | 回到 `00` 的阅读顺序、来源符号和复核说明。 |
| `01_基础画像.md` | 学生只知道姓名、机构路径和官网 URL。 | 尽量获得老师的全面事实信息：身份、机构、履历、论文集合、同名风险和资料边界。 | 说出老师是谁，哪些资料锁定，论文集合哪里稳、哪里需要复核。 | 回到身份锁定、论文归属、来源缺口和同名风险。 |
| `02_领域地图.md` | 学生有了基本事实和论文集合，但还没有专业范围感。 | 尽量获得领域全局，再定位导师方向在大领域、小领域或前沿方向中的位置。 | 说出导师方向适合先放在哪片领域读，旁边有哪些方向，哪些概念先抓。 | 回到领域层级、相邻方向、概念入口和证据边界。 |
| `03_论文路线.md` | 学生有领域假设，但还不知道论文群怎样互相关联。 | 回到当前方向相关履历阶段的论文集合，尽量理解论文内容，再分析问题链和角色分工。 | 用 3 到 5 句话复述论文群解决什么问题，哪些论文承担入口、核心、平台、背景或弱线索角色。 | 回到论文角色、目标论文、证据边界；必要时回 `02` 修领域坐标，回 `01` 核论文归属。 |
| `04_学习向导.md` | 学生知道当前研究重点和目标论文，但不知道怎样从基础课接过去。 | 从高数、线代、普物等基础课出发，逐步接到目标论文、核心图、方法、平台链路和进组前问题。 | 列出自己要补的课程知识、概念台阶、论文图读法和进组前问题。 | 回到 `03` 的目标论文和核心图，回 `02` 的概念入口，回 `01` 的事实边界。 |

这张表不当目录模板使用。它给每份文档规定“学习出口”。后续改 `D00-D04` 时，每份标准都要把自己的章节接到这一出口上。

<a id="s2-output-debug"></a>

## 输出失败时怎样倒查

学生输出失败时，先查文档设计，别急着归因于学生基础差。

| 输出失败表现 | 可能的文档问题 | 回到哪里修 |
|:---|:---|:---|
| 学生只能复述字段，不能说出主线。 | 正文把资料排成板块，没有写清本轮任务。 | 回到入口定位和任务收窄。 |
| 学生把弱线索当成确定结论。 | 证据边界没有靠近判断出现。 | 回到 S1 的证据分级和本文件的自检问题。 |
| 学生说不出论文之间的关系。 | `03` 没有完成论文内容分析，只按主题词分组。 | 回到论文集合、论文角色和问题链分析。 |
| 学生看到课程清单仍不知道先学什么。 | `04` 没有把课程知识接到目标论文、核心图或方法。 | 回到目标论文、核心图、课程缺口的桥接关系。 |
| 学生一读就开始查旁支，主线断掉。 | 学习块没有写暂时不展开的内容。 | 回到任务收窄和旁支暂存。 |
| 学生答不上自检题，也不知道回哪里。 | 自检只问结果，没有诊断失败来源。 | 回到失败诊断和回退补洞。 |
| 下一份文档用不上上一份。 | 上一份没有交出可用产物。 | 回到交给下一份的出口设计。 |

倒查时只改导致失败的那一层。学习块有问题就改学习块；证据边界有问题就回 S1；章节接力有问题交给 S3；图文支撑有问题交给 S4。

## 怎样写进正文

学习原则要藏进正文动作里，让学生自然完成任务。成品文档不讲教育学术语。

| 内部判断 | 正文里的写法 |
|:---|:---|
| 控制负荷 | “这一遍只抓当前方向和证据风险，先别追每篇论文的全部细节。” |
| 主动提取 | “读完本节，合上文档，用一句话说出这位老师当前方向适合先放在哪片领域里读。” |
| 自我解释 | “如果你说不出这篇论文为什么能当入口论文，回看它解决的问题、方法和图表位置。” |
| 元认知校准 | “答不上这个问题，说明你还缺领域坐标，先回到 `02_领域地图.md` 的概念入口。” |
| 旁支暂存 | “这轮先把术语写进暂存清单，等论文问题链清楚后再回头补。” |
| 出口对齐 | “本节输出会被下一节用来选择目标论文，所以只列论文题名还不够，要写出它承担的角色。” |

如果学生要让 AI 辅助阅读，可以给一个短提示：

```text
本轮只读：
本轮只解决：
我已经知道：
我卡在：
暂时不要展开：
请在结尾让我输出：
```

这个提示的作用是约束 AI 和学生留在同一个学习块里。它不能替代资料检索、论文阅读和证据判断。

<a id="s2-handoff"></a>

## 和其他支撑方法怎样交接

S1 交给 S2 的东西，是学生需要面对的证据难度。哪些判断稳、哪些是弱线索、哪些概念缺背景、哪些复核点会影响阅读信心，都要进入学习块设计。证据越弱，学生输出里越要看见限定语和复核位置。

S2 交给 S3 的东西，是章节和段落的学习任务。S3 写正文时，不能只追求顺滑，它要接住 S2 给出的入口、输出、自检和回退，把它们变成章节接力和段落工作。

S2 交给 S4 的东西，是学生工作记忆里的关系负担。学生需要看见范围、分工、路线或证据边界时，S4 再决定用正文、表格、矩阵、证据卡还是图。学生只是需要字段对照时，用表格；学生需要少背一段关系时，才考虑图。

S2 也会反向影响 M0。等 S3、S4 重构完，如果发现学习动作链需要改总方法的关卡命名或顺序，再回到 M0 统一处理。当前不要在 S2 里扩写总方法。

<a id="s2-quality-check"></a>

## 质量检查

使用或重构 S2 后，用这组问题审查。答不上时停在学习设计层，别继续润色正文。

| 检查 | 合格表现 |
|:---|:---|
| 真实失败 | 文档开头说清要拦住哪种“看过但不会用”的失败。 |
| 学生入口 | 说明学生进入本文件时已经有什么，还缺什么。 |
| 新增处理 | 写清本阶段还要调研、筛查和分析什么。 |
| 任务收窄 | 每个学习块只解决一个理解问题，旁支有暂存位置。 |
| 主动输出 | 学生读完要完成一个可检查动作。 |
| 自我解释 | 自检要追问判断为什么成立，不能只问结论。 |
| 回退路径 | 输出失败能回到具体文档、章节或证据缺口。 |
| 文档接力 | 本文件有产物交给下一份，不让下一份重新起头。 |
| 来源作用 | 参考资料改变了动作、边界或停写条件。 |

最低停写条件：

```text
学生出口说不清，停在入口定位。
学习块无法写出新增调研、筛查和分析，停在任务设计。
输出只写“了解”“掌握”“熟悉”，停在主动提取。
答不上时没有回看位置，停在失败诊断。
`04` 的学习资源接不到目标论文、核心图或方法，停在学习桥设计。
参考资料不能改变项目动作，移出正文主线。
```
