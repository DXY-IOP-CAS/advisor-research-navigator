# M0 总方法：从资料堆到五份成品理解路线

状态：工作台总方法稿。它位于四份支撑方法和 `00-04` 文档标准之上，用来规定资料怎样被加工成学生能读、能查、能继续学的五份成品。

本文件只做一件事：把公开资料、论文、领域背景、学习资源和图文形式，加工成学生能理解、能复述、能核查、能继续学习的路线。

工作台暂按三层命名：

```text
M0 总方法：规定从资料堆到五份成品的加工路线。
S1-S4 支撑方法：分别处理检索筛查、学习认知、文稿设计、可视化理解。
D00-D04 成品标准：分别规定五份学生文档怎样写。
```

这里先澄清标题和职责，不改文件名。后续要不要给四份支撑方法重命名或加编号，等支撑方法重修时一起处理。

`AI协作开发与质量控制.md` 属于执行协议。它要求执行 AI 在改本文件、支撑方法、文档标准、样本、skill、verifier 或 harness 前，先锁定层级、写任务合同、说明资料怎样改变判断，再做单层编辑、隔离 subagent 审查、反馈回灌和验证。这里吸收它的门禁，不展开成 AI 开发综述。

## 这份方法要解决的真实失败

工作台前几轮暴露的失败很具体。资料被收集了，文档也变长了，但学生仍然没有被带到理解出口。

第一种失败是资料转述。官网、论文、数据库、新闻和课程资料被排成章节，正文看起来有来源，实际只是把资料换成中文。学生读完能记住几个名词，却说不出这些资料共同说明什么。

第二种失败是原则堆砌。支撑方法会说“要有读者路径”“要控制认知负荷”“图文服务理解”“证据要分级”，但方法文档自己没有从问题走到做法。执行 AI 读完会点头，落地时仍然按模板填板块。

第三种失败是章节孤岛。文档标准有小节，有表格，有自检题，但每节像一个分类抽屉。上一节的结果没有交给下一节，下一节也用不上上一节。学生看到很多内容，脑子里没有路线。

第四种失败是图表装饰。表格被包装成图，平台图、时间图、学习图都画成相似流程图，论文角色和证据边界也都变成节点连线。图没有减少学生要记住的关系，反而增加页面噪声。

第五种失败是有来源但无判断边界。文末有参考资料，正文有引用，但关键判断仍然没有说明证据强弱。弱线索、交叉证据和直接证据被同一种语气写出来，读者无法判断哪里需要复核。

这份总方法要拦住这些失败。资料当然要整理，但整理只是起点；每一步都必须回答：

```text
这一步怎样让资料更接近学生可理解、可复述、可核查、可继续学习的状态？
```

答不上这个问题，就停在当前关卡，不继续写正文。

<a id="path-goal"></a>

## 总目标

导师调研材料的原料是一堆公开资料。成品文档要把这堆资料加工成学生能完成的具体动作：

```text
说出整套材料的阅读顺序，卡住时知道回看哪份文档；
指出老师身份、资料边界和仍需复核的事实风险；
把导师方向放回一片领域，并说出相邻方向和关键概念入口；
用 3 到 5 句话复述论文群围绕什么问题分工推进；
列出自己从基础课到目标论文要补的概念、方法、读图能力和进组前问题。
```

五份文档共享同一条加工路线，但每一份都是新的调研、筛查和分析阶段。上一份文档只提供输入和假设，不能替代下一份自己的资料处理。`02` 给 `03` 的是领域位置和问题候选，`03` 还要回到论文集合本身做尽量全面的阅读和归纳；`03` 的分析结果也可以反过来修正 `02` 对小领域或前沿位置的判断。

因此，写作顺序必须从学生出口倒推：定出口，筛证据，铺台阶，选图文，写正文，让学生输出，再用输出结果倒查全文。

这份总方法不是让执行者把七个词背下来。它要把每一关变成可交付的工作：执行者做完这一关，必须留下一个后文能消费的产物；审稿人看到这个产物，才能判断支撑方法是否真的落到文档里。

<a id="path-gates"></a>

## 七道关卡

一份导师文档从资料到成品，要过七道关卡。这里的关卡是写作门槛，不能当装饰性流程图。某一关不通过，后面不能靠润色补救。

```text
资料堆
  -> 出口关
  -> 证据关
  -> 台阶关
  -> 支撑关
  -> 成文关
  -> 输出与倒查关
```

| 关卡 | 执行者必须做什么 | 必须留下什么产物 | 不通过时回哪里 |
|:---|:---|:---|:---|
| 资料堆 | 把搜索结果按身份、论文、领域、路线、学习资源和弱线索分篮 | 候选资料篮、排除理由、待复核点 | 回 S1，不许把搜索结果改写成正文 |
| 出口关 | 把“了解 / 掌握”改成学生能说、画、核查或提问的动作 | 本文出口、章节出口、学生输出任务 | 回 S2，不许继续筛证据和排目录 |
| 证据关 | 逐条判断资料实际支撑什么、不支撑什么、证据等级是什么 | 关键判断清单、资料到判断卡、正文语气和复核点 | 回 S1，不许用弱线索承担结构判断 |
| 台阶关 | 写清本阶段新增调研、资料处理、学生理解增量和交给下一步的产物 | 章节推进链、学习块、反向修正点 | 回 S2/S3，不许直接扩写章节 |
| 支撑关 | 找出学生脑内硬扛的关系，并选择正文、表格、矩阵、证据卡或图像形式 | 视觉支撑规格、图文消费位置、降级方案 | 回 S4；证据不稳回 S1，出口不清回 S2，正文位置不清回 S3 |
| 成文关 | 把带边界判断和学习块写成章节接力、段落任务和证据语气 | 章节接力卡、段落任务、自检和局部支撑入口 | 回 S3，不许把并列材料堆成板块 |
| 输出与倒查关 | 让学生完成输出，并把失败分回对应关卡 | 学生输出、失败表现、回看路径、重写位置 | 回 S1/S2/S3/S4 的具体问题，不许只做语言润色 |

这七道关卡也是审稿顺序。输出失败时，不先改句子，而是问“这次失败暴露了哪一关没有留下可消费产物”。如果审稿人只能说“这里还可以优化”，说明倒查没有完成。

<a id="path-raw-material"></a>

## 资料堆：先承认资料还没有正文资格

执行 AI 拿到 `姓名 + 机构路径 + 官网 URL` 后，会找到官网、论文、数据库记录、新闻、项目页、综述和课程资料。这些只是原料。原料里混着同名噪声、过期机构、早期方向、旁支论文、弱线索和术语堆。

第一步要停止整理冲动。不要把搜索结果排成章节，不要把论文题名改写成方向判断，不要把几个领域词直接拼成地图。先建立资料篮：

```text
能确认身份的资料：
能确认论文归属的资料：
能支撑领域层级的资料：
能解释论文路线的资料：
能解释学习路径的资料：
只能作为弱线索的资料：
需要人工复核的资料：
```

资料篮的作用是提醒执行者：资料还没有获得正文资格。它只说明“可能有用”，不说明“已经能写”。

本关出口：

```text
我知道手里有哪些候选资料，也知道哪些资料暂时不能直接进入正文。
```

停写条件：

```text
如果还没区分身份资料、论文资料、领域资料、学习资料和弱线索，不能开始写正文。
```

<a id="path-exit"></a>

## 出口关：学生读完要能做什么

原料不能直接写，就要先定学生出口。出口要写成可检查动作，不能写成“了解”“熟悉”“掌握”这类空词。

五份成品的出口先按下面理解：

| 文档 | 学生读完后要能做什么 |
|:---|:---|
| `00_材料导读.md` | 说出五份材料的阅读顺序，知道读不懂时回看哪里 |
| `01_基础画像.md` | 说出老师是谁、资料是否锁定、论文集合有什么风险 |
| `02_领域地图.md` | 说出导师方向在大领域里的位置、相邻方向和关键概念入口 |
| `03_论文路线.md` | 用 3 到 5 句话复述论文群围绕什么问题分工推进 |
| `04_学习向导.md` | 说出自己从基础课到目标论文要补哪几段，并留下进组前问题 |

出口关要先问：

```text
读者进来时已经知道什么？
他此刻最容易卡在哪里？
本文只带他跨过哪一个卡点？
读完后他要能说出、画出、核查或提问什么？
```

本关出口：

```text
本文出口、章节出口和学生输出任务，三者至少各有一条可检查动作。
```

停写条件：

```text
如果出口只能写成“了解 / 掌握某个主题”这类抽象动词，不能继续。
```

<a id="path-evidence"></a>

## 证据关：哪些资料能支撑出口

有了学生出口，才知道该筛什么。以 `02_领域地图.md` 为例，如果出口是让学生建立领域位置感，优先资料应是综述、教材、课程体系、权威领域页面、导师近期论文和官网方向描述。新闻稿、讲座题目、数据库关键词可以提示检索方向，但不能单独定义领域层级。

每条资料进入正文前，都要被放进一个证据位置：

| 证据位置 | 能怎样写入正文 |
|:---|:---|
| 直接事实 | 可以稳定写出，并在附近标来源 |
| 交叉判断 | 写出多个来源怎样互相支撑 |
| 弱线索 | 保留限定语，不能承担结构性判断 |
| 冲突信息 | 写出冲突，不强行解释圆满 |
| 未确认信息 | 暂不进入正文主线，留给人工复核 |

证据关要避免一个常见错误：把“有来源”当成“能支撑”。来源只证明它说过某句话，不自动证明这句话能承担文档结构中的判断。

本关出口：

```text
能支撑学生出口的证据集合；
不能支撑出口的弱线索；
会影响后续结构的复核点。
```

停写条件：

```text
如果关键判断只能由新闻、题名关键词、数据库标签或单一弱来源支撑，不能写成主线。
```

<a id="path-steps"></a>

## 台阶关：每份文档怎样重新调研、处理并交接

证据筛完后，还不能直接写正文。台阶关要同时处理两件事：执行者这一阶段还要做哪些调研和分析，学生又怎样从上一份材料给出的入口走到本份文档的出口。少了前一件事，文档会变成下游转述；少了后一件事，文档会变成资料堆。

每一份文档、每一章、每一节都先回答五个问题：

```text
我要干什么？
上节交给了我什么？
我还需要调研和处理什么？
从而我能让学生多理解什么？
下节会拿走什么？
```

这五个问题要同时出现。只问“上节给什么、下节拿什么”，会把台阶关降级成材料交接；只问“我要写什么”，会把文档写成分类抽屉。台阶关真正要卡住的是：这一阶段新增的检索、筛查、论文阅读、领域定位、概念拆解或学习路径设计，怎样变成学生多理解的一步。

以 `02_领域地图.md` 为例，理解台阶可以是：

```text
我要干什么：建立整个大领域的范围感，再定位导师方向。
上节交给了我什么：01 给出可信身份、履历阶段、论文集合和官网方向线索。
我还需要调研和处理什么：查综述、教材、课程、领域页面和近期论文，分清大领域、问题域、相邻方向和容易混淆的词。
从而我能让学生多理解什么：学生能判断导师方向靠近哪片问题、旁边是什么、哪些词先别混用。
下节会拿走什么：03 拿走领域问题、关键概念入口和需要回到论文中核查的方向假设。
```

以 `03_论文路线.md` 为例，理解台阶可以是：

```text
我要干什么：理解导师当前方向相关履历阶段里的论文群，找出论文之间的问题链和角色分工。
上节交给了我什么：02 给出领域位置、问题候选和概念入口，只能作为阅读假设。
我还需要调研和处理什么：回到相关履历阶段的论文集合，尽量完整阅读题名、摘要、方法、对象、图表和引用线索，先归纳论文自己在解决什么，再判断它们之间的关系。
从而我能让学生多理解什么：学生能说出论文群怎样分工推进一个或几个问题，哪些论文是目标入口，哪些只是背景、平台或相邻语言。
下节会拿走什么：04 拿走目标论文、核心图、必须补的概念缺口和进组前问题。
```

`03` 要特别防止先射箭再画靶。`02` 的结论不能变成挑论文的靶子；`03` 必须回到相关履历阶段的论文集合做尽量全面的理解分析。分析后发现 `02` 的领域定位太宽、太窄或方向偏了，就回写复核点，不能把不合适的论文直接排除掉。

每个台阶都要留下这组记录：

```text
本阶段新增调研：
本阶段排除或降级的资料：
本阶段形成的学生理解台阶：
本阶段交给下一节的可用产物：
本阶段可能反向修正的前文判断：
```

本关出口：

```text
一组能说明新增调研、资料处理、学生理解增量和上下文交接的章节推进链。
```

停写条件：

```text
如果章节可以随意调换顺序，说明台阶没有成立。
```

<a id="path-support"></a>

## 支撑关：把学生脑内硬扛的关系外部化

台阶确定后，再选择局部支撑。这里的默认动作是主动寻找学生需要同时处理的关系。只要学生为了完成输出，需要在脑子里抓住多个对象之间的范围、上下位、命题关系、机制、时间、学习依赖、比较、证据强弱、论文图读法或真实数据模式，就优先考虑把关系外部化。

外部化不等于一律画 SVG。字段扫描用表格，两个判断维度共同起作用用矩阵，证据强弱容易误读用证据边界卡，范围和上下位需要看见时用范围图或层级图，论文角色和问题链需要看见时用带动词边的概念图，平台和方法流转用机制示意，基础课到目标论文用学习桥，目标论文核心图需要训练时用标注图。正文负责解释概念和证据边界；图表负责把关系放到纸面上。

| 关系负担 | 优先支撑形式 | 必须说清的边界 |
|:---|:---|:---|
| 字段重复，适合并排扫 | 表格 | 表格只管字段，表后正文必须拿走一个判断 |
| 两个维度共同决定判断 | 矩阵 | 行列必须都是判断维度，不能把字段堆成矩阵 |
| 范围、阅读位置、相邻方向难想象 | 范围图、区域图 | 不用坐标、面积、中心位置暗示无证据关系 |
| 上下位或概念层级混乱 | 层级树、缩进树、嵌套集合图 | 相邻方向不能硬画成上下级 |
| 论文、概念或问题之间有命题关系 | 带动词边的概念图 | 每条边都要能说出“提出 / 支撑 / 验证 / 复核”等关系 |
| 输入、处理、测量、输出混在一起 | 机制示意图 | 箭头只表示证据支持的流转，不把名词相邻画成因果 |
| 时间、阅读顺序或学习依赖混乱 | 时间线、路线、学习桥 | 箭头必须写明是时间、回看还是前置能力 |
| 证据强弱容易误读 | 证据边界卡 | 弱线索只进复核，不能被颜色、粗线或中心位置放大 |
| 论文核心图看不懂 | 标注图、读图顺序图 | 区分原图信息、本文解释、学生读图顺序和需回正文核查 |
| 有真实数量、趋势或分布 | 数据图 | 没有真实数据不画趋势、热力、桑基或比例 |

支撑关先写规格卡，但不要把轻量表格、自检题和复杂图都塞进同一张大表。所有支撑都先填最小卡；只有复杂图、证据卡或论文图标注，才加填扩展项。

```text
必填项：
学生输出：
学生卡住的关系：
关系中的对象：
选择的形式：
S1 证据允许的视觉语义：
禁止使用的视觉语义：
正文怎样提出、解释并消费这个支撑：
学生看完要复述什么：
答不上时回哪里：

复杂图 / 证据卡 / 论文图标注加填：
基础可视化出口：
为什么需要进一步画图或卡片：
图下怎样教读和标证据边界：
读图顺序或标注层类型：
重画、拆图或退回方案：
```

本关出口：

```text
每个难点都有一份能被正文消费的支撑规格；图、表、卡片或自检都知道自己回答什么问题。
```

停写条件：

```text
如果说不出学生要少背哪段关系，先回 S2/S3。
如果证据不足以支持位置、大小、颜色、粗细、箭头或中心性，先回 S1 或改成证据边界卡。
如果只是字段扫描，不把表格重画成图。
如果图前没有卡点、图下没有读法、图后没有消费，先回 S3/S4。
如果学生看完不能输出，重画、拆图或退回上游，不做美化。
```

<a id="path-draft"></a>

## 成文关：章节和段落怎样接力

到这里才进入正文写作。正文不能把前面材料重新扩写一遍。每节只推进一个主要动作，常见动作包括：

```text
定位、解释、比较、收束、核查、交接、输出
```

每节先写四句：

```text
本节接住上一节的什么：
本节只推进哪个动作：
本节需要什么局部支撑：
本节交给下一节什么：
```

每段也只承担一个主要任务：

```text
承接上一段；
引入一个判断；
解释一个关系；
指出证据边界；
教学生读图；
要求学生复述；
把读者交给下一节。
```

写完逐段检查：

```text
这一段接住了前面什么？
这一段让学生多理解了什么？
这一段里的判断证据够不够？
这一段有没有让学生硬扛关系？
下一段会怎样使用它？
```

本关出口：

```text
章节接力卡、段落任务清单和至少一个已嵌入正文的学生输出入口。
```

停写条件：

```text
如果一节删掉后前后文仍然能顺读，这节可能只是资料块。
如果一段只能让文章显得完整，删掉或移到内部材料。
```

<a id="path-output"></a>

## 输出与倒查关：学生输出失败时回哪里修

文档写完后，要让学生输出一个结果。输出承担校准作用，帮助学生判断自己有没有真的理解。

输出任务要具体：

```text
用一句话说出导师方向位置。
用 3 到 5 句话复述论文问题链。
指出哪条判断是弱证据。
看一张图，说出先看哪里、再看哪里。
列出进入下一份材料前还缺什么。
```

如果学生无法完成输出，先倒查文档，不先归因于学生基础差。

| 输出失败表现 | 可能原因 | 回到哪里修 | 必补产物 |
|:---|:---|:---|:---|
| 学生说不出主线 | 出口太虚，台阶断裂 | 出口关 / 台阶关；S2/S3 | 本文出口、章节推进链、学习块 |
| 学生把线索当结论 | 证据分级没进入正文 | 证据关；S1 | 资料到判断卡、证据等级、正文语气和复核点 |
| 学生只记住表格字段 | 表格替代了主线解释 | 支撑关 / 成文关；S3/S4 | 表后正文消费、段落任务、必要时支撑规格 |
| 学生看不懂图 | 图没有回答具体问题，或图下读法不足 | 支撑关；S4；若图前图后缺位并回 S3 | 关系负担、视觉语义边界、图文消费位置、降级方案 |
| 学生无法继续下一份 | 本文出口没有交给下一份可用产物 | 台阶关；S2/S3 | 下一份要拿走的产物、反向修正点、回看位置 |
| 学生输出空泛 | 自检题没有具体动作 | 输出关；S2 | 可检查输出任务、失败诊断、回退路径 |

本关出口：

```text
学生输出任务；
输出失败时的回看路径；
需要重写的关卡位置。
```

停写条件：

```text
如果审稿只列语言润色清单，说明还没有完成倒查。
```

<a id="path-methods"></a>

## 四份支撑方法怎样嵌入七道关卡

四份支撑方法不能当四个并列抽屉。它们嵌在同一条加工链里，各自给 M0 提供一类可交付产物。执行者不需要在写作时“想起所有原则”，而是遇到具体失败时能立刻分流。

| 关卡 | 主要调用 | 必须拿到的产物 | 失败分流 |
|:---|:---|:---|:---|
| 资料堆 | S1 信息检索与筛查 | 候选资料篮、排除理由、待复核点 | 资料混在一起时回 S1 分篮 |
| 出口关 | S2 学习与认知原则 | 可检查的学生动作、入口状态、暂不展开内容 | 出口写成“了解 / 掌握”时回 S2 |
| 证据关 | S1 信息检索与筛查 | 资料到判断卡、证据等级、正文语气、冲突降级 | 弱线索承担主线时回 S1 |
| 台阶关 | S2 学习与认知原则；S3 学术品味与文稿设计 | 学习块、章节接力卡、反向修正点 | 章节像摘要或抽屉时回 S2/S3 |
| 支撑关 | S4 可视化与理解；S3 学术品味与文稿设计 | 关系负担、支撑规格、图文消费位置、降级方案 | 学生硬扛关系时回 S4；图无正文位置时回 S3 |
| 成文关 | S3 学术品味与文稿设计；S1 证据边界 | 段落任务、证据语气、局部支撑入口和出口 | 句子顺但无推进时回 S3；语气越界时回 S1 |
| 输出与倒查关 | S2 学习与认知原则；S1/S3/S4 | 学生输出、失败表现、回看路径、重写关卡 | 只能语言润色时回对应方法，不许收尾 |

四份方法的职责边界：

| 编号与文件 | 它负责什么 | 它交给 M0 的最小产物 | 它不负责什么 |
|:---|:---|:---|:---|
| S1：`信息检索与筛查.md` | 资料怎样从候选变成正文判断 | 判断、来源、来源不支撑什么、证据等级、正文语气、复核点 | 不负责把文档写顺，也不替 S4 画关系 |
| S2：`学习与认知原则.md` | 学生出口、学习块、自检和回退 | 入口状态、单一学习任务、学生输出、失败诊断、回看位置 | 不负责堆课程或学习资源 |
| S3：`学术品味与文稿设计.md` | 章节接力、段落任务、证据语气和文稿主线 | 章节接力卡、段落任务、图前图后位置、证据边界写法 | 不负责替代证据判断，也不替 S4 选图 |
| S4：`可视化与理解.md` | 判断哪段关系需要被外部化，选择表、矩阵、证据卡或图 | 学生关系负担、可视化形式、视觉语义边界、图文消费、重画或退回方案 | 不负责为了完整感加图，也不靠图补证据或补出口 |

后续使用四份支撑方法时，都要按同一问题检查：

```text
这份方法要把执行者从哪种混乱带到哪种可执行状态？
它要求执行者留下什么产物？
这个产物会被 M0 的哪一关、哪一份 `00-04` 文档消费？
失败时应该回到哪份支撑方法，而不是继续润色？
```

<a id="path-docs"></a>

## `00-04` 怎样共用这套方法

五份成品共用同一套关卡，但每份文档都有自己的调研任务和学生出口。它们要连成一条理解路线，也允许后面的分析反向修正前面的判断。

```text
00 教学生怎样使用整套材料
  -> 01 尽量获得老师的全面事实信息
  -> 02 尽量获得领域全局，再定位导师方向
  -> 03 回到相关履历阶段的论文集合，分析论文问题链
  -> 04 从学生基础课一步步接到目标论文
  -> 回到 01 核查事实和证据风险
```

| 文档 | 我要干什么 | 上一份交给我什么 | 还要调研和处理什么 | 让学生多理解什么 | 下一份拿走什么 | 可能反向修正什么 |
|:---|:---|:---|:---|:---|:---|:---|
| `00_材料导读.md` | 说明五份材料怎样读、怎样自检、卡住时回哪里 | 最小输入和整套项目目标 | 梳理五份成品的职责、阅读顺序、证据符号和回看方式 | 学生知道这套材料怎样用，不把五份文档当五个孤立报告 | `01` 拿走读者入口和证据阅读习惯 | 后续文档如果职责变化，回修 `00` 的读法 |
| `01_基础画像.md` | 尽量获得老师的全面事实信息 | `00` 给出的阅读方式和证据规则 | 锁定身份、机构、履历、论文集合、同名风险、来源缺口和人工复核点 | 学生知道老师是谁，哪些资料稳定，哪些地方还不能确信 | `02` 拿走可信身份、履历阶段、论文集合边界和方向线索 | `02/03` 发现论文归属或履历阶段有问题时回修 `01` |
| `02_领域地图.md` | 尽量获得领域全局，再定位导师方向处在哪片大领域、小领域或前沿位置 | `01` 给出的身份、履历、论文集合和官网方向线索 | 查综述、教材、课程、领域页面、近期论文和相邻方向，先建立领域范围，再放入导师方向 | 学生能把导师方向放回一片领域里的具体位置 | `03` 拿走领域问题、概念入口、相邻方向和待核查假设 | `03` 论文分析显示定位偏差时回修 `02` |
| `03_论文路线.md` | 分析与当前方向有关的履历阶段论文，找出问题链和角色分工 | `02` 给出的领域位置和问题假设 | 回到相关论文集合，尽量全面理解论文的对象、问题、方法、图表、平台和相互关系；不能按 `02` 的结论只挑支持材料 | 学生知道论文群怎样推进问题，哪些论文是入口、核心、平台、背景或弱线索 | `04` 拿走目标论文、核心图、概念缺口、方法缺口和进组前问题 | 反向修正 `02` 的小领域判断，也可能回到 `01` 复核论文归属 |
| `04_学习向导.md` | 从学生已有基础课出发，接到导师当前重点、目标论文和核心图 | `03` 给出的目标论文、核心图、概念缺口和问题链 | 拆基础课程、数学物理工具、领域概念、论文片段、图表读法和阶段输出 | 学生知道自己从高数、线代、普物等基础怎样一步步接近目标论文 | 形成进组前问题和自学检查结果 | 如果发现论文入口不适合新手，回修 `03` 的目标论文选择 |

每份文档还要把“学生脑内硬扛的关系”写成可见支撑。这里不规定图形配额，只规定默认落点：

| 文档 | 常见关系负担 | 默认支撑形式 | 升级为图或卡片的条件 |
|:---|:---|:---|:---|
| `00_材料导读.md` | 五份材料的阅读顺序、卡住时回哪里 | 短清单、阅读顺序表 | 回看路径超过两层，或学生需要判断下一步读哪份时，用小决策图 |
| `01_基础画像.md` | 身份、履历、论文集合和证据风险并列出现 | 身份表、履历表、论文表、限定语 | 某个风险会影响 `02/03/04` 时，用证据边界卡靠近判断 |
| `02_领域地图.md` | 大领域、方法入口、问题域、相邻方向容易混用 | 分层正文、相邻方向表、概念入口表 | 位置感影响后续论文阅读时，用范围图；上下位影响理解时，用层级树；相邻方向容易混淆时，用比较图 |
| `03_论文路线.md` | 论文多、角色不清、问题链和平台链混在一起 | 论文角色表、问题链说明、证据边界 | 表格难以显示分工且证据足够时，用带动词边的概念图；方法流转清楚时，用机制图；核心论文图是入口时，用标注图 |
| `04_学习向导.md` | 基础课、概念、方法、目标论文图和进组问题断开 | 学习块、课程能力表、核心图读法清单 | 前置能力需要反复回看时，用学习桥；目标论文图承担训练入口时，用标注图或机制示意 |

改造 `00-04` 时，每份都要按七道关卡回答：

```text
我要干什么？
上一份交给了我什么？
我还需要调研和处理什么？
从而我能让学生多理解什么？
学生要少背哪段关系，怎样把它外部化？
下一份会拿走什么？
哪些判断可能被后文反向修正？
输出失败时回哪一关、哪份支撑方法？
```

这些问题不能只留在执行者脑子里。修 `D00-D04` 文档标准时，每份标准至少要留下七个可审查槽位：

| 槽位 | 标准里必须写出什么 | 缺失时停在哪里 |
|:---|:---|:---|
| 文档出口卡 | 本文出口、章节出口、学生输出任务 | 出口关 / S2 |
| 证据入口 | 本文使用哪些来源等级，哪些弱线索不能进入主线 | 证据关 / S1 |
| 章节推进链 | 每节接住什么、推进什么、交给下一节什么 | 台阶关 / S2/S3 |
| 关系负担与支撑规格入口 | 哪些关系要外部化，默认用正文、表格、矩阵、证据卡还是图 | 支撑关 / S4 |
| 段落任务要求 | 段落怎样承接、判断、解释、标边界、教读图或交接输出 | 成文关 / S3 |
| 学生输出任务 | 学生合上材料后要说、画、核查或提问什么 | 输出关 / S2 |
| 失败回退表 | 输出失败时回哪一节、哪份文档、哪份支撑方法补什么产物 | 输出与倒查关 / S1-S4 |

如果某份 D 标准写不出这些槽位，先停在 M0 和 D 标准接口，不继续扩写章节说明。标准文件可以有自己的结构，但不能缺这些产物；否则支撑方法仍然只是在上游讲得通，落不到成品写作。

<a id="path-references"></a>

## 支撑方法已吸收的方法判断怎样进入 M0

M0 不单独承载外部文献证据，也不把自己写成长综述。外部资料的 URL、采用要点和边界保留在 S1-S4；M0 只回收这些支撑方法已经吸收成项目动作的接口判断。后续 D 标准如果需要引用外部资料，必须回到对应支撑方法的参考资料表取 URL，不能只写泛化来源类别。

| 上游支撑方法 | 已吸收的方法判断 | M0 回收成什么接口 |
|:---|:---|:---|
| S2 学习与认知原则 | 新手学习需要控制认知负荷，主动提取、自我解释和元认知校准比重读更能暴露理解问题 | 每份文档必须有学生出口、输出任务和回退路径 |
| S3 学术品味与文稿设计 | 读者依赖句子位置、段落重心和文章结构来判断重点 | 每节必须有入口、推进、局部支撑和出口；段落只做一个主要动作 |
| S1 信息检索与筛查 | 单个来源不能自证，权威取决于问题场景，检索过程要可复查 | 身份、论文、领域、学习路径分别需要不同来源层级；弱线索不能承担结构判断 |
| S4 可视化与理解 | 图只有在任务、数据和视觉编码匹配时才降低认知负担；图形会把位置、颜色、大小、箭头读成语义 | 先写学生输出和关系负担，再决定正文、表格、矩阵、证据卡或图；视觉语义不能越过证据边界，图必须被正文消费 |

四份支撑方法保留具体参考资料和细节。总方法只记录这些资料如何被转成本项目的执行判断和标准接口。

<a id="path-agent-protocol"></a>

## 执行 AI 的最低工作协议

工作台被搞乱，常见原因不在文件数量本身，而在执行者没有先判断自己正在处理哪一层问题。`AI协作开发与质量控制.md` 的 G0-G7、G5a 和 G7a 在这里收成一组开工门禁；它们只管执行方式，不改变七道关卡本身。

每次修改前先写任务合同：

```text
当前层级：
本次目标：
输入和来源：
本次不碰：
完成条件：
验证方式：
人工审查边界：
隔离审查：
```

任务合同要短，但不能省。写不出“本次不碰”和“人工审查边界”，说明这次很可能会跨层扩散，先停下来缩小范围。

涉及调研、方法、论文路线、学习路径、可视化或 AI 协作规则时，再写资料到判断卡：

```text
读者或项目问题：
资料或证据：
从资料得到的判断：
这个判断改变哪份文档、哪一步写作或哪条执行规则：
边界或不确定性：
```

资料到判断卡的作用，是防止参考资料只留在文末。资料如果不能改变判断、结构、读者动作或执行规则，就不进入正文主线。

执行门禁按下面八条检查：

| 门禁 | 要做什么 | 不通过时怎样处理 |
|:---|:---|:---|
| G0 层级锁定 | 先判断当前问题属于 `M0`、`S1-S4`、`D00-D04`、成品样本、skill、verifier 还是 harness | 不能同时改多个层级；其他层的问题只记为后续回收 |
| G1 任务合同 | 写清目标、输入、约束、完成条件、验证和人工审查边界 | 合同写不清，就不开始编辑 |
| G2 资料到判断 | 说明资料怎样改变本项目判断或读者动作 | 资料只能证明“有人说过”，就降级为工作备注或参考线索 |
| G3 小步设计 | 改支撑方法或标准前，先定大纲、段落工作和出口 | 方向未确认时不整篇重写 |
| G4 单层编辑 | 一次只改当前层级 | 发现必须改别的层，先记录影响，当前里程碑结束后再决定 |
| G5 人工质量关 | 检查真实失败、资料依据、项目判断、执行动作、失败检查和未覆盖边界 | 只写原则、没有动作或停写条件，就退回重写 |
| G5a 隔离 Subagent 调度 | 高风险内容冻结后交给小白读者、证据审查或图文审查 subagent 只读回放；冻结前若已有明确并行侧线，可开探索型或验证型 subagent | 不把完整聊天和作者意图塞给 subagent，也不让 subagent 替主 agent 决定层级和最终路线 |
| G6 机械验证 | 跑 `git diff --check`、禁用表达检索、`git status`；成品样本另跑对应 verifier | 机械验证只证明格式底线，不声称学术质量通过 |
| G7 复盘回收 | 判断这次发现是单文件表达问题，还是跨导师会复现的方法问题 | 单文件问题不升格；跨文件问题才进入 M0、S、D、skill、verifier 或 harness |
| G7a Subagent 反馈回灌 | 把 subagent 实际读到和原设计路线对照，再决定删、改、拆、降级或回到上游方法 | 不能把 subagent 意见当成泛泛润色建议，也不能直接升格为长期规则 |

这组协议防止局部修补继续制造新支线。方法层没有稳定前，不直接改成品样本来“试压”；聊天里的共识也不能自动变成长期规则，必须先落到当前层级的可审查改动。

<a id="path-stop"></a>

## 质量检查：什么时候必须停下来

总方法的底线是知道什么时候不能继续写，不能把“文档写完”当作通过。

| 情况 | 必须停在哪里 |
|:---|:---|
| 学生出口说不清 | 停在出口关 |
| 来源只能支撑题名或关键词 | 停在证据关 |
| 章节可以随意换序 | 停在台阶关 |
| 图表说不出学生问题 | 停在支撑关 |
| 学生需要同时处理关系，正文却让他硬记 | 停在支撑关 |
| 图、表、卡片没有被后文消费 | 停在支撑关或成文关 |
| 段落只是在补完整感 | 停在成文关 |
| 学生输出空泛或答不上 | 停在输出与倒查关 |
| 支撑方法只写原则，没有产物和回退 | 停在四份支撑方法接口 |
| 讨论开始跨到样本、skill、verifier | 停在执行协议 |
| 没有任务合同就开始改文件 | 停在执行协议 |
| 高风险内容只靠主 agent 自审 | 停在 Subagent 回放门 |
| 参考资料说不清怎样改变判断 | 停在资料到判断卡 |
| 机械验证通过后想直接声称学术质量通过 | 停在人工质量关 |

每次停下后，不写“后续优化”，要写清回到哪一关、补什么判断、删什么形式、留下什么可消费产物。写不出产物，就说明这次修改还停在分析层，没有落到执行层。

## 待验证问题和后续回收

这份总方法已经吸收 S1-S4 的主要接口。下一步不是继续在支撑方法里打转，而是用它约束 `D00-D04` 五份文档标准。

后续回收顺序先按下面走：

```text
M0 冻结候选自查
  -> D00-D04 文档标准
  -> 真实导师样本试压
  -> skill / templates / verifier / harness 统一回收
```

每进入下一层，都只搬运已经能被执行和审查的规则。只会让文档显得更完整、但没有产物、停写条件和回退路径的内容，不回收。

待验证问题：

- `D00-D04` 是否都能写出七个最小落地产物槽位；写不出时停在 M0/D 标准接口。
- 四份支撑方法的产物是否真的能被文档标准消费，而不是只在 M0 里说得通。
- 每份 D 标准是否都能回答“我要干什么、上一份给什么、还要调研处理什么、学生多理解什么、下一份拿什么、可能反向修正什么”。
- 每份 D 标准是否都能回答“学生要少背哪段关系、怎样外部化、图文怎样消费、失败时回哪份支撑方法补什么产物”。
- 张鹏举黄金样本是否能用这套方法重修，而不再新增支线文件。
- 哪些判断需要回收进 `research-advisor` skill，哪些只留在 workbench。

在这些问题验证前，不回收进正式模板、verifier 或 harness。
