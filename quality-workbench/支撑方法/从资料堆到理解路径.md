<a id="path-main-question"></a>

## 中心问题

这份文档解决一个工作台反复暴露的问题：内容被整理成一块一块，看起来完整，读者却没有被带着往前走。章节像分类抽屉，图像任务清单，表格像资料仓库。读者能看到很多信息，却说不出这些信息怎样共同帮助自己理解导师方向。

导师材料的写作对象首先是一堆尚未加工的资料，章节只是在后面承载路径的外壳。写作要做的事，是把这堆资料加工成一条学生能走的理解路径。学生读完后应能复述关键判断，知道哪些地方可靠，知道哪里需要回看，也知道下一步怎样继续学习。

因此，本方法只追一个问题：

```text
这一步怎样让资料更接近学生可理解、可复述、可核查、可继续学习的状态？
```

答不上这个问题的章节、图、表、清单和段落，都不能进入主线。

<a id="path-main-chain"></a>

## 总链条

一份导师文档从资料到成品，应按这条链推进：

```text
资料堆
  -> 学生出口
  -> 证据筛选
  -> 理解台阶
  -> 局部支撑
  -> 正文推进
  -> 学生输出
  -> 反向审稿
```

这条链是写作顺序的约束。每一环都要交给下一环一个可使用的产物。

| 环节 | 这一环产出什么 | 下一环怎样使用 |
|:---|:---|:---|
| 资料堆 | 候选资料和风险意识 | 决定学生不能直接面对全部资料 |
| 学生出口 | 读完后要能完成的动作 | 反推哪些证据和章节有用 |
| 证据筛选 | 可写入主线的信息和风险信息 | 只用合格信息搭建理解台阶 |
| 理解台阶 | 从入口到出口的几步动作 | 每节只承担一个推进任务 |
| 局部支撑 | 文字、表格、图、例子、证据说明、自检的安排 | 正文按读者动作展开 |
| 正文推进 | 能沿着读下去的草稿 | 用学生输出检查是否成立 |
| 学生输出 | 复述、核查、提问或回退任务 | 反向发现主线、证据和图文问题 |
| 反向审稿 | 重写方向和删减判断 | 回到相应环节修正 |

四份支撑方法贯穿全链。它们不能各写各的：学术品味负责路径和接力，学习认知负责吸收和输出，信息筛查负责证据资格，可视化负责把难以靠文字承受的关系放到页面上。

<a id="path-four-methods"></a>

## 四种方法怎样合在一起

四份支撑方法对应同一条加工链里的四种判断。写作时不要把它们变成四个固定小节，而要在需要的位置一起发挥作用。

| 加工时刻 | 先问什么 | 主要调用 |
|:---|:---|:---|
| 资料进入视野 | 这条资料能不能支撑出口，还是只能当线索 | 信息检索与筛查 |
| 章节开始成形 | 这一节接住什么，又把什么交给下一节 | 学术品味与文稿设计 |
| 学生可能卡住 | 学生此刻需要少量输入、主动输出，还是回退补洞 | 学习与认知原则 |
| 关系难以靠文字承受 | 学生需要看见范围、层级、角色、机制、学习桥还是证据边界 | 可视化与理解 |

比如 `02_领域地图.md` 不能先画一个层级树就收工。信息筛查先确认“凝聚态物理、超快光谱、强关联材料”等词各自来自哪里；学术品味把这些词排成“从大领域到导师问题域”的接力；学习原则控制一次只解释当前台阶需要的少数概念；可视化再决定哪里用领域地形图，哪里用层级收束图，哪里只需要一句解释。

<a id="path-raw-material"></a>

## 资料还不能直接写

执行 AI 拿到 `姓名 + 机构路径 + 官网 URL` 后，会找到官网、论文、数据库记录、新闻、项目页、综述和课程资料。这些资料只是原料。原料里混着同名噪声、过期机构、早期方向、旁支论文、弱线索和术语堆。

写作的第一个动作是停止整理冲动。不要把搜索结果排成章节，不要把论文题名改写成方向判断，不要把几个领域词直接拼成地图。先建立一份资料篮：

```text
能确认身份的资料：
能确认论文归属的资料：
能支撑领域层级的资料：
能解释学习路径的资料：
只能作为弱线索的资料：
需要人工复核的资料：
```

本节交给下一节的产物是：一篮候选资料，以及“这些资料还不能直接写”的风险意识。

<a id="path-student-exit"></a>

## 先定学生出口

原料不能直接写，就要先定学生读完后要能做什么。出口要写成可检查动作，不能写成“了解”“熟悉”“掌握”这类空词。

五份成品的出口可以先这样定：

| 文档 | 学生读完后要能做什么 |
|:---|:---|
| `00_材料导读.md` | 说出五份材料的阅读顺序，知道读不懂时回看哪里 |
| `01_基础画像.md` | 说出老师是谁、资料是否锁定、论文集合有什么风险 |
| `02_领域地图.md` | 说出导师方向在大领域里的位置、相邻方向和关键概念入口 |
| `03_论文路线.md` | 用 3 到 5 句话复述论文群围绕什么问题分工推进 |
| `04_学习向导.md` | 说出自己从基础课到目标论文要补哪几段，并留下进组前问题 |

本节交给下一节的产物是：一个清楚的学生出口。后续所有证据、章节、图文和自检都要服务这个出口。

<a id="path-evidence"></a>

## 筛出能支撑出口的证据

有了学生出口，才知道该筛什么。以 `02_领域地图.md` 为例，如果出口是让学生建立领域位置感，优先资料应是综述、教材、课程体系、权威领域页面、导师近期论文和官网方向描述。新闻稿、讲座题目、数据库关键词可以提示检索方向，但不能单独定义领域层级。

每条资料进入正文前，都要被放进一个证据位置：

| 证据位置 | 能怎样写入正文 |
|:---|:---|
| 直接事实 | 可以稳定写出，并在附近标来源 |
| 交叉判断 | 写出多个来源怎样互相支撑 |
| 弱线索 | 保留限定语，不能承担结构性判断 |
| 冲突信息 | 写出冲突，不强行解释圆满 |
| 未确认信息 | 暂不进入正文主线，留给人工复核 |

本节交给下一节的产物是：可写入主线的信息，以及只能内部保留或风险提示的信息。

<a id="path-steps"></a>

## 把出口拆成理解台阶

证据筛完后，还不能直接写正文。要把学生出口拆成几级理解台阶。台阶的作用，是让学生从自己已有的东西出发，一步一步走到出口。

以 `02_领域地图.md` 为例，理解台阶可以是：

```text
从 01 拿到导师方向线索
  -> 看大领域里有哪些主要板块
  -> 把导师方向放进其中一块
  -> 区分相邻方向和容易混淆的词
  -> 把少数关键概念交给 03 的论文路线
```

每个台阶都要能回答三个问题：

```text
上一节交给我什么：
我让学生多理解什么：
下一节会拿走什么：
```

本节交给下一节的产物是：一条从入口到出口的章节推进链。

<a id="path-section-action"></a>

## 让每一节只推进一个动作

理解台阶确定后，每节只能推进一个主要动作。常见动作包括：

```text
定位、解释、比较、收束、核查、交接、输出
```

例如 `02_领域地图.md` 中，“领域全景”只负责让学生看到大领域里有哪些主要板块；“范围定位”负责把导师方向放进去；“概念入口”负责解释后续论文路线需要的少数概念。它们不能互相抢任务。

小节设计时先写四句：

```text
本节接住上一节的什么：
本节只推进哪个动作：
本节需要什么局部支撑：
本节交给下一节什么：
```

本节交给下一节的产物是：每节的单一任务清单。

<a id="path-local-support"></a>

## 给难理解的位置配支撑

每节任务确定后，再选择支撑形式。形式服从动作，不能先决定“这里放一张图”或“这里做一个表”。

| 读者卡点 | 合适支撑 | 使用边界 |
|:---|:---|:---|
| 字段重复，适合并排扫 | 表格 | 表格只管字段，不替代主线解释 |
| 空间位置难想象 | 领域地形图、二维定位图 | 只能表达证据允许的粗略位置 |
| 上下位关系不清 | 层级收束图 | 不负责表达比例和相邻关系 |
| 论文角色难分 | 论文角色地图、对比矩阵 | 线条和角色必须有语义 |
| 平台机制难想象 | 机制 / 平台链路图 | 箭头要表示输入输出、测量流或信息流 |
| 学习路线太抽象 | 学习桥图 | 每一段要接一个可做的小输出 |
| 证据强弱容易误读 | 证据边界说明 | 弱证据不能被图形语气放大 |
| 学生容易假懂 | 输出自检 | 自检要能指向回看位置 |

本节交给下一节的产物是：每节该用文字、表格、图、例子、证据说明还是自检的安排。

<a id="path-draft"></a>

## 写正文时让每段都有任务

到这里才进入正文写作。正文不能把前面材料重新扩写一遍。每段都要服务一个读者动作：

```text
承接上一段
引入一个判断
解释一个关系
指出证据边界
教学生读图
要求学生复述
把读者交给下一节
```

写作时逐段检查：

```text
这一段接住了前面什么？
这一段让学生多理解了什么？
这一段里的判断证据够不够？
这一段有没有让学生硬扛关系？
这一段交给下一段什么？
```

本节交给下一节的产物是：一份可以沿着读下去的正文草稿。

<a id="path-output"></a>

## 让学生把理解说出来

文档写完后，要让学生输出一个结果。输出承担校准作用，帮助学生判断自己有没有真的理解。

输出任务要具体：

```text
用一句话说出导师方向位置。
用 3 到 5 句话复述论文问题链。
指出哪条判断是弱证据。
看一张图，说出先看哪里、再看哪里。
列出进入下一份材料前还缺什么。
```

如果学生无法完成输出，就说明前面的主线、证据、图文或段落任务有问题。不要把这种失败归因于学生基础差，先回看文档有没有把台阶铺清楚。

本节交给下一节的产物是：学生输出任务和对应回看路径。

<a id="path-reverse-check"></a>

## 用学生输出倒查全文

最后的审稿从学生输出开始，不从格式检查开始。核心问题是：

```text
学生能不能完成本文定义的出口动作？
```

如果不能，就沿着总链条倒查：

| 输出失败表现 | 可能原因 | 回到哪里修 |
|:---|:---|:---|
| 学生说不出主线 | 出口太虚，台阶断裂 | 回到“学生出口”和“理解台阶” |
| 学生把线索当结论 | 证据分级没进入正文 | 回到“证据筛选” |
| 学生只记住表格字段 | 表格替代了主线解释 | 回到“局部支撑”和“正文推进” |
| 学生看不懂图 | 图没有回答具体问题，或图下读法不足 | 回到“局部支撑” |
| 学生无法继续下一份 | 本文出口没有交给下一份可用产物 | 回到“理解台阶” |
| 学生输出空泛 | 自检题没有具体动作 | 回到“学生输出” |

本节交给执行者的产物是：重写方向。只列润色清单，说明审稿还停在表面。

<a id="path-support-methods"></a>

## 怎样改造支撑方法

这份总方法要先用来改造四份支撑方法。支撑方法自己也必须从资料或问题出发，走向可执行的理解路径。

| 支撑方法 | 当前最容易的问题 | 改造方向 |
|:---|:---|:---|
| `学术品味与文稿设计.md` | 容易停在原则说明 | 增加坏段落到好段落的连续改造，展示章节怎样接力 |
| `学习与认知原则.md` | 容易变成学习原则清单 | 改成学生如何进入、输出、自检、回退的学习过程 |
| `信息检索与筛查.md` | 容易停在检索流程和清单 | 改成信息怎样从候选变成正文判断 |
| `可视化与理解.md` | 容易讲图形种类却不示范 | 保留总览图，每个小分类给独立局部图和读法 |

改造时不要问“这份支撑方法还缺什么内容”。先问：

```text
这份支撑方法要把读者从哪种混乱带到哪种可执行状态？
```

<a id="path-doc-standards"></a>

## 怎样改造 00-04

四份支撑方法能自洽后，再用本方法改造 `00-04` 文档标准。五份标准各自的出口不同，但加工链相同。

| 文档 | 要加工的原始状态 | 目标状态 |
|:---|:---|:---|
| `00_材料导读.md` | 学生不知道五份材料怎样读 | 学生知道阅读顺序、自检方式和回看路径 |
| `01_基础画像.md` | 老师资料分散，风险不清 | 身份、履历、论文集合和风险边界清楚 |
| `02_领域地图.md` | 方向名词混杂，缺少范围感 | 学生能定位大领域、问题域和相邻方向 |
| `03_论文路线.md` | 论文只是列表 | 学生能复述论文群的问题链和角色分工 |
| `04_学习向导.md` | 学生只有基础课，不知道怎样接前沿 | 学生知道课程缺口、目标论文片段和进组前输出 |

改造 `00-04` 时，每份文档都要回答：

```text
原料是什么？
学生出口是什么？
哪些证据能支撑出口？
理解台阶怎样排？
哪里需要局部图文？
学生怎样输出？
输出失败时回哪里修？
```

<a id="path-source-index"></a>

## 溯源索引

写作时先走本文总链条。需要深入某一类问题时，再回到对应支撑方法。

| 当前问题 | 回到哪份支撑方法 |
|:---|:---|
| 主线断裂、章节孤岛、段落任务不清 | `学术品味与文稿设计.md` |
| 学生看完不会复述、输出、自检或回退 | `学习与认知原则.md` |
| 身份、论文、领域、证据强弱不稳 | `信息检索与筛查.md` |
| 图形选择、图文位置、图下读法和渲染问题 | `可视化与理解.md` |

溯源索引放在最后。总方法先教执行者怎样加工资料，支撑方法再补每一类问题的细节。
