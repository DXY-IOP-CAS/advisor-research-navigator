## 这份方法要解决的真实失败

工作台前几轮暴露的失败很具体。资料被收集了，文档也变长了，但学生仍然没有被带到理解出口。

第一种失败是资料转述。官网、论文、数据库、新闻和课程资料被排成章节，正文看起来有来源，实际只是把资料换成中文。学生读完能记住几个名词，却说不出这些资料共同说明什么。

第二种失败是原则堆砌。支撑方法会说“要有读者路径”“要控制认知负荷”“图文服务理解”“证据要分级”，但方法文档自己没有从问题走到做法。执行 AI 读完会点头，落地时仍然按模板填板块。

第三种失败是章节孤岛。文档标准有小节，有表格，有自检题，但每节像一个分类抽屉。上一节的结果没有交给下一节，下一节也用不上上一节。学生看到很多内容，脑子里没有路线。

第四种失败是图表装饰。表格被包装成图，平台图、时间图、学习图都画成相似流程图，论文角色和证据边界也都变成节点连线。图没有减少学生要记住的关系，反而增加页面噪声。

第五种失败是有来源但无判断边界。文末有参考资料，正文有引用，但关键判断仍然没有说明证据强弱。弱线索、交叉证据和直接证据被同一种语气写出来，读者无法判断哪里需要复核。

这份总方法要拦住这些失败。它不追求把资料整理得更完整，而是要求每一步都回答：

```text
这一步怎样让资料更接近学生可理解、可复述、可核查、可继续学习的状态？
```

答不上这个问题，就停在当前关卡，不继续写正文。

<a id="path-goal"></a>

## 总目标

导师调研材料的原料是一堆公开资料。成品文档要把这堆资料加工成学生能完成的具体动作：

```text
知道怎样读整套材料；
知道老师是谁，资料边界在哪里；
知道导师方向大致处在哪片领域；
知道论文群围绕什么问题分工推进；
知道自己从基础课到目标论文要补哪几段。
```

因此，写作顺序必须从学生出口倒推：定出口，筛证据，铺台阶，选图文，写正文，让学生输出，再用输出结果倒查全文。

<a id="path-gates"></a>

## 七道关卡

一份导师文档从资料到成品，要过七道关卡。这里的关卡是写作门槛，不是装饰性流程图。某一关不通过，后面不能靠润色补救。

```text
资料堆
  -> 出口关
  -> 证据关
  -> 台阶关
  -> 支撑关
  -> 成文关
  -> 输出与倒查关
```

| 关卡 | 必须产出什么 | 不通过时禁止什么 |
|:---|:---|:---|
| 资料堆 | 候选资料篮和风险意识 | 禁止把搜索结果直接改写成正文 |
| 出口关 | 学生读完要完成的动作 | 禁止继续筛证据和排目录 |
| 证据关 | 能支撑出口的资料、弱线索和复核点 | 禁止用弱线索承担结构性判断 |
| 台阶关 | 从入口到出口的理解步骤 | 禁止直接写章节正文 |
| 支撑关 | 每个卡点所需的正文、表格、图、证据卡或自检 | 禁止为了完整感加图、加表、加清单 |
| 成文关 | 能接力推进的章节和段落 | 禁止把并列材料堆成板块 |
| 输出与倒查关 | 学生输出任务和回看路径 | 禁止只做格式检查或语言润色 |

这七道关卡也是审稿顺序。输出失败时，不先改句子，而是回到对应关卡。

<a id="path-raw-material"></a>

## 资料堆：先承认资料还不是正文

执行 AI 拿到 `姓名 + 机构路径 + 官网 URL` 后，会找到官网、论文、数据库记录、新闻、项目页、综述和课程资料。这些只是原料。原料里混着同名噪声、过期机构、早期方向、旁支论文、弱线索和术语堆。

第一步要停止整理冲动。不要把搜索结果排成章节，不要把论文题名改写成方向判断，不要把几个领域词直接拼成地图。先建立资料篮：

```text
能确认身份的资料：
能确认论文归属的资料：
能支撑领域层级的资料：
能解释论文路线的资料：
能解释学习路径的资料：
只能作为弱线索的资料：
需要人工复核的资料：
```

资料篮的作用是提醒执行者：资料还没有获得正文资格。它只说明“可能有用”，不说明“已经能写”。

本关出口：

```text
我知道手里有哪些候选资料，也知道哪些资料暂时不能直接进入正文。
```

停写条件：

```text
如果还没区分身份资料、论文资料、领域资料、学习资料和弱线索，不能开始写正文。
```

<a id="path-exit"></a>

## 出口关：学生读完要能做什么

原料不能直接写，就要先定学生出口。出口要写成可检查动作，不能写成“了解”“熟悉”“掌握”这类空词。

五份成品的出口先按下面理解：

| 文档 | 学生读完后要能做什么 |
|:---|:---|
| `00_材料导读.md` | 说出五份材料的阅读顺序，知道读不懂时回看哪里 |
| `01_基础画像.md` | 说出老师是谁、资料是否锁定、论文集合有什么风险 |
| `02_领域地图.md` | 说出导师方向在大领域里的位置、相邻方向和关键概念入口 |
| `03_论文路线.md` | 用 3 到 5 句话复述论文群围绕什么问题分工推进 |
| `04_学习向导.md` | 说出自己从基础课到目标论文要补哪几段，并留下进组前问题 |

出口关要先问：

```text
读者进来时已经知道什么？
他此刻最容易卡在哪里？
本文只带他跨过哪一个卡点？
读完后他要能说出、画出、核查或提问什么？
```

本关出口：

```text
一个可检查的学生动作。
```

停写条件：

```text
如果出口只能写成“了解导师方向”“掌握论文内容”，不能继续。
```

<a id="path-evidence"></a>

## 证据关：哪些资料能支撑出口

有了学生出口，才知道该筛什么。以 `02_领域地图.md` 为例，如果出口是让学生建立领域位置感，优先资料应是综述、教材、课程体系、权威领域页面、导师近期论文和官网方向描述。新闻稿、讲座题目、数据库关键词可以提示检索方向，但不能单独定义领域层级。

每条资料进入正文前，都要被放进一个证据位置：

| 证据位置 | 能怎样写入正文 |
|:---|:---|
| 直接事实 | 可以稳定写出，并在附近标来源 |
| 交叉判断 | 写出多个来源怎样互相支撑 |
| 弱线索 | 保留限定语，不能承担结构性判断 |
| 冲突信息 | 写出冲突，不强行解释圆满 |
| 未确认信息 | 暂不进入正文主线，留给人工复核 |

证据关要避免一个常见错误：把“有来源”当成“能支撑”。来源只证明它说过某句话，不自动证明这句话能承担文档结构中的判断。

本关出口：

```text
能支撑学生出口的证据集合；
不能支撑出口的弱线索；
会影响后续结构的复核点。
```

停写条件：

```text
如果关键判断只能由新闻、题名关键词、数据库标签或单一弱来源支撑，不能写成主线。
```

<a id="path-steps"></a>

## 台阶关：学生怎样一步步走到出口

证据筛完后，还不能直接写正文。要把学生出口拆成几级理解台阶。台阶的作用，是让学生从已有知识出发，一步一步走到出口。

以 `02_领域地图.md` 为例，理解台阶可以是：

```text
从 01 拿到导师方向线索
  -> 看大领域里有哪些主要板块
  -> 把导师方向放进其中一块
  -> 区分相邻方向和容易混淆的词
  -> 把少数关键概念交给 03 的论文路线
```

以 `03_论文路线.md` 为例，理解台阶可以是：

```text
从 02 拿到领域问题
  -> 筛出当前路线需要的论文
  -> 说明这些论文共同推进的问题
  -> 判断代表论文的角色
  -> 区分目标论文、回看论文、平台边界、相邻语言和弱线索
  -> 把目标论文、核心图和概念缺口交给 04
```

每个台阶都要能回答三句话：

```text
上一节交给我什么：
我让学生多理解什么：
下一节会拿走什么：
```

本关出口：

```text
一条从入口到出口的章节推进链。
```

停写条件：

```text
如果章节可以随意调换顺序，说明台阶没有成立。
```

<a id="path-support"></a>

## 支撑关：图、表、证据卡和自检怎样局部服务理解

台阶确定后，再选择局部支撑。形式服从读者卡点，不能先决定“这里放一张图”或“这里做一个表”。

| 读者卡点 | 合适支撑 | 使用边界 |
|:---|:---|:---|
| 字段重复，适合并排扫 | 表格 | 表格只管字段，不替代主线解释 |
| 两个维度共同决定判断 | 矩阵 | 只有两个维度都参与推理时才用 |
| 空间位置难想象 | 范围定位图 | 只能表达证据允许的粗略位置 |
| 对象分工难看清 | 结构关系图 | 节点和连线必须有语义 |
| 先后、机制、依赖混在一起 | 路线图 | 箭头必须说明时间、机制或学习依赖 |
| 证据强弱容易误读 | 证据边界卡 | 弱证据不能被图形语气放大 |
| 学生容易假懂 | 输出自检 | 自检要能指向回看位置 |

支撑关先写一小段规格：

```text
学生卡在哪里：
这种关系能否用正文讲清：
如果用表格 / 图 / 证据卡，它回答什么问题：
学生看完后要能复述什么：
如果图不合适，降级成什么形式：
```

本关出口：

```text
每个难点所需的最轻支撑形式。
```

停写条件：

```text
如果说不出这张图让学生少背哪段关系，就不画。
如果表格只是把正文切成格子，就改回正文。
如果证据不足以支持空间位置、大小或箭头关系，就不要画成图。
```

<a id="path-draft"></a>

## 成文关：章节和段落怎样接力

到这里才进入正文写作。正文不能把前面材料重新扩写一遍。每节只推进一个主要动作，常见动作包括：

```text
定位、解释、比较、收束、核查、交接、输出
```

每节先写四句：

```text
本节接住上一节的什么：
本节只推进哪个动作：
本节需要什么局部支撑：
本节交给下一节什么：
```

每段也只承担一个主要任务：

```text
承接上一段；
引入一个判断；
解释一个关系；
指出证据边界；
教学生读图；
要求学生复述；
把读者交给下一节。
```

写完逐段检查：

```text
这一段接住了前面什么？
这一段让学生多理解了什么？
这一段里的判断证据够不够？
这一段有没有让学生硬扛关系？
下一段会怎样使用它？
```

本关出口：

```text
一份能沿着读下去的正文草稿。
```

停写条件：

```text
如果一节删掉后前后文仍然能顺读，这节可能只是资料块。
如果一段只能让文章显得完整，删掉或移到内部材料。
```

<a id="path-output"></a>

## 输出与倒查关：学生输出失败时回哪里修

文档写完后，要让学生输出一个结果。输出承担校准作用，帮助学生判断自己有没有真的理解。

输出任务要具体：

```text
用一句话说出导师方向位置。
用 3 到 5 句话复述论文问题链。
指出哪条判断是弱证据。
看一张图，说出先看哪里、再看哪里。
列出进入下一份材料前还缺什么。
```

如果学生无法完成输出，先倒查文档，不先归因于学生基础差。

| 输出失败表现 | 可能原因 | 回到哪里修 |
|:---|:---|:---|
| 学生说不出主线 | 出口太虚，台阶断裂 | 回到出口关和台阶关 |
| 学生把线索当结论 | 证据分级没进入正文 | 回到证据关 |
| 学生只记住表格字段 | 表格替代了主线解释 | 回到支撑关和成文关 |
| 学生看不懂图 | 图没有回答具体问题，或图下读法不足 | 回到支撑关 |
| 学生无法继续下一份 | 本文出口没有交给下一份可用产物 | 回到台阶关 |
| 学生输出空泛 | 自检题没有具体动作 | 回到输出关 |

本关出口：

```text
学生输出任务；
输出失败时的回看路径；
需要重写的关卡位置。
```

停写条件：

```text
如果审稿只列语言润色清单，说明还没有完成倒查。
```

<a id="path-methods"></a>

## 四份支撑方法怎样嵌入七道关卡

四份支撑方法不是四个并列抽屉。它们嵌在同一条加工链里，各自负责一类判断。

| 关卡 | 主要调用 | 要拿走什么 |
|:---|:---|:---|
| 资料堆 | 信息检索与筛查 | 候选资料、排除理由、复核点 |
| 出口关 | 学习与认知原则 | 可检查的学生输出 |
| 证据关 | 信息检索与筛查 | 直接证据、交叉证据、弱线索和冲突处理 |
| 台阶关 | 学习与认知原则；学术品味与文稿设计 | 学习块、章节接力和回退路径 |
| 支撑关 | 可视化与理解；学术品味与文稿设计 | 文字、表格、图、证据卡和自检的安排 |
| 成文关 | 学术品味与文稿设计 | 章节入口、段落任务和下一节出口 |
| 输出与倒查关 | 学习与认知原则；其余三份方法 | 学生输出、证据回看、图文回看和重写位置 |

四份方法的职责边界：

| 支撑方法 | 它负责什么 | 它不负责什么 |
|:---|:---|:---|
| `信息检索与筛查.md` | 资料怎样从候选变成正文判断 | 不负责把文档写顺 |
| `学习与认知原则.md` | 学生出口、学习块、自检和回退 | 不负责堆课程或学习资源 |
| `学术品味与文稿设计.md` | 章节接力、段落任务、证据语气和文稿主线 | 不负责替代证据判断 |
| `可视化与理解.md` | 判断哪里需要图、表、矩阵、路线或证据卡 | 不负责为了完整感加图 |

后续改造四份支撑方法时，都要按同一问题检查：

```text
这份方法要把执行者从哪种混乱带到哪种可执行状态？
```

<a id="path-docs"></a>

## `00-04` 怎样共用这套方法

五份成品共用同一套关卡，但出口不同。五份文档要连成一条理解路线，不能写成五份并列报告。

```text
00 教学生怎样读
  -> 01 建立事实底座
  -> 02 建立专业范围感
  -> 03 看懂论文问题链
  -> 04 从基础课走到目标论文
  -> 回到 01 核查事实和证据风险
```

| 文档 | 原始混乱 | 本文出口 | 交给下一份什么 |
|:---|:---|:---|:---|
| `00_材料导读.md` | 学生不知道五份材料怎样读 | 知道阅读顺序、自检方式和回看路径 | 把读者送进 `01`，并知道之后怎样回看 |
| `01_基础画像.md` | 老师资料分散，论文集合和风险不清 | 身份、履历、论文集合和风险边界清楚 | 可信身份、履历阶段、论文集合边界 |
| `02_领域地图.md` | 方向名词混杂，缺少范围感 | 学生能定位大领域、问题域和相邻方向 | 当前领域问题、少数关键概念、目标入口候选 |
| `03_论文路线.md` | 论文只是列表 | 学生能复述论文群的问题链和角色分工 | 目标论文、核心图、论文角色、平台链路、概念缺口 |
| `04_学习向导.md` | 学生只有基础课，不知道怎样接前沿 | 学生知道课程缺口、目标论文片段和进组前输出 | 进组前问题；必要时回到 `01/02/03` 复核 |

改造 `00-04` 时，每份都要按七道关卡回答：

```text
原料是什么？
学生出口是什么？
哪些证据能支撑出口？
理解台阶怎样排？
哪里需要局部图文？
学生怎样输出？
输出失败时回哪里修？
```

<a id="path-references"></a>

## 参考文献怎样变成本项目的方法判断

参考文献不能只放在文末当摆设。总方法不写成长综述，只保存本项目从参考资料中吸收的判断。

| 来源方向 | 文献给出的通用判断 | 本项目吸收成的方法判断 |
|:---|:---|:---|
| 学习科学 | 新手学习需要控制认知负荷，主动提取、自我解释和元认知校准比重读更能暴露理解问题 | 每份文档必须有学生出口、输出任务和回退路径 |
| 科学写作 | 读者依赖句子位置、段落重心和文章结构来判断重点 | 每节必须有入口、推进、局部支撑和出口；段落只做一个主要动作 |
| 信息素养和系统检索 | 单个来源不能自证，权威取决于问题场景，检索过程要可复查 | 身份、论文、领域、学习路径分别需要不同来源层级；弱线索不能承担结构判断 |
| 可视化研究 | 图只有在任务、数据和视觉编码匹配时才降低认知负担 | 先写学生问题和图形规格，再决定正文、表格、矩阵、证据卡或图 |

四份支撑方法保留具体参考资料和细节。总方法只记录这些资料如何被转成本项目的执行判断。

<a id="path-agent-protocol"></a>

## 执行 AI 的最低工作协议

工作台被搞乱，常见原因通常不在文件数量，而在执行者没有先判断自己正在处理哪一层问题。以后执行 AI 先过这组协议。

1. 先判断当前问题属于哪一层：总方法、支撑方法、文档标准、成品样本、skill、verifier、harness。
2. 方法层没有稳定前，不许直接改成品样本来“试压”。
3. 如果只是单个导师样本的问题，先判断能否反推跨导师方法；不能反推，就只改样本，不新增规则。
4. 形成新规则前，先说明它解决哪种失败，替代或补强哪条旧规则。
5. 每次只推进一个层级。不要同时改方法、标准、样本、verifier 和 skill。
6. 聊天里的共识不能自动变成文件。先给结构方案，用户确认后再改。
7. 达到可审查里程碑后，检查 `git status`，需要保存时再提交。

这组协议的目的，是防止局部修补继续制造新支线。

<a id="path-stop"></a>

## 质量检查：什么时候必须停下来

总方法的底线是知道什么时候不能继续写，不能把“文档写完”当作通过。

| 情况 | 必须停在哪里 |
|:---|:---|
| 学生出口说不清 | 停在出口关 |
| 来源只能支撑题名或关键词 | 停在证据关 |
| 章节可以随意换序 | 停在台阶关 |
| 图表说不出学生问题 | 停在支撑关 |
| 段落只是在补完整感 | 停在成文关 |
| 学生输出空泛或答不上 | 停在输出与倒查关 |
| 讨论开始跨到样本、skill、verifier | 停在执行协议 |

每次停下后，不写“后续优化”，要写清回到哪一关、补什么判断、删什么形式。

<a id="path-open"></a>

## 待验证问题和后续回收

这份总方法先用于重构四份支撑方法。重构顺序建议是：

```text
信息检索与筛查
  -> 学习与认知原则
  -> 学术品味与文稿设计
  -> 可视化与理解
```

理由是：资料和证据先稳住，学生输出和回退随后确定，章节与段落接着处理，图文形式排在后面。可视化靠后处理，是为了防止图形再次变成装饰。

待验证问题：

- 七道关卡是否足够约束四份支撑方法。
- 四份支撑方法是否需要统一大纲，还是只统一关卡接口。
- `00-04` 文档标准是否都能写出“上一份给什么、本文做什么、下一份拿什么”。
- 张鹏举黄金样本是否能用这套方法重修，而不再新增支线文件。
- 哪些判断需要回收进 `research-advisor` skill，哪些只留在 workbench。

在这些问题验证前，不回收进正式模板、verifier 或 harness。
