<a id="workbench-role"></a>
## 文档职责

本文件记录支撑 `00-04` 五份导师文档的图文设计原则。它回答三个问题：

- 可视化在什么情况下能降低学生理解成本。
- 不同类型的信息关系适合用什么图形表达。
- 执行 AI 怎样标准化生成图，同时避免为完成任务而画装饰图。

它不替代 `文档标准/` 中每份文档自己的章节设计。每份文档仍要独立决定“这一节让读者做什么、为什么这样设计、读完后留下什么判断”。本文件只提供跨文档的图形选择方法。

<a id="core-judgment"></a>
## 核心判断

图的价值来自外部化关系。学生读一段文字时，要在工作记忆里同时保存对象、层级、顺序、证据强弱、因果和相邻关系；合适的图能把一部分关系放到页面上，让学生直接看见。

前面暴露的问题在这里：我把“可维护”和“稳定批量生成”理解得太窄，导致可视化常常退化成短表、文本链路或简单 Mermaid。这样的图容易维护，却把空间感、范围感、问题推进和学习桥接留给学生自己脑补。

本项目后续采用一条更明确的原则：

> 先判断学生要看见什么关系，再选择图形形式；工具和格式只服务这个关系。

图不是每份文档的装饰配额。删掉一张图后，如果学生理解几乎没有损失，这张图应删掉或重做。反过来，如果某个关系靠文字会让学生在脑中反复拼接，例如领域范围、论文角色、课程到论文的桥，图就应成为正文结构的一部分。

<a id="research-basis"></a>
## 研究依据

### 图文结合能帮学习，但前提是减少额外负担

多媒体学习研究强调，图文结合有效的条件是图和文字共同解释同一件事。图如果只是装饰、重复正文、离正文太远，学生需要额外寻找对应关系，理解负担会增加。

可用到本项目的结论：

- 图旁必须说明读法，不能让学生自己猜颜色、面积、箭头和距离代表什么。
- 图和解释要放在相邻位置，避免学生在页面上来回找。
- 图中只保留完成当前理解动作需要的信息，删掉装饰、过细标签和无关节点。
- 文本和图形表达同一关系时，要分工清楚。图表达结构，正文解释证据、边界和读完判断。

### 认知负荷决定图的复杂度

新手的工作记忆有限。复杂图可能把文字负担换成视觉负担。认知负荷理论提醒我们，教学材料要控制无关负担，让学生的注意力放在真正要理解的关系上。

可用到本项目的结论：

- 一个图只回答一个主问题。
- 长横向流程图容易在 Markdown / PDF 中缩小，学生看不清节点和标签。
- 大图可以拆成“总览图 + 局部图”。总览图给位置，局部图讲关系。
- 表格列太多也会增加横向扫描负担。表格不天然比正文更清楚。

### 多重表征需要分工

多重表征研究指出，同一内容可以用文字、表格、图、公式等多种方式表达。多种表征有三类作用：补充信息、限制误解、帮助学生建构更深理解。它们也可能互相干扰。

可用到本项目的结论：

- 图和正文不能只是重复。重复会占空间，却不增加理解。
- 图应承担正文难以承担的任务，例如空间位置、范围大小、问题链、依赖关系、证据层级。
- 新手需要表征之间的桥。图下说明应告诉学生：“先看哪里，再看哪里，最后得到什么判断。”

### 图适合表达关系，不适合替代证据

经典图示研究认为，图有时比文字有优势，因为它把相邻、包含、对齐、分组等关系直接编码在空间中。这个优势只对适合空间表达的关系成立。

可用到本项目的结论：

- 领域范围、包含关系、相邻方向和粗略大小适合用地图型图。
- 论文之间的推进关系适合用问题链、角色图或矩阵，不一定适合按年份流程图。
- 学习路径适合用依赖图和桥接图，不应只写成课程清单。
- 来源证据适合用证据链或证据表，不适合画成漂亮但无法核查的图。

### 信息可视化先看任务，再看图形

信息可视化设计通常先抽象任务：用户要比较、定位、分布、组成、关系、变化、流动，还是异常。图形选择应匹配任务，而不是从工具列表里挑一个好看的图。

可用到本项目的结论：

- `02` 的任务是定位和范围感，主图应像地图、treemap、气泡图或范围套层图。
- `03` 的任务是论文角色和问题推进，主图应表现“论文围绕什么问题分布、谁是底座、谁是扩展、谁是旁支”。
- `04` 的任务是从基础到目标论文，主图应表现“已有课程、缺口、目标图、论文任务怎样接上”。
- 同一图形在不同任务中可能有用，也可能误导。流程图只适合真实流程、依赖或平台链路。

<a id="selection-workflow"></a>
## 选择流程

每张图生成前，按这个顺序决策：

```text
读者问题 -> 关系类型 -> 证据精度 -> 图形形式 -> 图例与读法 -> 自检输出
```

### 第一步：写出读者问题

图必须先回答一个具体问题。问题应从学生视角写。

合适的问题：

- 这个方向在大专业里处在哪一片？
- 这些论文分别在研究路线中承担什么角色？
- 我从高数、线代、普物出发，还缺哪些知识才能看目标论文？
- 这条判断由官网、论文、综述还是弱线索支撑？

不合适的问题：

- 这里需要一个图。
- 让页面好看一点。
- 把这一段内容可视化。
- 满足验证器要求。

### 第二步：判断关系类型

先判断内容属于哪种关系，再选图。

| 关系类型 | 学生要看见什么 | 常见场景 |
|:---|:---|:---|
| 顺序 | 先后步骤、阅读轮次、阶段推进 | `00` 阅读顺序、`04` 学习阶段 |
| 层级 | 大类、小类、具体问题域 | `02` 学科到问题域 |
| 范围 / 组成 | 哪块大、哪块小、导师在哪一块 | `02` 领域全景 |
| 相邻 / 交叉 | 哪些方向靠近，哪些方向重叠 | `02` 相邻领域、交叉方向 |
| 网络 | 作者、机构、论文或概念之间的关系 | `01` 公开交叉线索、`03` 论文群 |
| 问题推进 | 一个问题怎样被多篇论文推进 | `03` 论文路线 |
| 依赖 | 学会 A 才能看懂 B | `04` 课程到论文 |
| 证据强弱 | 哪些判断稳，哪些需要复核 | `00/01/02` 来源核查 |
| 时间变化 | 领域重心或导师阶段怎样变化 | `01` 履历、`02` 领域发展 |
| 比较 | 多个方向、论文、方法的差异 | `02/03/04` 中的定位与选择 |

### 第三步：判断证据精度

图里的大小、位置、颜色和线条都暗含判断。生成前要分清证据能支撑到什么程度。

| 精度 | 可以怎么画 | 不该怎么画 |
|:---|:---|:---|
| 精确数据 | 柱状图、折线图、treemap 数值面积 | 改写或省略数据来源 |
| 半定量 | 大 / 中 / 小，主流 / 稳定 / 小众 | 写成百分比或精确排名 |
| 定性关系 | 高亮、分组、边框、靠近、包含 | 暗示精确距离或规模 |
| 弱线索 | 虚线、浅色、`需人工复核` 标签 | 画成确定节点或主线 |

本项目多数领域范围图属于半定量或定性关系。没有可复核数据时，不写百分比；用视觉权重和图例说明“这是范围感，不是统计图”。

### 第四步：选择最轻的有效形式

选择原则：能用小图解决，就不用大图；能用图解决关系，就不要让学生在文字里拼。

常用优先级：

1. 真正需要空间感或范围感，用 SVG、treemap、气泡图或范围地图。
2. 真正需要顺序和依赖，用竖向路线图、timeline 或小型流程图。
3. 真正需要多维比较，用矩阵或表格。
4. 真正需要概念关系，用小型概念图或 mindmap。
5. 只是列字段、来源和核查口径，用短表。

### 第五步：给图写读法

每张图附近必须有短说明：

- 这张图回答什么问题。
- 颜色、面积、箭头、位置或边框代表什么。
- 哪些地方是证据支撑，哪些地方只是视觉权重。
- 学生看完后应能复述哪句话。

### 第六步：设置自检输出

图的效果要能被检查。图后可以接一个小输出：

- “请用一句话说出导师方向在领域里的位置。”
- “请选出 2 篇论文，说明它们在路线中分别承担什么角色。”
- “请指出从你的基础到目标论文之间最先要补的 3 个缺口。”

没有输出检查，图很容易只被看一眼。

<a id="markdown-implementation"></a>
## Markdown 实现

Markdown 可以承载多种可视化。选择时同时考虑理解效果、批量生成、版本管理和 PDF 导出。

### 表格

表格适合重复字段、证据核查和多维比较。它的优势是稳定、可审查、适合批量生成。

适用场景：

- 来源符号说明。
- 论文角色矩阵。
- 数据库指标口径。
- 读后自检。
- 证据强弱。

表格的风险是“表单化”。如果一节连续出现多张表，学生可能只是在扫字段，无法形成主线。超过 5 列的表要谨慎，超过 6 行的表前要有读法提示。

### Mermaid

Mermaid 适合轻量、可版本管理的结构图。可以优先考虑这些类型：

- `flowchart`：真实流程、依赖或平台链路。
- `timeline`：领域发展、履历阶段、学习阶段。
- `mindmap`：少量概念关系。
- `quadrantChart`：两个轴都有明确含义的定位。
- `treemap`：组成、范围和粗略大小。
- `venn`：真实交叉关系。
- `block`：套层、模块和局部结构。

Mermaid 的风险是布局控制弱。长横向图、节点过多、标签太长时，导出 PDF 后常常变小。超过 9 个节点要考虑拆图；超过 12 个节点通常应改成总览图加局部图，或改成 SVG / 表格。

### SVG

SVG 适合本项目最缺的地图型图。它可以画范围图、气泡图、嵌套圆、标注图、论文星图和学习桥地图。

适用场景：

- `02` 领域范围图。
- `03` 论文问题地图。
- `04` 学习桥地图。
- 论文核心图的简化标注图。

SVG 的优势是表达力强、导出稳定、可精确控制布局。风险是维护成本高。后续批量化时应尽量用结构化字段生成 SVG，源数据放入 `_internal/figures/` 或同类位置，正文只引用成品图和图例。

### 位图

位图适合照片、论文原图截图或无法用 SVG 表达的真实图像。导师文档中原则上少用位图，因为可编辑性差、来源版权和清晰度风险更高。

若必须使用，应满足：

- 来源清楚。
- 图注说明只用于学习指引。
- 不截取过长版权内容。
- PDF 中清晰可读。

<a id="impact-on-docs"></a>
## 对五份文档的影响

### 00 材料导读

`00` 的主视觉应帮助学生知道五份文档怎样配合。适合使用一张轻量阅读地图或阅读闭环图：

```text
01 粗读事实 -> 02 建领域地图 -> 03 看论文问题链 -> 04 倒推学习准备 -> 01 回看核查
```

这张图不需要复杂。它的任务是防止学生把五份文档当成五篇并列报告。引用符号、读后自检和参考资料仍适合短表。

### 01 基础画像

`01` 的可视化应服务事实底座。主图不应复杂，但可以补两类小图：

- 履历阶段视图：让学生先看到哪个阶段是当前相关阶段。
- 身份证据链：让学生看到官网、作者档案、论文指纹和公开线索怎样互相支撑。

论文集合保留完整表，但长表前必须有阶段读法提示。连续表格过多时，优先把开头和收束改成短段，避免学生感觉自己在读数据库导出。

### 02 领域地图

`02` 的主视觉必须是领域范围图。它回答“导师方向在大专业里处在哪一片，这一片大概有多大，旁边有哪些方向”。

推荐组合：

1. 领域全景：SVG 范围地图、treemap 或气泡图。
2. 范围定位：范围套层图或方向重心图。
3. 概念入口：小型概念图。
4. 发展脉络：只有时间变化能帮助理解时才加入。

这里不能再用纯文字链条替代地图。层级关系只是地图的一部分，范围大小、相邻方向、导师高亮和证据边界同样重要。

### 03 论文路线

`03` 的主视觉应表现论文围绕问题怎样分布。它不应默认按年份画流程图。

推荐组合：

1. 论文问题地图：把目标论文、前史论文、方法论文、旁支论文放到同一张问题空间里。
2. 论文角色矩阵：列出论文、问题、对象、方法、结果、在路线中的角色。
3. 局部问题链：只在论文之间确实存在先后推进时使用。
4. 平台链路图：只在样品、平台、信号和瓶颈关系是理解关键时使用。

学生读完图后，应能说出：“这些论文共同在推进什么问题；每篇论文为什么放在这个位置。”

### 04 学习向导

`04` 的主视觉应是学习桥地图。它把学生已有基础、缺口、目标论文、核心图和进组问题连起来。

推荐组合：

1. 学习桥地图：高数、线代、普物等起点怎样接到目标论文。
2. 先修依赖图：哪些概念必须先补，哪些可以边读边补。
3. 核心图读法标注：把目标论文中的关键图拆成坐标、变量、现象和结论。
4. 学习块路线：每一阶段的目标、材料、输出和回退位置。

普通课程表只能作为辅助。若课程表无法连接到目标论文、核心图或具体输出任务，就还没有完成 `04` 的职责。

<a id="standard-fields"></a>
## 标准化字段

后续批量生成图时，建议每个视觉构件先写一份轻量规格。它可以放在蓝图或图源数据中。

| 字段 | 含义 | 示例 |
|:---|:---|:---|
| `visual_id` | 图的编号 | `02-field-landscape` |
| `reader_question` | 图回答的读者问题 | 导师方向在物理学里处在哪一片？ |
| `relation_type` | 关系类型 | 范围 / 层级 / 问题推进 / 依赖 |
| `recommended_form` | 推荐形式 | SVG 气泡地图 |
| `data_items` | 图中节点或条目 | 物理学、凝聚态、量子材料、超快动力学 |
| `scale_precision` | 大小精度 | 定性范围感 / 半定量 / 精确数据 |
| `evidence_basis` | 图形布局依据 | 学科分类、综述、课程、论文 |
| `encoding` | 视觉编码 | 面积表示范围感，颜色表示与导师关系 |
| `caption` | 图下读法 | 先看大块，再看高亮区域 |
| `self_check` | 看完后输出 | 用一句话说出导师方向位置 |
| `fallback` | 渲染失败时的替代 | 范围定位表 |

这些字段可以让执行 AI 不再凭感觉画图。图的样式可以变化，但读者问题、关系类型、证据依据和输出检查必须稳定。

<a id="quality-gates"></a>
## 质量门

每张图进入成品前，至少过这些检查：

- 读者问题明确：图回答了一个具体学生问题。
- 关系类型明确：图表达的是范围、层级、时间、依赖、问题推进、证据强弱或比较。
- 证据边界明确：大小、位置、颜色和高亮没有暗示无来源的精确判断。
- 图文相邻：图下有读法，不让学生猜。
- 形式合适：流程图只用于真实流程；范围图用于范围；矩阵用于多维比较。
- 信息克制：节点、标签和颜色数量不压垮新手。
- 删除测试：删掉图后，如果理解没有明显损失，图应删或重做。
- 复述测试：学生看完图后，能用一句话说出它支持的判断。
- 导出测试：Markdown / PDF 中能看清标签、线条和图例。
- 可回收：图能被字段、模板或生成规则复刻，不靠一次性灵感。

<a id="anti-patterns"></a>
## 常见反模式

- 为了满足“有图”而画图。
- 把正文句子拆成框，图中没有新增关系。
- 用长横向 Mermaid 表达简单顺序，导出后显示很小。
- 用层级链条冒充领域地图。
- 用流程图表达没有方向或依赖的并列关系。
- 用 mindmap 堆术语，却不说明对象、方法、问题和平台的关系。
- 用面积、距离或颜色暗示无证据的比例、热度或重要性。
- 同一张图回答太多问题。
- 图下没有读法，学生不知道先看哪里。
- 表格列太多，学生横向扫不动。
- 所有导师都套同一种图，忽略学科差异和证据差异。
