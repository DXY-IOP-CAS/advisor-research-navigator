<a id="vis-main-question"></a>

## 中心问题

可视化要解决学生脑子里同时装不下的关系。学生卡住时，常见情况是范围、先后、角色、证据和学习依赖同时挤在一段文字里。

本文件只追一个问题：

```text
这张图能让学生少背哪一段关系？
```

答不上这个问题，就先不画。普通字段对比用表格，短判断用正文，只有空间位置、结构关系、过程路线、证据边界或学习依赖会压住工作记忆时，才进入图形设计。

<a id="vis-gate"></a>

## 先判断要不要画图

执行 AI 不能先选 SVG、Mermaid 或表格。先过一组闸门：

```text
学生卡在哪里
  -> 这是否属于关系问题
  -> 文字或表格能不能更清楚
  -> 图里每个形状有没有语义
  -> 图旁能否给出一句读法
  -> 学生看完能否输出一句判断
```

下面这些情况优先不画图：

| 情况 | 更合适的形式 | 原因 |
|:---|:---|:---|
| 只是列字段、来源、论文题名 | Markdown 表格 | 表格已经能完成扫描任务 |
| 只是提醒学生回看哪里 | 短清单 | 决策节点太少，画图反而占空间 |
| 只是表达“这里很重要” | 正文 | 图不能替代判断理由 |
| 证据不支持空间位置或比例 | 限定性文字 | 图会放大无证据精度 |
| 关系少于 3 个对象 | 一两句话 | 画图成本高于理解收益 |

这一步要先承认取舍。可视化的质量不按数量算，按它是否减少学生的理解负担算。

<a id="vis-components"></a>

## 五类理解构件

这里把可视化收成五类“理解构件”。每一类都对应一种学生卡点。

| 学生卡点 | 可用构件 | 主要服务哪份文档 | 典型输出 |
|:---|:---|:---|:---|
| 不知道导师方向在大领域里处在哪儿 | 范围定位图 | `02_领域地图.md` | 用一句话说出大领域、邻近方向和当前入口 |
| 看不出几个对象之间的角色 | 结构关系图 | `03_论文路线.md` | 说出哪篇论文是底座、方法、目标或旁支 |
| 分不清先后、机制流转或学习依赖 | 路线图 | `01/03/04` | 说明箭头代表时间、机制还是学习前置 |
| 不知道正文判断靠什么来源 | 证据边界卡 | 所有文档 | 区分直接证据、交叉证据、弱线索和需复核 |
| 字段很多，需要并排扫描 | 表格 / 矩阵 | `01/03/04` | 找到关键差异，再回到正文主线 |

### 范围定位图

范围定位图回答“这个方向在大范围里靠哪儿”。它可以用地形式分区、二维定位或少量区域标注。它不负责精确比例；没有统计来源时，只能表达“大 / 中 / 小”“主流 / 相邻 / 新兴”这类粗略位置。

![范围定位图示意](_assets/visualization-guide/field-range-map.svg)

适合 `02_领域地图.md`。学生读完图后，应能说出：“导师方向大致落在什么大领域里，旁边哪些方向容易混淆，后续论文路线会从哪个入口进入。”

### 结构关系图

结构关系图回答“几个对象围绕一个问题怎样分工”。它适合论文角色、概念依赖、平台组成。节点和连线都要有语义；没有语义的网络图只会制造复杂感。

结构关系图不等于证据图。论文角色图的中心是“研究问题”，节点是论文或概念，线条表示支撑、提供方法、验证目标、延伸旁支。证据边界的中心是“正文判断”，旁边放来源和强弱标签，通常不需要画成网络。

### 路线图

路线图回答“从哪里走到哪里”。时间演化、平台链路和学习路线都可能长得像箭头图，区别在箭头语义。

![路线箭头语义](_assets/visualization-guide/route-semantics.svg)

三种箭头不能混用：

| 箭头语义 | 节点表示什么 | 箭头表示什么 | 适合位置 |
|:---|:---|:---|:---|
| 时间先后 | 阶段、履历、论文时期 | 先发生到后发生 | `01` 履历阶段、`03` 问题演化 |
| 机制流转 | 装置、方法、数据环节 | 输入、测量、处理、输出 | `03` 平台或方法链路 |
| 学习依赖 | 已有基础、概念缺口、目标论文片段 | 学会前一步才能做后一步 | `04` 学习向导 |

如果图中箭头说不清这三种语义，就先不画。

### 证据边界卡

证据边界卡回答“这句话靠什么站住”。它的默认形式是正文判断旁边的一小块证据说明，通常不画成网络。

![证据边界卡示意](_assets/visualization-guide/evidence-card.svg)

它适合放在关键判断附近，例如身份锁定、当前方向定位、论文主线判断。覆盖缺口图更多属于内部审查：检查资料篮缺什么。学生正文里通常只需要看到判断附近的证据强弱，不需要看完整采集面板。

### 表格 / 矩阵

表格就是表格，不要为了显得“可视化”把它画成 SVG。字段重复、适合并排扫描时，用 Markdown 表格即可。

只有两个维度共同决定判断时，才使用矩阵。例如：

```text
论文角色 × 证据强度
课程模块 × 目标论文图读法
资料来源 × 可支撑的判断类型
```

普通论文字段表、履历表、来源表都不需要包装成图。

## 几个容易混淆的位置

### 对比矩阵和表格

上一轮把“对比矩阵”画成 SVG 表格，是错误示范。字段并排比较时，Markdown 表格更清楚、更可维护。矩阵只有在两个维度都参与推理时才有必要。

### 平台图、学习路线图和时间图

它们可能都用箭头，但读法完全不同。平台图读“系统怎样运转”，学习路线图读“先补什么才能读什么”，时间图读“阶段怎样变化”。如果图题、箭头和图下注释没有说明这个语义，学生看到的只是普通流程图。

### 论文角色图和证据边界图

论文角色图服务 `03` 的问题链。证据边界服务所有文档的可靠性判断。前者帮助学生理解研究内容怎样分工，后者帮助学生判断一句话的证据强弱。把两者都画成节点连线，会抹掉这层差别。

### 覆盖缺口和阅读决策

覆盖缺口多用于内部质量审查，帮助执行者发现还缺官网、综述、课程或论文来源。阅读决策多半用短清单即可，例如“能复述就继续，不能复述就回看某节”。只有条件分支超过两层、文字会变绕时，才画小决策图。

<a id="vis-doc-roles"></a>

## 放进 00-04 的方式

五份导师材料不需要每份都塞满图。每份只放能推动当前出口的构件。

| 文档 | 当前学生最容易卡在哪里 | 优先构件 | 常见降级形式 |
|:---|:---|:---|:---|
| `00_材料导读.md` | 不知道怎么读、何时回看 | 阅读顺序短图或自检清单 | 清单优先，少画决策图 |
| `01_基础画像.md` | 身份、履历、论文集合和风险混在一起 | 履历时间线、来源表、证据边界卡 | 表格优先，少画网络图 |
| `02_领域地图.md` | 名词很多，没有范围感 | 范围定位图、少量收束链 | 证据不足时改成限定性文字 |
| `03_论文路线.md` | 论文列表看完没有问题链 | 结构关系图、必要的平台路线图 | 字段对比用表格 |
| `04_学习向导.md` | 课程、概念、论文图之间接不上 | 学习依赖路线、核心图读法标注 | 复杂路线拆成多个学习块 |

图必须嵌在正文里。正文先提出学生问题，图回答这个问题，图下给读法，后文消费这张图。如果图放完后正文再也不提它，这张图通常该删。

<a id="vis-spec"></a>

## 图源规格

每张图进入成品前，先写一份很短的规格。规格用来防止执行 AI 凭感觉画图。

| 字段 | 要写什么 |
|:---|:---|
| `reader_question` | 学生看完这张图要少迷糊哪一件事 |
| `component` | 范围定位、结构关系、路线、证据边界、表格 / 矩阵 |
| `evidence_basis` | 位置、大小、颜色、线条来自哪些来源或推断 |
| `encoding` | 形状、颜色、线型、箭头分别表达什么 |
| `caption` | 图旁读法，告诉学生先看哪里，再看哪里 |
| `student_output` | 学生看完后能复述的一句话或能完成的小任务 |
| `fallback` | 图不适合时，改成哪种正文、表格或清单 |

这份规格写不出来时，图先暂停。

<a id="vis-quality-check"></a>

## 质量检查

一张图进入 `00-04` 前，至少过这组检查：

| 检查项 | 通过标准 |
|:---|:---|
| 必要性 | 删除这张图后，学生理解会明显变差 |
| 问题绑定 | 图回答一个学生问题，不能只重复标题 |
| 语义清楚 | 每个颜色、形状、线条、箭头都有明确含义 |
| 证据克制 | 面积、坐标、距离不暗示无来源精度 |
| 图文接力 | 图前有问题，图下有读法，图后正文会使用它 |
| 新手负担 | 标签、节点和颜色数量不会压垮阅读 |
| 输出检查 | 学生看完能说出一句判断或完成一个小任务 |
| 渲染稳定 | VS Code Markdown Preview、HTML 和 PDF 中能看清 |

通过这张表只能说明图没有明显底线问题。真正的质量要在 `00-04` 的具体段落里看：这张图是否让学生更容易复述、更容易回看、更容易继续读。
