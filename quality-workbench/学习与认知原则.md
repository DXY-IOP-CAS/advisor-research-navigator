# 学习与认知原则

状态：原则主体完成，随 `01-04` workbench 继续校准。

## 文档职责

本文件记录支撑 `00-04` 五份导师文档设计的学习科学和用户经验模型。它不是最终 Harness 规则，不是教育学综述，也不是给学生直接阅读的成品章节。

它要回答：

- 学生怎样从“看过材料”走向“真的理解并能行动”。
- 哪些学习方法有较强证据，哪些常见说法需要谨慎。
- 用户提出的“主线剧情 + 费曼输出 + 苏格拉底提问”怎样映射到可执行的文档设计。
- 这些原则如何反推 `00-04` 的章节职责、图文配合、自检问题和 Harness 质量门。

## 核心判断

导师调研材料不是资料包，而是一个带脚手架的学习环境。学生真正缺的不是更多信息，而是：

1. 能降低认知负荷的进入顺序。
2. 能组织零散知识的主线结构。
3. 能把输入转成输出的自检任务。
4. 能发现误解和弱点的提问机制。
5. 能回到证据来源的核查路径。

因此，五份文档不应只回答“这里有什么内容”，还应在自然语言中告诉学生“为什么这样读、这样做能帮助理解”。但不要把教育学术语直接堆给学生；学生需要的是可执行的阅读动作、自检问题和回退路径。

## 原则总览

| 原则 | 对学习的含义 | 对本项目的设计含义 |
|:---|:---|:---|
| 降低认知负荷 | 新手工作记忆有限，不能一开始承受过多术语、论文和分支 | 先给主线、分段、短表和回退路径；避免长段堆叠和大而全图 |
| 先建组织框架 | 先有领域坐标和问题主线，后续细节才有挂靠位置 | `02` 建领域地图，`03` 建论文问题链，`04` 把学习路径挂回目标论文 |
| 输出促进理解 | 能看懂不等于会；主动复述、解释和提问才能暴露弱点 | 每份文档都要有出口检查，`00` 设置读后自检，`04` 设置进组前输出任务 |
| 自我解释 | 学生要解释“为什么这样分类、为什么这篇论文接上那篇” | 正文不能只给结论，要给判断理由；图表旁说明读法和应复述的判断 |
| 元认知校准 | 学生要知道自己哪些懂了，哪些只是以为懂了 | 设置自检问题、证据强弱标记和“答不上就回看哪一节”的路径 |
| 苏格拉底式提问 | 通过小问题逐步暴露前提、因果和边界，避免信息过载 | 每节用少量关键问题引导，不一次塞满全部信息 |
| 单任务沉浸 | 一段学习时间最好只处理一个主问题，频繁切换会留下注意残留 | `00` 把每轮阅读设计成学习块，`04` 把学习路径拆成可进入状态的任务块 |
| 收尾与复现 | 中断前留下上下文，下一次才能快速接回主线 | 每个学习块结束时生成小输出：当前理解、卡点、下次入口 |
| 目标倒推 | 先定义学生最后要能做什么，再设计内容和检查方式 | 从目标论文、核心图、平台链路倒推 `04` 学习路线 |
| 图文服务理解 | 图表应减少理解负担，而不是满足“有图”要求 | 领域用层级/定位表，论文用问题矩阵，学习用课程-能力-论文桥 |
| 警惕学习风格神话 | “视觉型/听觉型学习者”式分类证据弱 | 不按人设堆图；按内容结构选择最合适的表达媒介 |

## 用户经验模型

用户当前的潜意识经验可以整理为三件事：

1. 主线剧情：先建立一条贯穿全局的主线，避免在 AI 对话或资料阅读中被细枝末节带偏。
2. 费曼学习法：通过输出解释来发现自己真正不会的地方。输入时觉得懂，输出时才暴露结构缺口。
3. 苏格拉底式提问：用一轮一轮小问题推进理解，降低认知过载，同时迫使自己解释前提、因果、边界和反例。

这三件事和学习科学证据基本同向：

- 主线剧情对应 schema 建构、advance organizer、constructive alignment 和认知负荷控制。
- 费曼学习法对应 retrieval practice、self-explanation、learning by teaching 和 constructive engagement。
- 苏格拉底式提问对应 metacognitive monitoring、guided questioning 和分段脚手架。

需要注意的是，三者不能只停留在口号上。若文档只说“请用费曼学习法复述”，学生仍然可能不知道复述什么。更好的做法是给出具体输出对象：

- 用一句话定位方向。
- 用 3-5 句话解释论文问题链。
- 用一张表列出自己缺的课程知识和方法能力。
- 用 2-3 个问题请教导师或师兄师姐。
- 对关键判断标注“直接证据、交叉证据、弱推断、需人工复核”。

## 学习效率与沉浸原则

这一节不是给 `00-04` 之外再加一个独立成品板块，而是在“学习与认知原则”内部把学习时段如何组织讲清楚。之后它应被融入阅读顺序、出口检查、学习任务和 AI 辅助学习提示中。

核心判断：学习导师方向时，学生不只需要“内容顺序”，还需要“进入状态的方式”。如果一边读文档、一边查很多旁支、一边刷手机、一边开 AI 问无限问题，表面上很忙，实际上主线很难沉淀。

设计含义：

1. 单个学习块只解决一个主问题。例如本轮只粗读 `01` 建立事实轮廓，或只读 `03` 复述论文问题链。不要在同一轮里同时追求查完所有术语、读懂所有论文和规划全部课程。
2. 学习块开始时先写清目标：本轮读哪一份文档、只抓什么、暂时不展开什么、最后要留下什么输出。
3. 学习块中允许记录旁支问题，但不要立刻全部展开。可把旁支放入“稍后查”清单，避免主线被打断。
4. 休息不是切到高刺激信息流。短休息应降低干扰，帮助注意力恢复；如果休息变成刷信息流，下一轮需要重新加载上下文。
5. 中断前必须做收尾记录。最小格式是：“我刚才在读什么；我现在理解到什么；我卡在哪里；下次从哪里继续。”
6. 每个学习块用一个小输出收束，例如一句话方向定位、3-5 句话论文问题链、一个课程缺口表、一个请教问题清单。

AI 辅助学习时尤其需要主线防跑偏。推荐在对话开始时使用一个轻量提示：

```text
本轮主线：
本轮只解决：
我已经知道：
我卡在：
不要展开：
最后请给我：
```

这不是为了把学习变成机械流程，而是为了让 AI 的发散能力服务主线。导师文档应承担“指南针”和“字典”的作用：学生可以和 AI 讨论，但文档要帮助学生知道当前讨论属于哪一层、是否偏离目标、应该回到哪份材料。

## 对五份文档的影响

### 00 材料导读

`00` 是学习入口，不是正文摘要。它应承担四个学习功能：

1. 建立阅读契约：告诉学生这套材料是阅读路线图，不是导师评价或教材。
2. 控制阅读负荷：告诉学生每一步读到什么程度即可继续。
3. 设置读后自检：让学生知道读完后要能输出什么，而不是只看完。
4. 说明证据核查：让学生知道顺滑叙述不等于可靠结论。
5. 管住学习节奏：告诉学生每一轮阅读只解决一个主问题，不要同时展开所有旁支。

因此，`00_材料导读.md` 已采用 `读后自检`，不再使用偏被动或读音上接近 `来源核查` 的旧命名。`阅读顺序` 管过程检查点，`读后自检` 管整套材料的最低输出能力。

### 01 基础画像

`01` 是事实底座。它的学习功能不是让学生立刻理解研究方向，而是让学生知道“这个老师是谁、资料从哪里来、论文集合大概是什么、哪些信息有风险”。

设计重点：

- 第一遍只抓身份、机构、履历阶段、论文集合和来源风险，避免陷入“每篇论文都想读懂”的资料泥潭。
- 不让学生一开始就用论文列表判断当前方向。
- 缩写和术语第一次出现要解释。
- 履历、论文、来源风险要排版清楚，避免一长段压成一坨。
- 读完后学生应能说出身份、机构、履历阶段、论文集合和来源缺口。

### 02 领域地图

`02` 是组织框架。它帮助学生给后续论文找到“挂靠位置”。

设计重点：

- 不写成百科词条。
- 从大领域到小问题域，再定位导师当前方向。
- 用层级树、定位表或小地图帮助学生建立坐标。
- 每个领域概念都要解释“为什么它对理解这位导师重要”。
- 本文的学习块目标是建立坐标，不是穷尽领域知识；不懂的旁支概念先记录，只有影响导师定位时才展开。

### 03 论文路线

`03` 是问题链。它不能是论文摘要列表，而要让学生看出论文之间如何共同推进一个研究问题。

设计重点：

- 先讲当前论文群在追什么问题，再讲单篇论文角色。
- 用问题-方法-对象-作用矩阵替代长串论文介绍。
- 引导学生自我解释：为什么这篇论文是底座、机制、平台、扩展或旁支。
- 读完后学生应能用 3-5 句话复述论文问题链。
- 阅读时保持连续问题链，不要在每篇论文题名上无限展开背景；先判断它在路线中承担什么角色，再决定是否需要深挖。

### 04 学习向导

`04` 是目标倒推。它不能只是课程表，而要从目标论文、核心图和平台链路倒推出学习路径。

设计重点：

- 从“高数 + 线代 + 普物”起点出发，逐步连接课程知识、方法能力、领域概念和目标论文。
- 每一步说明为什么要学、学到什么程度、怎样接到论文或核心图。
- 设置输出任务：复述目标论文、解释核心图、列出学习缺口、提出请教问题。
- 允许回退：读不懂目标论文回 `03`，概念位置不清回 `02`，事实来源不清回 `01`。
- 把学习路线设计成若干单任务学习块：每块有进入目标、核心材料、输出任务和下次入口，避免把路线写成一张没有执行节奏的课程清单。

## 融入成品的方式

这些原则应融入 `00-04`，但不应把成品写成“学习科学讲义”。推荐方式：

1. 用一句自然语言解释设计原因。例如：“第一遍不要求读懂所有论文，因为此时你还没有领域坐标，过早精读容易把早期方向和当前主线混在一起。”
2. 每节提供出口检查。例如：“读完本节，你应能用一句话说出当前方向处在哪个问题域。”
3. 图表旁写清读法。例如：“先看左列的问题，再看中列的方法，最后看右列说明这篇论文在路线中的作用。”
4. 对新手设置回退路径。例如：“如果这里看不懂，先回到 `02_领域地图.md` 的领域坐标。”
5. 避免教育学术语外显化。内部可以用 retrieval practice、self-explanation、cognitive load；正文面向学生时写成“试着不看原文复述”“用自己的话解释”“答不上就回看”。
6. 用学习块控制发散。例如：“这一轮只做一件事：先用自己的话复述论文问题链；旁支术语先记下来，复述不通时再回查。”

## 对 Harness 的影响

后续统一回收时，可以把这些原则分流：

- `docs/计划书.md`：沉淀为什么五份文档必须是学习脚手架，而不是资料堆叠。
- `research-advisor references/`：沉淀每份文档如何设置主线、输出、自检和回退。
- `assets/templates/`：把“出口检查”“读后自检”“图表读法”做成骨架。
- `scripts/` 或 verifier：只做机械底线，例如是否存在自检表、图表读法、术语首次解释；不宣称能验证教学质量。

## 后续校准问题

- 每份成品是否都需要一个短的“读完本节应能……”出口检查。
- `04_学习向导.md` 中输出任务应做到多具体，才既能驱动学习，又不造成认知过载。
- 如何让可视化支持自我解释，而不是成为装饰。
- 如何设计 AI 辅助学习时的“主线防跑偏”提示。
- 学习块长度是否需要给推荐范围，还是只给“单目标、可收尾、可复现”的判断标准。
- 成品文档中学习效率提示应写到什么程度，才不会喧宾夺主。
