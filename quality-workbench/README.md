# 质量工作台

`quality-workbench/` 是当前阶段的设计源头。这里先收束 `00-04` 五份导师文档的标准、学习设计、信息检索和证据筛查方法；正式 Harness 等五份文档标准稳定后，再从这里统一回收。

旧计划书、旧模板、旧 skill reference 和旧 verifier 暂时只作历史参考。它们仍可能保留有用约束，但不反过来压过这里已经确认的文档设计结论。

## 阅读顺序

1. 先读 `文档标准/00_材料导读.md`，确认五份文档怎样组成一条阅读路径。
2. 再按 `01 -> 02 -> 03 -> 04` 阅读 `文档标准/` 中的各份标准。
3. 遇到学习路径、认知负荷、读后自检等问题，查 `支撑方法/学习与认知原则.md`。
4. 遇到图文搭配、可视化选择、主图设计和图质量门问题，查 `支撑方法/可视化与理解.md`。
5. 遇到身份核查、论文归属、来源冲突和证据强弱问题，查 `支撑方法/信息检索与筛查.md`。

## 工作原则

- 先讨论成品质量，再反推 Harness。
- 只记录跨导师可复用的结论，不记录聊天流水账。
- 未讨论清楚的内容先留在工作台，不急着升格为正式规则。
- 不为每个小反馈新增文件；能并入现有文档的，先并入现有文档。
- 五份文档标准完成后，统一回收进成品、模板、skill reference、verifier、计划书或删除清单。

## 边界

`文档标准/` 管五份文档各自怎样写。它回答“这份文档为什么存在、内部怎样组织、读者读完能做什么”。

`支撑方法/` 管跨文档方法。它回答“为什么这样设计学习路径、怎样查资料、怎样判断证据可靠、怎样避免顺滑但错误的结论”。

README 只管入口和边界。不要把具体小节规范、长篇讨论或过程记录塞进 README。
