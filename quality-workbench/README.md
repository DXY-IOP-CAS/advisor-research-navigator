# 质量工作台

`quality-workbench/` 是当前阶段的设计手册。它先把 `00-04` 五份导师文档怎样服务学生理解讨论清楚；等标准稳定后，再从这里抽取正式 Harness 规则、模板、脚本检查和计划书内容。

这里不追求把所有讨论都塞进文档。真正要沉淀的是可复用的判断链：学生为什么卡住，这份文档怎样带他跨过去，哪些图文形式能降低理解成本，执行 AI 怎样稳定复刻。

旧计划书、旧模板、旧 skill reference 和旧 verifier 暂时只作历史参考。它们可能有可用约束，但不能压过这里已经确认的成品设计结论。

## 当前主线

五份成品文档要组成一条连续的理解路线：

```text
00 教学生怎样读
  -> 01 建立事实底座
  -> 02 建立专业范围感
  -> 03 看懂论文问题链
  -> 04 从基础课走到目标论文
  -> 回到 01 核查事实和证据风险
```

`quality-workbench/` 的任务，是把这条路线拆成可写、可审、可复刻的设计标准。每份标准都要回答同一个核心问题：这一份文档让学生完成哪一步理解动作，上一份交给它什么，它又把什么交给下一份。

## 阅读方式

新读者先按这个顺序看：

1. 先读本 README，理解 workbench 的主线和写作纪律。
2. 再读 `支撑方法/从资料堆到理解路径.md`，先把“资料怎样变成学生能走的理解路径”这条总链条建立起来。
3. 然后读 `文档标准/00_材料导读.md`，确认五份成品怎样组成一条阅读路径。
4. 继续按 `01 -> 02 -> 03 -> 04` 阅读 `文档标准/`。
5. 遇到具体问题，再回到四份支撑方法查细节：文稿主线看 `学术品味与文稿设计.md`，学习节奏看 `学习与认知原则.md`，图文搭配看 `可视化与理解.md`，证据判断看 `信息检索与筛查.md`。

如果某份支撑方法读起来像资料堆砌，应先回到 `从资料堆到理解路径.md` 的总链条，检查它有没有把读者从混乱带到可执行状态。支撑方法服务五份成品文档的设计，不写成综述论文或规则仓库。

## 方法接入

改 `文档标准/` 时，先用总链条定位问题，再打开对应支撑方法。这样能避免“遇到问题就加一节”的惯性。

| 当前卡点 | 先看哪里 | 需要拿走什么 |
|:---|:---|:---|
| 不知道该写什么、删什么、怎样排 | `支撑方法/从资料堆到理解路径.md` | 学生出口、证据筛选、理解台阶、局部支撑、反向审稿 |
| 章节各说各话，读完没有主线 | `支撑方法/学术品味与文稿设计.md` | 读者路径、章节接力、局部支撑和坏味道检查 |
| 学生看完材料但不会复述、不会行动 | `支撑方法/学习与认知原则.md` | 学习块、输出任务、自检问题和回退路径 |
| 身份、论文、领域判断怕查错或写满 | `支撑方法/信息检索与筛查.md` | 查全、消歧、证据分级、冲突降级和正文边界 |
| 图放了但没帮学生理解 | `支撑方法/可视化与理解.md` | 先判断要不要画图，再选择范围定位、结构关系、路线、证据边界或表格 / 矩阵 |

每份 `文档标准/` 也要反向写清自己调用了哪些方法。比如 `02_领域地图.md` 主要调用信息检索来确认领域边界，调用可视化来建立范围感，调用学习原则来控制概念数量，调用学术品味来检查章节是否接力。

## 写作纪律

以后修改 `quality-workbench/`，先守住这些规则。

每份文档先写中心主线。开头不要先放形式化“阅读入口”。先用短段说明：这份文档解决什么问题，读者为什么会卡住，本文怎样带他往前走。

章节必须接力。每个主要章节都要能说清四件事：

- 上一节交给我什么。
- 我让读者完成什么理解动作。
- 我产出什么判断、问题或检查结果。
- 下一节怎样接着使用这个结果。

标准化服务主线。可以有固定骨架、清单和模板，但它们只能节约时间、统一底线和帮助批量化，不能替代文档自己的推进逻辑。

表格要克制。表格只用于字段、对比、来源、检查清单和多维矩阵。主线解释优先用短段落；如果表格只是把正文切成格子，就改回正文或小列表。

图文先问读者问题。任何图、表、Mermaid 或 SVG，都要先回答“学生需要看见什么关系”。普通字段用表格；范围感用范围定位图；对象分工用结构关系图；先后、机制和学习依赖用路线图，但必须写清箭头语义；证据可靠性优先用判断旁边的证据边界卡。

待确认和回收放到文末。“待确认问题”“后续落地清单”“参考资料”属于工作区，应避开正文推进线。

## 推荐骨架

`文档标准/` 和 `支撑方法/` 可以有不同细节，但建议都按这条路组织：

```text
中心主线
  -> 读者困难
  -> 推进链
  -> 关键章节或关键方法
  -> 图文与证据怎样服务主线
  -> 最低质量检查
  -> 待验证问题
  -> 参考资料或回收位置
```

其中最重要的是“推进链”。如果一份文档只有章节并列，没有从入口到出口的连续动作，就还没有写成。

`文档标准/` 还应额外说明五份成品之间的接力关系：

```text
上一份文档给了什么事实、坐标或问题
本文件怎样继续推进
下一份文档需要拿走什么
```

`支撑方法/` 应额外说明它如何服务 `00-04`，不要写成外部资料综述。例如：

- `从资料堆到理解路径.md` 的主线是：先定学生出口，再筛证据、铺台阶、配支撑、写正文、让学生输出，并用输出倒查全文。
- `学习与认知原则.md` 的主线是：把主线、少量输入、主动输出、自检和回退写进阅读动作。
- `可视化与理解.md` 的主线是：先判断要不要画图，再让图回答一个学生卡点。
- `信息检索与筛查.md` 的主线是：候选资料经过消歧、分级和冲突处理后，才生成正文判断。
- `学术品味与文稿设计.md` 的主线是：先设计读者路径和章节接力，再安排段落、图表、证据和输出。

## 目录

- `文档标准/00_材料导读.md`：设计整套材料的阅读入口、阅读顺序、读后自检和来源核查。
- `文档标准/01_基础画像.md`：设计事实底座，回答老师是谁、资料是否可靠、履历和论文集合怎样读。
- `文档标准/02_领域地图.md`：设计专业范围感，回答导师方向处在哪片领域、旁边有哪些方向、学生先抓哪些概念。
- `文档标准/03_论文路线.md`：设计论文问题链，回答论文群怎样共同推进导师当前方向。
- `文档标准/04_学习向导.md`：设计学习桥，回答学生怎样从基础课走到目标论文和进组问题。
- `支撑方法/从资料堆到理解路径.md`：总方法，记录资料怎样被加工成学生能理解、能复述、能核查、能继续学习的路径。
- `支撑方法/学习与认知原则.md`：记录学生怎样从看过材料走到能复述、能自检、能回退。
- `支撑方法/学术品味与文稿设计.md`：记录资料怎样变成读者路径、章节接力和段落任务。
- `支撑方法/可视化与理解.md`：记录哪些学生卡点需要图，哪些应改用正文、表格或证据卡。
- `支撑方法/信息检索与筛查.md`：记录搜索结果怎样经过消歧、证据分级和冲突处理后进入正文判断。

## 边界

`quality-workbench/` 暂不承担最终成品或正式 Harness 职责。它先服务讨论、试写和设计校准。

只有跨导师可复用、能稳定提高成品质量、归属清楚的结论，才在统一回收时进入模板、skill reference、verifier、计划书或脚本。单次讨论里的想法先放在对应文档的待验证区；如果无法接到主线，就不要沉淀。

README 只管主线、阅读方式和写作纪律。具体章节规范留在各文件内部；长篇过程记录不进 README。
