# 质量工作台

这个目录用于集中记录 `00-04` 五份成品文档的讨论结论。

它不是最终 Harness 规则，不是成品文档，也不是项目计划书。它的作用是先把用户和 AI 对每份文档的中心思想、大纲、章节逻辑、图文设计和待确认问题集中放在一起。等五份文档讨论清楚后，再统一决定哪些内容进入成品、模板、skill reference、verifier 或计划书。

工作原则：

- 先讨论成品质量，再反推 Harness。
- 每份文档单独记录，避免规则散落。
- 只写可复用结论，不记录聊天流水账。
- 未讨论清楚的内容先放在工作台，不急着升格为项目规则。
- 五份文档都讨论完后，做一次统一回收和瘦身。
