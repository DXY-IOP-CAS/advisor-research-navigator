# 输出文件模板索引

本目录的 `templates/` 段定义每个阶段产出文件的字段。这些是**结构骨架**——每个具体导师的产出文件必须包含这里列的所有字段。

## 00_身份验证卡.md

用途：让用户**一眼看清**"我们找到了谁、要爬什么"并决策。

**必填字段**：
- 姓名（中/英）
- 机构
- 邮箱（如可提取）
- 职称
- 各学术平台的候选 ID + 命中率
- 数据来源列表（4-7 项）
- AI 推断的学科归类
- 用户决策点："确认是这个人吗？"

**禁止字段**：不要列论文摘要、不要列研究综述——这些是阶段 1-3 的内容。

## 01_基础画像.md

用途：完整学者 profile。

**frontmatter（YAML）**：
```yaml
---
affiliation: <机构>
department: <部门>
tags: [<方向列表>]
source_updated: <YYYY-MM-DD>
orcid: <id 或 null>
openalex_id: <id 或 null>
arxiv_id: <id 或 null>
inspire_id: <id 或 null>
s2_id: <id 或 null>
google_scholar_url: <url 或 null>
---
```

**9 个标准节**：

| 节 | 标题 | 数据来源 | 字段 |
|:--:|:-----|:--------|:-----|
| 1 | 身份标识 | 全部 | 中英文姓名、职称、机构、邮箱、主页、各 ID |
| 2 | 学术履历时间线 | ORCID、官网 | 教育（年份+学校+学位）、工作（年份+机构+职位） |
| 3 | 研究方向 | 官网、S2 领域标签、论文标题聚类 | 总体概览 + 具体方向分点 |
| 4 | 代表性论文 | OpenAlex works | 表格：标题/期刊/年份/DOI/引用数/一句话说明 |
| 5 | 学术影响力指标 | OpenAlex 作者页 | 总引用、h-index、人才项目、获奖 |
| 6 | 合作网络 | OpenAlex 论文合作者字段 | 博导/博后老板/高频合作者 |
| 7 | 公开多维信息 | Web Search（新闻/采访/讲座）+ 机构主页 | 每条带 URL |
| 8 | 技术关键词 | 论文标题+摘要提取 | 实验技术/理论工具/研究对象/应用方向 |
| 9 | 来源说明与多源比对 | 汇总所有数据源 | URL 出处、一致性表、缺失标注 |

**铁则**：无数据留空不编、所有外部引用必须有 URL。
