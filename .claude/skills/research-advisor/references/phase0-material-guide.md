# 阶段零：材料导读

## 目的

`00_材料导读.md` 是成品材料的读者入口。它不是项目说明、不是开发交接、也不是正文摘要合集；它负责让第一次打开导师目录的学生知道五份文档如何阅读、每份文档解决什么问题、引用符号如何理解，以及怎样从事实画像走到领域、论文和学习路径。

导读默认读者只看到当前导师的成品目录，不知道项目规则，也不一定会读其他导师材料。因此导读必须自包含，不能写“详见项目说明”来替代解释。

## 必须回答的问题

1. 这套材料不做什么：不评价导师、不做匹配推荐、不替代论文和教材。
2. 第一遍应该怎样读：先用 `01_基础画像.md` 建立导师初印象，再读 `02_领域地图.md`、`03_论文路线.md`、`04_学习向导.md`，最后回看 `01_基础画像.md` 核查事实和来源风险。
3. 为什么要这样读：第一遍画像给人物和论文集合的粗轮廓；领域、论文和学习路径补足后，回看画像能更好理解履历、合作网络和论文分组。
4. 引用符号如何读：`[O#]` 是官方/身份来源，`[P#]` 是论文来源，`[R#]` 是课程、教材、综述和学习资源，`[B#]` 是背景辅助来源。
5. 起步闭环是什么：用一篇目标论文、一组核心图和一条平台链路，把“看过资料”转成可讨论的问题。
6. 内部风险如何看：遇到 `[未找到]` 和 `需人工复核` 不要当成已确认结论。

## 写作规则

- 用面向读者的说明，不使用 pipeline、harness、skill、agent 交接等内部工程词。
- 建议阅读顺序必须包含两次 `01_基础画像.md`：先粗读建立印象，最后回读做事实核查。
- 建议阅读顺序不能只写一句话或只放表格。必须先用分步文字解释为什么 `01 -> 02 -> 03 -> 04 -> 01` 这样读，再给一个短表格方便快速扫描。
- 导读可以比其他文档短，但不能省略引用说明和使用边界。
- 起步闭环只给最小可讨论入口，不要替代 `04_学习向导.md`。
- 正文原则上使用上标引用键，完整 URL 放到文末 `参考文献与资料`。
