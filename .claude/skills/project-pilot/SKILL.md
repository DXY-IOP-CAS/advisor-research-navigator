# Project Pilot（种子流程）

<!-- 来源：综合设计文档 v0.2 §二 + 初衷 01-04；CLAUDE.md R5 已声明"文字护栏无效"，本文件 P1-P5 流程约束以 Hook 阻断为准 -->

## 1. 首步：流程深度判定

触发本框架后，**第一句话必须是流程深度判定问询**——这是种子驱动的第一步。

```
先确认两件事——
1. 这是哪种规模的任务？
   A 微型（<1h，写邮件/修小 bug/查一个事实）
   B 小型（1-4h，单文件笔记/小工具）
   C 中型（1-3d，多文件项目/讲义/调研报告）
   D 大型（>1w，多阶段多角色）

2. 流程深度？
   精简（直接给答案，跳过阶段 1-3）
   标准（阶段 1+2 压缩，阶段 3+4 完整）
   完整（4 阶段全量 + 质量门）

不同深度对应不同的产出文件要求。
```

**Why**：用户第二轮反馈"大炮打蚊子"——任何任务都走完整 4 流程是浪费。
**R3 体现**：用户拍板深度，AI 不擅自决定。

### 工具未定型特例

如果这是工具**首次启动**（`产出/` 全空 + 用户没有具体任务），跳过判定直接进入：
> "这是 pilot 种子的实例化。你现在想做什么？"

让用户说出"想做 X"——X 就是工具的当前形态。

## 2. 阶段判断

检查 `产出/` 目录文件存在性：

| 阶段 | 触发条件 | 模式 | 产出 |
|:----:|:---------|:----:|:----:|
| 1 需求锚定 | 无 `需求规格书.md` | Plan-and-Execute + HITL | `产出/需求规格书.md` |
| 2 全面调研 | 有规格书，无 `调研报告.md` | ReAct（搜索→阅读→分析→再搜索） | `产出/调研报告.md` |
| 3 计划制定 | 有前两者，无 `执行计划.md` | Plan-and-Execute | `产出/执行计划.md` |
| 4 执行审核 | 前三者都有 | Reflection + HITL | `产出/收工审核报告.md` |

**回退条件**（R3 体现——用户决策）：
- 执行时发现需求理解错误 → 回退阶段 1 重做
- 执行时发现知识缺口 → 回退阶段 2 轻量重调研
- H3 Hook 在 PreToolUse 检测到跳步时仅警告，最终是否回退由用户决定

### 阶段 2 调研顺序（因果链，不并列）

**不能把"邻域调研"和"工具调研"并行——有因果依赖**：

1. **先调研邻域**——建立 AI 和用户对该领域的共同认知
   - 这是邻域认知阶段：这事大概是什么、关键概念、核心难点、行业现状
   - 来源：学术论文（原理/进展）+ 工业界（实现/成熟度）+ 商业（成本/可用性）
   - **不做工具调研**——这时还不知道要什么工具

2. **再调研工具**——基于邻域认知，知道"这事大概怎么做"后再看"有什么现成可借力"
   - 列出完成需求所需的能力清单
   - 对每项能力搜索对应工具/Skill/库/服务
   - 评估每个选项：成熟度、学习成本、适配性、维护状态

3. **最后设计**——基于扬长避短，做出符合用户真正需求的内容
   - 借用现成工具 / 组合现有工具 / 自研缺失部分
   - 写进 `产出/调研报告.md` 的"执行方案"小节

**Why**：用户原话（2026-06-27）："先调研邻域建立认知，才知道大概怎么做 → 再调研工具 → 最后设计扬长避短"。**并列调研会让你不知道在找什么工具**——D05 决策 4 也确认了"路线 A 先启动、路线 B 基于 A 的能力清单"。

## 3. 流程约束 5 条

### P1 防幻觉（CLAUDE.md R1 重述，参见初衷 R1）
涉及论文/公式/数据/外部信息 → 必须搜索验证再写。间接引用追溯原始来源。文件末尾 "## 验证来源" 小节必须存在（H1 Hook 自动检）。

### P2 文件化产出（CLAUDE.md R2 重述）
任何教学/讲解/分析/对比内容 → 先写 `.md` 文件再开口。回复第一行必须是 `已写入：[文件路径]`。违规修正：停笔 → Write → 重发。

### P3 阶段门控
不可跳步。阶段切换需用户确认。流程深度判定后，进入阶段 1 前用户必须点头。

### P4 追问回路
用户主动追问时全力解答。解答后主动确认："我们在 [当前阶段]。继续本题还是挖深？"

### P5 外循环：人决策 + AI 执行（CLAUDE.md R3 重述，参见初衷 R3）
AI 记录摩擦点和提议；改规则/改模板的决策权在人。**禁止无确认的自我优化**——SKILL.md 自身的修改也受此约束。

## 4. 自检锚点 4 条

### A1 写完文件 → 检查"## 验证来源"节（H1 自动）
涉及外部信息的文件，末尾必须含 `## 验证来源` 小节，每条标 URL。H1 Hook 自动检测并警告。

### A2 笔记/规则/建议 → 检查"怎么来的"字段（CLAUDE.md R8）
每条建议必须标注出处（`<!-- 来源 -->` 或 `**出处**`）。来源缺失视为违反 CLAUDE.md R7。

### A3 收尾 → 检查待定项更新
每次收尾盘点时，`产出/` 里相关编号必须更新状态。

### A4 每轮对话前 → 检查"是否需要回查初衷"
不确定"为什么要这么做"时，**必须先读 `初衷/`** 再回答用户。不靠记忆、不靠脑补。
