# 学习讲义写作规范

主文档：[`../SKILL.md`](../SKILL.md)

## 总体原则

学习讲义不是教科书——它是**字典 + 路径索引**，不是内容。

## 大字典 vs 小字典的关系

本科+研一的教材体系是大字典（用户已有或可参考），讲义是给用户看"我从哪一页开始读"的**路径指引**。

| 层面 | 大字典（教材） | 小字典（本讲义） |
|:-----|:------------|:---------------|
| 范围 | 通用人才培养，覆盖完整体系 | 只覆盖导师方向相关的部分 |
| 深度 | 一字不差的体系 | 关键概念+直觉，跳过严格证明 |
| 用途 | 字典参考 | 学习路径指引 |

## 编排原则

### 起承转合五步叙事流（每章节）

每章节内容按"起承转合"组织：

- **起**：本章要解决什么问题（一个具体的研究问题）
- **承**：在前一章基础上，怎么构建这个问题的语言
- **转**：核心概念是什么，给直觉和形式化定义
- **合**：本章学完后能做什么（用一个真实例子）
- **注**：本章的关键定理/方法在导师论文中怎么用

### 章节顺序——拓扑序

按依赖关系的拓扑序排列。如果 A 依赖 B，A 必须在 B 后面。

允许章节并行（如"线性代数"和"概率论"可同时学），但都在共同的先修之后。

## 资源索引约束

### 必须验证

- 所有视频链接播放过
- 所有教材 ISBN 验证过
- 所有综述论文真的写在该方向

### 不确定不写

找不到合适资源宁可留空，不强行填。

## 长度约束

- 每章节 ≤ 800 字（不含代码/公式）
- 整本讲义 ≤ 10000 字
- 表格优先于段落（信息密度更高）

## 排版约束

- h2：起点声明 / 领域全景 / 章节目录 / 学习路径正文 / 资源索引
- h3：每个章节
- h4：每个章节内的"起承转合"小段
- 不用 h5+

代码块限三类：公式、引用论文标题、伪代码示例。

## 写作风格

- 第二人称（"你"）——和读者对话
- 不写"读者需要"、"我们应该"——直接"你从 X 学到 Y"
- 不客套不总结
- 不出现的常见 AI 句式（详略）
