# paper-locate（阶段 3 — 论文定位）

## 本 skill 是什么

输入：`02_领域脉络.md`（领域发展树）+ `01_基础画像.md` 的论文列表
输出：`项目/导师/<姓名>/03_论文定位.md`

任务：用领域框架当坐标系，把导师的每一篇论文标注到领域树的某个节点上。

## 工作流概览

```
输入：领域树 + 论文列表
              │
              ▼
┌─────────────────────────────────────────────────┐
│ Step 1: 论文聚类                                  │
│  按主题/技术/合作者把导师论文分组                  │
└─────────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────┐
│ Step 2: 在领域树上标注                             │
│  每组论文对应领域树的哪个分支                      │
└─────────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────┐
│ Step 3: 时间线叙事                                 │
│  按年份组织：入职/早期/转型/成熟/当前前沿           │
└─────────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────┐
│ Step 4: 贡献归属                                   │
│  导师在哪几篇是通讯作者？在哪几篇是关键贡献者？    │
└─────────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────┐
│ Step 5: 输出 03_论文定位.md                       │
└─────────────────────────────────────────────────┘
```

## 输出文件结构

参见 `assets/03_论文定位.md` 模板。

要点节：
- **论文时间线**（年份+论文组+主题+领域树节点）
- **导师角色**（通讯 vs 合作 vs 学生时代贡献）
- **转型节点分析**（什么时候从 A 方向转到 B 方向）
- **当前前沿位置**（导师现在做领域树的哪一分支）

## 前置依赖

- `02_领域脉络.md` 必须存在
- `01_基础画像.md` 中的论文列表必须完整

## 调用脚本

| 脚本 | 用途 |
|:-----|:-----|
| `scripts/cluster_papers.py` | 按主题聚类导师论文 |
| `scripts/locate_on_tree.py` | 把每组论文映射到领域树节点 |
| `scripts/build_paper_locate.py` | 合成 03_论文定位.md |
