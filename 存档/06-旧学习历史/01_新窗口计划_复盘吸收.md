## 3. 执行流程（AI 必须按这个走）

### 第 1 步：列输入清单
AI 第一轮响应**只做一件事**——列出"我将读取的文件清单"让你确认。**不要立刻开始写**。

### 第 2 步：逐节写（关键）
按 7 节顺序，**每节一个 Write 动作**——写完一节停一下，输出"§X 已写入，请审核"。

**理由**：避免一次性写 15 页你审核不动。

### 第 3 步：每节内容要求

| 要求 | 说明 |
|:----|:----|
| **来源标注** | 每节末尾或关键论断后标 `<!-- 来源：[文件名] [章节/决策] -->` |
| **不复述老文档** | 吸收 + 提炼 + 重写，不是复制粘贴 |
| **保留冲突** | 不同文档有冲突时**两方都列**，标"未解决"，不强行调和 |
| **不发明** | 没有依据的内容**不写**——宁可少写，不假装有结论 |

### 第 4 步：质量门（写完前自查）
- [ ] 总长度 ≤ 15 页？
- [ ] 每节都有来源标注？
- [ ] 没有"可能"、"也许"、"大概"等模糊词？
- [ ] 待定项**明确标"未解决"**，不假装解决？
- [ ] 老文档的核心信息**没丢失**（用户回看老文档能找到对应章节）？

### 第 5 步：完成
7 节全部写完后，AI 输出"核心文档完成"——你回当前窗口告诉原窗口的 AI"复盘文档写好了"。
