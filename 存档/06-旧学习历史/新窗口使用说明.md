## 四、预期会"看起来不工作"的地方

**先讲清楚，避免误判为 bug**：

1. **H0 reminder 没出现**：可能是 SessionStart 文档建议"静态上下文用 CLAUDE.md"。我们已经把"先读初衷"写进 CLAUDE.md 第 0 节——即使 H0 没生效，AI 仍应在第一轮响应前意识到这件事。如果 H0 没生效但 CLAUDE.md 文字生效，工具仍能用。

2. **H3 不阻断**：H3 是警告不阻断——故意设计的。CLAUDE.md R3（人决策）要求最终是否回退由用户决定。

3. **PowerShell 执行策略**：Windows 默认 Restricted 不允许运行 `.ps1`。如果脚本拒绝执行：
   ```powershell
   Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
   ```
   这一步在 settings.json 调用前可能需要。

4. **`${CLAUDE_PROJECT_DIR}` 不存在**：如果环境变量没传，H2 的绝对路径检查会跳过——只检查 `..` 上跳。这是**已知降级行为**，不是 bug。

5. **Hook 报错但流程继续**：exit code 1 = 非阻塞错误，工具调用继续。这是 Claude Code 官方定义。
