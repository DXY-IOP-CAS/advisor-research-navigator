# 复盘吸收任务卡

## 用法

1. VS Code 新建窗口 → 打开 `pilot-test/` 文件夹
2. 启动 Claude Code，第一轮说：
   > "按 `02_复盘吸收任务卡.md` 执行复盘吸收。先读完这个文件再开始。"
3. AI 读完本文件后，按 §1 读文件、按 §2 写文件、按 §3 逐节产出

**工作目录**：新窗口中 Claude Code 的工作目录是 `pilot-test/`。以下所有路径都是相对于 `pilot-test/` 的。

## §2 输出

**一个文件**：`产出/PILOT_种子设计_核心文档_v1.md`

**格式**：Markdown，≤ 15 页（5000-8000 字），中等等长。不要写 30 页。

**7 节结构**：

| # | 标题 | 核心来源 | 字数 |
|:-:|:-----|:--------|:----:|
| 1 | 为什么做这个种子 | 初衷/01 + D08 决策 | ~1 页 |
| 2 | 种子是什么 / 不是什么 | 初衷/02 + D08 决策 7/11 | ~1 页 |
| 3 | 已落地的核心决策（现状） | 综合设计 v0.1 + D08 收尾 | ~3 页 |
| 4 | 当前真实痛点 | 反馈日志 + F1/F2/F3 | ~2 页 |
| 5 | 开放待定问题 | 待定项总览 P01-P12 | ~2 页 |
| 6 | pilot-test/ 当前现状 | 实际文件树 + Hook 状况 | ~3 页 |
| 7 | 后续可能方向 | 综合设计 + 中间结论 | ~3 页 |

## §3 执行规则

### 3.1 执行流程
1. **先列输入清单**——第一轮响应只做一件事：列出"我将读哪些文件"让用户确认。**不立刻开始写。**
2. **逐节写**——按 7 节顺序，每写一节用一个 Write 动作。写完一节输出"§X 已写入，请审核"，等用户确认再写下一节。
3. **每节内容要求**：
   - 来源标注：关键论断后标 `<!-- 来源：[文件名] [章节] -->`
   - 不复述老文档：吸收 + 提炼，不是复制粘贴
   - 保留冲突：不同文档冲突时两方都列，标"未解决"
   - 不发明：没有依据的**不写**

### 3.2 禁止事项
- 不在 §3/§4/§6 添加新内容（只吸收不创造）
- 不在 §5 假装解决问题
- 不在 §7 做承诺
- 不试图塞进所有信息（15 页硬上限）
- 不提问超过 1 个澄清问题（用户已明确目标）

### 3.3 质量自检（写完每节自查）
- [ ] 总长度 ≤ 15 页？
- [ ] 本节有来源标注？
- [ ] 没有"可能""也许""大概"等模糊词？
- [ ] 待定项明确标"未解决"？
- [ ] 老文档的核心信息没丢失？

### 3.4 收尾
7 节全部写完并通过审核后，最后更新 `待定项总览.md`：将本复盘吸收任务标记为完成。

输出"**复盘吸收完成，核心文档已就位**。恭喜，老文档们光荣退休。"

## §4 新窗口 AI 的额外注意事项

你在新窗口运行，**没有根项目 CLAUDE.md 的全量上下文**。因此：

- **搜索验证**：涉及论文/公式/数据/外部引用时，必须搜索验证再写。工具选 Exa（学术搜索）优先。
- **路径安全**：不要修改 `pilot-test/` 子树以外的任何文件。Hook H2 会阻断越界写入。
- **用户身份**：物理专业本科，正在学习 project-pilot 框架设计。当前学习阶段基本完成，转向实践。
- **沟通风格**：中文 + 英文术语。开口即要点，不写引言/过渡/客套。不用 `|` 表格外的 Markdown 装饰。
