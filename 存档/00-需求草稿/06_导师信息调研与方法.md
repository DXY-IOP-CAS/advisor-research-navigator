# 导师信息调研方法与数据源全景

**生成日期**：2026-06-30
**用途**：Block 2 工具设计中第1段"导师基础信息调研"的方法论支撑
**调研方法**：Exa 语义搜索 + Google 交叉验证，覆盖国内外学者画像工具、学术平台、中文信息源

---

## 1. 核心结论

目前市面上对导师信息的采集，没有超出以下数据源类型的组合。每一类能拿到的信息不同，适合公开渠道获取。

---

## 2. 数据源全景

### 2.1 学术数据库（核心层）

这是导师画像的**骨架**——论文列表、引用指标、合作者网络。

| 数据源 | 协议 | 额度 | 覆盖 | 能拿到的信息 | 局限 |
|:-------|:----|:----:|:-----|:-----------|:-----|
| **OpenAlex** | REST 无 key | ∞ | 2.5亿+论文 | 论文标题/DOI/引用数/期刊/合作者/机构/概念标签 | 中国物理学者覆盖率未知 |
| **Semantic Scholar** | REST 无 key | 100 req/s | 2亿+论文 | 论文+TLDR+影响因子+引用图+领域分类 | CS/AI 偏重，物理略弱 |
| **arXiv API** | REST | ∞ | 预印本 | 最新论文（比正式发表早6-12月） | 只有预印本，非正式发表 |
| **CrossRef** | REST | 5 req/s | 1.5亿+ DOI | 论文元数据+引用关系 | 无摘要，纯元数据 |
| **ORCID** | REST 无 key | ∞ | 研究者profile | 教育/工作经历、论文ID列表、funding | 中国学者使用率低 |
| **Web of Science** | 付费 | — | — | 权威引用数据 | ❌ 不开放 |
| **Scopus** | 付费 | — | — | 权威学者profile | ❌ 不开放 |
| **CNKI 知网** | 浏览器自动化 | — | 中文学术 | 中文论文/学位论文/基金信息 | 需机构访问权限 |

**关键发现**：OpenAlex + Semantic Scholar 组合可覆盖 90% 以上公开论文元数据需求。两者都免费、无 key、够快。中国物理学者覆盖率需要实测验证。

### 2.2 中文原生信息源（差异化层）

这是导师画像的**血肉**——中文背景、新闻、采访、学生视角。supervisor 等工具做不到的部分。

| 数据源 | 访问方式 | 内容类型 | 可获取性 | 局限 |
|:-------|:--------|:--------|:--------:|:-----|
| **机构主页** | HTTP 直连 | 简介、研究方向、招生 | ✅ 公开 | 信息更新不及时 |
| **百度百科** | Web 搜索 | 人物词条 | ✅ 公开 | 覆盖率低，含AI学者信息较少 |
| **知乎** | 搜索 | 导师评价/经验分享 | ✅ 公开搜索 | 零散，非结构化 |
| **Google 新闻/中文新闻** | 搜索 | 采访/报道/获奖 | ✅ 公开 | 需要定向搜索 |
| **小红书** | 搜索 | 学生经验分享 | ✅ 公开搜索 | 信息零散 |
| **科学网博客** | 搜索 | 学者自写博客 | ✅ 公开 | 活跃用户少 |
| **微信公众号** | ⚠️ 复杂 | 课题组/机构动态 | ⚠️ 难爬取 | 详见第4节 |
| **微博** | 搜索 | 学术社交动态 | ✅ 公开搜索 | 学者活跃度低 |
| **研究生贴吧** | 搜索 | 选导师讨论 | ✅ 公开 | 零散 |

### 2.3 学术社交网络（辅助层）

| 平台 | 信息类型 | 可获取性 |
|:-----|:--------|:--------:|
| ResearchGate | 论文+问答+合作网络 | ✅ 公开（部分功能需登录） |
| Google Scholar | 论文+引用+h-index | ✅ 公开 |
| Academia.edu | 论文+profile | ✅ 公开 |
| Loop（Frontiers） | 学者profile | ✅ 公开 |
| 学者网（中国） | 中文学者profile | ✅ 公开 |
| 百度学术 | 中文学者profile | ✅ 公开 |

### 2.4 导师评价类平台（参考层）

| 平台 | 内容 | 说明 |
|:-----|:-----|:-----|
| 导师评价网（daoshipingjia.net） | 学生匿名评分/评价 | 有争议，不主动采集，仅供参考 |
| GradChoice（研选） | 六维度评分 | 开源+匿名，不主动采集 |
| RateYourSupervisor | 评价数据 | 同上 |
| 历史存档（GitHub） | 原导师评价网备份 | 2014-2020年数据，不更新 |

**我们的立场**：不做口碑评价，不在工具中主动采集这些平台的数据。但如果用户自己找到了这些信息，可以补充到第1段画像中作为参考——标注来源和可信度。

### 2.5 非学术信息（人物全貌）

用户提到了"导师除了学术外还有爱好等其他行为"——这类信息对判断"与什么样的人共事"有帮助，但获取渠道有限：

| 信息类型 | 渠道 | 可获取性 | 可靠性 |
|:---------|:-----|:--------:|:------:|
| 采访/报道 | 新闻网站、学校官网 | ✅ 高 | ✅ 高 |
| 讲座/演讲记录 | YouTube/B站/学校视频站 | ✅ 中高 | ✅ 高 |
| 个人博客/自媒体 | 独立博客、知乎专栏 | ✅ 中 | ✅ 中高 |
| 学术八卦/逸事 | 知乎、小红书 | ✅ 中 | ⚠️ 低 |
| 社交媒体（微博/推特） | 搜索 | ✅ 中 | ✅ 高（本人发言） |
| 性格/管理风格 | 导师评价网、知乎 | ⚠️ 零散 | ⚠️ 主观 |
| 实验室氛围 | 知乎、小红书、贴吧 | ⚠️ 零散 | ⚠️ 主观 |

**实用建议**：非学术信息通过 Web 搜索关键词覆盖（如 `张鹏举 采访`、`张鹏举 讲座`），不针对特定平台写爬虫。每次搜索本身也是公开的，不会触发反爬。

## 3. 同类工具对比

调研覆盖 18+ 工具，其中与第1段"导师画像"直接相关的主要有以下：

### 3.1 supervisor（GitHub: ybq22/supervisor）

**采集策略**：4 种 Google 搜索（一般+Wikipedia+Google Scholar+个人主页）→ Puppeteer 浏览器搜 → 去重 → 深度论文分析

**信息维度**：
- 纸质信息：论文列表、机构页面、Wikipedia
- 研究风格：理论/应用偏好、方法论特点
- 公开信息：性格、沟通风格、学术理念（从新闻/访谈/社交媒体分析）
- 论文深度：近3年+Top 10引用论文 → 研究演变轨迹

**输出**：结构化 JSON（含profile+research+style）+ Claude Code Skill

**可借鉴**：4 种搜索的组合策略、多源交叉比对、信息质量评分

**局限**：Puppeteer 浏览器搜在我们环境不可用（无浏览器）；分析侧侧重"性格"类信息，我们不需要

### 3.2 ScholarGraph（中国人民大学 WAMDM 实验室）

**采集策略**：20+ 学术领域的半结构化/非结构化数据 → 数据集成（ScholarSpace）→ 知识图谱

**数据源**：个人主页、研究机构网页、会议网页、期刊网页

**覆盖**：900万+ 实体、1000万+ 三元组

**可借鉴**：数据源分类方法（非结构化→结构化→清洗→提取→存储）

### 3.3 scholar-wendao-skill（GitHub: tizzy916）

**采集策略**：四层自动化（Tier 1: OpenAlex/Crossref/Semantic Scholar → Tier 2: OA出版平台 → Tier 3: 浏览器辅助 → Tier 4: 手动导入）

**核心设计**：多源采集 → PDF证据提取 → 概念地图蒸馏

**可借鉴**：四层采集策略的分级设计——先全自动 API，再逐级 fallback 到手动

### 3.4 WeAreGeng（GitHub: 425776024）

**采集策略**：OpenAlex + Semantic Scholar 双源去重合并 + 联网搜索补充

**特色**：内置 1 万+ 中国学者数据（院士/杰青/优青）

**可借鉴**：双源合并去重策略、中国学者名单的公开数据获取方式

### 3.5 ai-talent-graph（GitHub: rrrrrredy/ai-talent-graph）

**采集策略**：OpenAlex + Semantic Scholar + ORCID + arXiv 四 API 并行采集

**可借鉴**：4 API 并行调用的流水线实现

### 3.6 SouWen（GitHub: BlueSkyXN/SouWen）

**特色**：91 个异构数据源统一接口（论文17源/专利8源/网页29源/社交5源/中文技术9源）

**可借鉴**：数据源注册表架构思路——但不适合我们直接用（太重了）

## 4. 关于微信公众号调研

用户问到微信公众号可否调研。以下是技术现状：

**搜狗微信搜索**（weixin.sogou.com）是唯一的公开入口。但：
- 反爬极强——连续请求会弹验证码，IP 被封禁
- 只能搜到最近的文章，历史文章受限
- 文章链接是临时链接（几小时失效）
- 不能精确搜索指定公众号

**现有的方案**（全部不推荐给非CS用户）：
1. **WechatSogou 库** — 基于搜狗搜索的爬虫，需要处理验证码/代理/IP池
2. **mitmproxy 中间人攻击** — 拦截手机微信流量，需要物理设备或模拟器
3. **微信公众号后台接口** — 需要有公众号权限 + Selenium 模拟登录
4. **wechat-article-exporter** — PC 微信客户端 hook，需要运行二进制工具

**结论**：微信公众号内容作为导师调研数据源，技术门槛高、稳定性差、法律边界模糊。不适合作为工具的标准数据源。

**建议策略**：不直接爬取。但如果用户**自己**知道目标导师的公众号名称，可以在 SKILL.md 中给出一条提示："如果你知道该导师的公众号名称，请告知，我将尝试通过公开渠道查找相关文章"。实际执行依赖搜索工具搜索公众号名称，不写爬虫。

## 5. 行业组织方式调研

你之前问的"学校→学院→部门→老师"结构是否通用。调研结果如下：

| 机构类型 | 分类方式 | 示例 |
|:---------|:--------|:-----|
| **中科院物理所** | 实验室/中心 | 超快物质科学中心、超导国家重点实验室、光物理重点实验室 |
| **综合性大学（北大/清华）** | 学院→系 | 物理学院→凝聚态物理系、光学系 |
| **理工科大学（中科大）** | 学院→方向 | 物理学院→量子信息方向 |
| **研究机构（其他所）** | 实验室/部门 | 各大研究所各不相同的命名体系 |

**结论**：没有统一结构。你选的"以学校为最外层，之内按该机构自己的分类方式"是对的——不对所有机构强加统一模板，而是在 `按机构.md` 索引中每个机构单独写它的分类方式。`build_index.py` 不假定结构统一，只按 frontmatter 的 `affiliation` + `department` 字段分组。

## 6. 推荐第1段调研流程（基于调研结论）

综合以上调研，建议的第1段执行流程：

```
用户输入：导师姓名 + 学校 + 官网 URL（可选）
  │
  ├─ Step 1: 解析机构主页
  │   → 读取官网 URL → 提取姓名、职称、部门、研究方向、论文列表
  │
  ├─ Step 2: 学术数据库采集
  │   → OpenAlex：按姓名+机构搜作者ID → 获取论文列表
  │   → Semantic Scholar：补充引用数、TLDR
  │   → arXiv：补充最新预印本
  │   → ORCID：补充工作经历（如果存在）
  │
  ├─ Step 3: 中文网络搜索
  │   → 百度百科词条（如果有）
  │   → 新闻/采访/报道
  │   → 知乎相关讨论
  │   → 学校/B站上的讲座视频
  │
  ├─ Step 4: 交叉比对与去重
  │   → 官网列表 vs OpenAlex 列表 → 合并去重
  │   → 中文源 vs 英文源 → 字段合并
  │   → 每条信息带来源 URL
  │
  └─ 输出：01_基础画像.md
      → frontmatter（YAML 元数据）
      → 基础信息（姓名/职称/机构/部门）
      → 论文列表（带 DOI/引用数/年份/来源标注）
      → 研究领域标签
      → 公开信息（新闻/采访/讲座）
      → 多源比对备注
```

流程中的所有步骤都可以由 AI 在 Claude Code 中完成，不需要用户写代码或部署服务。
