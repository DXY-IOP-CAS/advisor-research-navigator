## Block 2: 工具设计

### 2.1 整体工作流与技术概览

#### 工作流程

用户提供导师信息，AI 驱动的流水线自动执行：

```
第0步 -- 用户输入
  提供：导师姓名 + 学校/机构 + 机构官网导师URL
  可选补充：方向关键词、已知论文列表

第1步 -- 身份验证（AI自动执行）
  浅调研 -> 输出 00_身份验证卡.md
  用户读卡 -> 确认"是这个人"或"不是"
  确认后进入下一步，否则调整输入重试

第2步 -- 基础画像（AI自动执行）
  深调研 -> 输出 01_基础画像.md
  用户可选修改 -> 进入下一步

第3步 -- 领域脉络（AI自动执行）
  分析 -> 输出 02_领域脉络.md
  用户可选修改 -> 进入下一步

第4步 -- 论文定位（AI自动执行）
  分析 -> 输出 03_论文定位.md
  用户可选修改 -> 进入下一步

第5步 -- 学习讲义（AI自动执行）
  设计 -> 输出 04_学习讲义.md
  用户终审
```

**核心规则**：用户可在任意步骤后介入修正。每步完成后 AI 自检（来源可验证、信息一致），自检不通过则不进入下一步。

#### 技术概览

| 层级 | 选型 | 说明 |
|:-----|:-----|:------|
| 执行引擎 | Claude Code Skill（SKILL.md） | 流水线指令写在 SKILL.md 中，AI 按步骤执行 |
| 数据采集 | 免费学术 API（OpenAlex/Semantic Scholar/ORCID/arXiv） | 零费用，无需 API Key |
| 页面抓取 | MCP HTTP fetch / Tavily Extract | 读机构官网、中文百科、新闻页面 |
| 身份消歧 | ORCID API 三级搜索（名称+机构+email） | 配合官网论文列表交叉比对 |
| 信息搜索 | Web Search / Exa / Serper | 中文网络信息（新闻、采访、知乎） |
| 输出格式 | Markdown + YAML frontmatter | 人可读、机器可索引 |
| 辅助脚本 | Python（可选） | fetch_papers.py / build_index.py |

不引入：Web 服务器、数据库、Docker。

五份文件从验证到讲义逐层递进：

```
                     +----------------------+
    用户输入 ------+ | 00_身份验证卡.md       | <- Phase 1a
                     |（确认身份后再继续）     |
                     +----------+-----------+
                                | 用户确认
                     +----------v-----------+
                     | 01_基础画像.md        | <- Phase 1b
                     |（完整学者 profile）    |
                     +----------+-----------+
                                |
                     +----------v-----------+
                     | 02_领域脉络.md        | <- Phase 2
                     |（领域发展树+里程碑）    |
                     +----------+-----------+
                                |
                     +----------v-----------+
                     | 03_论文定位.md        | <- Phase 3
                     |（论文时间线+位置标注）  |
                     +----------+-----------+
                                |
                     +----------v-----------+
                     | 04_学习讲义.md        | <- Phase 4
                     |（从零到前沿学习路线）   |
                     +----------------------+
```

前一段输出作为后一段的认知上下文。信息量逐段收窄、理解深度逐段加深。

#### 设计原则

1. **所有信息必须标注来源 URL**。不标注则不写入产出。
2. **缺失比编造好**。API 返回为空 -> 标注"未找到"，不生成 AI 推断。
3. **交叉比对**。同一条信息至少 2 个来源才写入"已验证"；单一来源标注"仅单源"。
4. **多源冲突**。列出多个来源的不同信息，不加 AI 仲裁定论。
5. **用户确认前置**。身份消歧不允许全自动 -- 必须输出验证卡等人确认。
6. **每段自检**。每段产出后 AI 检查：所有外部引用是否有 URL、是否有事实性矛盾。

### 2.2 第1段：身份验证与基础画像

**工作流程**

第1段是整个流水线的入口。用户给了"张鹏举 + 中科院物理所 + 官网 URL"后，AI 需要做两件事：

1. **确认这是谁** -- 官网 URL 指向的页面是可靠的"身份锚点"。从页面提取的邮箱、论文标题、研究方向可用来在学术数据库中锁定唯一 ID。这一步不可全自动，必须让用户确认。

2. **画出完整画像** -- 确认身份后，用唯一的 Author ID 从各平台拉取所有公开数据，组织成结构化画像。

两段之间插了一个"人闸"：AI 先做浅调研输出验证卡，用户确认后才做深调研。

#### 技术原理与执行

**先讲清楚一个概念："爬虫"在我们工具里是什么意思。**

我们不做传统意义上的爬虫（写 Python 脚本遍历网页）。我们让 AI 做三件事：
- 调用免费 API（OpenAlex、ORCID、Semantic Scholar）拿结构化数据
- 调用 MCP fetch 工具抓指定网页内容
- 用 Web Search 搜中文信息

这三件事 AI 都能直接做，不需要你装任何爬虫框架。

**身份消歧的核心原理**：

学术界通用的消歧方法（OpenAlex/Semantic Scholar/AMiner）基于 ML 聚类 -- 把同名作者按合作者网络、机构归属、研究主题一致性等特征归类。但我们有独特优势：用户提供了机构官网 URL，指向的老师是确定的，不需要靠聚类猜。

所以我们用的消歧方法更简单：**用官网 URL 提取的论文标题做 ground truth，跟 API 返回的候选人的论文列表做直接比对**。匹配度高的就是本人。不需要 ML 模型。

**Phase 1a：身份验证（6 步）**

```
Step 1 -- 抓机构官网页面
  做什么：用 MCP fetch 工具读取用户提供的 URL
  URL 例子：https://edu.iphy.ac.cn/detail_teacher.php?id=6368
  从页面文字中提取：
    email   -> pengju.zhang@iphy.ac.cn        -> 给 Step2 用
    论文标题 -> Time-resolved multi-electron... -> 给 Step3/5 用
    姓名    -> 张鹏举 (Pengju Zhang)
    职称    -> 特聘研究员
  异常处理：页面返回 404 -> 让用户确认 URL

Step 2 -- 搜索 ORCID
  做什么：用 email 查 ORCID 注册库
  请求（AI自动执行）：
    GET https://pub.orcid.org/v3.0/expanded-search?q=email:pengju.zhang@iphy.ac.cn
  返回格式（JSON）：
    {
      "expanded-result": [
        {"orcid-id": "0000-0002-...", "given-name": "Pengju", "family-name": "Zhang",
         "institution-name": ["Institute of Physics, CAS"]}
      ]
    }
  结果判断：
    返回 1 人且机构匹配 -> 拿 ORCID ID，这是最可靠的标识
    返回 0 人 -> 张鹏举没注册 ORCID，跳过
    返回多人但机构不匹配 -> 标注"ORCID 存在但机构不对"

Step 3 -- 搜索 OpenAlex
  做什么：用"姓名+机构"查 OpenAlex 作者库
  请求：
    GET https://api.openalex.org/authors?search=pengju+zhang
    或者用中文：search=张鹏举（注意需要 URL encode）
    同时用 filter 限定机构：
    filter=last_known_institutions.id:https://ror.org/05qbk4x57（中科院物理所 ROR）
  返回格式（JSON）：
    {
      "results": [
        {
          "id": "https://openalex.org/A5031655767",
          "display_name": "Pengju Zhang",
          "works_count": 65,
          "cited_by_count": 1565,
          "last_known_institutions": [
            {"display_name": "ETH Zurich"},
            {"display_name": "Chinese Academy of Sciences"}
          ]
        },
        ... 其他同名候选人
      ]
    }
  结果判断：
    返回多人 -> 保留所有候选人，交给 Step5 做论文比对
    返回 0 人 -> 尝试去掉中文名，换成拼音重试
    注意：我们在测试中发现张鹏举返回 25 个结果，消歧不干净
    所以必须做 Step5 的论文标题比对来确定

Step 4 -- 搜索 Semantic Scholar（辅助）
  做什么：从另一个源交叉验证
  请求：
    GET https://api.semanticscholar.org/graph/v1/author/search?query=Pengju+Zhang&fields=name,affiliations,paperCount
  返回：候选列表，每人有 paperCount
  作用：辅助验证，不单独做决定

Step 5 -- 论文标题交叉比对（核心消歧手段）
  做什么：把从官网抓到的论文标题，与每个候选人的论文列表对比
  具体操作：
    AI 用 OpenAlex 的 /authors/{id}/works 接口拉取每位候选人的论文标题
    然后计算：官网论文标题在候选人论文列表中出现的比例
  判定标准：
    >=80% 命中 -> 锁定，高置信度
    50%-80% -> 最可能候选人，标注置信度
    <50% -> 多个候选人各命中部分论文 -> 让用户手动选

Step 6 -- 输出 00_身份验证卡.md
  让用户看到：我们找到了什么ID、要爬哪些源、要拿什么数据
  用户确认后 Phase 1b 才开始
```

**Phase 1b：深调研（用户确认后执行）**

```
身份已确认，Author ID 已锁定（例：OpenAlex ID = A5031655767）
现在拉取这个 ID 下的所有公开数据。

(1) 拉论文列表 + 引用数据
  OpenAlex /authors/A5031655767/works?per_page=200&sort=cited_by_count:desc
  返回 200 条论文记录，每条包含：title, doi, publication_year, cited_by_count, authors
  -> 这部分是 "爬虫" 的主体——不是写爬虫，是调一次 REST API
  -> AI 自动解析 JSON 里的论文列表

(2) 拉作者信息 + 引用总数
  OpenAlex /authors/A5031655767
  返回：works_count, cited_by_count, counts_by_year, last_known_institutions
  -> AI 填入画像的 "学术影响力指标" 节

(3) 拉 TLDR 摘要（可选）
  Semantic Scholar /author/2400405493/papers?fields=title,tldr
  返回每篇论文的 1 句话 TLDR 摘要
  -> AI 填入画像的 "代表性论文" 节

(4) 拉最新预印本
  arXiv API search?query=au:Pengju+Zhang&sortBy=submittedDate&sortOrder=descending
  返回 arXiv 上该作者的预印本列表
  -> 补充 OpenAlex 可能遗漏的最新论文

(5) 搜中文网络信息
  Web Search "张鹏举 中科院物理所 采访/新闻/讲座"
  返回：学术报告通知、新闻稿、个人博客
  -> AI 填入画像的 "公开多维信息" 节

AI 把以上数据整理合并 -> 输出 01_基础画像.md
```

**"爬虫"真正做了什么？**

```
整个 Phase 1b 的爬虫 = 5 次 API 调用 + 1 次 Web Search

每次调用格式完全固定：
  API URL + 参数 -> 返回 JSON -> AI 解析 -> 填入对应章节

不需要你写任何代码。AI 在 SKILL.md 的指令下自动执行。
如果某个 API 返回空或报错，AI 跳过该项并标注"未获取到"。
```

**从实际测试跑张鹏举所得的重要经验**：

1. **OpenAlex 对中文名消歧不够可靠**。同名跨多机构、无清晰主条目的情况常见。必须用官网论文交叉验证。
2. **邮箱是 ORCID 搜索的黄金锚点**。ORCID expanded-search 支持按 email 搜索，有邮箱则命中率极高。
3. **Semantic Scholar 在物理领域的论文覆盖率不如 OpenAlex**。建议 OpenAlex 为主、S2 为辅。
4. **arXiv API 对中文名搜索差**。arXiv 上中文作者名通常是拼音，直接匹配困难。

### 2.6 跨段公共设计

#### 交付形态

工具不是网站/App，是一套 AI 可读的指令文件 + 极简脚本。

用户使用流程：

```
git clone 仓库
安装 Skill
/research-advisor "张鹏举" "中科院物理所" "https://..."
-> Agent 先输出 00_身份验证卡.md -> 用户确认 ->
-> Agent 跑完整流水线 -> 项目/张鹏举/ 下生成五份 .md
```

仓库结构：

```
仓库根/
|-- CLAUDE.md
|-- .claude/skills/research-advisor/
|   |-- SKILL.md                     # 流水线指令（核心）
|   +-- 模板/
|       |-- 00_身份验证卡.md
|       |-- 01_基础画像.md
|       |-- 02_领域脉络.md
|       |-- 03_论文定位.md
|       +-- 04_学习讲义.md
|-- 脚本/
|   |-- fetch_papers.py
|   +-- build_index.py
+-- 项目/
    |-- 导师/
    |   +-- 张鹏举/
    |       |-- 00_身份验证卡.md
    |       |-- 01_基础画像.md
    |       |-- 02_领域脉络.md
    |       |-- 03_论文定位.md
    |       +-- 04_学习讲义.md
    +-- 索引/
        |-- 按机构.md
        +-- 按方向.md
```

零服务器、零数据库、零部署。SKILL.md 是核心。

**数据源**

| 源 | 协议 | 免费额度 | 用途 |
|:---|:----|:--------:|:-----|
| OpenAlex | REST API（无 key） | 无限 | 论文/引用/机构/作者消歧 |
| Semantic Scholar | REST API（无 key） | 100 req/s | 论文影响力/领域分类 |
| arXiv API | REST 搜索 | 无限 | 最新预印本 |
| ORCID | REST API（无 key） | 无限 | 职业历史交叉验证 |
| 机构主页 | HTTP 抓取 | 无限 | 中文/机构特有信息 |
| 中文百科/新闻 | Web 搜索 | 取决于工具 | 背景/采访/报道 |

**数据组织**

每个老师一个独立目录（项目/导师/<姓名>/），全身只在这里存一份。01_基础画像.md 顶部用 YAML frontmatter 标注分类元数据：

```yaml
---
affiliation: 中科院物理所
department: 超快物质科学中心
tags: [超快光学, 强场物理, 阿秒科学]
---
```

build_index.py 扫描项目/导师/下所有 frontmatter，自动重建两个索引：

- 按机构.md：学校->学院->部门->老师（树形）
- 按方向.md：研究方向->老师（跨机构散点网络）

新增一个老师 -> 脚本重跑 -> 索引自动更新。

**贡献与迭代模式**

| 层 | 门槛 | 内容 |
|:---|:---:|:-----|
| 数据修正 | 极低 | PR 改 .md 文件里的论文标题/DOI/URL |
| 新增导师 | 低 | 跑一次流水线，核实产出，PR 合入 |
| 方法论改进 | 中 | 优化 SKILL.md / 模板结构 |

迭代循环：用户跑实例 -> 发现问题 -> 修正 -> PR 合入 -> 下一个人拿到修正版。PR 天然零冲突（每人独立目录）。
