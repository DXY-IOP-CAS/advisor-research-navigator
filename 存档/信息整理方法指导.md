## 1. 为什么需要专门的方法论

信息整理不是"收集完再读一遍"就能完成的工作。没有结构化方法，常见问题包括：

- 收集了大量资料，不知道从哪条开始读
- 读完后笔记散落各处，无法串联
- 分析停留在表面描述，无法提炼深层结构
- 需要用到时找不到之前读过什么

方法论提供一个系统框架，让信息从"收集"到"分析"到"输出"有章可循。本章介绍的方法，按"先传统后AI"排列——传统方法提供分析原理（为什么这么做），AI工具提供执行效率（如何更快做到）。

### 2.1 框架分析法 Framework Analysis

**一句话说明**：用预设的分析框架对定性数据进行结构化整理，适合有明确研究问题的应用型研究。

**来源**：Ritchie & Spencer, 1994，"Qualitative data analysis for applied policy research"（Sage 出版社）。

**核心五阶段**：

1. **数据熟悉**：通读原始资料（访谈转录/笔记/文献），在边缘做初步批注，记录整体印象
2. **框架识别**：基于研究问题和熟悉过程中浮现的模式，构建主题框架——一套用于分类和索引的类别体系。框架可以来自理论（演绎）或数据本身（归纳）
3. **索引**：将框架中的类别应用到全部数据上——逐段标注属于哪个类别。不要求数据只属于一个类别
4. **图表化**：将索引后的数据按类别提取出来，填入矩阵表格——每行一个案例/文献，每列一个类别/维度。保留原始语境和引用
5. **映射和解释**：分析矩阵中的模式——比较行间差异、找列间关联、识别典型和非典型案例。提出解释性结论

**优缺点**：
- 优点：结构清晰，适合团队协作（多人可独立索引同一框架）；透明度高（分析过程可追溯）；适合政策研究和应用型评估
- 缺点：框架预设可能限制新发现；耗时长，需要手动处理大量数据；矩阵表格在数据量大时变得笨重

**适合场景**：有明确研究问题的信息调研、政策分析、多案例比较分析。项目工具的需求分析和竞品调研阶段适用。

### 2.2 Zettelkasten 卡片盒笔记法

**一句话说明**：将知识拆解为原子笔记，通过链接建立知识网络，让知识在连接中自然生长。

**来源**：Niklas Luhmann（德国社会学家），1981 年论文 "Kommunikation mit Zettelkasten"（《与卡片盒交流》）。Luhmann 用这套方法写了 70 多本书和 400 多篇学术文章。

**核心原则**：

1. **原子性**：每张笔记只包含一个想法/概念/事实。一个笔记是对一个信息的独立表达，不依赖其他笔记就能理解
2. **链接优先**：新笔记必须与已有笔记建立链接。链接不是分类标签，而是具体的语义关系——A 支持 B、A 反驳 B、A 是 B 的特例、A 是 B 的前提
3. **用自己的话写**：不做原文摘抄，用自己的语言重构理解。重构的过程就是理解的过程
4. **汇聚**：从多张链接的笔记中，汇聚出新的洞察——笔记本身不解释什么，链接模式才解释

**优缺点**：
- 优点：知识积累可复用（笔记是"活"的，不是"读完即弃"）；促进跨领域连接；长期效果显著
- 缺点：入门成本高（需要练习原子化思维）；短期看不到回报；不适合线性任务（写论文提纲时反而低效）

**适合场景**：长期知识积累（建立个人知识库）、跨学科交叉研究、学术阅读笔记。项目中用于长期积累"导师调研"的经验和模式。

### 2.3 文献分析矩阵 / 编码表

**一句话说明**：将多篇文献按统一维度编码，形成可横向比较的结构化记录。

**核心步骤**：

1. **确定分析维度**：常见维度包括——理论框架（用了什么理论）、研究方法（定性/定量/混合）、样本/数据集、关键发现、局限性、未来方向。维度根据研究问题调整
2. **设计编码表**：每篇文献一行，每列一个维度。预留"备注"列用于记录你自己的反思
3. **逐篇编码**：读一篇填一篇。编码不是摘抄标题/摘要，而是提取该文针对每个维度的实质内容
4. **比较分析**：完成编码后，对每列做纵向比较（所有文献在这个维度上的共识和分歧）

**优缺点**：
- 优点：结构标准化，结果可验证；便于多人大规模协作；编码表本身就是文献综述的输出框架
- 缺点：维度选择有主观性；编码质量依赖读者理解水平；维度固定后难以容纳意外发现

**适合场景**：系统文献综述、开题报告、研究背景调研。项目工具的第2段（领域发展调研）和第3段（老师论文串联）均适用。

### 2.4 MECE + 金字塔原理

**一句话说明**：信息分类必须"相互独立且完全穷尽"（MECE），表达时结论先行（金字塔原理）。

**来源**：Barbara Minto，1960 年代末在 McKinsey & Company 开发，后整理为《金字塔原理》一书。

**核心原则**：

**MECE（Mutually Exclusive, Collectively Exhaustive）**：
- **相互独立**：同一层级的不同类别之间不重叠——任何一条信息只属于一个类别
- **完全穷尽**：所有类别合起来覆盖了全部可能——没有信息被遗漏

**金字塔原理**：
- **结论先行**：先说核心结论/观点，再展开支撑论据
- **纵向支持**：下一层是上一层的直接支撑——问"为什么"得到下层，问"所以呢"得到上层
- **横向独立**：同一层的支撑点之间 MECE

**优缺点**：
- 优点：大幅降低信息冗余和遗漏；金字塔结构的表达效率极高
- 缺点：MECE 思维在复杂问题上难以做到（真正的"穷尽"几乎不可能）；过度强调分类可能忽略灰色地带

**适合场景**：报告写作、信息分类、问题拆解。项目工具中用于需求规格书的结构化和调研报告的输出组织。

### 2.5 PARA 方法

**一句话说明**：将信息按"可操作程度"划分为四个层级——项目、领域、资源、归档，解决"信息放哪"和"找不回来"的根本问题。

**来源**：Tiago Forte（Forte Labs），详见其《Building a Second Brain》一书和官网 fortelabs.com。

**四大类**：

| 类别 | 定义 | 示例 |
|:----|:----|:-----|
| **Projects**（项目） | 有明确截止日期和目标的任务 | "完成导师调研工具v1.0" "写系统综述论文" |
| **Areas**（领域） | 需持续维护但不设截止的责任范围 | "学术写作能力" "量子蒙特卡洛知识" |
| **Resources**（资源） | 将来可能用到的主题资料 | "AI Agent 论文合集" "方法论笔记" |
| **Archives**（归档） | 已完成/已暂停的旧内容 | 已完成的项目、已经不活跃的领域 |

**核心规则**：信息只存放在这四个顶层文件夹中。不建深层嵌套，只用搜索+链接找内容。定期整理（每周/每月）将活跃内容向 Archives 移动。

**优缺点**：
- 优点：极其简单（只有4类）；以行动为导向（项目优先于资源）；归档清理不心痛
- 缺点：Projects 和 Areas 的边界模糊（长期写一篇论文是项目还是领域？）；不适合纯知识管理场景（更偏任务管理）

**适合场景**：个人知识库组织、Obsidian/Notion 信息整理。项目工具中用于管理调研过程中积累的文件和笔记。

### 2.6 渐进式总结 Progressive Summarization

**一句话说明**：对笔记做五层递进总结，每次只加工"看起来最重要"的部分，让笔记随使用次数自然浓缩。

**来源**：Tiago Forte（Forte Labs），2017 年提出。详见其《Building a Second Brain》和博客。

**五层递进**（L0-L4）：

- **L0 原文**：收集的原始资料全文。放在笔记中，不做修改
- **L1 粗体**：读原文时，把"最有趣/最有用"的句子加粗。第一次加工，降低回读成本
- **L2 高亮**：把粗体句子中"**最最核心**"的句子再高亮。第二次过滤
- **L3 摘要**：用自己的话写一段总结——这段资料到底在说什么？为什么它重要？写完后未来不需要回读原文
- **L4 重述**：把摘要糅合进自己的知识体系中——"这件事我理解的程度是？还缺什么？它和我知道的其他东西有什么关联？" 最终变成你自己的洞察

**核心机制**：不要一次性把五层全做。每层只在"回看觉得不够"时才推进。大多数笔记停在 L1，只有真正重要的才推到 L3-L4。

**优缺点**：
- 优点：工作量和信息价值匹配（高价值笔记自动获得更多加工）；渐进决策消除选择困难（一次只做一层）
- 缺点：需要"回看"习惯支撑——从来不回看的人永远停留在 L0-L1；逐层界限主观

**适合场景**：大量阅读 + 定期回顾的研究场景。项目中用于第2-3段调研资料的整理——调研报告和论文全文进入 L0-L1，关键发现推到 L3-L4。

### 2.7 概念图 / 知识图谱法

**一句话说明**：用图结构表示知识——节点=概念，边=关系，可视化展示知识结构。

**核心原则**：

1. **节点代表概念**：每个节点是一个独立的知识点（一个术语、一个方法、一个原理）
2. **边代表关系**：边有方向和标签——A "导致" B、A "是" B 的一种、A "支持" B、A "矛盾" B
3. **分层组织**：从根概念（高抽象层级）向下展开为子概念（具体细节），形成概念层级
4. **重构而非记录**：绘制概念图的行为本身就是理解过程——画不出来说明没真懂

**常用工具**：手绘、Miro/FigJam（白板）、Obsidian Graph View、CmapTools

**优缺点**：
- 优点：直观展示知识结构（一眼看全貌）；发现知识盲区（没有连边的地方就是你不懂的地方）；促进大图景理解
- 缺点：绘制成本高；复杂领域的概念图迅速变得杂乱；边的关系类型难以标准统一

**适合场景**：学习初期构建领域认知地图、跨学科知识连接、团队知识共享。项目工具的第2段（领域发展调研）产出领域认知地图，概念图是核心输出形式。

### 2.8 批判性阅读三阶分析

**一句话说明**：阅读学术文本时，按"现象描述→争议点追问→证据评估"三阶递进，从被动阅读转向主动评估。

**核心步骤**：

1. **现象描述（读懂）**：回答——这篇论文做了什么？用了什么方法？主要发现是什么？不需要评价，只复述
2. **争议点提问（深挖）**：回答——这篇论文和已有认知有什么冲突？它假设了什么（是否合理）？它没回答什么？它隐含的前提是什么？激活批判性思维
3. **证据对比（判断）**：回答——证据本身是否支持结论？样本是否代表性？方法论是否有缺陷？其他文献怎么看这个问题？形成自己的判断

**优缺点**：
- 优点：防止被动阅读（三连问强制进入思考）；结构化阅读记录便于后续检索
- 缺点：耗时长（不适合粗筛海量文献）；需要领域基础知识才能提出好问题

**适合场景**：精读阶段的高质量文献评估。项目中用于对筛选出的核心论文做深度分析。

### 2.9 比较矩阵法

**一句话说明**：将多篇论文按指定维度列成交叉对比表，让模式和差异一目了然。

**核心步骤**：

1. **确定行和列**：行=被比较对象（文献、方法、工具等），列=比较维度
2. **选择比较维度**：常见维度——方法/算法、研究对象/样本量、理论框架、关键结论、局限性、可用资源/代码、与我们的相关性
3. **逐格填充**：每个格子是对应文献在该维度的实质内容摘要。保持一致性和可比性
4. **分析矩阵**：逐列问"大家在这一点上是否一致？"；逐行问"这篇文献最特别的特征是什么？"；整体问"哪些维度上差异最大？这些差异说明什么？"

**输出形式**：纸质矩阵、Excel/Google Sheets 表格、Notion 数据库

**优缺点**：
- 优点：高度可视化（一眼看到差异）；强制标准化（不标准化无法填入格子）；适合团队共享进展
- 缺点：维度设计决定了视野——选错了维度等于浪费时间；格子太大会变成原文粘贴，失去比较价值

**适合场景**：文献综述的初步合成阶段、竞品分析、工具评估。项目中用于第2段（领域调研）和各工具/方法的横向对比。

### 2.10 主题分析法 Thematic Analysis

**一句话说明**：从数据中识别、编码、归类、提炼出研究主题，是定性研究中最基础也最通用的分析方法。

**来源**：Braun & Clarke, 2006，"Using thematic analysis in psychology"（Qualitative Research in Psychology，第3卷第2期第77-101页）。后发展为"反思性主题分析法"（Reflexive Thematic Analysis）。

**六阶段**：

1. **熟悉数据**：反复阅读，做初步笔记。通读全部数据至少一次
2. **生成初始编码**：逐行/逐段标注"这段在说什么"——编码是对数据的最小解释单位，不是主题本身
3. **搜索主题**：将编码归类为候选主题——哪些编码可以组合成一个更大的概念？编码和主题的关系是"多对一"
4. **审查主题**：检查候选主题内部是否一致（所有编码都属于这个主题吗？）和外部是否有区分（不同主题之间边界清晰吗？）。必要时拆分、合并、丢弃
5. **定义和命名主题**：每个主题写一个定义——它的本质是什么？它回答什么问题？它和其他主题什么关系？
6. **撰写报告**：用主题+证据（引用原文）的方式呈现分析结果

**优缺点**：
- 优点：灵活通用（几乎所有定性数据都适用）；六阶程序化降低新手焦虑；不绑定理论假设（适合各种立场）
- 缺点：编码的质量依赖分析者洞察力；很容易变成"做完了六步但什么都没发现"的机械操作；主题数量缺乏客观标准

**适合场景**：开放式的信息分析方法，适合没有预设框架的探索性调研。项目中用于从用户访谈/调研材料中发现"隐形的模式"——比如导师调研中用户真实痛点的分类。

## 3. AI 时代工具（进阶）

以下工具不取代上述方法论，而是让方法论执行更快、覆盖率更广。建议配合使用：**方法论定框架，AI 工具提效率**。

---

### 3.1 NotebookLM

**一句话说明**：Google 出品的多源信息整合工具，支持上传文档/网页/视频后以问答方式提取和分析信息。

**核心能力**：
- 上传多份源材料（PDF、网站、YouTube 转录等），AI 自动索引
- 基于上传材料回答用户问题——回答有引用出处可追溯
- 自动生成简报（Briefing Doc）——按主题汇总多个来源的关键内容
- 支持从特定来源生成学习指南/FAG/时间线

**优缺点**：
- 优点：多源交叉引用防幻觉；出处可追溯（每句回答标来源）；降低大量阅读的认知负担
- 缺点：分析深度有限（不适合做批判性分析）；需要上传到云端可能有隐私顾虑

**适合场景**：快速从多篇论文/多源信息中提取一致信息点。项目中用于调研初筛阶段的快速概览。

---

### 3.2 NVivo / ATLAS.ti

**一句话说明**：专业定性数据分析软件，支持编码、主题建立、可视化和团队协作。

**核心能力**：
- 导入各类数据（文本、音频、视频、图片、PDF、社交媒体）
- 手动/自动编码：选定文本片段，分配编码标签
- 编码矩阵：交叉表显示不同案例/维度上的编码分布
- 词频统计、情感分析、聚类分析
- 查询功能：按编码/属性/关系组合搜索

**优缺点**：
- 优点：功能全面，从编码到可视化一站式解决；适合大规模定性项目；支持团队协作
- 缺点：学习曲线陡峭；许可证昂贵（NVivo 学术版约 $200+，ATLAS.ti 约 $99+/年）；过度依赖软件可能弱化分析思维

**适合场景**：大规模定性研究项目、系统综述的编码阶段。项目工具中用于多轮调研材料的统一编码管理（如果规模足够大）。

---

### 3.3 Obsidian + AI 插件

**一句话说明**：本地知识库 + 双向链接笔记 + AI 辅助（通过插件接入 LLM）。

**核心能力**：
- 本地 Markdown 存储（完全隐私可控）
- 双向链接：查看"哪些笔记引用了当前笔记"
- Graph View：可视化链接网络
- AI 插件（如 Copilot、Smart Connections）：在笔记库中搜索、总结、连接
- 配合插件实现 Zettelkasten、PARA、渐进式总结等工作流

**优缺点**：
- 优点：本地化（隐私、纯文本格式无 vendor lock-in）；双向链接天然支持 Zettelkasten；社区插件生态活跃
- 缺点：插件质量参差（依赖社区维护）；AI 能力取决于接口配置（非内置）；组织和维护成本在前

**适合场景**：个人学术知识管理、长期笔记系统。项目中作为个人知识库，存放调研过程中积累的方法论笔记和调研发现。

---

### 3.4 Scite

**一句话说明**：引文上下文分析工具——查看论文间"支持"、"反对"或"提及"关系。

**核心能力**：
- 输入一篇论文，查看哪些论文引用了它，以及引用的上下文
- 自动标注引用类型：支持（supporting）、反对（contrasting）、提及（mentioning）
- Citation Statements：每篇引用被标注了具体语境和对该文的态度
- Smart Citation 可视化：展示一篇论文被引的"支持 vs 反对"比例

**优缺点**：
- 优点：快速判断论文学术影响力（不仅仅看被引次数，还看被引态度）；发现学术争议点；节省追踪引文的时间
- 缺点：覆盖范围有限（不是所有期刊和会议都收录）；需要订阅

**适合场景**：论文深度的引文分析、判断领域内共识与争议。项目工具中用于第2段（确认领域内的关键争议点和核心共识）和第3段（验证老师论文被引情况和学术影响力）。

---

### 3.5 Elicit

**一句话说明**：AI 自动从论文中提取研究设计、样本量、方法、关键结果等结构化数据。

**核心能力**：
- 根据研究问题搜索相关论文（基于 Semantic Scholar）
- 从论文中自动提取标准化信息（目标、方法、结果、局限等列在表格中）
- 对提取结果排序——按相关性、样本量、时效性等
- 从列表中直接选中论文，Elicit 输出该领域的综合摘要

**优缺点**：
- 优点：大幅减少人工数据提取时间（尤其文献综述初期）；结构化输出直接可用
- 缺点：提取准确率不稳定（特殊领域可能经常出错）；无法提取隐性的方法论缺陷

**适合场景**：系统文献综述的数据提取阶段。项目中用于第2段（快速提取多篇论文的结构化信息）。注意：关键数据仍需人工验证。

---

### 3.6 Claude / GPT 长篇总结

**一句话说明**：利用大语言模型的长上下文窗口对多篇论文/长文档进行综合分析和模式识别。

**核心能力**：
- 长上下文窗口（Claude 支持近 200K tokens，可一次处理数百页论文）
- 模式识别：从多篇输入中识别共同主题、研究趋势、未解决问题
- 按指定格式输出：比对表、结构化摘要、研究进展时间线
- 多轮对话深入分析（不适合一次性完整的分析，适合逐步深入的人机对话）

**优缺点**：
- 优点：处理速度快、上下文容量极大、可直接交互调整分析方向
- 缺点：可能产生"看起来合理但实质错误"的幻觉；上下文限制（200K tokens 对海量文献仍不够）；不能完全替代人工判断

**适合场景**：调研中的快速初步分析、长文档的模式提取、假设生成。项目中用于第2段的长文本分析。

---

### 3.7 AI 辅助系统综述

**一句话说明**：2025-2026年新趋势——用 AI 工具辅助系统文献综述的数据提取、筛选和分析。

**主要进展**：
- AI 辅助筛选：用 LLM 初步筛选标题和摘要，人工审核 AI 排除的论文（单向审查可减少人工筛选量 30-50%）
- AI 辅助数据提取：从 PDF 中自动化提取研究特征（样本量、方法、结果）到预定义模板
- AI 辅助质量评估：用 LLM 对研究进行初步质量评分（如 Cochrane Risk of Bias）
- 代表性研究："Large Language Models for Systematic Reviews: A Comprehensive Guideline"（arXiv, 2025）

**注意事项**：
- AI 辅助不等于 AI 自主——当前最佳实践是"AI 初筛 + 人工复核"
- 数据提取需要预设模板和明确提示词，模板质量直接影响提取效果
- 透明报告：任何 AI 辅助的筛选/提取/评估都需要记录使用方法和时间

**适合场景**：大规模系统综述或荟萃分析。项目中直接相关——导师调研工具需要处理大量论文数据，这是其核心工作流。

## 4. 你的"多层汇总法"在学术界对应的位置

你在对话中提到的"小范围汇总 → 逐层聚合"方法（小范围分组先总结，再将多组结果聚合为更高级总结，逐层递进）——在学术界有明确的方法论对应：

### 4.1 直接对应：Meta-Aggregation（元聚合）

**来源**：Joanna Briggs Institute (JBI)，一种系统性定性证据合成的方法论。

**核心思想**：将原始研究发现（findings）逐层聚合为类别（categories），再聚合为综合发现（synthesized findings）。每一步聚合都忠实于原始数据，不添加新的理论解释。

**三层结构与你的对照**：

| JBI 元聚合层级 | 你的"多层汇总法"对应 |
|:---|:---|
| 第一层：提取原始发现（Findings） | 小范围汇总——每个来源/每个主题的独立总结 |
| 第二层：将类似含义的发现归入类别（Categories） | 中间层聚合——同主题/同方向的信息合并 |
| 第三层：将类别综合为全局发现（Synthesized findings） | 最高层聚合——产出可操作的全局结论 |

**关键一致性**：两种方法都强调 **自下而上、逐层浓缩**、不跳过中间层级、每一步都保留可回溯的引用链。

### 4.2 间接对应：Grounded Theory 的编码层次

Grounded Theory（扎根理论，Glaser & Strauss, 1967）的三级编码过程与多层汇总形式相似但目的不同：

- **开放编码**（Open coding）：对数据进行逐行编码——逐条信息提炼→对应"小范围汇总"
- **轴心编码**（Axial coding）：建立编码间的关联和归类→对应"中间层聚合"
- **选择性编码**（Selective coding）：提炼核心范畴，统合全部→对应"高层的全局结论"

**区别**：Grounded Theory 是要"从数据中生成新理论"，多层汇总更偏向"忠实汇总已有信息"——这一点上更接近 Meta-Aggregation。

### 4.3 间接对应：Framework Analysis 的 Charting → Mapping 阶段

Framework Analysis 的"图表化（Charting）→ 映射和解释（Mapping/Interpretation）" 也是逐层抽象的过程——先把原始数据提炼填入矩阵表格（Charting），再在表格基础上做跨案例的模式识别（Mapping）。

### 4.4 你的方法独特在哪

多层汇总法与上述方法的最大不同在于**操作层面的设计原则**：

- **小范围先行**：先做最小可操作单元的综合（一篇论文、一个话题），再做跨单元聚合。这避免了"一次性面对海量信息"的认知过载
- **层次比绝对大小重要**：不预设汇总层数，根据实际信息量决定需要多少层——从 2 层（简单）到 5+ 层（复杂）弹性调整
- **可回溯性**：每一层汇总保留指向原信息的引用，高层结论可以一直追溯到底层原始资料

### 4.5 推荐参考

- Lockwood et al., 2015, "Qualitative research synthesis: methodological guidance for systematic reviewers utilizing meta-aggregation", JBI Evidence Implementation, 13(3), 179-187.
- Joanna Briggs Institute, "JBI Manual for Evidence Synthesis", 2024 edition, Chapter 2: Meta-Aggregation.

## 5. 导师调研工具的具体建议——信息整理 SOP

基于前述方法，针对"导师调研工具"的四段嵌套结构，提出以下信息整理阶段的推荐操作流程。

### 总体框架

采用**Framework Analysis + 多层汇总**的组合策略：
- Framework Analysis 提供整体结构（五阶段流程）
- 多层汇总提供分析操作（逐层浓缩信息）

### 信息整理 SOP（四段适用）

#### 第一阶段：信息收集与原始索引

| 步骤 | 做什么 | 使用的方法/工具 | 产出 |
|:----|:------|:--------------|:-----|
| 1.1 收集全部原始资料 | 按来源分类收集：（a）老师官网（b）学术平台（Google Scholar/ResearchGate）（c）中文百科（d）采访/报道（e）论文PDF列表 | 无特别方法，重在天罗地网 | 原始资料文件夹（按来源分） |
| 1.2 原始资料粗读 | 通读一遍，标注"这是什么、讲了什么" | Frame Analysis 第1步（数据熟悉）+ 渐进式总结 L1（粗体关键句） | 粗读批注（每份资料第一层标注） |
| 1.3 建立分析框架 | 根据调研目标，确定分析维度——例如： | Frame Analysis 第2步（框架识别）+ MECE 原则确保维度无重叠无遗漏 | 分析框架表（维度定义表） |
| | 第一段维度：姓名/职位/单位/研究方向/代表论文/资金/荣誉/合作网络 | | |
| | 第二段维度：研究问题/领域起源/技术路线/当前状态/开放问题 | | |
| | 第三段维度：每篇论文-核心问题/方法/结论/与该老师整体方向关系 | | |
| | 第四段维度：前置知识链（高数→线代→...→该方向前沿） | | |

#### 第二阶段：逐层信息编码与初次汇总

| 步骤 | 做什么 | 使用的方法/工具 | 产出 |
|:----|:------|:--------------|:-----|
| 2.1 按框架编码 | 将每份资料的粗读结果填入分析框架，逐段标注属于哪个维度 | Frame Analysis 第3步（索引）+ Thematic Analysis 初始编码 | 编码文件（每份资料的维度标注） |
| 2.2 小范围汇总（第一层） | 每个维度内，将同主题的信息合并为一段独立摘要——这是多层汇总的第一层 | 多层汇总第一层 + 渐进式总结 L3（摘要） | 维度摘要（每段信息一句话或一小段） |
| 2.3 跨维度关系记录 | 记录不同维度之间的关联——例如"他的研究方向A直接受导师B的影响" | 概念图法（记录节点间关系）+ Zettelkasten 链接 | 关系记录表（A→B型条目） |

#### 第三阶段：图表化和跨源对比

| 步骤 | 做什么 | 使用的方法/工具 | 产出 |
|:----|:------|:--------------|:-----|
| 3.1 构建比较矩阵 | 每篇论文/每份资料为一行，分析维度为列，填充编码摘要 | 比较矩阵法 + Frame Analysis 第4步（图表化） | 比较矩阵（可放在 Excel 或 AI 工具中） |
| 3.2 横向对比分析 | 对每列问三个问题：所有来源一致吗？有矛盾吗？留下了什么空白？ | 批判性阅读三阶分析 + 比较矩阵法 | 矛盾点记录 + 空白点记录 |
| 3.3 中间层聚合 | 将小范围汇总按主题/模式合并为更高一层的综合总结——多层汇总的第二层 | 多层汇总第二层 | 主题级总结（多个维度合并为一个主题总结） |

#### 第四阶段：高层次综合与输出

| 步骤 | 做什么 | 使用的方法/工具 | 产出 |
|:----|:------|:--------------|:-----|
| 4.1 跨主题模式识别 | 对所有主题级总结做综合分析，识别贯穿多个主题的全局模式 | Frame Analysis 第5步（映射和解释）+ 多层汇总最高层 | 全局结论（可选1-3条核心发现） |
| 4.2 可视化知识结构 | 将分析结果绘制为概念图/时间线 | 概念图/知识图谱法 | 认知地图（Miro/FigJam/Obsidian Graph） |
| 4.3 输出组织 | 采用金字塔原理组织输出——核心结论在前，逐级展开证据 | MECE + 金字塔原理 | 最终报告/讲义结构 |

### 各段独有的信息整理注意点

**第1段（老师基础信息）**：
- 使用 Toolkit: Elicit + Claude 快速提取论文结构化数据
- 矩阵维度：官网、Google Scholar、知乎/博客、采访稿等来源的交叉验证
- 难点：中英文信息混合

**第2段（领域发展调研）**：
- 使用 Toolkit: Scite 看争议点和共识 + Elicit 提取论文信息
- 额外产出：领域发展时间线（概念图法）
- 推荐先做概念图（全貌）再做详细编码（细节）

**第3段（老师论文串联）**：
- 最大挑战：需要将论文序列放在领域框架中重新定位
- 操作手法：以第2段的概念图为底图，将老师的每篇论文标注在对应位置
- 推荐：用比较矩阵法对比老师不同阶段的论文

**第4段（学习讲义设计）**：
- 这一步不再收集信息，而是以第1-3段的信息为基础做路线设计
- 核心操作：知识图谱的"路径提取"——在领域知识图谱中提取一条从基础到前沿的最优路径
