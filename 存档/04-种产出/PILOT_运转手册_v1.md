# pilot-test 运转手册

> 不是零件清单，是系统怎么转。

---

## §1 五块积木，每块一个角色

| 组件 | 一句话角色 | 管辖范围 |
|:----|:---------|:-------|
| `CLAUDE.md` | **宪法**——约束所有 AI 行为，铁律不可违 | 全局始终生效 |
| `初衷/01-04` | **基因**——告诉 AI 种子的根本立场：先问清楚，大量调研，不闭门造车 | 新窗口第一轮前必读 |
| `SKILL.md` | **流程**——当用户说"我想做X"时，告诉 AI 按什么步骤走 | 按 Skill 触发 |
| `HOOKS` | **机械警察**——AI 违反规则时拉警报或直接挡住 | 特定事件触发 |
| `产出/` | **结果篮**—AI 干活的唯一输出目录 | 所有文件写到这里 |

这 5 个组件的工作关系：**宪法始终约束 AI → 基因在 AI 第 一句回答前就喂给 AI → 流程在用户说出"我想做X"时启动 → 机械警察在 AI 做事时暗中盯梢 → 结果全部落进产出目录。**

不是"5 个文件"，是"5 层同时生效的机制"。

## §2 运转全景

下面跟踪一次完整的 pilot-test 使用：从新窗口打开，到 AI 做完一件事，到反馈注入下一次。

### 2.1 阶段 0：新窗口启动（加载中）

**你做的事**：VS Code → 新建窗口 → 打开 pilot-test 文件夹 → 启动 Claude Code

**背景自动发生**：

```
Claude Code 启动
  ├─ 加载 pilot-test/CLAUDE.md        → 宪法就位（R1-R10）
  ├─ 加载 .claude/settings.json        → Hooks 就位（5 条）
  └─ 加载 .claude/skills/              → SKILL.md 就位（等待触发词）
```

此时**所有约束已加载，但还没开始工作**。CLAUDE.md 要求 AI 第一轮响应前读 `初衷/01-04`。

### 2.2 阶段 1：第一句话之前（基因注入）

**你做的事**：输入第一句话

**AI 做的事**（严格遵守 CLAUDE.md 第 0 节）：

1. 先强行读 `初衷/01_为什么要建种子.md` → 知道两个不可跳步：先问需求、先调研
2. 读 `初衷/02_种子的本质.md` → 知道种子不是模板不是框架不是 Agent
3. 读 `初衷/03_使用原则.md` → 知道 R1-R10 每条的为什么
4. 读 `初衷/04_克隆与生长说明.md` → 知道种子怎么复制到新文件夹

**你感知到的**：AI 第一句回答里，已经理解了这个项目不是普通的"随便问随便答"，需要搞清楚你真正想做什么。

**谁在起作用**：CLAUDE.md §0 + 初衷 4 份文件 + H0 可选加强

### 2.3 阶段 2：你说出"我想做X"

**你做的事**：说"我想做一个读书笔记工具"

**AI 做的事**：

1. `SKILL.md` 触发词匹配 →"我想做X"匹配"需要结构化推进的意图"
2. SKILL.md 生效 → **第一句话必须是流程深度判定问询**
3. AI 问你："先确认两件事——这是哪种规模的任务？A/B/C/D？流程深度精简/标准/完整？"

**你感知到的**：AI 没有直接开始做，而是先问你"这东西有多大？想做到什么程度？"

**谁在起作用**：SKILL.md 第 1 节（流程深度判定）+ CLAUDE.md R6/R7（不凭空造车）

### 2.4 阶段 3：你回答判定

**你可能选的几个典型回答**：

| 你说 | AI 怎么做 |
|:----|:--------|
| "就一个笔记工具，C 中型，标准" | 先走阶段 1（需求锚定）和阶段 2（调研），再写计划和执行 |
| "只是改个小 bug，A 微型，精简" | 跳过分阶段，直接给你答案 |
| "这是我第一次用 pilot-test" | AI 跳过判定，直接问"你现在想做什么" |

**你感知到的**：AI 根据你的实际需求调整行为，不把所有问题当大项目处理。

### 2.5 阶段 4：逐阶段推进

假设你选了"C 中型，标准"。AI 按 SKILL.md 的阶段判断表推进：

```
阶段 1 ────> 写 产出/需求规格书.md
  │
  ∨
阶段 2 ────> 写 产出/调研报告.md
  │
  ∨
阶段 3 ────> 写 产出/执行计划.md
  │
  ∨
阶段 4 ────> 写 产出/收工审核报告.md
```

**每个阶段的推进过程**：

- AI 检查 `产出/` 目录 → 看缺哪个阶段的文件
- 缺哪个的阶段 → 问你是否进入该阶段 → 你确认 → 执行
- 执行完后 产出/ 出现新文件
- 你审核 → 通过或回退

**回退机制**（R3 人决策）：执行时发现理解有误，AI 问你"要回退到阶段 1 重做吗？"——你不会点头，AI 不擅自退。

**你的角色**：每阶段切换时确认一次。不是"完全让 AI 跑"，是"每段关键产出人看一次"。

### 2.6 阶段 5：机械警察暗中工作

AI 执行 4 阶段的过程中，Hook 在背后自动触发，你不需要知道它们的存在，除非触发警告：

| Hook | 什么时候触发 | 做了什么 | 你感知到 |
|:----|:----------|:-------|:-------|
| H1 | AI 写完一个文件后 | 检查文件末尾有没有 `## 验证来源` | 如果漏了 → 终端提示"来源节缺失" |
| H2 | AI 正要写文件时 | 检查路径有没有 `..` 上跳 | 如果越界 → 写入被阻断 |
| H3 | AI 正要写文件时 | 检查是否跳过了前置阶段 | 如果跳过 → 终端警告，但不阻断 |
| H4 | AI 写完文件后 | 检查 SKILL.md 是否超过 200 行 | 如果超了 → 终端警告 |

**关键设计原则**：H1/H3/H4 只警告不阻断——R3 规定决策权在你。**唯一真正阻断的是 H2（隔离边界）**——物理上不让 AI 写到 pilot-test 外面去。

### 2.7 阶段 6：收工与反馈

一次完整的 4 阶段走完后：

1. AI 产出物在 `产出/` 目录下
2. 你审核全部产出，给出判断
3. AI 记下摩擦点和提议（R3）
4. 你决定改什么规则、不改什么
5. **下一次启动**：AI 读到的宪法/流程/初衷和上次一样——但有了上次项目经验（产出/ 目录里的东西），它会知道"这个项目在做什么"

### 2.8 全局运转总结图

```
新窗口启动
  │
  ├─ CLAUDE.md 加载（宪法就位）
  ├─ Hooks 加载（警察就位）
  └─ SKILL.md 加载（流程就位）
  │
  ∨
用户说第一句话前 → AI 强读初衷 4 份 → 知道"项目立场"
  │
  ∨
用户说"我想做 X"
  │
  ├─ SKILL.md 触发 → 流程深度判定
  │     │
  │     ∨
  │  用户选深度 → AI 按阶段执行
  │     │
  │     ├─ 阶段1 → 写 产出/需求规格书.md
  │     ├─ 阶段2 → 写 产出/调研报告.md
  │     ├─ 阶段3 → 写 产出/执行计划.md
  │     └─ 阶段4 → 写 产出/收工审核报告.md
  │
  ├─ Hooks 全程暗中监听
  │     ├─ H1 检查引用
  │     ├─ H2 阻止越界
  │     ├─ H3 警告跳步
  │     └─ H4 防臃肿
  │
  ∨
审核 → 反馈 → 下一轮
```

## §3 配合关系：什么时候谁在管

### 3.1 "用户说我想做X" 时的三层响应

```
用户："我想做..."
  │
  ① SKILL.md 匹配触发词
     → 启动流程深度判定
     │
  ② 受 CLAUDE.md 约束：
     ├─ R6 不生搬硬套 → 不从外部找模板，从项目已有出发
     ├─ R7 不凭空造车 → 每个建议必须有项目内依据
     └─ R3 人决策     → 深度判定是问用户，不是 AI 猜
     │
  ③ Hooks 警戒就位：
     ├─ H2 准备拦截越界写入
     └─ H3 准备警告跳步
```

### 3.2 "AI 写文件" 时的分工

```
AI 写 产出/xxx.md
  │
  ① CLAUDE.md R1、R2、R9 约束内容：
     ├─ R1 防幻觉 → 涉及外部引用先搜索再写
     ├─ R2 Output Gate → 回复第一行必须是"已写入：路径"
     └─ R9 联网搜索 → 按意图分类选工具
  │
  ② Hooks 触发：
     ├─ H1 写完后检查 `## 验证来源` 节
     ├─ H2 写入前检查路径是否越界
     └─ H3 写入前检查是否跳阶段
  │
  ③ SKILL.md 约束步骤：
     ├─ 阶段内按顺序执行
     └─ 阶段切换需用户确认
```

### 3.3 "AI 想改规则" 时的阻断

```
AI 觉得规则不好想改...
  │
  ① CLAUDE.md R3 直接阻断：
     "禁止无确认的自我优化——任何'为了更好用所以我改了 X'视为违规"
  │
  ② AI 只能做：记录摩擦点 + 提出修改建议
    用户决定：改不改、怎么改
  │
  ③ 如果用户同意改：
     ├─ 产物层改动（产出/ 下）→ 随时改
     ├─ 机制层改动（Hook/触发条件）→ 需两次反馈
     └─ 基因层改动（初衷/ + R1-R10）→ 需用户明确表态
```

### 3.4 CLAUDE.md vs SKILL.md vs Hook 的约束力

| 来源 | 约束类型 | 可绕过吗 | 阻断方式 |
|:----|:-------|:-------:|:-------:|
| CLAUDE.md | 提示词约束 | 可能 | AI 在重复生成时概率性遗漏 |
| SKILL.md | 提示词流程引导 | 可能 | AI 可能在长对话中忽略流程 |
| **Hook** | **确定性代码** | **不能** | 条件匹配就执行，exit 2 真阻断 |

**这就是为什么 R5 说"文字护栏无效"**——CLAUDE.md + SKILL.md 都是提示词，AI 可能不遵循。Hook 是唯一真正安全的。但当前 Hook 只覆盖了 4 条检查，大量规则还只有提示词层。

## §4 第一次启动指南

你在新窗口第一次打开 pilot-test，看到的是这样的：

```
pilot-test/
├── 初衷/          → 基因（AI 读，你不用读）
├── CLAUDE.md      → 宪法（如果你好奇的话可看，但不用记）
├── .claude/       → 流程 + 警察（自动工作）
├── 产出/          → 空的（还没做事）
└── 转接文档.md     → 旧基线（已经用不上了）
```

**你的操作就一句话**：说你想做什么。

场景 1——你已经有想法：
> "我想做一个读书卡片工具"

AI 会先问你是哪种规模、多深，然后开始走阶段。

场景 2——你还不知道具体想做什么：
> "我第一次用，不知道 pilot 能做什么"

AI 跳到工具未定型特例，直接问"你现在想做什么"——帮你理清。

**不需要做的事**：
- 不用去读初衷/（AI 会读）
- 不用去手动运行什么
- 不用先搞懂 Hooks 怎么配置
- 不用看转接文档

**你需要做的事**：
- 每阶段切换时点头确认
- 审核 AI 产出的文件
- 如果 AI 跑偏了，说"不对，回退"

一次完整的 4 阶段跑完后：`产出/` 下会有 4 个文件。你的第一个工具就长出来了。

## §5 这个机制解决了什么

对应 D09 三方对齐报告中说的"纸上谈兵"问题，这个运转方式这样做：

| 之前的问题 | 运转机制怎么解决 |
|:---------|:--------------|
| 需要时框架不主动出来 | SKILL.md "宁可误触不可漏触"，触发词宽 |
| 规则全靠 AI 自觉 | Hooks 机械检查，真阻断越界，真警告跳步 |
| 文件散落不记得在哪 | 所有产出在 `产出/`，一个目录够 |
| 修改规则不用改代码不改 | CLAUDE.md + SKILL.md 改了就生效，Hook 改了要验证 |
| 每次新窗口记忆清零 | 初衷/ 和 CLAUDE.md 始终加载，种子基因不丢 |
| 不知道 AI 跳过什么步骤 | H3 警告跳步，你立即知道 |

**但也要说实话——有些问题还没有解决**：Hooks 只覆盖 4 条检查、流程弹性全靠提示词、阶段间反馈回路还没设计、自进化回路不存在。这些在 `待定项总览.md` 的开放问题里记着。

当前版本能跑的，是 **从"新窗口打开"到"AI 产出第一份文件"** 这段路走通了。之后的迭代靠你在实际使用中反馈来推。
