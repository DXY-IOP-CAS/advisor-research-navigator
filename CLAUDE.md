# pilot-test CLAUDE.md

> 本文件是 pilot 种子在本工具项目实例（`pilot-test/`）中的宪法层。新窗口启动第一轮响应前，必须先读 `初衷/` 下 4 份文件，读完再行动。

---

## 0. 项目身份

本实例承载的是**导师研究方向调研工具**。学生输入目标导师的信息（学校+姓名+方向关键词），AI 按四段嵌套（基础信息 → 领域发展 → 论文串联 → 学习讲义）产出从调研到讲义的完整成果到 `项目/<老师>/` 下。第一个测试实例为张鹏举（IPHY 超快物质科学中心）。

---

## 1. 核心纪律

**① 防幻觉**：涉及论文/公式/定理/数据/时效信息 → 必须搜索验证再写。每条外部信息末尾标 URL 和"验证来源"小节。

**② Output Gate**：任何教学/讲解/对比/分析内容 → 先写 `.md` 文件再开口。回复第一行必须是 `已写入：[文件路径]`。违规修正：停笔 → Write → 重发。

**③ 人决策**：AI 执行，人决策。禁止无确认的自我优化——任何"为了更好用所以我改了 X"都违规。

**④ 隔离边界**：不修改 `初衷/` 下的任何文件。所有改动必须在 `pilot-test/` 子树内。

**⑤ 不凭空造车**：每个方案从项目已有内容出发，生搬硬套和空口提方案均违规。必须标注"这个方案怎么来的"。

**⑥ 搜索规则**：详见 [指导/身份与学术标准.md](指导/身份与学术标准.md) ——搜索意图分类、工具选择、URL 溯源要求。

**⑦ 归档**：非主线任务产物归 `历史/方案/[日期]_[主题]/`，子文件夹必须含 `README.md`。

**⑧ 写作规范**：第4段讲义严格遵循 [指导/讲义写作规范.md](指导/讲义写作规范.md) 的起承转合五步叙事流和排版格式。

---

## 2. 操作红线

- CLAUDE.md / SKILL.md 的规则变动前必须先问用户，用户拍板才能改。
- 修改 SKILL.md 后必须对应改 `settings.json` 的 Hook 触发条件——否则视为新规则未生效。
- 不可直接修改 `项目/<老师>/` 下已有的产出——必须经用户审核确认后改动。
