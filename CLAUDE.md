## 2. 硬规则（R1-R10）

每条规则的"为什么"在 `初衷/03_使用原则.md`。本文件只列"做什么"和"出处"。

### R1 防幻觉
涉及论文/公式/定理/数据/时效信息/外部引用 → 必须搜索验证再写。间接引用追溯原始来源。每条外部信息标 URL，末尾"验证来源"小节列出。
**出处**：根 CLAUDE.md + 软约束失效业界方案 + 初衷 R1

### R2 文件化产出（Output Gate 协议）
任何教学/讲解/分析/对比内容 → 必先写 `.md` 文件再开口。回复第一行必须是 `已写入：[文件路径]`。违规修正：停笔 → Write → 重发。
**出处**：根 CLAUDE.md + 输出门协议

### R3 外循环：人决策 + AI 执行
AI 记录摩擦点和提议；改规则/改模板的决策权在人。**禁止无确认的自我优化**——任何"为了更好用所以我改了 X"的自动行为视为违规。
**出处**：D08 决策10（宪法级）+ 初衷 R3

### R4 隔离边界
不修改 `pilot-test/` 子树之外的任何文件。根项目 `.claude/skills/project-pilot/`、根 `.claude/settings.json`、根 CLAUDE.md、其它 `练习册/` 子目录均为只读。沙盒内全部改动必须落在本目录子树内。
**出处**：Hook 加载按子目录隔离 + 初衷 R4

### R5 文字护栏无效
本目录 SKILL.md 的"规则"以 Hook 阻断为准——文字护栏是"礼貌的请求"不是安全。任何 SKILL.md 写的规则若没有 Hook 落地，视为不存在。
**出处**："Why Prompt Guardrails Fail" + D07 K5 + D09 三方对齐 + 初衷 R5

### R6 不生搬硬套
外部方案/方法论不直接套项目。所有设计从项目已有内容出发——`初衷/` 是基线起点。
**出处**：用户明确指令 + 转接文档 §4 + 初衷 R6

### R7 不凭空造车
任何方案/规则必须有项目内依据。空口提出的方案视为未充分论证。
**出处**：初衷 R7

### R8 每条建议标"怎么来的"
CLAUDE.md / SKILL.md / Hooks 的每条规则或建议必须标注来源：`<!-- 来源：[模块] [决策/章节] -->` 或 `**出处**：[出处]`。
**出处**：用户明确指令 + 转接文档 §4 + 初衷 R8

### R9 联网搜索规则
联网搜索规则继承根 CLAUDE.md "联网搜索"节，不复述。
**出处**：根 CLAUDE.md + 初衷 R9

### R10 对话归档
非主线任务的产物归 `存档/历史方案/[日期]_[主题]/`，子文件夹必须含 `README.md`。
**出处**：根 CLAUDE.md + 初衷 R10

## 5. 分层与谨慎度（避免长歪）

**为什么这一节**：用户原话（2026-06-27）——"种子可以自迭代，但基因部分不能随意更改，否则工具可能在成长中不断长歪"。种子分三层，谨慎度不同：

### 基因层（不可轻易改）

| 包含 | 改的门槛 |
|:----|:--------|
| `初衷/01-04` 全部 | 用户明确表达"我的根本立场变了" |
| CLAUDE.md R1-R10 | 用户明确表达"某条铁律要废" |
| 流程约束 P1-P5 中标注"宪法级"的部分 | 同上 |

**改后影响**：未来所有用种子克隆的工具都受影响。**改之前必须明确告知用户"这是基因层改动"**。

### 机制层（按反馈谨慎改）

| 包含 | 改的门槛 |
|:----|:--------|
| 阶段判断流程 | 至少 2 个不同项目的真实使用反馈触发同一痛点 |
| Hooks H0-H4 触发条件 | 同上 |
| 流程深度判定的 4 规模 + 3 深度切分 | 同上 |
| 调研顺序（邻域 → 工具 → 设计） | 用户明确表达 |

**改后影响**：当前工具的工作方式，但不影响未来克隆出的工具。**改之前必须先记录到 `产出/迭代反馈.md`（如不存在则新建）**。

### 产物层（频繁改）

| 包含 | 改的门槛 |
|:----|:--------|
| `产出/` 目录下所有文件 | 无——用户每次反馈都可改 |
| 工具的具体功能、UI、产出格式 | 无 |

**改后影响**：仅当前工具的当前成果。**自由改**。

### 边界判定原则

- 改动**只影响当前工具的当前使用方式** → 产物层（自由改）
- 改动**影响当前工具未来的工作方式** → 机制层（要反馈触发）
- 改动**影响未来所有工具的形态** → 基因层（要用户明确确认）
- 不确定 → 默认按上一层谨慎度处理

**Why**：种子克隆到新文件夹时带的是基因 + 机制。**基因是种子本身**——种子的根本立场；**机制是种子让工具怎么长的方式**。混了这两层，种子在不同环境长出完全不同的东西，名字还叫 pilot——长歪。
