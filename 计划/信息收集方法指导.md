# 传统方法论（底子）

以下 7 种方法构成信息收集的**基础能力**。AI 工具可以加速执行，但不能替代对原则的理解。不掌握底子就使用 AI 工具，容易在筛选时漏关键文献、在综合时产生偏见、在输出时缺乏结构。

---

## 1. 系统性文献综述 5 步法

*系统性文献综述的信息收集骨架。5 步标准化流程，确保文献检索可复现、可审计。*

### 核心步骤

**Step 1: 明确研究问题与关键词**
- 将宽泛问题拆分为具体子问题
- 利用 PICO（Population, Intervention, Comparison, Outcome）或类似框架结构化问题
- 生成同义词、近义词、上下位词列表以扩展检索覆盖

**Step 2: 多数据库检索**
- 至少覆盖 2-3 个数据库（如 PubMed、Web of Science、Google Scholar、Scopus、CNKI）
- 保存每个数据库的检索式（search string）、检索日期、结果数量
- 使用布尔运算符（AND/OR/NOT）组合关键词

**Step 3: 筛选**
- 设定纳入/排除标准（可信度、年代、语种、研究类型、主题相关性）
- 两阶段筛选：标题/摘要筛选 -> 全文筛选
- 记录每阶段的排除原因（PRISMA 流程图可在此步使用）

**Step 4: 深度阅读与结构化笔记**
- 每篇论文提取：研究问题、方法、样本、核心发现、局限
- 使用统一模板记录，便于后续比较

**Step 5: 分析与综合**
- 叙事综合（thematic synthesis）：按主题归纳发现
- 或定量综合（meta-analysis）：统计合并效应量

### 适合场景
- 文献综述章节
- 研究领域的全景调研
- 需要向他人证明"搜索是系统性的"的场景

## 2. PRISMA 标准

*系统性综述的报告规范。告诉你在论文中必须报告什么，间接推动搜索过程的规范化。*

### 核心内容

PRISMA 2020 版包含 27 项检查清单，分布在 7 个部分：

| 部分 | 核心要求 |
|:----|:--------|
| 标题 | 标明是系统性综述还是 Meta 分析 |
| 摘要 | 结构化摘要 |
| 引言 | 明确目的+已知知识 gap |
| 方法 | 检索策略、纳入标准、筛选流程、质量评估方法 |
| 结果 | 文献筛选流程图+各研究特征 |
| 讨论 | 证据总结+局限性 |
| 资金 | 资金来源+利益冲突声明 |

PRISMA 流程图的关键节点：
```
数据库检索记录 (n=...)
    -> 去重后 (n=...)
    -> 标题/摘要筛选 (n=...)
    -> 全文获取 (n=...)
    -> 全文评估 (n=...)
    -> 纳入综述 (n=...)
```

2025 年扩展 PRISMA-trAIce：要求透明报告 AI 在文献综述中的使用——包括 AI 用于哪些步骤、使用的工具名称和版本、人如何验证 AI 输出。

### 适合场景
- 计划发表系统性综述
- 需要高可信度的文献调研
- 任何需要让调研过程"可见"的场景

## 3. 滚雪球法

*从一篇高质量文献出发，沿引用链扩展文献库。不需要复杂的检索式，适合冷门领域。*

### 核心做法

**前向滚雪球**：找到引用了目标论文的所有论文（Google Scholar 的"被引用次数"链接即此功能）
**后向滚雪球**：阅读目标论文的参考文献列表

实施步骤：

1. 找到 2-3 篇核心种子论文（高被引、最近综述、领域经典）
2. 读参考文献列表，标记看起来相关的标题
3. 回到数据库检索这些标题，读摘要确认相关性
4. 获取全文后重复步骤 2-3
5. 当连续 10 篇新增文献不再带来新信息时停止（理论饱和）

### 适合场景
- 刚入门一个领域，还没有系统关键词
- 冷门领域，数据库检索结果过少
- 补充系统检索的遗漏

## 4. 批判性阅读三阶分析

*不只"读懂了"，而是对信息做结构性判断。防止被低质量或偏颇的信息源误导。*

### 三阶流程

**第一阶：现象描述**
- 作者声称发现了什么？
- 用了什么方法？样本是什么？
- 结论是什么？

**第二阶：争议点提问**
- 作者假设了什么？这些假设合理吗？
- 有没有替代解释？
- 方法选择是否支持结论？
- 样本是否有偏差？

**第三阶：证据对比**
- 这篇文章的发现与同领域其他研究是否一致？
- 不一致的话，原因是方法差异还是真正的争议？
- 引用这篇文章的其他论文是支持还是反对它？

**结论提炼**
- 在自己的知识体系中，这个信息处于什么地位：事实、主流观点、少数派意见，还是已被推翻？
- 标注可信度等级（高/中/低 + 理由）

### 适合场景
- 核心文献深度阅读
- 争议性话题的调研
- 信息综合前的质量评估

## 5. Framework Method（框架分析法）

*对大量文本信息的系统化分类与比较方法。特别适合半结构化信息（访谈记录、开源文本、多源资料）。*

### 五阶段

**阶段 1: 数据熟悉**
- 通读全部素材，在边缘做初步笔记
- 记印象、初步主题想法

**阶段 2: 框架识别**
- 从数据中归纳初始主题/类别
- 也允许从研究问题出发设定初始类别（deductive + inductive 结合）
- 形成初步的编码框架（coding framework）

**阶段 3: 索引**
- 用编码框架给全文做标记
- 每段文字标注对应的类别
- 允许新增类别，但需回溯重新标注之前的数据

**阶段 4: 图表化**
- 将编码后的数据整理成矩阵表（行=案例，列=类别）
- 每个单元格是原始数据的提炼摘要（附来源位置）

**阶段 5: 映射和解释**
- 在矩阵中找模式、关系、例外
- 提炼核心主题
- 生成结论

### 适合场景
- 多源定性数据的综合（访谈、文档、调查）
- 导师画像构建（从论文、官网、社交信息中提取特征）
- 研究方向的横向对比

## 6. MECE 原则 + 金字塔原理

*信息的分类组织原则。确保分类不重叠、不遗漏，输出时有层次感。*

### MECE（Mutually Exclusive, Collectively Exhaustive）

- **Mutually Exclusive**：同一层级的类别之间不重叠
- **Collectively Exhaustive**：所有类别加起来覆盖全部可能性

MECE 的常见分类维度：

| 分类维度 | 示例 |
|:--------|:-----|
| 二分法 | 理论 vs 实验，开放获取 vs 付费 |
| 流程法 | 输入 -> 处理 -> 输出 |
| 公式法 | 收入 = 价格 x 数量 |
| 矩阵法 | 2x2 矩阵（重要-紧急，熟悉-陌生） |

### 金字塔原理（Barbara Minto）

写作/表达的信息组织方式：

1. 结论先行——先说核心结论
2. 上层抽象，下层具体——每层是对下层的概括
3. 横向 MECE——同一层的分组相互独立
4. 纵向支撑——下层事实支撑上层结论

金字塔结构的检查：阅读每层标题即可理解全貌，但支撑细节能追溯到底层事实。

### 适合场景
- 信息综合后的分类框架设计
- 调研报告的提纲搭建
- 将零散信息组织成可表达的结构
- 导师分类体系设计

## 7. Zettelkasten（卡片盒笔记法）

*将信息原子化存储，通过连接产生新见解。本质是一套个人知识管理系统。*

### 核心原则

**原子性**：每张卡片只记录一个概念/想法/事实
**连接性**：每张卡片至少连接 1-3 张已有卡片（通过链接/标签）
**用自己的话写**：不是照抄原文，而是用自己的语言重述
**出处必标**：每张卡片标注信息来源（论文 DOI、URL、页码）

### 实践操作

卡片类型：

| 类型 | 内容 | 写法 |
|:----|:----|:----|
| 文献笔记 | 论文/书的要点 | 用自己的话缩写，标出处 |
| 概念笔记 | 一个概念的定义+理解 | 定义 + 例子 + 与其他概念的关系 |
| 项目笔记 | 与某个项目相关的所有笔记 | 临时性，项目结束归档 |

装配流程：

1. 读一篇文章 -> 写文献笔记（每篇 2-5 张卡片）
2. 从文献笔记中提炼概念 -> 写概念笔记
3. 在概念笔记之间加链接（"X 与 Y 的矛盾在于…""A 提供了一个例子证明 B 的假设"）
4. 定期回顾，发现模式 -> 触发新的问题 -> 去搜索 -> 回到步骤 1

### 适合场景
- 长期积累的领域知识管理
- 跨论文的思想连接（"这篇论文引了那篇论文说 X，但实际数据是 Y"）
- 导师调研中的信息碎片整理（论文特征、项目经历、合作网络）

# AI 时代工具（进阶）

传统方法是底子，AI 工具是加速器。以下工具按功能分组，**每组的第一个是推荐首选**。

---

## 8. 引文图谱工具

*可视化论文之间的引用关系网络，快速发现领域结构。*

### Connected Papers

一句话：输入一篇论文，生成其引文关系图谱——节点是论文，边是引用关系。

核心功能：
- 相似论文聚类：相邻节点是研究主题相近的论文
- 自动标出关键论文：大节点 = 高被引或与输入论文关系最紧密
- 支持调整年代范围

适合场景：刚进入新领域时 5 分钟建立文献地图。先找 2-3 篇核心论文输入，从图谱中识别出领域的关键工作、经典论文和最新趋势。

局限：依赖 Semantic Scholar 数据库，中文论文覆盖差。

### Research Rabbit

一句话：学术发现的"推荐系统"——关注研究脉络，新相关论文发布时自动推送。

核心功能：
- 建立论文收藏集，工具自动推荐相似论文
- 追踪某个作者或某条研究线的更新
- 可视化研究脉络的时间线

适合场景：长期追踪某个领域的最新进展。与 Connected Papers 配合使用：先用 Connected Papers 建立初始图，再用 Research Rabbit 持续追踪。

## 9. AI 辅助文献搜索与提取

*用自然语言搜索文献，AI 自动提取论文的关键信息（研究设计、样本量、结果）。*

### Semantic Scholar API

一句话：语义搜索的学术论文数据库，支持 API 调用的结构化搜索。

核心功能：
- API 级别的搜索接口（适合程序化调用）
- 论文之间的引用/被引关系
- 研究领域标签、影响力评分
- 开放获取 PDF 链接

适合场景：程序化的文献检索需求（如导师调研工具的后端检索层）。

### Elicit

一句话：用问题搜论文，自动提取研究设计、样本量、结果等结构化信息到表格。

核心流程：

1. 输入一个研究问题（而非关键词）
2. Elicit 返回相关论文列表
3. AI 自动提取每篇论文的：研究设计、干预措施、样本量、结果、局限性

适合场景：一次性快速了解一个方向有哪些研究、结论是否一致。特别适合有明确因果/比较问题的场景（"X 对 Y 的效果"、"A 和 B 哪个更好"）。

局限：社会科学与生物医学覆盖好，理论物理/数学等理论性强的领域覆盖弱。

### Consensus AI

一句话：学术版搜索引擎——输入问题，直接给出带引用的答案。

核心功能：
- 回答附带引用论文
- 支持过滤（研究类型、样本量、开放获取）
- 给出"共识程度"指标（多篇论文一致 vs 有争议）
- 可直接跳到原文的相关段落

适合场景：快速验证一个学术主张是否是领域共识。用于信息收集阶段的"第一轮扫射"——先看 AI 怎么说，再去读原文。

### Perplexity

一句话：综合搜索引擎，整合搜索结果并附来源链接。

核心场景：作为辅助工具使用——在系统性检索后发现信息缺口，用 Perplexity 快速补充。不适合作为主检索工具（结果可能不全面、不稳定）。

## 10. AI 辅助文献综合与写作

*阅读后阶段——帮助综合多源信息，组织写作结构。*

### NotebookLM（Google）

一句话：多源信息综合工具——上传文档，AI 基于上传的内容回答问题。

核心功能：
- Audio Overview：自动生成音频对话总结
- 支持多源综合（论文、网页、笔记）
- 回答严格基于上传内容（减少幻觉）

适合场景：文献综合——把所有相关论文上传，让 AI 帮你整理共性、差异、争议点。适合信息收集完成后的综合阶段。

### Scite

一句话：引文上下文分析——显示一篇论文被引用时，是支持、反对还是提及。

核心功能：
- 对每篇论文显示 citation statements（引用语境）
- 标出引用极性：支持（supporting）、反对（contrasting）、提及（mentioning）
- 可追踪某篇论文被争议的过程

适合场景：评估一篇论文的可信度和影响力。高被引不等于高共识——Scite 能告诉你引用这篇论文的人究竟是拿它当证据还是当靶子。

### Paperpal

一句话：AI 学术写作助手，协助语法润色、结构优化、引用格式。

适合场景：文献综合完成后的写作阶段。不是信息收集工具，但在输出环节有用。

## 11. AI 辅助的综述筛选（2026 进展）

*2026 年的重要进展：AI 辅助系统综述筛选已被验证在多个场景中显著节省时间。*

核心发现：
- AI 可辅助标题/摘要筛选阶段，减少人工需要筛选的数量
- 敏感度（recall）可与双人筛选媲美，但需谨慎设置阈值
- 目前最佳实践是"human + AI"混合——AI 做初筛，人确认排除决策
- 仍需透明报告 AI 的使用方式（PRISMA-trAIce）

对工具开发的启发：
- 导师调研工具的前期筛选可以引入 AI 辅助
- 但排除决策需要人确认（或记录排除理由供人追溯）
- 最好的定位是"加速而非替代"

# 针对"导师调研工具"场景的建议

基于以上方法论，以下是针对导师调研工具**信息收集阶段**的具体操作规程。

---

## 阶段划分

导师调研的信息收集分为 3 个递进阶段：

| 阶段 | 目标 | 核心产出 |
|:----|:----|:--------|
| 广撒网 | 快速建立候选导师池 | 候选名单（~10-15 人） |
| 深钻 | 深入理解每位导师的研究方向和水平 | 每人的结构化画像 |
| 校验 | 交叉验证信息准确性 | 确认后的最终选校名单 |

---

## 信息收集阶段 SOP

### 阶段 A: 广撒网（以领域定候选）

**输入**：研究生入学方向 / 兴趣关键词列表

**步骤 A1 - 关键词膨胀**
- 使用 MECE 原则分解方向关键词（例如 "量子蒙特卡洛" -> [量子蒙特卡洛, 量子蒙卡, QMC, path integral Monte Carlo, tensor network, quantum simulation]）
- 产出：关键词字典（含中英文 + 近义词 + 上下位词）

**步骤 A2 - 初筛检索**
- 首选：Semantic Scholar API（程序化，批量）或 Google Scholar（手动）
- 搜索目标领域综述文章（review paper）-> 获取领域全景
- 从综述的参考文献中提取高频引用作者 -> 候选列表
- 辅助：Connected Papers -> 输入一篇核心综述 -> 观察聚类图谱中的核心节点

**步骤 A3 - 滚雪球补漏**
- 找到领域内 2-3 个顶级组 -> 前向滚雪球（看谁在引用他们）
- 找到领域最新会议论文集 -> 查看新兴方向的工作者

**输出**：10-15 人的候选导师名单（含姓名、机构、主要研究方向、1-2 篇代表作）

### 阶段 B: 深钻（以人定画像）

**输入**：候选导师名单

**步骤 B1 - 论文扫描**
- 下载每位候选导师近 5 年的一作/通讯论文
- 使用 Framework Method 的编码理念，统一提取每篇论文：

| 字段 | 提取内容 |
|:----|:--------|
| 问题 | 这篇论文要解决什么物理/数学问题？ |
| 方法 | 用了什么技术手段？（特别关注导师本人提出的方法） |
| 基线 | 他们与谁竞争/比较？ |
| 结论 | 核心发现 |
| 后续 | 论文说了"未来工作"是什么 |

**步骤 B2 - 引文分析**
- 使用 Scite 查看这些论文的引用上下文（被支持还是被质疑？）
- 使用 Semantic Scholar 查看每篇论文的被引量趋势（快速上升还是平稳？）
- 识别导师工作的"影响质量"而非"影响数量"

**步骤 B3 - 网络识别**
- 从论文的合作者列表中识别导师的长期合作网络
- 注意区分：独立方向 leader vs 跟随大组工作
- 关注作者的归属单位变更（反映学术流动性）

**步骤 B4 - 结构化笔记**
- 每张"导师卡片"使用 Zettelkasten 原子化原则：一人一卡
- 卡片内容包含：研究方向 + 代表作 + 合作网络 + 引用表现 + 代码/数据开放情况
- 在卡片间加连接（"A 和 B 做过合作"、"C 的方法与 D 竞争"）

**输出**：每位候选导师的结构化画像（研究方向的术语层级、代表作分析、合作网络图、学术声誉指标）

### 阶段 C: 校验（以证据定结论）

**输入**：候选导师画像

**步骤 C1 - 信息源交叉验证**
- 每项核心结论至少 2 个来源确认（如下图）
  - 官网信息 -> 论文中确认
  - 论文结论 -> 引用分析中确认支持程度
  - 学术活动 -> 组会/报告记录中确认

**步骤 C2 - 争议检测**
- 使用批判性阅读三阶分析：导师的核心工作是否有被质疑/修正过？
- 使用 Scite 查看关键论文被反对的比例
- 如果核心工作存在争议 -> 标记为高风险

**步骤 C3 - 综合排名**
- 使用金字塔原理组织信息：先给出推荐结论，再展开支撑证据
- 使用 Framework Method 的映射阶段：建立"候选导师 x 评估维度"矩阵

| 评估维度 | 权重 | 信息来源 |
|:---------|:----|:--------|
| 研究方向匹配度 | 高 | 画像+论文 |
| 研究水平（引用/认可度） | 高 | Scite+引文分析 |
| 导师风格（经验/资源/自由度） | 中 | 学生评价+论文产出节奏 |
| 毕业条件/年限 | 中 | 课题组毕业生去向 |
| 地理位置/待遇 | 低 | 官网+生活成本数据 |

**输出**：最终推荐列表（每个候选人的推荐理由 + 风险点 + 建议下一步（套磁/约谈/联系组内学生））

## 反模式清单

1. 只搜一个数据库：遗漏领域内的关键工作
2. 只依赖 AI 筛选不校对：AI 可能漏掉冷门但重要的论文
3. 不记录检索过程：后续无法回溯"为什么选了 C 没选 D"
4. 只看引用量不看引用质量：被引高可能是因为被批评（Scite 可解决）
5. 把所有论文当同等证据：综述论文、方法论文、应用论文的质量和定位不同
6. 不标注信息源：信息失效或发现矛盾时无法溯源
7. 过早停止滚雪球：有时候最相关的论文在第 2-3 层引用链里
8. 只用传统方法不用 AI：耗时且容易漏（Top-down 偏见——只看到大牛，漏了快速上升的新组）
