## 章 1：OpenAlex 数据质量关键警告（所有设计的前提）

以下 4 篇 2024-2025 年的同行评议研究揭示 OpenAlex 对中文/中国数据的系统性缺陷。这些缺陷直接影响工具的数据源策略。

### 1.1 中文论文覆盖率：不完整且不连续

- **来源**：Zheng et al. (2025) "Understanding discrepancies in the coverage of OpenAlex: the case of China", *JASIST* 76(11):1591-1601。
- **发现**：OpenAlex 对中国论文的覆盖存在"不完整和不连续"问题。虽声称覆盖非英语论文，但实际覆盖波动剧烈，且已基本停止从 CNKI 收录。CNKI 收录的中文论文仅 46 篇被索引。
- **影响**：对中国导师的中文论文，OpenAlex 覆盖严重不足。

### 1.2 中文语言元数据：63.7% 误分类率

- **来源**：Céspedes et al. (2025) "Evaluating the linguistic coverage of OpenAlex", *JASIST* 76(6):884-895。
- **发现**：中文是 OpenAlex 中最常被误分类的语言（63.7%）。约 550 万篇实际中文论文被标记为英文。
- **影响**：依赖 `language` 字段筛选中文论文不可靠。

### 1.3 中文核心期刊索引率：仅 37%

- **来源**：arXiv 2512.16339 (2025) "Beyond openness: Inclusiveness and usability of Chinese scholarly data in OpenAlex"。
- **发现**：OpenAlex 仅索引中国核心期刊目录中 37% 的期刊、24% 的论文。元数据质量不均匀：DOI 覆盖有限，作者所属机构频繁缺失，引用数据不完整。
- **影响**：OpenAlex 不能作为中文论文搜索的唯一源。

### 1.4 作者消歧：中文名不可靠

- **来源**：Zhao & Chen (2025) "Accuracy Assessment of OpenAlex and Clarivate Scholar ID with an LLM-Assisted Benchmark", arXiv:2502.11610。
- **源 URL**：https://arxiv.org/html/2502.11610v2
- **发现**：OpenAlex 的 XGBoost 消歧对中文名的精度和召回率均低于英语和欧洲作者。大规模 ORCID 覆盖（92% vs WoS 的 16%）部分源于过度激进的消歧——部分中文作者被 link 到超过 10,000 篇论文。
- **影响**：我们的"论文标题比对"策略（官网 ground truth 做锚点）是正确的。只有这个方案能在 OpenAlex 不可靠的环境下锁定身份。

**结论**：OpenAlex 是"最大但最脏"的开放数据集。必须多源互补。工具当前的策略（官网 ground truth + 标题比对 + 多源共证）完全正确。

## 章 2：学者画像工具生态

### 2.1 学术身份 ID 体系

| ID 类型 | 提供方 | 性质 | 覆盖 | API |
|:-------|:-------|:----|:----|:---:|
| **ORCID** | 自注册 | 自认证，一对一人 | 注册率低 | 免费公共 API |
| **OpenAlex ID** | 自动消歧 | ML 聚类 | 100M 作者，中文不准 | 免费，100K/天 |
| **S2 Author ID** | 算法聚类 | ML | 覆盖广 | 免费，100/5min |
| **ArXiv Author ID** | 自 claim | 手动关联，物理/数学常用 | 注册率一般 | 免费 |
| **Google Scholar ID** | 自建 profile | 自维护 | 全球最广 | 无官方 API |
| **INSPIRE-HEP ID** | 高能物理 | 业内事实标准 | 仅 HEP | 免费 |
| **NASA ADS** | 天体物理 | 业内事实标准 | 仅天文 | 需 token |
| **Clarivate Scholar ID** | 商业消歧 | 付费 | WoS 覆盖 | 付费 |
| **Scopus Author ID** | 商业消歧 | 付费 | Scopus 覆盖 | 付费 |
| **VIAF** | OCLC | 图书馆标准 | 名人级别 | 免费 |
| **DBLP PID** | 人工策展 | CS 领域最准 | 仅 CS | 免费 |

**OpenAlex 的新消歧模型**：2023 年 8 月宣布了新的 Author Disambiguation 模型，改善"作者分裂"（同一人被误认为多个）问题。但六维特征（姓名相似度、机构、合著网络、引用模式、ORCID、主题分布）对中文名的效果仍存疑。

**工具当前的 ID 源选择**：strategy 正确。但我建议增加 **OpenAlex 官网直接搜姓名主页**（https://openalex.org/authors?search= 获取候选池）作为人工辅助验证的额外渠道。

### 2.2 学者画像聚合平台

| 平台 | 覆盖 | 数据源 | 定位 | 是否竞品 |
|:----|:-----|:-------|:----|:--------:|
| **ScholarGPS** | 30M+ 学者，50K+ 机构 | 多种源聚合 | 学者排名+profile，可 claim | 部分重叠阶段1 |
| **Lens.org** | 200M+ 学术记录+专利 | MAG + PubMed + Crossref + OpenAlex | 学术+专利，开放引用图 | 间接参考 |
| **find-research-prof** (GitHub) | — | OpenAlex + S2 + ORCID | Wikipedia 式教授发现 | 部分重叠 |
| **UKIP**（GitHub: keilynrp） | — | 8个 adapter（OA/S2/WoS/Scopus等） | 研究情报平台，388个 API 端点 | 参考架构 |
| **merita**（Codeberg） | — | OpenAlex + S2 | 个人学术成就数据库，引用指标 | 非竞品 |

**关键观察**：所有画像聚合平台都停在同一层面——"导师长什么样"。没有工具继续推进到"学懂这个方向怎么做"。工具的四段嵌套（调研→综述→教学）仍然是独有的。

## 章 5：项目策略评估

### 5.1 当前工具设计 vs 竞品对比

| 维度 | 同类工具 | 我们 |
|:-----|:---------|:-----|
| **深度** | 停在"导师什么样" | 走到"学懂这个方向怎么做" |
| **用户** | 通用研究生 | 物理本科生 |
| **形式** | 网站/App | Claude Code Skill + 项目夹 |
| **身份消歧** | 信 OpenAlex/ML 聚类 | 官网 ground truth + 论文标题比对 |
| **论文组织** | 按引用数排序 | 按学术履历阶段分组 |
| **阶段1→4** | ❌ 无 | 逐段递进 |
| **数据源** | 单一或已选好 | 多源互补+中文 Web 搜索 |
| **费用** | 多数付费 | 全免费（API polite pool） |

### 5.2 关键发现汇总

1. **OpenAlex 对中文数据不可靠已被学术文献证实**。工具的策略（官网 ground truth + 多源交叉）是正确且必要的。

2. **Academix MCP 和 orcid-mcp 是 "低挂水果"**。如果后续 identity_resolver.py 的 bug 修起来费劲，可以用现成 MCP 替代。

3. **Academic Detective (174⭐)** 是目前最接近阶段 3 的开源项目——论文产出核实 + 学科基准线检测 + 六维质量评分。可以读它的方法论作为参考。

4. **导师.skill (ybq22)** 做了"蒸馏导师为 AI skill"的事，和阶段 4 的教学讲义有概念重叠——它生成可对话的 skill，我们生成递进式 Markdown 讲义。两种形态可以并存。

5. **现有的 7 个导师匹配平台没有一家走到 "领域发展脉络 → 论文在领域树上定位 → 学习讲义" 的流水线**。工具的差异化定位未被侵蚀。
