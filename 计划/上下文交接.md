## 3. 关键设计决策摘要（第五轮修订后）

- **数据收集策略（v3.2 核心修订）**：由固定四层顺序改为自适应两步走——先调 OpenAlex 看 works_count，< 20 则降级到 Google Scholar 做论文列表主源（GS 无 API，爬公开 profile 可行性已验证）
- **同名干扰过滤三部曲**（v3.2 新增）：合著者网络白名单 → 期刊范围 → 领域关键词。尤其 GS 和 arXiv 搜索返回大量同名论文时必须用
- **覆盖预期修正**（v3.2）：OpenAlex 充足时 90-95%+，不足时依赖 GS profile 的维护程度。对中文作者不再承诺 95%+
- **覆盖检测机制**：works_count < 20 自动触发 GS 降级路径。阈值在 config/sources.json 中可配
- **巨人出处表**（2.1.1 新增）：每层数据来源明确标注"巨人归属"和"不自造的部分"
- **论文全量合并算法**（2.3.2 修订）：P0 DOI → P1 arXiv ID → P2 归一化标题，合并后跨源取最优字段，标注共证可信度 N/4
- **覆盖预期**（2.3.2 新增）：独立后 5 年 95%+ / 15 年+ 90-95%+ / 高产 PI 90%+，缺失部分标注边界
- **OpenAlex 中文缺陷已被学术界实证**：Zhao & Chen 2025（消歧不准）+ Zheng et al. 2025（覆盖不连续）——但物理方向英文论文为主，此缺陷不致命
- **阶段0流程**：抓官网 → 学科识别 → 列调研计划给你审（链接/锚点/数据源）→ 确认后批量查ID → 论文比对 → 输出验证卡 → STOP等yes
- **阶段1流程**：用确认ID拉profile → OpenAlex/S2/ORCID → 不需ID的并行采集（arXiv/Web Search/百度百科）→ 去重排序 → 论文按学术履历阶段分组 → 输出9节画像
- **论文分组**：按学术履历阶段（博士/博后/独立后），不是按引用数排序
- **回环设计**：阶段2（领域脉络）建立的认知，在阶段3（论文串联）中用于标注每篇论文在领域树上的位置
- **脚本策略**：只写必要的脚本（discipline_classifier + identity_resolver），渲染/决策由AI手动完成
- **人闸两次**：阶段0 Step 3（审调研计划）+ Step 6（确认身份）
