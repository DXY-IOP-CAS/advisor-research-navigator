## 2. 第1段：老师基础信息调研 —— 学者画像

### 2.1 `supervisor`（GitHub: ybq22/supervisor）

**简介**：2026-03 发布的中文学者画像工具。"把导师蒸馏成随时可问的 AI Skill"。支持 Google 搜索 + 机构主页 + Wikipedia + Google Scholar 多源信息采集，AI 分析研究风格、学术观点，最终生成结构化 JSON 档案和可直接对话的 Claude Code Skill。

**核心功能**：
- 多源信息采集（机构主页、Wikipedia、Google Scholar、个人主页）
- AI 深度论文分析（研究主题、方法论偏好、写作风格）
- 公开信息分析（性格、沟通风格、学术理念）
- 聊天记录/邮件/PDF 多渠道上传
- 输出结构化 JSON + Claude Code Skill

**对我们的价值**：**最直接相关**。目标完全一致。差异点在于它输出到"可对话的 AI Skill"为止，不做领域调研和学习讲义。我们的差异化正好在它停下的地方继续往前推。

**可借鉴**：多源信息采集流水线设计、增量更新模式。
**局限**：单 Agent 非流水线结构、无领域认知层。

### 2.2 SciAtlas / SciScholar（GitHub: zjunlp/SciScholar）

**简介**：2026-04 更新的学术知识图谱工具，CLI 驱动。内置 `researcher-review` 子命令，从学术知识图谱提取学者研究轨迹和代表论文。

**核心功能**：
- 四层研究分类（domain → field → subfield → topic）
- 7 个 CLI 研究流程：文献综述、想法接地、趋势分析、学者画像等
- Agent Skill 包，可移植到 Codex/Claude Code
- 输出 `report.md` / `response.json` / `summary.txt`

**对我们的价值**：第1段的信息聚合骨架。四层研究分类法可借鉴到第2段的领域认知框架。
**局限**：只基于学术论文/引文数据，缺少机构背景、新闻采访、中文资源等维度。英文输出，需翻译流水线。

### 2.3 ai-talent-graph（GitHub: rrrrrredy/ai-talent-graph）

**简介**：2026-04 发布的 AI 学者画像工具，整合 4 个免费公开 API。

**核心功能**：
- OpenAlex（论文/引用/机构）、Semantic Scholar（h-index）、ORCID（职业历史）、arXiv（最新预印本）
- 学者搜索、全景 profile、机构分析、最新论文追踪
- JSON 输出，支持程序化处理
- 可选 GitHub 工程数据 + X/Twitter 影响力

**对我们的价值**：代码架构参考——4 个免费 API 的数据采集流水线实现，Python 脚本化。
**局限**：CS/AI 领域偏重，物理领域适配需要验证数据覆盖率。

### 2.4 ORCID MCP Server（GitHub: cyanheads/orcid-mcp-server）

**简介**：2026-05 发布的 MCP 协议工具，通过 ORCID 公共 API 获取研究者 profile/works/affiliations/funding。

**核心功能**：
- 研究者消歧（姓名+机构交叉验证）
- Profile 摘要（传记、关键词、外部 ID）
- Works 列表（含 DOI/PMID/arXiv ID，可直接链式查询下游服务）
- Funding 和 peer review 记录

**对我们的价值**：数据源层的一个 MCP 模块。标准化 ORCID 数据获取通道。
**局限**：仅 ORCID 单一数据源，需要与其他源组合。

### 2.5 同类对比

| 工具 | 数据源覆盖 | 中文学者 | 输出格式 | 与我们第1段的匹配度 |
|:-----|:----------:|:--------:|:--------:|:------------------:|
| supervisor | Google + Scholar + 主页 + 多文件 | ✅ 中文 | JSON + Skill | ⭐⭐⭐⭐⭐ |

| ORCID MCP | ORCID 单一源 | 无关 | JSON (MCP) | ⭐⭐ |
| OmniScholar | Semantic Scholar | ❌ 英文 | Excel + web | ⭐⭐ |

## 3. 第2段：相关领域发展调研 —— 知识图谱与领域地图

### 3.1 GoAI（arXiv 2503.08549, 2025）

**简介**："Graph of AI Ideas"——用知识图谱 + LLM 为 AI 学生生成个性化学习路径。

**核心方法**：
1. 从论文构建研究知识图谱（节点 = 论文/概念/技能/工具；边 = 语义注释的引文关系）
2. beam search 引导的领域演化路径追踪
3. 按前提概念复杂度排序 → 渐进课程
4. Idea Studio：形成性反馈 + 创意评估

**对我们的价值**：**高度相关**，特别是第2段的"领域发展脉络"和第4段的"学习路径排序"。beam search + 引文语义的方法可直接借鉴到第2/3段。

**可借鉴**：beam search 路径生成方法、前提概念复杂度排序算法。
**局限**：领域限定在 AI/CS；物理领域知识图谱需要重新构建。

### 3.2 PaperCircle（GitHub: MAXNORM8650/papercircle, ACL 2026 Oral）

**简介**：开源多智能体论文发现与分析框架。

**核心功能**：
- 多源论文发现（arXiv/Scopus/IEEE）
- **Paper Mind Graph**：LLM 从论文提取概念/方法/实验为结构化知识图谱
- **Paper Lineage**：关系映射（extends/applies/evaluates/contradicts/survey/prerequisite）
- 论文审核生成（ICLR/NeurIPS/ICML 格式）

**对我们的价值**：第3段的"论文串联"直接对应 Paper Lineage 的论文关系映射。6 种关系类型可以作为我们第3段论文时间线的关系分类模板。

### 3.3 Meta Knowledge Graph（GitHub: Seaual/meta-knowledge-graph, 2026-03）

**简介**：LLM 赋能的学术知识图谱引擎。上传 PDF → LLM 两阶段概念抽取 → 交互式知识图谱。

**核心功能**：
- 两阶段概念提取：阶段1 论文理解 → 阶段2 分层概念提取（8种分类）
- Semantic Scholar 元数据增强
- 研究空白发现（4 种方法论：Gap Filling / Leaf Extension / Bottleneck / Transfer）
- 多格式导出（HTML / Obsidian Canvas / Markdown）
- 双语支持（中/英概念名）

**对我们的价值**：第2段的概念提取工程参考。两阶段提取流程可以用于从论文中提取领域核心概念。
**局限**：需要论文 PDF 输入，不适合批量学者场景。

### 3.4 Paper Pulse（GitHub: Shahir-47/Paper-Pulse）

**简介**：AI 学术研究平台，从 4 个源聚合论文，构建知识图谱，通过 RAG 回答论文相关问题。

**核心功能**：
- 每日自动论文发现（4 源聚合）
- Cohere 神经重排序
- Neo4j 知识图谱（paper/author/concept/institution 节点）
- 3 种文献综述模式（Quick / Publication / Deep Analysis）

**对我们的价值**：知识图谱构建流水线参考——如何从论文元数据自动构建概念/作者/机构关系。

## 4. 第3段：老师论文串联与定位

这个环节的专有工具最少。最相关的方法集中在上面提到的**GoAI beam search 路径生成**和**PaperCircle lineage mapping**。

此外有几个参考：

| 工具 | 可借鉴点 |
|:-----|:--------|
| GeneticFlow (arXiv 2304.12217) | 自引图的时变层级可视化，展示学者影响的时间演化 |
| SCIOS (GitHub: HuberyLL/SCIOS) | "Tech Tree" + "Collaboration Network" 技术演化可视化 |
| Paper Agent (GitHub: KingdeGuo/paper-agent) | GraphRAG 引文网络遍历 + 10 维论文交叉对比表格 |

**现状判断**：第3段很大程度上需要**自研**。没有现成工具能把"老师从入职到现在的论文"串联成一个发展叙事。GoAI 的 beam search 在技术上最接近。

## 5. 第4段：从零到前沿学习讲义

### 5.1 paper-to-course（GitHub: KaguraTart/paper-to-course, 2026-04）

**简介**：Claude Code Skill。输入论文 PDF，输出 6 模块交互式 HTML 课程。

**6 模块结构**：
1. Problem & Motivation
2. Literature Timeline（领域关键里程碑动画）
3. SOTA Comparison（方法对比表，hover 显示细节）
4. Method Deep-Dive（公式逐行翻译、算法流）
5. Experiments & Results（基准/消融/SOTA 对比）
6. Limitations & Future Work

**对我们的价值**：**章节模板高度相关**。第4段学习讲义的每章结构可以直接借鉴。其叙事顺序（问题→历史→对比→深挖→结果→展望）与我们的"起承转合"一致。

**可借鉴**：6 模块顺序、公式逐行翻译模式、"Literature Timeline" 动画。
**局限**：单论文级，我们要做的是"从基础到前沿的完整路径"，比单论文更长程。

### 5.2 InstructKG（GitHub: aalrabah/instructkg, 2026）

**简介**：从教学材料自动构建 instructor-aligned 知识图谱。

**方法**：PDF → Chunking → 概念提取（LLM）→ UMAP+HDBSCAN 聚类 → 证据聚合 → LLM 关系判断（prerequisite/related/unrelated）→ 知识图谱

**对我们的价值**：如果第4段讲义包含"推荐教材章节"，InstructKG 的方法可以用于从教材 PDF 中自动提取概念依赖图，验证学习路径的零跳步。

**局限**：需要教材 PDF 作为输入；非实时，需要预构建。

### 5.3 GoAI（再次提及）

GoAI 的"前提概念复杂度排序 → 渐进课程"方法直接对应第4段的核心需求：**从高数+线代到老师前沿的递进路径**。其 beam search 路径生成可以理解为"从某个目标论文反向搜索前置知识，形成学习链"。

### 5.4 claude-paper（GitHub: AhsanZiaFarooqui/claude-paper）

**简介**：Claude Code 论文学习插件。解析 PDF → 按难度生成自适应学习材料。

**核心产出**：summary.md / insights.md / method.md / mental-model.md / qa.md / code demo。
**对我们的价值**：单论文学习材料的输出格式参考。

## 7. 综合结论

### 7.1 四段的技术可行性

| 段 | 现成工具可用性 | 自研占比 | 推荐策略 |
|:---|:-------------:|:-------:|:--------|
| 第1段 老师基础信息 | **高**（supervisor, SciAtlas, 4-API 聚合） | 20% | 借鉴 supervisor 的采集流程 + 自研多源交叉比对 |
| 第2段 领域发展调研 | **中**（GoAI, PaperKG 方法） | 60% | GoAI beam search 方法借鉴 + 物理领域适配 |
| 第3段 论文串联定位 | **低**（GeneticFlow 可视化, PaperCircle lineage） | 80% | 主要是叙事设计 + 引文关系分类 |
| 第4段 学习讲义 | **中**（paper-to-course 模板, GoAI 排序） | 60% | 结构借鉴 paper-to-course, 排序借鉴 GoAI |

### 7.2 关键差异化

- `supervisor` 停在"可对话的导师画像"——我们继续做领域+讲义
- `SciAtlas` 停在"学术KG学者报告"——我们加入中文/机构/新闻等多维信息
- `paper-to-course` 停在"单论文课程"——我们做多论文+从基础开始的完整路径
- `GoAI` 停在"AI 领域"——我们推广到物理等自然科学

### 7.3 不建议照搬的

- 不直接用 SciAtlas 的 CLI 输出（英文、缺中文数据源、缺机构背景信息）
- 不直接用 paper-to-course 的 HTML 输出格式（我们要的是 .md 可编辑文档）
- 不搭建 Neo4j 等重型图数据库（Paper Pulse/PaperCircle 的架构太重）

### 7.4 值得优先试用的

1. **supervisor 的采集流程** — 看它的多源信息聚合怎么做的
2. **GoAI 的 beam search 路径生成** — 看能不能迁移到物理文献
3. **paper-to-course 的 6 模块结构** — 作为第4段章节模板的起点
