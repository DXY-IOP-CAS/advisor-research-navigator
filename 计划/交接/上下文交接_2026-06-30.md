## 3. 本轮关键决策

### 3.1 定位确认

- **不做**：导师口碑/评价、学生→导师匹配、搭建网站
- **做**：信息调研 + 教学。给定导师→产出五份文档
- **交付形态**：Claude Code Skill + Markdown 模板，零服务器零部署

### 3.2 流水线结构（重要变化）

从四段改为五份文档：

```
00_身份验证卡.md  ← 新增前置"人闸"
01_基础画像.md
02_领域脉络.md
03_论文定位.md
04_学习讲义.md
```

关键设计：Phase 1a（浅调研→输出验证卡）→ 用户确认 → Phase 1b（深调研）。身份消歧不允许全自动。

### 3.3 身份消歧方案（已实测验证）

技术路径：用户提供官网URL → 用 MCP fetch 抓页面提取 email/论文标题 → 搜 ORCID（用 email） → 搜 OpenAlex（用姓名+机构ROR） → 论文标题交叉比对锁定唯一ID → 输出验证卡让用户确认。

实测发现的问题已记录在 `计划/工具设计.md` 的 2.2 节。

### 3.4 文件规范

需求确认文档统一规则：
- h2 为 Block，h3 为小节，h4 用于每节三个子段（工作流程/技术原理与执行/输出规格）
- 禁止 h5+
- 待填充内容统一标记 `[待设计]`

### 3.5 数据组织

- `项目/导师/<姓名>/` 每个老师独立目录，frontmatter 标注 affiliation/department/tags
- `build_index.py` 扫描 frontmatter 自动生成按机构/按方向索引
- 机构树为根，方向视图为派生

## 6. 技术细节备忘

### 身份消歧的 API 调用（实测有效）

```
ORCID:  GET https://pub.orcid.org/v3.0/expanded-search?q=email:{邮箱}
OpenAlex: GET https://api.openalex.org/authors?search={姓名}&filter=last_known_institutions.id:{机构ROR}
Semantic Scholar: GET https://api.semanticscholar.org/graph/v1/author/search?query={姓名拼音}
```

中科院物理所 ROR: `https://ror.org/05qbk4x57`

### 实测教训

- OpenAlex 对中文名消歧不可靠（张鹏举返回 25 个同名）
- 邮箱是 ORCID 搜索的黄金锚点
- Semantic Scholar 在物理领域覆盖率不如 OpenAlex
- Edit 工具对中文 old_string 匹配敏感——替换前从文件复制精确文本
